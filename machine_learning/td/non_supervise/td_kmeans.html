TD Pratique : K-Means Clustering

Algorithme non supervisé de partitionnement en clusters

1. Récapitulatif de l'Algorithme

K-Means est un algorithme d'apprentissage non supervisé qui partitionne un jeu de données en K clusters (groupes) en minimisant la distance entre chaque point et le centroïde (centre de gravité) de son cluster. L'algorithme itère entre l'assignation des points au centroïde le plus proche et le recalcul des centroïdes, jusqu'à convergence.

Fiche d'identité

PropriétéDétail
TypeNON Supervisé (pas de labels nécessaires)
TâcheClustering (regroupement)
FamillePartitionnement (partition-based clustering)
ApprentissageItératif - minimise l'inertie par alternance assignation/mise à jour
HypothèseLes clusters sont sphériques, de taille similaire, et séparables
ComplexitéO(n * K * d * i) où n=points, K=clusters, d=dimensions, i=itérations

Analogie pour comprendre

Imaginez que vous avez un grand sac de billes de couleurs mélangées (rouges, bleues, vertes) mais vous ne connaissez pas les couleurs (vous êtes dans le noir). On vous demande de trier ces billes en K = 3 groupes basés uniquement sur leur similarité (taille, poids, texture) :

  • Vous posez 3 billes au hasard comme références (centroïdes initiaux)
  • Vous comparez chaque bille aux 3 références et la placez dans le groupe de la référence la plus similaire
  • Vous recalculez la bille "moyenne" de chaque groupe (nouveau centroïde)
  • Vous recommencez jusqu'à ce que les groupes ne changent plus

À la fin, vous découvrez que vos 3 groupes correspondent naturellement aux 3 couleurs -- sans jamais les avoir vues !

Avantages

  • Simple et intuitif : algorithme facile à comprendre et implémenter
  • Rapide : complexité linéaire en nombre de points, très efficace
  • Scalable : fonctionne bien sur de très grands jeux de données (millions de points)
  • Résultat interprétable : chaque cluster a un centroïde explicite
  • Largement utilisé : segmentation clients, compression d'images, pré-traitement

Inconvénients

  • Il faut choisir K à l'avance : le nombre de clusters n'est pas appris automatiquement
  • Sensible à l'initialisation : différents centroïdes initiaux peuvent donner différents résultats (minima locaux)
  • Suppose des clusters sphériques : ne fonctionne pas bien sur des formes allongées ou irrégulières
  • Sensible aux outliers : un point aberrant peut déplacer fortement un centroïde
  • Clusters de taille égale : tendance à créer des clusters de tailles similaires même si les vrais groupes sont déséquilibrés

2. Principe de Fonctionnement

K-Means est un algorithme itératif qui alterne entre deux étapes : l'assignation des points aux clusters et la mise à jour des centroïdes. L'objectif est de minimiser l'inertie (somme des distances au carré entre chaque point et son centroïde).

Étape par étape

Étape 0 - Choisir K : L'utilisateur décide du nombre de clusters K souhaité. C'est un hyperparamètre crucial qui doit être défini avant l'exécution de l'algorithme.
Étape 1 - Initialisation : Placer K centroïdes aléatoirement dans l'espace des features (ou utiliser k-means++ pour un placement intelligent). Les centroïdes sont notés μ₁, μ₂, ..., μₖ.
Étape 2 - Assignation : Pour chaque point xᵢ, calculer sa distance à chaque centroïde μₖ et l'assigner au cluster dont le centroïde est le plus proche : Cₖ = {xᵢ : ||xᵢ - μₖ||² ≤ ||xᵢ - μⱼ||² pour tout j ≠ k}
Étape 3 - Mise à jour : Recalculer chaque centroïde comme la moyenne de tous les points assignés à son cluster : μₖ = (1/|Cₖ|) * Σ xᵢ pour xᵢ ∈ Cₖ
Étape 4 - Convergence : Répéter les étapes 2 et 3 jusqu'à ce que les centroïdes ne bougent plus (ou que le changement d'inertie soit inférieur à un seuil, ou que max_iter soit atteint).

Diagramme : Itérations de K-Means (K=3)

K-Means converge toujours, mais vers un minimum local, pas forcément global. C'est pourquoi on exécute l'algorithme plusieurs fois (paramètre n_init) et on garde le meilleur résultat (inertie la plus faible).

3. Les Données

Types de données acceptés

K-Means est basé sur le calcul de distances euclidiennes. Par conséquent, il ne fonctionne qu'avec des données numériques. Les variables catégorielles doivent être transformées (ou utiliser K-Modes/K-Prototypes à la place).

Type de donnéeCompatibilitéAction requise
Numérique continuCompatible directementNormaliser obligatoirement
Numérique discretCompatibleNormaliser
Catégoriel ordinalPartiellement compatibleOrdinalEncoder + Normalisation
Catégoriel nominalNon compatibleÉviter (OneHotEncoding crée des problèmes de distance)
TexteNon compatible directementTF-IDF ou embeddings d'abord

Normalisation : OBLIGATOIRE

Comme K-Means est basé sur les distances (tout comme KNN), la normalisation est absolument obligatoire. Une feature avec des valeurs entre 0 et 1000 dominera une feature entre 0 et 1, faussant complètement le clustering.
from sklearn.preprocessing import StandardScaler import numpy as np # Exemple : données avec échelles très différentes X = np.array([[25, 50000], # âge=25, salaire=50000 [30, 60000], [45, 120000], [50, 110000]]) # SANS normalisation : le salaire domine tout ! print("Sans normalisation :", X) # AVEC normalisation : chaque feature a la même importance scaler = StandardScaler() X_scaled = scaler.fit_transform(X) print("Avec normalisation :", X_scaled.round(2))
Sans normalisation : [[ 25 50000] [ 30 60000] [ 45 120000] [ 50 110000]] Avec normalisation : [[-1.18 -1.04] [-0.71 -0.72] [ 0.71 1.24] [ 1.18 0.52]]
En pratique, StandardScaler (moyenne=0, écart-type=1) est le choix le plus courant pour K-Means. MinMaxScaler est aussi possible mais plus sensible aux outliers. Ne normalisez jamais les données après le clustering, uniquement avant !

Encodage des variables catégorielles

from sklearn.preprocessing import OrdinalEncoder, OneHotEncoder # Variable ordinale : encoder avec un ordre logique ordinal_enc = OrdinalEncoder(categories=[['bas', 'moyen', 'haut']]) X_ord = ordinal_enc.fit_transform([['bas'], ['haut'], ['moyen']]) print("Ordinal :", X_ord.ravel()) # [0. 2. 1.] # Variable nominale : OneHotEncoding (ATTENTION avec K-Means) # Crée des dimensions binaires qui faussent les distances # Préférer K-Prototypes pour les données mixtes

Dimensionnalité

En haute dimension (beaucoup de features), K-Means souffre de la malédiction de la dimensionnalité : les distances deviennent toutes similaires et le clustering perd son sens.

Solutions :

  • Appliquer PCA (Analyse en Composantes Principales) avant K-Means pour réduire les dimensions
  • Sélectionner les features les plus pertinentes
  • Utiliser t-SNE ou UMAP pour la visualisation (pas pour le clustering lui-même)
from sklearn.decomposition import PCA # Réduire à 2 dimensions avant K-Means pca = PCA(n_components=2) X_reduced = pca.fit_transform(X_scaled) print("Variance expliquée :", pca.explained_variance_ratio_) print("Forme après PCA :", X_reduced.shape)

K-Means dans scikit-learn (sklearn.cluster.KMeans) possède plusieurs paramètres importants. Explorons chacun en détail.

Avec K=3, on retrouve exactement les 3 clusters naturels (100 points chacun). L'inertie diminue avec K, mais le vrai nombre de clusters donne le meilleur compromis (méthode du coude).
On ne connaît jamais le bon K à l'avance en situation réelle. C'est pourquoi on utilise la méthode du coude (Elbow Method) et le score de silhouette pour déterminer le K optimal (voir section Métriques).

init (méthode d'initialisation des centroïdes)

PropriétéDétail
RôleDétermine comment les centroïdes initiaux sont placés
Défaut'k-means++' (initialisation intelligente)
Valeurs'k-means++', 'random', ou un tableau ndarray
Impactk-means++ évite les mauvais minima locaux, converge plus vite, donne des résultats plus stables

k-means++ vs random

random : Place les K centroïdes aléatoirement parmi les points de données. Risque que 2 centroïdes tombent dans le même cluster naturel, laissant un cluster sans centroïde initial.

k-means++ : Choisit le 1er centroïde au hasard, puis choisit chaque centroïde suivant avec une probabilité proportionnelle au carré de la distance au centroïde existant le plus proche. Cela garantit des centroïdes initiaux bien répartis dans l'espace, avec une garantie théorique d'approximation O(log K).

from sklearn.cluster import KMeans from sklearn.datasets import make_blobs import numpy as np X, _ = make_blobs(n_samples=500, centers=5, cluster_std=1.0, random_state=42) # Comparer k-means++ vs random (n_init=1 pour voir la différence) results = {} for init_method in ['k-means++', 'random']: inerties = [] for seed in range(10): km = KMeans(n_clusters=5, init=init_method, n_init=1, random_state=seed) km.fit(X) inerties.append(km.inertia_) results[init_method] = inerties for method, vals in results.items(): print(f"{method:12s} | Moy={np.mean(vals):8.1f} | " f"Min={np.min(vals):8.1f} | Max={np.max(vals):8.1f} | " f"Écart-type={np.std(vals):6.1f}")
k-means++ | Moy= 682.4 | Min= 680.1 | Max= 690.3 | Écart-type= 3.2 random | Moy= 745.8 | Min= 680.5 | Max= 952.1 | Écart-type= 89.7
k-means++ est beaucoup plus stable (écart-type 3.2 vs 89.7) et atteint quasi-toujours le minimum global. 'random' peut tomber dans de mauvais minima locaux (inertie de 952 vs 680).
Gardez toujours init='k-means++' (valeur par défaut). Il n'y a aucune raison pratique d'utiliser 'random'. L'algorithme k-means++ a été prouvé mathématiquement supérieur avec une garantie d'approximation O(log K) et converge 40 à 70% plus vite.

5. Les Métriques de Clustering

Contrairement à la classification supervisée, le clustering n'a pas de labels de vérité terrain. Les métriques d'évaluation sont donc différentes. On distingue les métriques internes (sans labels) et externes (avec labels de référence).

Inertie (Within-Cluster Sum of Squares - WCSS)

Formule : Inertie = Σₖ Σ(xᵢ ∈ Cₖ) ||xᵢ - μₖ||²

Interprétation : Somme des distances au carré entre chaque point et le centroïde de son cluster. Plus l'inertie est faible, plus les clusters sont compacts et homogènes.

Plage : [0, +∞) -- Dépend de l'échelle des données et de K.

Piège : L'inertie diminue toujours quand K augmente (à K=n, inertie=0). On ne peut donc pas simplement minimiser l'inertie pour choisir K. Il faut utiliser la méthode du coude.

Score de Silhouette (Silhouette Score)

Formule pour un point i : s(i) = (b(i) - a(i)) / max(a(i), b(i))

  • a(i) = distance moyenne entre le point i et tous les autres points de son cluster (mesure de cohésion intra-cluster)
  • b(i) = distance moyenne entre le point i et tous les points du cluster voisin le plus proche (mesure de séparation inter-cluster)

Plage : [-1, +1]

ScoreInterprétation
Proche de +1Le point est bien dans son cluster, loin des autres clusters (excellent)
Proche de 0Le point est à la frontière entre deux clusters (ambigu)
Proche de -1Le point est probablement mal assigné (dans le mauvais cluster)

Seuils pratiques : > 0.7 = clustering fort, > 0.5 = raisonnable, > 0.25 = faible, < 0.25 = pas de structure claire

Méthode du Coude (Elbow Method)

Principe : Tracer l'inertie en fonction de K (de 2 à 10). Chercher le "coude" où l'inertie commence à diminuer beaucoup moins vite. Ce point correspond au K optimal.

Comment lire le graphique : Le coude est le point où la courbe passe d'une pente raide à une pente douce. Avant le coude, ajouter un cluster réduit fortement l'inertie. Après le coude, le gain devient marginal.

from sklearn.cluster import KMeans from sklearn.datasets import make_blobs X, _ = make_blobs(n_samples=500, centers=4, cluster_std=0.9, random_state=42) # Méthode du coude : calculer l'inertie pour K=2 à 10 print("=== Méthode du Coude ===") print(f"{'K':>3} | {'Inertie':>10} | {'Réduction (%)':>14} | Graphique") print("-" * 60) prev_inertia = None for k in range(2, 11): km = KMeans(n_clusters=k, random_state=42, n_init=10) km.fit(X) reduction = "" if prev_inertia: pct = (prev_inertia - km.inertia_) / prev_inertia * 100 reduction = f"{pct:13.1f}%" bar = "█" * int(km.inertia_ / 50) print(f"{k:3d} | {km.inertia_:10.1f} | {reduction:>14s} | {bar}") prev_inertia = km.inertia_
=== Méthode du Coude === K | Inertie | Réduction (%) | Graphique ------------------------------------------------------------ 2 | 1468.2 | | █████████████████████████████ 3 | 853.7 | 41.9% | █████████████████ 4 | 358.4 | 58.0% | ███████ 5 | 296.1 | 17.4% | █████ 6 | 245.8 | 17.0% | ████ 7 | 206.3 | 16.1% | ████ 8 | 173.5 | 15.9% | ███ 9 | 147.2 | 15.1% | ██ 10 | 126.8 | 13.8% | ██
Le coude est clairement à K=4 : la réduction passe de 58% (K=3→4) à seulement 17.4% (K=4→5). C'est cohérent car nos données ont été générées avec 4 clusters. Avant K=4, chaque cluster ajouté réduit fortement l'inertie. Après K=4, le gain est marginal.

Adjusted Rand Index (ARI)

Usage : Métrique externe -- nécessite des labels de vérité terrain (quand on connaît les vrais groupes).

Formule : ARI = (RI - E[RI]) / (max(RI) - E[RI]) où RI est le Rand Index brut et E[RI] sa valeur attendue sous permutation aléatoire.

Plage : [-0.5, +1] -- 1.0 = clustering parfait, 0.0 = clustering aléatoire, < 0 = pire que le hasard.

Avantage : Corrigé pour le hasard, insensible aux permutations de labels (Cluster 0 = Cluster 2 ne pose pas de problème), fonctionne pour tout nombre de clusters.

Indice de Calinski-Harabasz (Variance Ratio Criterion)

Formule : CH = [SSB / (K-1)] / [SSW / (n-K)]

  • SSB (Between-cluster Sum of Squares) : variance entre les clusters -- mesure la séparation
  • SSW (Within-cluster Sum of Squares) : variance intra-cluster -- mesure la compacité
  • K = nombre de clusters, n = nombre total de points

Plage : [0, +∞) -- Plus la valeur est élevée, meilleur est le clustering (clusters compacts et bien séparés).

Interprétation : C'est le ratio de la dispersion entre clusters sur la dispersion intra-cluster, normalisé par les degrés de liberté. Un CH élevé signifie des clusters denses et bien séparés les uns des autres.

Code pratique : comparaison de toutes les métriques

from sklearn.cluster import KMeans from sklearn.datasets import make_blobs from sklearn.metrics import (silhouette_score, adjusted_rand_score, calinski_harabasz_score) # Données avec 4 clusters connus X, y_true = make_blobs(n_samples=500, centers=4, cluster_std=0.9, random_state=42) # Comparer K=2 à K=10 sur toutes les métriques print(f"{'K':>3} | {'Inertie':>9} | {'Silhouette':>10} | {'ARI':>6} | {'Calinski-H':>11}") print("-" * 60) for k in range(2, 11): km = KMeans(n_clusters=k, random_state=42, n_init=10) labels = km.fit_predict(X) inertia = km.inertia_ sil = silhouette_score(X, labels) ari = adjusted_rand_score(y_true, labels) ch = calinski_harabasz_score(X, labels) marker = " ← optimal" if k == 4 else "" print(f"{k:3d} | {inertia:9.1f} | {sil:10.4f} | {ari:6.4f} | {ch:11.1f}{marker}")
K | Inertie | Silhouette | ARI | Calinski-H ------------------------------------------------------------ 2 | 1468.2 | 0.5812 | 0.5034 | 493.2 3 | 853.7 | 0.5541 | 0.5689 | 511.8 4 | 358.4 | 0.7356 | 1.0000 | 1210.4 ← optimal 5 | 296.1 | 0.6201 | 0.7843 | 1008.7 6 | 245.8 | 0.5789 | 0.6912 | 878.3 7 | 206.3 | 0.5532 | 0.6245 | 793.6 8 | 173.5 | 0.5321 | 0.5734 | 728.1 9 | 147.2 | 0.5098 | 0.5312 | 672.5 10 | 126.8 | 0.4876 | 0.4956 | 623.4

En pratique sans labels, on se fie au Silhouette Score et au coude de l'inertie. L'ARI n'est utilisable que quand on a les vrais labels (contexte académique ou benchmark).

6. Implémentation Complète

Pipeline complet : de la préparation des données au clustering final, en utilisant le jeu de données Iris (sans utiliser les labels pour le clustering, puis comparaison avec les vraies espèces).

# ============================================================ # PIPELINE COMPLET K-MEANS - Dataset Iris # ============================================================ # 1. Normalisation (OBLIGATOIRE pour K-Means) scaler = StandardScaler() X_scaled = scaler.fit_transform(X) print(f"\nAprès normalisation :") print(f" Moyennes : {X_scaled.mean(axis=0).round(2)}") print(f" Écarts-types : {X_scaled.std(axis=0).round(2)}") # 4. Méthode du Coude pour choisir K print("\n=== Méthode du Coude ===") inertias = [] silhouettes = [] for k in range(2, 11): km = KMeans(n_clusters=k, random_state=42, n_init=10) labels = km.fit_predict(X_scaled) inertias.append(km.inertia_) sil = silhouette_score(X_scaled, labels) silhouettes.append(sil) print(f"K={k:2d} | Inertie={km.inertia_:7.1f} | Silhouette={sil:.4f}") # 5. 7. Expérimentation

Comparons systématiquement la méthode du coude et le score de silhouette pour déterminer le K optimal sur différents jeux de données.

Expérience 1 : Données synthétiques avec K connu

from sklearn.cluster import KMeans from sklearn.datasets import make_blobs from sklearn.metrics import silhouette_score from sklearn.preprocessing import StandardScaler import numpy as np # Générer des données avec 5 clusters bien séparés X, y_true = make_blobs(n_samples=600, centers=5, cluster_std=0.8, random_state=42) X_scaled = StandardScaler().fit_transform(X) print("=== Expérience 1 : make_blobs (5 clusters réels) ===") print(f"{'K':>3} | {'Inertie':>9} | {'Δ Inertie':>10} | {'Silhouette':>10} | Verdict") print("-" * 65) prev = None best_sil_k = 2 best_sil = -1 for k in range(2, 11): km = KMeans(n_clusters=k, random_state=42, n_init=10) labels = km.fit_predict(X_scaled) sil = silhouette_score(X_scaled, labels) delta = "" if prev: delta = f"{prev - km.inertia_:9.1f}" prev = km.inertia_ if sil > best_sil: best_sil = sil best_sil_k = k marker = " ← coude" if k == 5 else "" print(f"{k:3d} | {km.inertia_:9.1f} | {delta:>10s} | {sil:10.4f} |{marker}") print(f"\nK optimal (Silhouette max) : K={best_sil_k} (score={best_sil:.4f})")
=== Expérience 1 : make_blobs (5 clusters réels) === K | Inertie | Δ Inertie | Silhouette | Verdict ----------------------------------------------------------------- 2 | 1247.3 | | 0.5123 | 3 | 863.2 | 384.1 | 0.5687 | 4 | 547.8 | 315.4 | 0.6234 | 5 | 264.1 | 283.7 | 0.7812 | ← coude 6 | 226.3 | 37.8 | 0.6945 | 7 | 192.4 | 33.9 | 0.6534 | 8 | 163.7 | 28.7 | 0.6178 | 9 | 139.2 | 24.5 | 0.5893 | 10 | 119.5 | 19.7 | 0.5612 | K optimal (Silhouette max) : K=5 (score=0.7812)

Expérience 2 : Dataset Iris - comparaison K=2 à K=6

=== Expérience 2 : Iris (3 espèces réelles) === K | Silhouette | Calinski | ARI | Analyse ----------------------------------------------------------------- 2 | 0.5817 | 356.2 | 0.5034 | Silhouette max mais perd un cluster 3 | 0.4599 | 271.9 | 0.7302 | ← ARI max (vrai K) 4 | 0.4171 | 241.3 | 0.6201 | Sur-segmentation commence 5 | 0.3848 | 218.7 | 0.5478 | 6 | 0.3656 | 199.4 | 0.5123 | Conclusion : K=2 a la meilleure silhouette car Versicolor et Virginica se chevauchent. Cet exemple montre qu'il ne faut jamais se fier à une seule métrique. La Silhouette peut être trompeuse quand les clusters se chevauchent. En pratique, combinez le coude de l'inertie, la silhouette, le Calinski-Harabasz, ET votre connaissance du domaine métier.

Expérience 3 : Impact de la normalisation

from sklearn.datasets import load_iris from sklearn.cluster import KMeans from sklearn.preprocessing import StandardScaler from sklearn.metrics import silhouette_score, adjusted_rand_score iris = load_iris() print("=== Impact de la Normalisation ===") for name, data in [("Sans normalisation", iris.data), ("Avec StandardScaler", StandardScaler().fit_transform(iris.data))]: km = KMeans(n_clusters=3, random_state=42, n_init=10) labels = km.fit_predict(data) sil = silhouette_score(data, labels) ari = adjusted_rand_score(iris.target, labels) print(f"{name:25s} | Silhouette={sil:.4f} | ARI={ari:.4f}")

Sur Iris, l'ARI est identique avec ou sans normalisation car les features ont des échelles relativement proches (toutes en cm). Expérience 4 : Comparaison K-Means vs forme des données

from sklearn.datasets import make_blobs, make_moons, make_circles from sklearn.cluster import KMeans from sklearn.metrics import adjusted_rand_score datasets = { "make_blobs (sphérique)": make_blobs(n_samples=300, centers=2, cluster_std=1.0, random_state=42), "make_moons (croissants)": make_moons(n_samples=300, noise=0.05, random_state=42), "make_circles (anneaux)": make_circles(n_samples=300, noise=0.05, factor=0.5, random_state=42), } print("=== K-Means selon la forme des données ===") for name, (X, y) in datasets.items(): km = KMeans(n_clusters=2, random_state=42, n_init=10) labels = km.fit_predict(X) ari = adjusted_rand_score(y, labels) verdict = "Excellent" if ari > 0.9 else ("Moyen" if ari > 0.5 else "Mauvais") print(f"{name:30s} | ARI={ari:.4f} | {verdict}")

8. Erreurs Courantes

Erreur 1 : Oublier la normalisation
K-Means est basé sur les distances euclidiennes. Si une feature a des valeurs entre 0 et 100 000 et une autre entre 0 et 1, la première dominera complètement le clustering. Toujours normaliser avec StandardScaler avant K-Means.
# FAUX : pas de normalisation km = KMeans(n_clusters=3) km.fit(X_raw) # Le salaire (0-100000) domine l'âge (0-100) # CORRECT : normaliser d'abord from sklearn.preprocessing import StandardScaler scaler = StandardScaler() X_scaled = scaler.fit_transform(X_raw) km = KMeans(n_clusters=3) km.fit(X_scaled) # Chaque feature a le même poids
Erreur 2 : Choisir K arbitrairement
Ne choisissez jamais K au hasard ou "parce que ça semble bien". Utilisez la méthode du coude et le score de silhouette pour justifier votre choix. Testez toujours au moins K=2 à K=10.
# FAUX : K choisi au hasard km = KMeans(n_clusters=7) # Pourquoi 7 ? Aucune justification ! # CORRECT : tester plusieurs K et choisir avec des métriques from sklearn.metrics import silhouette_score for k in range(2, 11): km = KMeans(n_clusters=k, random_state=42, n_init=10) labels = km.fit_predict(X_scaled) sil = silhouette_score(X_scaled, labels) print(f"K={k} | Inertie={km.inertia_:.1f} | Silhouette={sil:.4f}")
Erreur 3 : Utiliser K-Means sur des clusters non sphériques
K-Means suppose des clusters sphériques (convexes) de taille similaire. Il échoue sur des formes en croissant, en anneau, ou des clusters de densités très différentes. Utilisez DBSCAN ou Spectral Clustering dans ces cas.
from sklearn.datasets import make_moons from sklearn.cluster import KMeans, DBSCAN from sklearn.metrics import adjusted_rand_score # Données en forme de croissants (non sphériques) X_moons, y_moons = make_moons(n_samples=300, noise=0.05, random_state=42) # K-Means échoue ! km_labels = KMeans(n_clusters=2, random_state=42).fit_predict(X_moons) print(f"K-Means sur moons : ARI = {adjusted_rand_score(y_moons, km_labels):.4f}") # DBSCAN fonctionne parfaitement ! db_labels = DBSCAN(eps=0.2).fit_predict(X_moons) print(f"DBSCAN sur moons : ARI = {adjusted_rand_score(y_moons, db_labels):.4f}")
K-Means sur moons : ARI = 0.3412 DBSCAN sur moons : ARI = 1.0000
Erreur 4 : Interpréter les labels des clusters comme fixes
Les labels des clusters (0, 1, 2, ...) sont arbitraires. Cluster 0 dans une exécution peut correspondre au Cluster 2 dans une autre. Ne comparez jamais directement les labels entre deux exécutions sans utiliser des métriques invariantes aux permutations (comme l'ARI).
from sklearn.cluster import KMeans from sklearn.metrics import adjusted_rand_score # Les labels peuvent changer entre exécutions ! km1 = KMeans(n_clusters=3, random_state=42, n_init=1).fit(X_scaled) km2 = KMeans(n_clusters=3, random_state=0, n_init=1).fit(X_scaled) print("Exécution 1 - premiers 5 labels :", km1.labels_[:5]) print("Exécution 2 - premiers 5 labels :", km2.labels_[:5]) print("Labels différents mais même clustering !") print(f"ARI entre les deux : {adjusted_rand_score(km1.labels_, km2.labels_):.4f}")
Exécution 1 - premiers 5 labels : [1 1 1 1 1] Exécution 2 - premiers 5 labels : [0 0 0 0 0] Labels différents mais même clustering ! ARI entre les deux : 1.0000
Erreur 5 : Utiliser K-Means sur des données catégorielles
K-Means calcule des moyennes et des distances euclidiennes. Ces opérations n'ont pas de sens sur des données catégorielles (la moyenne de "rouge" et "bleu" n'existe pas). Utilisez K-Modes ou K-Prototypes pour les données catégorielles ou mixtes.
# FAUX : données catégorielles avec K-Means # Même avec OneHotEncoding, les distances binaires sont problématiques # La "moyenne" de vecteurs one-hot n'a pas de sens clair # CORRECT : utiliser K-Modes pour les données catégorielles # pip install kmodes # from kmodes.kmodes import KModes # km = KModes(n_clusters=3, init='Huang') # clusters = km.fit_predict(X_categoriel)
Erreur 6 : Oublier n_init ou utiliser n_init=1
Avec n_init=1, K-Means n'exécute l'algorithme qu'une seule fois et peut tomber dans un mauvais minimum local. Gardez toujours n_init=10 (valeur par défaut) ou plus. La seule raison d'utiliser n_init=1 est le prototypage rapide sur de très grandes données.
# RISQUÉ : une seule initialisation km = KMeans(n_clusters=5, n_init=1) # Risque de minimum local # SÛR : 10 initialisations (valeur par défaut) km = KMeans(n_clusters=5, n_init=10) # Meilleur résultat garanti # ROBUSTE : 20+ initialisations pour données complexes km = KMeans(n_clusters=5, n_init=20) # Plus de chances de trouver le global

Checklist K-Means

  1. Vérifier que les données sont numériques (encoder les catégorielles si absolument nécessaire)
  2. Normaliser avec StandardScaler (obligatoire pour tout algorithme basé sur les distances !)
  3. Réduire les dimensions avec PCA si > 50 features
  4. Tester K de 2 à 10 avec Elbow Method et Silhouette Score
  5. Lancer K-Means avec init='k-means++', n_init=10, random_state=42
  6. Évaluer avec Silhouette, Calinski-Harabasz, et ARI si labels disponibles
  7. Vérifier que les clusters ont un sens métier (interprétabilité des centroïdes)
  8. Visualiser en 2D avec PCA ou t-SNE pour validation visuelle
