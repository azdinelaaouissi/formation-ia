TD Pratique : DBSCAN Clustering

Density-Based Spatial Clustering of Applications with Noise - Trouver les clusters par la densité

Objectif : Maîtriser DBSCAN. Comprendre eps, min_samples, les points noyaux / frontières / bruit, la méthode du k-distance plot, et pourquoi DBSCAN surpasse K-Means sur les formes non-convexes. Aucun label nécessaire : on découvre la structure cachée ET les anomalies.

1. Récapitulatif de l'Algorithme

DBSCAN (Density-Based Spatial Clustering of Applications with Noise) est un algorithme de clustering qui regroupe les points qui sont proches les uns des autres en termes de densité, tout en identifiant comme bruit les points isolés dans des régions de faible densité. Contrairement à K-Means, il n'a pas besoin du nombre de clusters K, et il peut trouver des clusters de forme arbitraire.

Analogie : La fête

Imaginez une fête dans une grande salle. Les gens qui se tiennent proches les uns des autres forment naturellement des groupes de conversation. Si quelqu'un se tient à côté d'un groupe mais ne parle pas beaucoup, c'est un point frontière. Et la personne seule dans un coin, loin de tout le monde ? C'est du bruit.

DBSCAN fonctionne exactement comme ça : il regarde la densité locale (combien de personnes dans un rayon donné) pour décider qui fait partie d'un groupe et qui est isolé. Pas besoin de décider à l'avance combien de groupes il y a !

Avantages

  • Pas besoin de spécifier K : DBSCAN découvre automatiquement le nombre de clusters.
  • Formes arbitraires : Trouve des clusters en croissant, spirale, anneau... contrairement à K-Means.
  • Détection d'anomalies intégrée : Les points isolés sont marqués comme bruit (label = -1).
  • Robuste aux outliers : Les points aberrants ne déforment pas les clusters.

Inconvénients

  • Sensible à eps et min_samples : Un mauvais choix peut tout casser.
  • Densités variables : Si les clusters ont des densités très différentes, un seul eps ne suffit pas.
  • Haute dimension : La notion de distance perd son sens en très haute dimension (curse of dimensionality).
  • Plus lent que K-Means : O(n²) dans le pire cas vs O(n*K*i) pour K-Means.

2. Principe de Fonctionnement

Concepts clés

eps-voisinage (epsilon-neighborhood) : L'ensemble de tous les points situés à une distance ≤ eps d'un point p. Noté N_eps(p) = {q ∈ D | distance(p, q) ≤ eps}.
Point noyau (Core Point) : Un point p est un point noyau s'il a au moins min_samples points dans son eps-voisinage (y compris lui-même). |N_eps(p)| ≥ min_samples.
Point frontière (Border Point) : Un point qui n'est PAS un point noyau mais qui se trouve dans l'eps-voisinage d'un point noyau. Il est en bordure d'un cluster.
Point bruit (Noise Point) : Un point qui n'est ni un point noyau ni un point frontière. Il ne se trouve dans l'eps-voisinage d'aucun point noyau. Label = -1.
Densité-accessible (Density-Reachable) : Un point q est densité-accessible depuis p s'il existe une chaîne de points p1, p2, ..., pn où chaque pi est un point noyau et pi+1 est dans l'eps-voisinage de pi.
Densité-connecté (Density-Connected) : Deux points p et q sont densité-connectés s'il existe un point o tel que p et q sont tous deux densité-accessibles depuis o. Un cluster DBSCAN = tous les points densité-connectés.

Étapes de l'algorithme

Étape 1 : Pour chaque point p non visité, calculer son eps-voisinage N_eps(p). Compter les voisins.
Étape 2 : Si |N_eps(p)| ≥ min_samples, p est un point noyau. Créer un nouveau cluster et y ajouter p.
Étape 3 : Pour chaque voisin q de p dans N_eps(p) : si q n'est pas encore visité, marquer q comme visité et calculer N_eps(q). Si q est aussi un point noyau, ajouter ses voisins à la file. Ajouter q au cluster courant.
Étape 4 : Répéter jusqu'à ce que tous les points soient visités. Les points qui n'ont été ajoutés à aucun cluster sont du bruit.
L'algorithme est glouton : une fois qu'un point est assigné à un cluster, il ne change jamais de cluster. L'expansion se fait par propagation de densité : de point noyau en point noyau, tant qu'ils sont connectés par leurs eps-voisinages.

Diagramme : Points noyaux, frontières et bruit

Un point noyau a au moins min_samples voisins dans son rayon eps (cercles en pointillé). Un point frontière est dans le rayon d'un point noyau mais n'a pas assez de voisins pour être noyau. Un point bruit est trop isolé : il n'est dans le voisinage d'aucun point noyau.

3. Les Données

Types de données acceptées

DBSCAN travaille avec des données numériques car il repose sur le calcul de distances. Les features catégorielles doivent être encodées numériquement au préalable (ou utiliser une métrique adaptée comme Gower).

Normalisation OBLIGATOIRE ! DBSCAN utilise des distances. Si une feature est sur l'échelle 0-1000 et une autre sur 0-1, la première dominera complètement le calcul de distance. Utilisez StandardScaler ou MinMaxScaler AVANT d'appliquer DBSCAN.
from sklearn.preprocessing import StandardScaler import numpy as np # Exemple : donnees avec echelles differentes X_raw = np.array([[1000, 0.1], [1050, 0.2], [5000, 0.8], [5100, 0.9]]) print("AVANT normalisation :") print(f" Feature 0 : min={X_raw[:,0].min()}, max={X_raw[:,0].max()}") print(f" Feature 1 : min={X_raw[:,1].min()}, max={X_raw[:,1].max()}") scaler = StandardScaler() X_scaled = scaler.fit_transform(X_raw) print("APRES normalisation :") print(f" Feature 0 : min={X_scaled[:,0].min():.2f}, max={X_scaled[:,0].max():.2f}") print(f" Feature 1 : min={X_scaled[:,1].min():.2f}, max={X_scaled[:,1].max():.2f}")
AVANT normalisation : Feature 0 : min=1000, max=5100 Feature 1 : min=0.1, max=0.9 APRES normalisation : Feature 0 : min=-0.93, max=1.19 Feature 1 : min=-1.16, max=1.16

Ce que DBSCAN gère nativement

  • Bruit / Outliers : Détectés automatiquement (label = -1). Pas besoin de pré-traitement.
  • Clusters non-sphériques : Formes arbitraires (croissants, spirales, anneaux).
  • Nombre de clusters inconnu : DBSCAN le découvre tout seul.

Ce que DBSCAN ne gère PAS bien

  • Densités variables : Un seul eps ne peut pas capturer des clusters de densités très différentes.
  • Haute dimension : Les distances perdent leur sens (curse of dimensionality). Réduire d'abord avec PCA/UMAP.

4. Les Paramètres

eps (epsilon) - Le rayon de voisinage

Rôle : Définit la distance maximale entre deux points pour qu'ils soient considérés comme voisins.

Défaut : 0.5

Impact : Trop petit = tout est bruit. Trop grand = un seul cluster géant. C'est LE paramètre le plus critique.

from sklearn.cluster import DBSCAN from sklearn.datasets import make_moons from sklearn.preprocessing import StandardScaler import numpy as np # Donnees en forme de croissants X, y_true = make_moons(n_samples=300, noise=0.08, random_state=42) X = StandardScaler().fit_transform(X) # Tester differentes valeurs de eps print(f"{'eps':>6} | {'Clusters':>8} | {'Bruit':>6} | Commentaire") print("-" * 58) for eps in [0.1, 0.3, 0.5, 1.0, 2.0]: db = DBSCAN(eps=eps, min_samples=5) labels = db.fit_predict(X) n_clusters = len(set(labels) - {-1}) n_noise = list(labels).count(-1) if eps == 0.1: comment = "TROP PETIT - presque tout est bruit !" elif eps == 0.3: comment = "BON - 2 clusters bien separes" elif eps == 0.5: comment = "OK - commence a fusionner" elif eps == 1.0: comment = "TROP GRAND - 1 seul cluster" else: comment = "BEAUCOUP TROP GRAND - tout dans 1 cluster" print(f"eps={eps:.1f} | {n_clusters:>8} | {n_noise:>6} | {comment}")
eps | Clusters | Bruit | Commentaire ---------------------------------------------------------- eps=0.1 | 12 | 187 | TROP PETIT - presque tout est bruit ! eps=0.3 | 2 | 4 | BON - 2 clusters bien separes eps=0.5 | 1 | 0 | OK - commence a fusionner eps=1.0 | 1 | 0 | TROP GRAND - 1 seul cluster eps=2.0 | 1 | 0 | BEAUCOUP TROP GRAND - tout dans 1 cluster
Avec eps=0.1, le rayon est trop petit : les points ne trouvent pas assez de voisins, 187 points sur 300 sont classés comme bruit. Avec eps=0.3, on obtient exactement 2 clusters (les 2 croissants). Avec eps ≥ 1.0, le rayon est si grand que tous les points deviennent un seul cluster.
Le choix de eps est critique. Il dépend de l'échelle des données. C'est pourquoi normaliser d'abord est essentiel : après normalisation, les valeurs typiques de eps sont entre 0.1 et 2.0. Sans normalisation, eps pourrait devoir être 1000 !

min_samples - Le seuil de densité

Rôle : Nombre minimum de points dans l'eps-voisinage pour qu'un point soit considéré comme point noyau.

Défaut : 5

Impact : Plus min_samples est grand, plus les clusters doivent être denses, et plus de points sont classés comme bruit.

# Impact de min_samples (eps fixe a 0.3) print(f"{'min_samples':>11} | {'Clusters':>8} | {'Bruit':>6} | {'% Bruit':>7}") print("-" * 50) for ms in [2, 5, 10, 20]: db = DBSCAN(eps=0.3, min_samples=ms) labels = db.fit_predict(X) n_clusters = len(set(labels) - {-1}) n_noise = list(labels).count(-1) pct_noise = n_noise / len(labels) * 100 print(f"min_samples={ms:>2} | {n_clusters:>8} | {n_noise:>6} | {pct_noise:>6.1f}%")
min_samples | Clusters | Bruit | % Bruit -------------------------------------------------- min_samples= 2 | 2 | 0 | 0.0% min_samples= 5 | 2 | 4 | 1.3% min_samples=10 | 2 | 18 | 6.0% min_samples=20 | 2 | 52 | 17.3%
Avec min_samples=2, presque rien n'est bruit (trop permissif). Avec min_samples=20, 17% des points sont bruit (trop strict). min_samples=5 est un bon compromis. Règle générale : min_samples ≥ nombre_de_dimensions + 1, ou min_samples = 2 * n_features.
Pour des données 2D, min_samples=4 ou 5 est un bon départ. Pour des données haute dimension (50 features), essayez min_samples=100 ou plus. Augmenter min_samples rend DBSCAN plus conservateur : il exige des zones plus denses pour former un cluster.

metric - La mesure de distance

Rôle : Définit comment la distance entre deux points est calculée.

Défaut : 'euclidean'

Impact : Change la forme des eps-voisinages (cercles vs losanges).

# Comparaison des metriques de distance from sklearn.metrics import adjusted_rand_score for met in ['euclidean', 'manhattan']: db = DBSCAN(eps=0.3, min_samples=5, metric=met) labels = db.fit_predict(X) n_clusters = len(set(labels) - {-1}) n_noise = list(labels).count(-1) ari = adjusted_rand_score(y_true, labels) print(f"metric={met:>12} | Clusters: {n_clusters} | Bruit: {n_noise:>3} | ARI: {ari:.4f}")
metric= euclidean | Clusters: 2 | Bruit: 4 | ARI: 0.9536 metric= manhattan | Clusters: 2 | Bruit: 2 | ARI: 0.9687
La métrique euclidienne (par défaut) fonctionne bien dans la plupart des cas. Manhattan peut être meilleure en haute dimension. Pour des données spécifiques (texte, géolocalisation), envisagez cosine ou haversine.

algorithm - L'algorithme de recherche de voisins

Rôle : Détermine comment les voisins sont recherchés (impact sur la vitesse, PAS sur les résultats).

Défaut : 'auto'

Options : 'auto', 'ball_tree', 'kd_tree', 'brute'

import time # Comparer les temps de calcul from sklearn.datasets import make_blobs X_big, _ = make_blobs(n_samples=5000, n_features=2, random_state=42) X_big = StandardScaler().fit_transform(X_big) print(f"{'algorithm':>10} | {'Temps (ms)':>10} | Clusters") print("-" * 40) for algo in ['auto', 'ball_tree', 'kd_tree', 'brute']: start = time.time() db = DBSCAN(eps=0.3, min_samples=5, algorithm=algo) labels = db.fit_predict(X_big) elapsed = (time.time() - start) * 1000 n_c = len(set(labels) - {-1}) print(f"{algo:>10} | {elapsed:>8.1f}ms | {n_c}")
algorithm | Temps (ms) | Clusters ---------------------------------------- auto | 42.3ms | 3 ball_tree | 45.1ms | 3 kd_tree | 38.7ms | 3 brute | 312.5ms | 3
Les résultats sont identiques quel que soit l'algorithme. Seule la vitesse change. 'auto' choisit intelligemment. 'brute' est beaucoup plus lent car il calcule toutes les paires de distances. Gardez 'auto' (défaut).

Comment choisir eps : Méthode du k-distance plot

La méthode du k-distance plot est l'approche standard pour choisir eps :

  1. Fixer k = min_samples (ex: 5)
  2. Pour chaque point, calculer la distance au k-ème voisin le plus proche
  3. Trier ces distances par ordre croissant et les tracer
  4. Chercher le coude dans la courbe : c'est la valeur optimale de eps

Avant le coude : distances faibles = points dans des clusters. Après le coude : distances élevées = bruit.

from sklearn.neighbors import NearestNeighbors import numpy as np # Donnees : 2 croissants + bruit X_moons, y_moons = make_moons(n_samples=300, noise=0.08, random_state=42) X_moons = StandardScaler().fit_transform(X_moons) # Calculer la distance au 5eme voisin (min_samples=5) k = 5 nn = NearestNeighbors(n_neighbors=k) nn.fit(X_moons) distances, _ = nn.kneighbors(X_moons) # Prendre la distance au k-eme voisin (derniere colonne) k_distances = np.sort(distances[:, k-1]) # Afficher les stats pour trouver le coude print("k-distance plot (min_samples=5) :") print(f" Percentile 80% : {np.percentile(k_distances, 80):.4f}") print(f" Percentile 90% : {np.percentile(k_distances, 90):.4f}") print(f" Percentile 95% : {np.percentile(k_distances, 95):.4f}") print(f" Max : {k_distances.max():.4f}") print(f"\n => Le coude est autour de {np.percentile(k_distances, 90):.2f}") print(f" => Choisir eps ~ 0.30")
k-distance plot (min_samples=5) : Percentile 80% : 0.2134 Percentile 90% : 0.2876 Percentile 95% : 0.3512 Max : 0.8234 => Le coude est autour de 0.29 => Choisir eps ~ 0.30
La méthode du k-distance plot est l'équivalent de la méthode du coude pour K-Means. Elle est semi-automatique : vous devez interpréter visuellement le coude. En cas de doute, testez plusieurs valeurs de eps autour du coude et comparez les résultats (nombre de clusters, pourcentage de bruit, silhouette score).

5. Les Métriques d'Évaluation

DBSCAN est non supervisé : pas de labels pour calculer accuracy ou F1. On utilise des métriques de clustering :

Silhouette Score

Formule : Pour chaque point i : s(i) = (b(i) - a(i)) / max(a(i), b(i))

  • a(i) = distance moyenne de i aux autres points de SON cluster (cohésion)
  • b(i) = distance moyenne de i aux points du cluster le plus PROCHE (séparation)

Plage : -1 à +1. Proche de 1 = bon clustering. Proche de 0 = chevauchement. Négatif = mauvaise affectation.

Avec DBSCAN, il faut exclure les points bruit (label=-1) du calcul du silhouette score ! Sinon, les points isolés faussent la métrique.

Nombre de clusters trouvés

DBSCAN découvre automatiquement le nombre de clusters. C'est une information en soi : si DBSCAN trouve 50 clusters sur des données où vous en attendez 3, vos paramètres sont probablement mauvais.

Calcul : n_clusters = len(set(labels) - {-1})

Nombre de points bruit

Le nombre de points classés comme bruit (label=-1). À interpréter comme des anomalies ou outliers. Si 80% des points sont bruit, eps est trop petit. Si 0% sont bruit, eps est peut-être trop grand.

Calcul : n_noise = list(labels).count(-1)

Adjusted Rand Index (ARI)

Quand l'utiliser : Uniquement si vous avez les vrais labels (données synthétiques ou données annotées).

Plage : -1 à +1. ARI = 1 = correspondance parfaite. ARI = 0 = aléatoire.

Formule : ARI = (RI - E[RI]) / (max(RI) - E[RI]), où RI est le Rand Index et E[RI] son espérance sous l'hypothèse nulle.

DBCV (Density-Based Clustering Validation)

Concept : Métrique conçue spécifiquement pour les clusterings basés sur la densité. Contrairement au silhouette score qui préfère les clusters compacts et sphériques, DBCV évalue la qualité en comparant la densité intra-cluster et la séparation inter-cluster locale.

Plage : -1 à +1. Plus c'est proche de 1, meilleur est le clustering.

Avantage : Ne nécessite aucun paramètre d'entrée, et fonctionne bien sur des clusters non-convexes. C'est la métrique la plus adaptée pour évaluer DBSCAN, mais elle n'est pas disponible dans scikit-learn (nécessite une librairie externe).

Code pratique : Évaluation complète

from sklearn.cluster import DBSCAN from sklearn.datasets import make_moons from sklearn.preprocessing import StandardScaler from sklearn.metrics import silhouette_score, adjusted_rand_score import numpy as np # Donnees X, y_true = make_moons(n_samples=300, noise=0.08, random_state=42) X = StandardScaler().fit_transform(X) # DBSCAN db = DBSCAN(eps=0.3, min_samples=5) labels = db.fit_predict(X) # Metriques n_clusters = len(set(labels) - {-1}) n_noise = list(labels).count(-1) pct_noise = n_noise / len(labels) * 100 print(f"Nombre de clusters : {n_clusters}") print(f"Points bruit : {n_noise} ({pct_noise:.1f}%)") # Silhouette (EXCLURE les points bruit !) mask = labels != -1 if len(set(labels[mask])) > 1: sil = silhouette_score(X[mask], labels[mask]) print(f"Silhouette Score : {sil:.4f} (sans bruit)") # ARI (si on a les vrais labels) ari = adjusted_rand_score(y_true, labels) print(f"Adjusted Rand Index: {ari:.4f}") # Taille de chaque cluster for c in sorted(set(labels)): count = list(labels).count(c) label_name = f"Cluster {c}" if c != -1 else "Bruit" print(f" {label_name:>12} : {count} points")
Nombre de clusters : 2 Points bruit : 4 (1.3%) Silhouette Score : 0.4812 (sans bruit) Adjusted Rand Index: 0.9536 Bruit : 4 points Cluster 0 : 148 points Cluster 1 : 148 points
Le silhouette score de 0.48 peut sembler moyen, mais c'est normal pour des clusters non-sphériques ! Le silhouette score préfère les clusters convexes et compacts. L'ARI de 0.95 confirme que DBSCAN retrouve quasi parfaitement les 2 croissants. Pour des clusters non-convexes, fiez-vous davantage à l'ARI (si disponible) ou au DBCV qu'au silhouette score.

6. Implémentation Complète

Pipeline complet : Génération de données → Normalisation → K-distance plot → DBSCAN → Évaluation → Analyse des résultats.

7. Expérimentation : K-Means vs DBSCAN

Objectif : Prouver que DBSCAN surpasse K-Means sur les données non-convexes (make_moons), et montrer quand chacun brille.

from sklearn.cluster import DBSCAN, KMeans from sklearn.datasets import make_moons, make_blobs from sklearn.preprocessing import StandardScaler from sklearn.metrics import silhouette_score, adjusted_rand_score import numpy as np # ============================================================ # TEST 1 : Donnees en croissants (non-convexes) # ============================================================ X_moons, y_moons = make_moons(n_samples=500, noise=0.07, random_state=42) X_moons_s = StandardScaler().fit_transform(X_moons) # K-Means sur les croissants km = KMeans(n_clusters=2, random_state=42, n_init=10) km_labels = km.fit_predict(X_moons_s) km_ari = adjusted_rand_score(y_moons, km_labels) km_sil = silhouette_score(X_moons_s, km_labels) # DBSCAN sur les croissants db = DBSCAN(eps=0.3, min_samples=5) db_labels = db.fit_predict(X_moons_s) db_ari = adjusted_rand_score(y_moons, db_labels) mask = db_labels != -1 db_sil = silhouette_score(X_moons_s[mask], db_labels[mask]) db_noise = list(db_labels).count(-1) print("=" * 60) print("TEST 1 : Croissants (make_moons) - DBSCAN GAGNE") print("=" * 60) print(f"{'Metrique':<20} | {'K-Means':>10} | {'DBSCAN':>10} | Gagnant") print("-" * 60) print(f"{'ARI':<20} | {km_ari:>10.4f} | {db_ari:>10.4f} | {'DBSCAN' if db_ari > km_ari else 'K-Means'}") print(f"{'Silhouette':<20} | {km_sil:>10.4f} | {db_sil:>10.4f} | {'DBSCAN' if db_sil > km_sil else 'K-Means'}") print(f"{'Clusters':<20} | {'2':>10} | {len(set(db_labels)-{-1}):>10} |") print(f"{'Bruit detecte':<20} | {'0':>10} | {db_noise:>10} |")

8. Erreurs Courantes

Erreur 1 : Mauvais choix de eps (trop petit)
Si eps est trop petit, presque tous les points sont classés comme bruit. Vous obtenez des dizaines de micro-clusters ou aucun cluster. Solution : Utilisez le k-distance plot pour choisir eps, ou augmentez progressivement eps jusqu'à obtenir un nombre raisonnable de clusters.
# Erreur : eps trop petit db_bad = DBSCAN(eps=0.05, min_samples=5) labels_bad = db_bad.fit_predict(X_moons_s) n_noise_bad = list(labels_bad).count(-1) print(f"eps=0.05 : {len(set(labels_bad)-{-1})} clusters, {n_noise_bad} bruit ({n_noise_bad/len(labels_bad)*100:.0f}%)") print("=> Presque tout est bruit ! eps est TROP PETIT.")
eps=0.05 : 0 clusters, 500 bruit (100%) => Presque tout est bruit ! eps est TROP PETIT.
Erreur 2 : Oublier la normalisation
DBSCAN calcule des distances. Si les features ont des échelles différentes (salaire en milliers vs âge en dizaines), le clustering sera dominé par la feature à grande échelle. Solution : Toujours appliquer StandardScaler() AVANT DBSCAN.
# Erreur : pas de normalisation sur donnees a echelles differentes X_raw = X_moons.copy() X_raw[:, 0] *= 1000 # Feature 0 sur echelle 0-1000 db_raw = DBSCAN(eps=0.3, min_samples=5) labels_raw = db_raw.fit_predict(X_raw) n_noise_raw = list(labels_raw).count(-1) db_norm = DBSCAN(eps=0.3, min_samples=5) labels_norm = db_norm.fit_predict(StandardScaler().fit_transform(X_raw)) n_noise_norm = list(labels_norm).count(-1) print(f"SANS normalisation : {len(set(labels_raw)-{-1})} clusters, {n_noise_raw} bruit") print(f"AVEC normalisation : {len(set(labels_norm)-{-1})} clusters, {n_noise_norm} bruit")
SANS normalisation : 0 clusters, 500 bruit AVEC normalisation : 2 clusters, 3 bruit
Erreur 3 : Utiliser DBSCAN sur des données à densités variables
Si un cluster est très dense et un autre très épars, un seul eps ne peut pas capturer les deux. Le cluster épars sera classé comme bruit. Solution : Utiliser HDBSCAN ou OPTICS qui gèrent les densités variables.
# Erreur : densites variables # Cluster 1 : tres dense (std=0.3), Cluster 2 : tres epars (std=2.0) from sklearn.datasets import make_blobs X_var, y_var = make_blobs(n_samples=[200, 200], centers=[[0,0],[10,10]], cluster_std=[0.3, 2.0], random_state=42) X_var_s = StandardScaler().fit_transform(X_var) for eps in [0.2, 0.5, 0.8]: db_var = DBSCAN(eps=eps, min_samples=5) labels_var = db_var.fit_predict(X_var_s) n_c = len(set(labels_var) - {-1}) n_b = list(labels_var).count(-1) ari_var = adjusted_rand_score(y_var, labels_var) print(f"eps={eps} : {n_c} clusters, {n_b} bruit, ARI={ari_var:.3f}") print("=> Aucun eps ne donne un ARI parfait. HDBSCAN serait meilleur ici.")
eps=0.2 : 1 clusters, 178 bruit, ARI=0.234 eps=0.5 : 2 clusters, 45 bruit, ARI=0.712 eps=0.8 : 1 clusters, 8 bruit, ARI=0.567 => Aucun eps ne donne un ARI parfait. HDBSCAN serait meilleur ici.
Erreur 4 : Ignorer les points bruit dans l'analyse
Les points bruit (label=-1) ne sont pas des erreurs, ce sont des informations précieuses. Ils représentent des anomalies ou des outliers. Ne les supprimez pas aveuglément : analysez-les ! Solution : Examinez les points bruit séparément et incluez-les dans votre rapport.
Erreur 5 : Comparer DBSCAN uniquement avec le silhouette score
Le silhouette score favorise les clusters convexes et compacts. DBSCAN excelle sur les clusters non-convexes, mais son silhouette score sera souvent plus bas que K-Means. Solution : Utiliser l'ARI (si labels disponibles), le DBCV, ou une analyse visuelle pour évaluer DBSCAN.
Erreur 6 : Appliquer DBSCAN en très haute dimension sans réduction
En haute dimension (>50 features), toutes les distances se ressemblent (curse of dimensionality). DBSCAN ne peut plus distinguer les voisins des non-voisins. Solution : Réduire la dimension avec PCA ou UMAP AVANT d'appliquer DBSCAN.
# Erreur 6 demontrée : haute dimension from sklearn.datasets import make_blobs # 2D : DBSCAN fonctionne X_2d, y_2d = make_blobs(n_samples=300, n_features=2, centers=3, random_state=42) X_2d = StandardScaler().fit_transform(X_2d) db_2d = DBSCAN(eps=0.5, min_samples=5).fit_predict(X_2d) # 100D : DBSCAN echoue X_100d, y_100d = make_blobs(n_samples=300, n_features=100, centers=3, random_state=42) X_100d = StandardScaler().fit_transform(X_100d) db_100d = DBSCAN(eps=0.5, min_samples=5).fit_predict(X_100d) print(f"2D : {len(set(db_2d)-{-1})} clusters, ARI={adjusted_rand_score(y_2d, db_2d):.3f}") print(f"100D : {len(set(db_100d)-{-1})} clusters, ARI={adjusted_rand_score(y_100d, db_100d):.3f}") print("=> En 100D, les distances perdent leur sens. Reduire d'abord avec PCA !")
2D : 3 clusters, ARI=0.978 100D : 0 clusters, ARI=0.000 => En 100D, les distances perdent leur sens. Reduire d'abord avec PCA !

Les 5 règles d'or de DBSCAN

  1. Toujours normaliser les données avant DBSCAN (StandardScaler).
  2. Utiliser le k-distance plot pour choisir eps (pas au hasard !).
  3. Commencer avec min_samples = 2 * n_features et ajuster.
  4. Analyser les points bruit : ce sont des anomalies, pas des erreurs.
  5. Ne pas se fier au silhouette seul pour des clusters non-convexes.
