TD Pratique : Arbre de Decision (Decision Tree)

Algorithme supervise de classification et regression - le modele le plus interpretable du machine learning

1. Comprendre l'Algorithme

L'Arbre de Decision est un algorithme supervise qui apprend une serie de questions if/else sur les features pour classer les donnees. Chaque question divise les donnees en deux groupes, creant une structure en arbre qui va de la racine (toutes les donnees) jusqu'aux feuilles (la prediction finale).

Analogie : Le jeu des 20 questions

Imaginez le jeu des 20 questions : vous pensez a un animal, et votre ami pose des questions pour le deviner.

  • Question 1 : "Est-ce qu'il vit dans l'eau ?" → Oui/Non
  • Question 2 (si Non) : "Est-ce qu'il a des ailes ?" → Oui/Non
  • Question 3 (si Oui) : "Est-ce qu'il est plus grand qu'un chat ?" → Oui/Non
  • ... jusqu'a identifier l'animal

L'arbre de decision fait exactement la meme chose, mais avec des donnees numeriques : "Est-ce que petal_length ≤ 2.45 ?" au lieu de questions en langage naturel. L'algorithme apprend automatiquement les meilleures questions a poser et dans quel ordre.

Comment ca marche ? Les 4 etapes

Etape 1 : Choisir la meilleure feature et le meilleur seuil pour diviser les donnees (celle qui separe le mieux les classes)
Etape 2 : Diviser les donnees en 2 groupes selon cette question (gauche = Oui, droite = Non)
Etape 3 : Repeter recursivement pour chaque groupe (chaque sous-groupe devient un nouveau probleme)
Etape 4 : S'arreter quand un noeud est pur (une seule classe) ou quand les contraintes sont atteintes (max_depth, min_samples...)

Comment l'arbre choisit la meilleure feature ?

Le coeur de l'algorithme est de trouver, a chaque noeud, la question qui separe le mieux les classes. Pour mesurer "le mieux", on utilise des mesures d'impurete : Gini ou Entropie.

Formule de Gini (impurete de Gini)

Gini(noeud) = 1 - ∑ pi²

Ou pi est la proportion de la classe i dans le noeud.

Exemple concret : Un noeud contient 40 Setosa et 60 Versicolor (100 samples).

  • p(Setosa) = 40/100 = 0.4
  • p(Versicolor) = 60/100 = 0.6
  • Gini = 1 - (0.4² + 0.6²) = 1 - (0.16 + 0.36) = 1 - 0.52 = 0.48

Interpretation : Gini = 0 signifie un noeud pur (une seule classe). Gini = 0.5 signifie impurete maximale (2 classes a 50/50). Plus Gini est bas, plus le noeud est pur.

Formule de l'Entropie

H(noeud) = - ∑ pi × log2(pi)

Ou pi est la proportion de la classe i dans le noeud.

Meme exemple : 40 Setosa et 60 Versicolor.

  • H = -(0.4 × log2(0.4) + 0.6 × log2(0.6))
  • H = -(0.4 × (-1.322) + 0.6 × (-0.737))
  • H = -(-0.529 + (-0.442)) = -(-0.971) = 0.971

Interpretation : H = 0 signifie un noeud pur. H = 1 signifie impurete maximale (2 classes a 50/50). Plus H est bas, plus le noeud est pur.

Information Gain (Gain d'Information)

IG = H(parent) - [ (n_gauche/n_total) × H(gauche) + (n_droite/n_total) × H(droite) ]

L'arbre choisit le split qui maximise le gain d'information : la plus grande reduction d'impurete entre le noeud parent et ses enfants.

Exemple : Un noeud parent a H = 0.971. Apres un split :

  • Enfant gauche (30 samples) : H = 0.0 (pur, 30 Setosa)
  • Enfant droit (70 samples) : H = 0.863 (melange)
  • IG = 0.971 - [(30/100) × 0.0 + (70/100) × 0.863]
  • IG = 0.971 - [0 + 0.604] = 0.367

L'arbre compare l'IG de tous les splits possibles et choisit celui avec le plus grand IG.

En pratique, Gini et Entropie donnent des resultats tres similaires dans ~98% des cas. La difference est surtout theorique. Sklearn utilise Gini par defaut car le calcul est plus rapide (pas de logarithme).

Avantages et Inconvenients

Avantages

  • Tres interpretable : on peut lire les regles comme un humain
  • Pas de normalisation necessaire (unique parmi les algorithmes !)
  • Gere les types mixtes : numeriques ET categoriques
  • Rapide a entrainer et a predire
  • Feature importance integree
  • Gere les relations non-lineaires

Inconvenients

  • Overfitting facile : sans contraintes, memorise tout
  • Instabilite : un petit changement dans les donnees peut donner un arbre completement different
  • Biais vers les features avec beaucoup de valeurs
  • Frontieres rectangulaires seulement (paralleles aux axes)
  • Pas optimal globalement : approche gloutonne (greedy)

2. Les Donnees

Pas de normalisation necessaire !

C'est l'avantage unique de l'arbre de decision : aucune normalisation n'est necessaire. L'arbre ne calcule pas de distances entre les points (contrairement a KNN ou SVM). Il pose simplement des questions du type "feature X ≤ seuil ?".

Que petal_length soit en centimetres (0-7) ou en kilometres (0-0.00007), la question "petal_length ≤ 2.45 cm" ou "petal_length ≤ 0.0000245 km" donnera exactement le meme split.

Types de features geres

Sensibilite aux changements de donnees

L'arbre de decision est tres instable : retirer ou ajouter quelques samples peut changer completement la structure de l'arbre. C'est l'une des raisons principales pour lesquelles on utilise des ensembles d'arbres (Random Forest, XGBoost) en pratique.
# Demonstration de l'instabilite from sklearn.tree import DecisionTreeClassifier from sklearn.datasets import load_iris import numpy as np X, y = load_iris(return_X_y=True) # Arbre 1 : toutes les donnees tree1 = DecisionTreeClassifier(random_state=42) tree1.fit(X, y) print("Feature importances (toutes donnees):", tree1.feature_importances_.round(3)) # Arbre 2 : on retire 5 samples idx = np.random.RandomState(0).choice(150, 145, replace=False) tree2 = DecisionTreeClassifier(random_state=42) tree2.fit(X[idx], y[idx]) print("Feature importances (5 samples en moins):", tree2.feature_importances_.round(3))
Feature importances (toutes donnees): [0. 0.013 0.563 0.424] Feature importances (5 samples en moins): [0. 0.027 0.927 0.046]
On voit que retirer seulement 5 samples sur 150 change completement l'importance des features. petal_width passe de 0.424 a 0.046 ! C'est le probleme principal de l'arbre de decision seul.

3. Les Parametres (Detailles)

Les parametres de l'arbre de decision controlent sa complexite. Sans contraintes, l'arbre grandit jusqu'a ce que chaque feuille soit pure → il memorise les donnees d'entrainement (overfitting). Les parametres sont des gardes-fous.

max_depth (LE parametre le plus important)

RoleLimite la profondeur maximale de l'arbre (nombre de niveaux de questions)
DefautNone (pas de limite = l'arbre grandit jusqu'a purete ou min_samples)
Typeint ou None
AugmenterArbre plus profond → plus complexe → plus d'overfitting
DiminuerArbre moins profond → plus simple → plus d'underfitting
RegleCommencer avec max_depth=3 a 5, puis ajuster avec cross-validation
# Impact de max_depth from sklearn.tree import DecisionTreeClassifier from sklearn.datasets import load_iris from sklearn.model_selection import train_test_split X, y = load_iris(return_X_y=True) X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42) for depth in [1, 2, 3, 5, None]: tree = DecisionTreeClassifier(max_depth=depth, random_state=42) tree.fit(X_train, y_train) train_acc = tree.score(X_train, y_train) test_acc = tree.score(X_test, y_test) print(f"max_depth={str(depth):>4} | Train: {train_acc:.3f} | Test: {test_acc:.3f}")
max_depth= 1 | Train: 0.667 | Test: 0.667 max_depth= 2 | Train: 0.962 | Test: 0.956 max_depth= 3 | Train: 0.981 | Test: 0.956 max_depth= 5 | Train: 1.000 | Test: 0.956 max_depth=None | Train: 1.000 | Test: 0.956
max_depth=1 est clairement en underfitting (66.7%). max_depth=3 atteint un bon equilibre. Au-dela, le train monte a 100% sans ameliorer le test → memorisation inutile.

4. Les Metriques

Metriques de classification

Accuracy (Exactitude)

Proportion de predictions correctes sur le total. Simple mais trompeuse sur des donnees desequilibrees.

Accuracy = (VP + VN) / (VP + VN + FP + FN)

Precision

Parmi toutes les predictions positives, combien sont vraiment positives ? Importante quand les faux positifs sont couteux.

Precision = VP / (VP + FP)

Recall (Rappel)

Parmi tous les vrais positifs, combien ont ete detectes ? Important quand les faux negatifs sont couteux (ex: diagnostic medical).

Recall = VP / (VP + FN)

F1-Score

Moyenne harmonique de precision et recall. Bon compromis quand on veut equilibrer les deux.

F1 = 2 * (Precision * Recall) / (Precision + Recall)

Matrice de confusion

Tableau qui montre les predictions vs les vraies classes. Permet de voir exactement ou le modele se trompe.

Feature Importance

L'arbre de decision fournit automatiquement l'importance de chaque feature via tree.feature_importances_. Cette importance est calculee a partir de la reduction totale d'impurete (Gini ou Entropy) apportee par chaque feature dans tous les splits ou elle est utilisee.

Interpretation : Une feature avec importance 0.9 a contribue 90% de la reduction d'impurete totale. Une feature avec importance 0.0 n'a jamais ete utilisee pour un split.

export_text : lire l'arbre comme des regles

La fonction export_text de sklearn permet de visualiser l'arbre sous forme de regles if/else lisibles par un humain. C'est l'avantage majeur de l'arbre de decision : l'interpretabilite totale.

Cross-validation pour la selection de parametres

Ne jamais choisir les parametres sur le test set ! Utiliser la cross-validation (validation croisee) : diviser le train set en K parties, entrainer sur K-1, valider sur la partie restante, repeter K fois. Le score moyen est une estimation fiable de la performance reelle.

X_train, X_test, y_train, y_test = train_test_split( X, y, test_size=0.3, random_state=42, stratify=y ) print(f"\nTrain : {X_train.shape[0]} samples") print(f"Test : {X_test.shape[0]} samples")

Exercice 10 : Explication complete des resultats

Analyse detaillee des resultats

1. L'overfitting est reel : Sans contraintes, l'arbre atteint 100% en train mais seulement 95.6% en test. Avec max_depth=3 et min_samples_leaf=3, on perd 1.9% en train (98.1%) mais on gagne 2.2% en test (97.8%). C'est le principe fondamental : un modele plus simple generalise mieux.

2. Les parametres les plus impactants : max_depth est le parametre le plus puissant. Passer de None a 3 reduit la complexite de moitie (9 feuilles → 5). min_samples_leaf=3 empeche les feuilles trop specifiques.

3. Gini vs Entropy : Aucune difference mesurable sur Iris. C'est typique - les deux criteres produisent presque toujours le meme arbre. Gardez 'gini' (plus rapide a calculer).

4. Feature importance : petal_width (93.1%) est la feature dominante. Cela signifie que la largeur des petales suffit presque a elle seule pour classifier les 3 especes d'iris. sepal_length et sepal_width ne sont pas utilisees du tout.

5. Matrice de confusion : Une seule erreur : un Virginica classe comme Versicolor. Les deux especes sont proches botaniquement, ce qui explique cette confusion. Setosa est parfaitement separe (petal_width ≤ 0.80).

6. Interpretabilite : On peut expliquer la decision pour chaque prediction avec seulement 4 regles simples. C'est impossible avec la plupart des autres algorithmes de ML.

6. Erreurs Courantes

Erreur 1 : Ne pas limiter la profondeur de l'arbre

# ERREUR : arbre sans contrainte = overfitting garanti tree = DecisionTreeClassifier() # max_depth=None par defaut ! tree.fit(X_train, y_train) # Train: 100%, Test: 95.6% = overfitting

Correction : Toujours definir au minimum max_depth ou max_leaf_nodes.

# CORRECT : contraindre l'arbre tree = DecisionTreeClassifier(max_depth=3, min_samples_leaf=3) tree.fit(X_train, y_train)

Erreur 2 : Normaliser les donnees avant un arbre de decision

# INUTILE : normaliser avant un arbre ne sert a rien from sklearn.preprocessing import StandardScaler scaler = StandardScaler() X_train_scaled = scaler.fit_transform(X_train) # Inutile !

Correction : L'arbre ne calcule pas de distances. La normalisation ne change rien aux splits. Economisez cette etape.

Erreur 3 : Se fier a un seul arbre pour des donnees complexes

# RISQUE : un arbre seul est instable tree = DecisionTreeClassifier(max_depth=5) # Changer 3 samples peut donner un arbre completement different !

Correction : Pour des donnees reelles complexes, utiliser Random Forest (ensemble de nombreux arbres) qui moyennise l'instabilite.

# MIEUX : Random Forest = moyenne de 100 arbres instables from sklearn.ensemble import RandomForestClassifier rf = RandomForestClassifier(n_estimators=100, max_depth=5)

Erreur 4 : Ignorer la feature importance

# ERREUR : ne pas verifier quelles features sont utiles tree.fit(X_train, y_train) # On passe directement a la prediction sans analyser...

Correction : Toujours analyser tree.feature_importances_. Cela peut reveler que certaines features sont inutiles ou que le modele repose sur une seule feature (signe de fragilite).

# CORRECT : analyser les importances for name, imp in zip(feature_names, tree.feature_importances_): print(f"{name}: {imp:.3f}")

Erreur 5 : Utiliser max_depth trop eleve en pensant "plus = mieux"

# ERREUR : max_depth=50 sur un petit dataset tree = DecisionTreeClassifier(max_depth=50) # Sur Iris (150 samples), profondeur max reelle ~5 # max_depth=50 est equivalent a None = aucune contrainte

Correction : Adapter max_depth a la taille des donnees. Pour Iris : 2-5. Pour grands datasets : 5-20. Toujours verifier avec cross-validation.

Erreur 6 : Choisir les parametres sur le test set

# ERREUR : tester plein de parametres et garder le meilleur score test for d in range(1, 20): tree = DecisionTreeClassifier(max_depth=d) tree.fit(X_train, y_train) print(tree.score(X_test, y_test)) # Choisir le meilleur = tricher !

Correction : Utiliser GridSearchCV avec cross-validation sur le TRAIN set uniquement. Le test set ne sert qu'une seule fois, a la toute fin.

# CORRECT : GridSearchCV sur le train set grid = GridSearchCV(tree, param_grid, cv=5) grid.fit(X_train, y_train) # Evaluation finale (une seule fois !) print(grid.score(X_test, y_test))

Quand utiliser l'Arbre de Decision ?

Utiliser quand...

  • L'interpretabilite est cruciale (medecine, finance, legal)
  • Vous devez expliquer chaque decision individuellement
  • Les donnees sont de types mixtes (numeriques + categories)
  • Comme base pour comprendre avant d'utiliser Random Forest
  • Exploration rapide des donnees (quelles features sont utiles ?)
  • Petit dataset ou prototype rapide

NE PAS utiliser quand...

  • La performance prime sur l'interpretabilite → Random Forest, XGBoost
  • Le dataset est volumineux et complexe → preferer les ensembles
  • Les frontieres sont diagonales ou circulaires → SVM avec kernel, KNN
  • La stabilite est critique → Random Forest (moyenne de 100+ arbres)
  • Relations tres complexes → reseaux de neurones

Guide de choix : Arbre seul vs Ensemble ?

L'arbre de decision est rarement utilise seul en production. Mais il est indispensable de le comprendre car il est la brique fondamentale de Random Forest (TD suivant) et de XGBoost. Maitrisez l'arbre seul, et vous comprendrez naturellement ces algorithmes plus puissants.
