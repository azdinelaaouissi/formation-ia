TD Pratique : SVM (Support Vector Machine)

Algorithme supervise de classification et regression par maximisation de marge

1. Comprendre l'Algorithme

SVM (Support Vector Machine) est un algorithme supervise qui cherche l'hyperplan optimal separant les classes avec la marge maximale. Les points les plus proches de cet hyperplan, appeles vecteurs de support, determinent sa position et son orientation.

Analogie pour comprendre

Imaginez que vous devez tracer une ligne sur le sol pour separer des billes rouges et des billes bleues. Vous pourriez tracer cette ligne n'importe ou entre les deux groupes, mais SVM cherche la meilleure ligne possible : celle qui laisse le plus grand espace vide (la marge) de chaque cote. Plus cet espace est grand, plus on est confiant que de nouvelles billes seront correctement classees.

Et si les billes sont melangees et qu'aucune ligne droite ne peut les separer ? SVM utilise alors le kernel trick : il "projette" les billes dans un espace de dimension superieure ou elles deviennent separables par un plan.

Comment ca fonctionne, etape par etape

Etape 1 : Trouver l'hyperplan separateur
Dans un espace a N dimensions, un hyperplan est une surface de dimension N-1. En 2D c'est une ligne, en 3D c'est un plan. SVM cherche l'hyperplan qui separe les deux classes. L'equation est : w . x + b = 0, ou w est le vecteur normal et b le biais.
Etape 2 : Maximiser la marge
Parmi tous les hyperplans possibles, SVM choisit celui qui maximise la marge = la distance entre l'hyperplan et les points les plus proches de chaque classe. La marge vaut 2 / ||w||. Maximiser la marge revient a minimiser ||w||.
Etape 3 : Identifier les vecteurs de support
Les vecteurs de support sont les points qui se trouvent exactement sur les frontieres de la marge. Ce sont eux qui "soutiennent" l'hyperplan. Si on enlevait un de ces points, l'hyperplan changerait. Les autres points n'ont aucune influence.
Etape 4 : Le kernel trick pour les donnees non lineaires
Quand les donnees ne sont pas separables lineairement, SVM utilise une fonction noyau (kernel) pour projeter les donnees dans un espace de dimension superieure ou elles deviennent separables. L'astuce est que cette projection est faite implicitement via le calcul de produits scalaires.

Formules cles expliquees simplement

Equation de l'hyperplan : w . x + b = 0

Ou w est le vecteur de poids (direction perpendiculaire a l'hyperplan) et b le biais.


Largeur de la marge : 2 / ||w||

Maximiser la marge revient a minimiser ||w|| (la norme du vecteur de poids).


Probleme d'optimisation (soft margin) :

Minimiser : (1/2) ||w||^2 + C * somme(xi_i)

Ou xi_i est la variable de relachement (tolere quelques erreurs) et C controle le compromis marge/erreurs.


Kernel trick : K(x_i, x_j) = phi(x_i) . phi(x_j)

On calcule le produit scalaire dans l'espace transforme sans jamais calculer phi. Exemple RBF : K(x_i, x_j) = exp(-gamma * ||x_i - x_j||^2)

Avantages

  • Performant en haute dimension : fonctionne meme quand le nombre de features depasse le nombre d'exemples
  • Kernel trick : capture des frontieres non-lineaires complexes
  • Robuste au sur-apprentissage en haute dimension grace a la regularisation
  • Modele parcimonieux : la decision ne depend que des vecteurs de support
  • Theoriquement fonde : base sur la theorie de Vapnik (apprentissage statistique)

Inconvenients

  • Lent sur les gros datasets : complexite O(n^2) a O(n^3), inadapte si plus de 100k points
  • Sensible aux features non normalisees : la normalisation est OBLIGATOIRE
  • Pas de probabilites natives : probability=True est lent et approximatif
  • Choix du kernel difficile : le reglage des hyperparametres peut etre delicat
  • Pas ideal pour les donnees tres bruitees avec beaucoup de chevauchement entre classes

2. Les Donnees

La normalisation est OBLIGATOIRE

SVM calcule des distances et des produits scalaires. Si une feature a des valeurs de 0 a 1 et une autre de 0 a 100 000, la deuxieme dominera totalement. Le modele sera biaise et les performances catastrophiques.

Exemple dramatique : classification avec 2 features a echelles tres differentes.

from sklearn.svm import SVC from sklearn.preprocessing import StandardScaler from sklearn.datasets import make_classification from sklearn.model_selection import train_test_split from sklearn.metrics import accuracy_score import numpy as np # Donnees avec echelles tres differentes X, y = make_classification(n_samples=200, n_features=2, n_redundant=0, random_state=42) X[:, 0] = X[:, 0] * 1 # Feature 1 : entre -3 et 3 X[:, 1] = X[:, 1] * 10000 # Feature 2 : entre -30000 et 30000 X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42) # SANS normalisation svm_sans = SVC(kernel='rbf') svm_sans.fit(X_train, y_train) print(f"SANS normalisation : {accuracy_score(y_test, svm_sans.predict(X_test)):.2f}") # AVEC normalisation scaler = StandardScaler() X_train_sc = scaler.fit_transform(X_train) X_test_sc = scaler.transform(X_test) svm_avec = SVC(kernel='rbf') svm_avec.fit(X_train_sc, y_train) print(f"AVEC normalisation : {accuracy_score(y_test, svm_avec.predict(X_test_sc)):.2f}")
SANS normalisation : 0.62 AVEC normalisation : 0.92
La difference est enorme : 62% vs 92%. Sans normalisation, SVM ignore completement la feature 1 car ses valeurs sont negligeables par rapport a la feature 2.

SVM excelle en haute dimension

  • Classification de texte (TF-IDF, milliers de features) : SVM lineaire est souvent le meilleur choix
  • Genomique : milliers de genes pour quelques centaines de patients
  • Reconnaissance d'images (avant le deep learning) : descripteurs HOG, SIFT

Limites en taille de dataset

SVM n'est pas adapte aux tres gros datasets. Au-dela de 100 000 exemples, l'entrainement devient tres lent. Utilisez SGDClassifier(loss='hinge') ou LinearSVC qui approximent un SVM lineaire en O(n).

from sklearn.svm import SVC from sklearn.datasets import make_classification from sklearn.model_selection import train_test_split from sklearn.preprocessing import StandardScaler from sklearn.metrics import accuracy_score X, y = make_classification(n_samples=300, n_features=2, n_redundant=0, n_clusters_per_class=1, random_state=42) X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42) scaler = StandardScaler() X_train_sc = scaler.fit_transform(X_train) X_test_sc = scaler.transform(X_test) for c_val in [0.001, 0.01, 0.1, 1, 10, 100, 1000]: svm = SVC(C=c_val, kernel='rbf') svm.fit(X_train_sc, y_train) tr = accuracy_score(y_train, svm.predict(X_train_sc)) te = accuracy_score(y_test, svm.predict(X_test_sc)) n_sv = svm.n_support_.sum() print(f"C={c_val:>7} | Train: {tr:.3f} | Test: {te:.3f} | SV: {n_sv}")

4. Les Metriques

SVM etant principalement utilise pour la classification, on utilise les memes metriques que pour la regression logistique, interpretees dans le contexte des vecteurs de support.

Accuracy (exactitude)

Formule : (TP + TN) / (TP + TN + FP + FN)

Proportion de predictions correctes. Utile quand les classes sont equilibrees.

Precision

Formule : TP / (TP + FP)

Parmi les predictions positives, combien sont vraiment positives ? Important quand les faux positifs sont couteux.

Recall (rappel)

Formule : TP / (TP + FN)

Parmi les vrais positifs, combien ont ete identifies ? Important quand les faux negatifs sont dangereux.

F1-Score

Formule : 2 * (Precision * Recall) / (Precision + Recall)

Moyenne harmonique de Precision et Recall. Utile avec des classes desequilibrees.

Nombre de Support Vectors

Metrique specifique a SVM. Indique combien de points definissent la frontiere de decision.

  • Beaucoup de SV (proche de n_samples) = modele trop simple ou donnees mal separees
  • Peu de SV = modele efficient, separation claire

Matrice de confusion

Validation croisee pour la selection de modele

from sklearn.model_selection import cross_val_score svm = SVC(kernel='rbf', C=1.0, gamma='scale') scores = cross_val_score(svm, X_train_sc, y_train, cv=5, scoring='accuracy') print(f"Scores par fold : {scores}") print(f"Accuracy moyenne : {scores.mean():.3f} (+/- {scores.std():.3f})")
Scores par fold : [0.952 0.976 0.952 0.952 0.976] Accuracy moyenne : 0.962 (+/- 0.012)
La validation croisee est essentielle pour SVM car les performances dependent fortement des hyperparametres. Un faible ecart-type (0.012) indique un modele stable.

Etape 1 : Charger et preparer les donnees

On utilise le dataset Breast Cancer : 30 features numeriques pour classifier des tumeurs (benignes ou malignes).

import numpy as np from sklearn.datasets import load_breast_cancer from sklearn.model_selection import train_test_split from sklearn.preprocessing import StandardScaler # Charger les donnees data = load_breast_cancer() X, y = data.data, data.target print(f"Shape : {X.shape}") print(f"Classes : {np.unique(y)} (0=malin, 1=benin)") print(f"Distribution : {np.bincount(y)}") # Split train/test avec stratification X_train, X_test, y_train, y_test = train_test_split( X, y, test_size=0.3, random_state=42, stratify=y) # Normalisation OBLIGATOIRE scaler = StandardScaler() X_train_sc = scaler.fit_transform(X_train) X_test_sc = scaler.transform(X_test) print(f"\nAvant : mean={X_train[:, 0].mean():.1f}, std={X_train[:, 0].std():.1f}") print(f"Apres : mean={X_train_sc[:, 0].mean():.4f}, std={X_train_sc[:, 0].std():.4f}")

Etape 7 : GridSearchCV - trouver la meilleure combinaison

Pourquoi ? C, gamma et kernel interagissent. Il faut les optimiser ensemble.

6. Erreurs Courantes

Erreur 1 : Oublier la normalisation
L'erreur la plus frequente et la plus grave. SVM est base sur les distances. Sans normalisation, les features avec les plus grandes valeurs dominent. Toujours utiliser StandardScaler avant SVM.
Erreur 2 : Normaliser avant le split train/test
Si vous faites scaler.fit_transform(X) avant le split, le scaler "voit" les donnees de test = fuite d'information. Faire fit_transform sur X_train puis transform sur X_test.
# MAUVAIS : fuite d'information X_scaled = scaler.fit_transform(X) # Le scaler voit TOUT X_train, X_test = train_test_split(X_scaled) # BON : pas de fuite X_train, X_test = train_test_split(X) X_train_sc = scaler.fit_transform(X_train) # fit sur train seulement X_test_sc = scaler.transform(X_test) # transform sur test
Erreur 3 : Utiliser un gamma trop grand
Un gamma eleve donne 100% en train mais echoue en test. C'est trompeur car on croit que le modele est parfait. Utiliser gamma='scale' par defaut.
Erreur 4 : Utiliser SVM sur un dataset trop gros
SVM a une complexite O(n^2) a O(n^3). Sur 1 million de points, l'entrainement prend des jours. Pour n > 100k, utiliser LinearSVC ou SGDClassifier(loss='hinge').
Erreur 5 : Ne pas tester plusieurs kernels
Le choix du kernel est crucial. Un lineaire echouera sur des donnees non-lineaires, un RBF sera inutilement lent sur des donnees lineaires. Toujours comparer au moins 'linear' et 'rbf'.
Erreur 6 : Confondre les coefficients SVM avec l'importance des features
Les coefficients du SVM (svm.coef_ pour kernel lineaire) ne representent pas directement l'importance des variables. Utiliser permutation_importance pour mesurer l'importance des features.

Check-list avant d'utiliser SVM

  1. Normaliser les features (StandardScaler)
  2. Commencer avec un kernel lineaire (rapide, interpretable)
  3. Si insuffisant, passer au kernel RBF
  4. Utiliser GridSearchCV pour optimiser C et gamma ensemble
  5. Verifier le nombre de vecteurs de support (trop = probleme)
  6. Comparer train et test accuracy (ecart = overfitting)
  7. Si dataset trop gros, utiliser LinearSVC ou SGDClassifier
