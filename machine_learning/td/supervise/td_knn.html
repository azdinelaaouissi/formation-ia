TD Pratique : KNN (K-Nearest Neighbors)

Algorithme supervise de classification et regression

1. Recapitulatif de l'Algorithme

KNN (K-Nearest Neighbors) est un algorithme supervise qui predit la classe d'un nouveau point en regardant les K points les plus proches dans les donnees d'entrainement, puis en effectuant un vote majoritaire parmi ces voisins.

Fiche d'identite

ProprieteDetail
TypeSupervise (necessite des labels)
TacheClassification (principalement) et Regression
FamilleAlgorithmes a base d'instances (instance-based learning)
ApprentissageParesseux (lazy) - pas de phase d'entrainement, tout se fait a la prediction
HypotheseLes points proches dans l'espace des features ont des labels similaires
ComplexiteNon-parametrique (pas de forme fixe pour la frontiere de decision)

Analogie pour comprendre

Imaginez que vous arrivez dans un nouveau quartier et vous voulez savoir si c'est un quartier riche ou pauvre. Vous regardez les K maisons les plus proches de la votre :

  • Si 4 maisons sur 5 sont grandes et luxueuses => le quartier est probablement riche
  • Si 3 maisons sur 5 sont modestes => le quartier est probablement modeste

C'est exactement ce que fait KNN : il regarde les voisins les plus proches et fait un vote majoritaire.

Avantages

  • Tres simple a comprendre et implementer
  • Pas d'entrainement : on stocke juste les donnees
  • Non-parametrique : pas d'hypothese sur la forme des donnees
  • Frontiere de decision flexible : s'adapte a toute forme de distribution
  • Fonctionne bien sur des petits/moyens datasets

Inconvenients

  • Lent a la prediction : doit calculer la distance a TOUS les points d'entrainement
  • Sensible aux echelles : une feature avec de grandes valeurs domine les autres
  • Malediction de la dimensionnalite : en haute dimension, les distances perdent leur sens
  • Sensible au bruit : un point mal etiquete affecte ses voisins
  • Memoire : doit stocker TOUTES les donnees d'entrainement

2. Principe de Fonctionnement

KNN est un algorithme dit "paresseux" (lazy learning) car il ne construit aucun modele pendant l'entrainement. Il stocke simplement les donnees. Tout le travail se fait au moment de la prediction.

Etape par etape

Etape 1 - Stocker les donnees : On garde en memoire tous les points d'entrainement avec leurs labels. C'est la phase "d'entrainement" (qui n'en est pas vraiment une).
Etape 2 - Recevoir un nouveau point : Un point X_nouveau arrive, on ne connait pas sa classe. On veut la predire.
Etape 3 - Calculer les distances : On calcule la distance entre X_nouveau et CHAQUE point d'entrainement. La distance la plus courante est la distance euclidienne : d(A, B) = sqrt((a1-b1)^2 + (a2-b2)^2 + ... + (an-bn)^2)
Etape 4 - Selectionner les K voisins : On trie les distances et on garde les K points les plus proches.
Etape 5 - Vote majoritaire : Parmi ces K voisins, on regarde quelle classe apparait le plus souvent. Cette classe est la prediction.
Dans ce schema, le nouveau point (etoile verte) a 3 voisins les plus proches : 2 bleus (classe A) et 1 orange (classe B). Le vote majoritaire donne : classe A (2 contre 1). Si on avait pris K=1, seul le plus proche compterait. Si K=5, on consulterait 5 voisins.

Formules de distance

Distance Euclidienne (la plus utilisee) :

d(A, B) = sqrt( (a1 - b1)^2 + (a2 - b2)^2 + ... + (an - bn)^2 )

C'est la distance "en ligne droite" entre deux points dans l'espace.

Distance Manhattan :

d(A, B) = |a1 - b1| + |a2 - b2| + ... + |an - bn|

C'est la distance "en grille", comme si on marchait dans les rues de Manhattan (pas de diagonales).

Distance Minkowski (generalisation) :

d(A, B) = ( |a1-b1|^p + |a2-b2|^p + ... + |an-bn|^p )^(1/p)

Avec p=2 c'est Euclidienne, avec p=1 c'est Manhattan.

Preparation obligatoire : la Normalisation

KNN mesure des DISTANCES. Si une feature va de 0 a 10 000 (ex: salaire) et une autre de 0 a 1 (ex: taux), le salaire va DOMINER totalement le calcul de distance. La normalisation met toutes les features a la meme echelle. C'est OBLIGATOIRE pour KNN.

Deux methodes de normalisation :

  • StandardScaler : transforme en moyenne=0, ecart-type=1. Le plus courant.
  • MinMaxScaler : transforme entre 0 et 1. Utile si les donnees n'ont pas de distribution normale.
Erreur classique : Appliquer fit_transform sur le test. Il faut faire scaler.fit_transform(X_train) puis scaler.transform(X_test). Le scaler doit apprendre les parametres (moyenne, ecart-type) UNIQUEMENT sur le train, sinon c'est du data leakage.

Taille des donnees

  • Petit dataset (100-10 000 points) : KNN fonctionne tres bien
  • Moyen (10 000-100 000) : Commence a etre lent a la prediction
  • Grand (>100 000) : Trop lent. Preferer Random Forest ou XGBoost
  • Peu de features (<20) : Ideal pour KNN
  • Beaucoup de features (>50) : Malediction de la dimensionnalite, KNN perd en efficacite

4. Les Parametres de KNN (en detail)

KNN a peu de parametres, mais chacun est important. Voici les 4 principaux :

Parametre 1 : n_neighbors (K) - LE plus important

Role : Determine combien de voisins on consulte pour le vote. C'est le parametre qui controle la complexite du modele.

Valeur par defaut : 5

Impact :

  • K = 1 : On regarde seulement le plus proche voisin. La frontiere de decision est tres irreguliere. Le modele memorise les donnees d'entrainement (100% accuracy train). Mais il est tres sensible au bruit => OVERFITTING.
  • K petit (3-7) : Bon compromis. Le modele capture les patterns locaux tout en etant assez robuste au bruit.
  • K grand (20-50) : La frontiere de decision est tres lisse. Le modele ignore les details locaux => risque de UNDERFITTING.
  • K = N (total de points) : Tous les points votent. Le modele predit toujours la classe majoritaire (inutile).

Comment choisir K : Utiliser la validation croisee (cross-validation). Tester K de 1 a 30 et garder celui qui donne la meilleure accuracy de validation. Astuce : pour la classification binaire, choisir un K impair pour eviter les egalites.

from sklearn.neighbors import KNeighborsClassifier from sklearn.datasets import load_iris from sklearn.model_selection import train_test_split, cross_val_score from sklearn.preprocessing import StandardScaler from sklearn.metrics import classification_report, confusion_matrix import numpy as np # 1. Charger les donnees Iris (3 especes de fleurs, 4 mesures) iris = load_iris() X, y = iris.data, iris.target # 2. Separer train (70%) et test (30%) X_train, X_test, y_train, y_test = train_test_split( X, y, test_size=0.3, random_state=42 ) # 3. Normaliser les donnees (OBLIGATOIRE pour KNN) scaler = StandardScaler() X_train_scaled = scaler.fit_transform(X_train) # fit + transform sur le train X_test_scaled = scaler.transform(X_test) # transform SEULEMENT sur le test # 4. Tester differentes valeurs de K print(f"{'K':>3} | {'Acc Train':>10} | {'Acc Test':>10} | Analyse") print("-" * 55) for k in [1, 3, 5, 7, 10, 15, 20, 30, 50]: knn = KNeighborsClassifier(n_neighbors=k) knn.fit(X_train_scaled, y_train) acc_train = knn.score(X_train_scaled, y_train) acc_test = knn.score(X_test_scaled, y_test) # Diagnostic automatique if acc_train - acc_test > 0.05: diag = "OVERFITTING (train >> test)" elif acc_test < 0.90: diag = "UNDERFITTING (test trop bas)" else: diag = "BON" print(f"K={k:>2} | {acc_train:.4f} | {acc_test:.4f} | {diag}")

Lecture des resultats :
- K=1 : 100% sur le train car chaque point est son propre voisin le plus proche. Mais le test est plus bas (95.6%) => le modele a memorise au lieu d'apprendre.
- K=3 a 7 : Zone optimale ! Le train et le test sont equilibres et eleves (~97-98%).
- K=50 : Trop de voisins diluent le vote. Sur 105 points de train, consulter 50 voisins revient a demander l'avis de presque la moitie du dataset => perte de precision locale.

Parametre 2 : weights - Poids des voisins

Role : Determine si tous les voisins ont le meme poids dans le vote, ou si les voisins plus proches comptent davantage.

Valeur par defaut : 'uniform'

Les 2 options :

  • 'uniform' : Tous les K voisins ont le meme poids. C'est un vote democratique simple. Voisin a distance 0.1 = voisin a distance 10.
  • 'distance' : Les voisins proches comptent plus. Le poids est 1/distance. Un voisin a distance 0.1 a 100x plus de poids qu'un voisin a distance 10.

Quand utiliser 'distance' : Quand K est grand. Plus K est grand, plus on inclut des voisins lointains qui polluent le vote. 'distance' compense ce probleme en donnant plus de poids aux proches.

# Comparer uniform vs distance pour different K print(f"{'K':>3} | {'uniform':>10} | {'distance':>10}") print("-" * 32) for k in [3, 5, 10, 20]: scores_u = cross_val_score( KNeighborsClassifier(n_neighbors=k, weights='uniform'), X_train_scaled, y_train, cv=5 ) scores_d = cross_val_score( KNeighborsClassifier(n_neighbors=k, weights='distance'), X_train_scaled, y_train, cv=5 ) print(f"K={k:>2} | {scores_u.mean():.4f} | {scores_d.mean():.4f}")
K | uniform | distance -------------------------------- K= 3 | 0.9524 | 0.9524 K= 5 | 0.9619 | 0.9619 K=10 | 0.9524 | 0.9619 K=20 | 0.9429 | 0.9524
Pour K petit (3, 5), 'uniform' et 'distance' sont equivalents car tous les voisins sont proches. Pour K=20, 'distance' gagne (+1%) car il attenue l'influence des voisins lointains.

Parametre 3 : metric - Type de distance

Role : Definit comment on mesure la "proximite" entre deux points.

Valeur par defaut : 'minkowski' (avec p=2, ce qui equivaut a la distance euclidienne)

Les principales metriques :

MetriqueFormule intuitiveQuand l'utiliser
euclideanDistance en ligne droiteCas general. Le plus courant.
manhattanDistance en grille (pas de diagonale)Donnees en haute dimension, features independantes.
chebyshevDistance maximale sur un seul axeQuand une seule feature tres differente suffit.
minkowskiGeneralisation (p=1=manhattan, p=2=euclidean)Quand on veut tester differents p.

metric=euclidean | Accuracy: 0.9619 (+/- 0.0216) metric=manhattan | Accuracy: 0.9524 (+/- 0.0289) metric=chebyshev | Accuracy: 0.9429 (+/- 0.0338) metric=minkowski | Accuracy: 0.9619 (+/- 0.0216) # = euclidean (p=2 par defaut)
En pratique, euclidean fonctionne bien dans la majorite des cas. Ne changez la metrique que si vous avez une raison specifique (ex: manhattan est plus robuste aux outliers en haute dimension).

Parametre 4 : algorithm - Methode de recherche

Role : Determine comment l'algorithme cherche les voisins les plus proches. N'affecte PAS les resultats, seulement la vitesse.

Valeur par defaut : 'auto'

  • 'brute' : Calcule la distance a TOUS les points. Simple mais lent sur gros datasets.
  • 'kd_tree' : Structure de donnees en arbre pour accelerer la recherche. Efficace en basse dimension (<20 features).
  • 'ball_tree' : Similaire a kd_tree mais meilleur en haute dimension.
  • 'auto' : Sklearn choisit automatiquement la meilleure methode. Garder 'auto'.

5. Les Metriques d'Evaluation

KNN etant un algorithme de classification, voici les metriques a utiliser pour evaluer ses performances :

1. Accuracy (Exactitude)

Formule : Accuracy = Predictions correctes / Total des predictions

Quand l'utiliser : Quand les classes sont equilibrees (meme nombre d'exemples par classe).

Piege : Si 95% des donnees sont de classe A, un modele qui predit toujours A a 95% d'accuracy sans rien avoir appris !

2. Precision

Formule : Precision = Vrais Positifs / (Vrais Positifs + Faux Positifs)

Question a laquelle elle repond : "Parmi toutes les predictions positives, combien sont vraiment positives ?"

Importante quand : Le cout d'une fausse alarme est eleve (ex: spam detecte a tort, faux positif medical).

3. Recall (Rappel / Sensibilite)

Formule : Recall = Vrais Positifs / (Vrais Positifs + Faux Negatifs)

Question a laquelle il repond : "Parmi tous les vrais positifs, combien ont ete detectes ?"

Important quand : Le cout de rater un positif est eleve (ex: cancer non detecte, fraude non detectee).

4. F1-Score

Formule : F1 = 2 * (Precision * Recall) / (Precision + Recall)

Quand l'utiliser : Quand on veut un equilibre entre Precision et Recall. C'est la moyenne harmonique des deux.

Utile quand : Les classes sont desequilibrees.

5. Matrice de Confusion

C'est quoi : Un tableau montrant pour chaque classe : combien de predictions correctes et combien d'erreurs, et QUELLES erreurs.

Pourquoi c'est utile : Elle montre non seulement combien d'erreurs, mais quelles classes sont confondues entre elles.

6. Cross-Validation Score

C'est quoi : On decoupe le dataset en K parties (folds). On entraine sur K-1 parties et on teste sur la derniere, K fois de suite. On fait la moyenne des scores.

Pourquoi c'est indispensable : Un seul train/test split peut donner un resultat chanceux ou malchanceux. La cross-validation donne une estimation beaucoup plus fiable.

Application pratique des metriques

# Modele KNN optimal knn_best = KNeighborsClassifier(n_neighbors=5, weights='distance') knn_best.fit(X_train_scaled, y_train) y_pred = knn_best.predict(X_test_scaled) # 1. Accuracy print(f"Accuracy : {knn_best.score(X_test_scaled, y_test):.4f}") # 2. Classification Report (precision, recall, f1 pour chaque classe) print("\nRapport de Classification :") print(classification_report(y_test, y_pred, target_names=iris.target_names)) # 3. Matrice de confusion print("Matrice de Confusion :") print(confusion_matrix(y_test, y_pred)) # 4. Cross-validation (estimation plus fiable) scores_cv = cross_val_score(knn_best, X_train_scaled, y_train, cv=5) print(f"\nCross-Validation (5-fold) : {scores_cv.mean():.4f} (+/- {scores_cv.std():.4f})")

6. Implementation Complete (de A a Z)

Voici le code complet pour utiliser KNN sur un nouveau probleme, avec chaque etape commentee :

# ============================================= # PIPELINE COMPLET KNN # ============================================= # Etape 1 : Imports from sklearn.neighbors import KNeighborsClassifier from sklearn.model_selection import train_test_split, cross_val_score from sklearn.preprocessing import StandardScaler from sklearn.metrics import classification_report from sklearn.datasets import load_iris # Etape 2 : Charger les donnees iris = load_iris() X, y = iris.data, iris.target # Etape 3 : Separer train/test X_train, X_test, y_train, y_test = train_test_split( X, y, test_size=0.3, random_state=42, stratify=y ) # stratify=y : garde la meme proportion de classes dans train et test # Etape 4 : Normaliser (OBLIGATOIRE pour KNN) scaler = StandardScaler() X_train_s = scaler.fit_transform(X_train) X_test_s = scaler.transform(X_test) # Etape 5 : Trouver le meilleur K par cross-validation best_k, best_score = 1, 0 for k in range(1, 31): scores = cross_val_score( KNeighborsClassifier(n_neighbors=k, weights='distance'), X_train_s, y_train, cv=5 ) if scores.mean() > best_score: best_score = scores.mean() best_k = k print(f"Meilleur K = {best_k} (accuracy CV = {best_score:.4f})") # Etape 6 : Entrainer le modele final avec le meilleur K knn_final = KNeighborsClassifier(n_neighbors=best_k, weights='distance') knn_final.fit(X_train_s, y_train) # Etape 7 : Evaluer sur le test print(f"\nAccuracy Test : {knn_final.score(X_test_s, y_test):.4f}") print(classification_report(y_test, knn_final.predict(X_test_s), target_names=iris.target_names)) # Etape 8 : Faire des predictions sur de nouvelles donnees nouvelle_fleur = [[5.1, 3.5, 1.4, 0.2]] nouvelle_fleur_s = scaler.transform(nouvelle_fleur) prediction = knn_final.predict(nouvelle_fleur_s) print(f"Prediction pour {nouvelle_fleur[0]} : {iris.target_names[prediction[0]]}")

8. Erreurs Courantes de Debutant

Erreur 1 : Oublier la normalisation. KNN utilise des distances. Sans normalisation, les features avec de grandes valeurs dominent. Toujours normaliser avec StandardScaler.
Erreur 2 : Utiliser fit_transform sur le test. Il faut faire scaler.fit_transform(X_train) puis scaler.transform(X_test). Le scaler apprend les statistiques (moyenne, ecart-type) uniquement sur le train. Utiliser fit_transform sur le test est du data leakage (le modele "triche" en voyant les statistiques du test).
Erreur 3 : Choisir K sans cross-validation. Ne jamais choisir K "au feeling". Toujours utiliser la validation croisee pour tester systematiquement K de 1 a 30 et garder le meilleur.
Erreur 4 : Utiliser KNN sur un gros dataset. KNN calcule la distance a TOUS les points d'entrainement pour chaque prediction. Sur 1 million de points, chaque prediction parcourt 1 million de distances. C'est trop lent. Au-dela de ~50 000 points, preferer Random Forest ou XGBoost.
Erreur 5 : Trop de features (haute dimension). En haute dimension (>50 features), toutes les distances deviennent similaires. Les "voisins" ne sont plus vraiment proches. C'est la malediction de la dimensionnalite. Solution : reduire les dimensions avec PCA avant KNN.
Erreur 6 : K pair en classification binaire. Avec K pair (ex: K=4), on peut avoir 2 votes pour chaque classe => egalite. Utiliser un K impair (3, 5, 7...) pour eviter les egalites.
