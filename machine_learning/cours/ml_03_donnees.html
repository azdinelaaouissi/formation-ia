📊 Comprendre les Données

Cours 3/12 - Les données sont le carburant du ML

1. Les données = le carburant du ML

Règle n°1 du ML : Un modèle ne sera JAMAIS meilleur que ses données. Données pourries = modèle pourri. C'est la règle "Garbage In, Garbage Out" (poubelle en entrée = poubelle en sortie).

Le secret que personne ne dit : Un Data Scientist passe 80% de son temps à préparer les données, et seulement 20% sur les modèles.

3. Features et Labels

Retiens :

  • Features (X) = les informations qu'on donne au modèle (ce qu'il voit)
  • Label (Y) = ce qu'on veut prédire (la bonne réponse)
  • En apprentissage non supervisé, il n'y a PAS de label

5. Train / Test / Validation Split

Problème : Si tu testes le modèle sur les mêmes données qu'il a utilisées pour apprendre, c'est comme faire un examen avec les réponses devant toi. Le résultat ne veut rien dire.

Solution : On divise les données en 2 ou 3 groupes.

🏫 Analogie : L'école

  • Train = les exercices d'entraînement avec corrections (l'élève apprend)
  • Validation = les devoirs maison (l'élève s'ajuste)
  • Test = l'examen final (on découvre le vrai niveau)

6. Données manquantes

En pratique, les données sont rarement parfaites. Il manque souvent des valeurs.

7. Outliers (valeurs aberrantes)

Un outlier = une donnée qui est très loin des autres. Exemple : dans un dataset de salaires, tout le monde gagne 30-50k€ sauf une personne qui gagne 5 millions.

StratégieQuand
Supprimer l'outlierSi c'est une erreur de saisie (âge = 999)
Le garderSi c'est une vraie valeur (un PDG gagne vraiment beaucoup)
Le transformerAppliquer un log pour réduire l'écart

8. Normalisation et Standardisation

Le problème : Si "surface" va de 30 à 200 m² et "chambres" va de 1 à 5, le modèle va penser que la surface est plus importante juste parce que les chiffres sont plus grands. Il faut mettre toutes les features sur la même échelle.

TechniqueComment ça marcheQuand l'utiliser
Normalisation (Min-Max)Ramène toutes les valeurs entre 0 et 1Quand les données n'ont pas de distribution normale
Standardisation (Z-score)Centre à 0 avec écart-type = 1Quand les données suivent une distribution normale

9. Encodage catégoriel

Le problème : Les modèles ML ne comprennent que les NOMBRES. Comment transformer "Paris", "Lyon", "Marseille" en nombres ?

Label Encoding

Ville (texte)Ville (encodée)
Paris0
Lyon1
Marseille2

Problème : Le modèle pense que Marseille (2) > Lyon (1) > Paris (0). Mais il n'y a pas d'ordre ! Solution : One-Hot Encoding.

One-Hot Encoding

Villeest_Parisest_Lyonest_Marseille
Paris100
Lyon010
Marseille001

Chaque catégorie devient une colonne. 1 = oui, 0 = non. Pas de faux ordre. C'est la méthode la plus utilisée.

10. Feature Engineering

Feature Engineering = créer de nouvelles features à partir des existantes pour aider le modèle.

Feature originaleNouvelle feature crééePourquoi c'est utile
Date de naissanceÂgeL'âge est plus utile que la date brute
Surface + ChambresSurface par chambreIndicateur d'espace
Date d'achatJour de la semaineLes achats varient selon le jour
Texte d'un emailNombre de mots, "!" countLes spams utilisent beaucoup de "!"
Latitude + LongitudeDistance au centre-villePlus concret que des coordonnées brutes
HeureMatin/Après-midi/SoirRegrouper les heures en catégories utiles

🍳 Analogie : La cuisine

Les features originales = les ingrédients bruts (farine, oeufs, lait).

Le feature engineering = préparer les ingrédients (mélanger les oeufs, tamiser la farine). Avec des ingrédients bien préparés, le gâteau (le modèle) sera meilleur.

11. Biais dans les données

Le biais = quand les données ne représentent pas la réalité de manière équitable. Le modèle apprend alors des préjugés.

Type de biaisExempleConséquence
Biais de sélectionDataset de visages avec 90% de personnes blanchesLe modèle reconnaît mal les autres ethnies
Biais historiqueDonnées d'embauche où 80% des ingénieurs sont des hommesLe modèle pénalise les candidates femmes
Biais de survieÉtudier seulement les entreprises qui ont survécuOn ignore les facteurs d'échec
Biais de mesureThermomètre mal calibréToutes les températures sont fausses

Comment éviter les biais :

  • Diversifier les sources de données
  • Vérifier la distribution des catégories (équilibré ?)
  • Tester le modèle sur des groupes différents
  • Avoir une équipe diverse qui vérifie les résultats

12. Data Augmentation

Data Augmentation = créer de nouvelles données à partir des existantes quand on n'en a pas assez.

Type de donnéeTechniques d'augmentation
ImagesRotation, flip horizontal, zoom, changement de luminosité, recadrage
TexteSynonymes, paraphrase, traduction aller-retour, suppression de mots aléatoires
AudioChangement de vitesse, ajout de bruit, changement de pitch
TableauSMOTE (créer des exemples synthétiques pour les classes minoritaires)

📸 Analogie : Photos de chat

Tu as 100 photos de chat mais il t'en faut 1000. Tu peux prendre chaque photo et la tourner, la retourner, zoomer, changer la luminosité... Tu obtiens 10 variations de chaque photo = 1000 photos.

Le modèle voit un chat sous tous les angles et apprend mieux.

13. Checklist qualité des données

Avant de lancer un modèle ML, vérifie :

  1. Taille suffisante ? Au minimum quelques centaines d'exemples. Idéalement des milliers.
  2. Équilibré ? Si 95% classe A et 5% classe B, le modèle ignorera B.
  3. Valeurs manquantes ? Les traiter avant de lancer le modèle.
  4. Outliers ? Vérifier et décider quoi en faire.
  5. Doublons ? Supprimer les lignes identiques.
  6. Types corrects ? L'âge est bien un nombre, pas du texte "trente".
  7. Fuites de données ? Le label ne se trouve pas caché dans les features.
  8. Représentatif ? Les données représentent bien le problème réel.
