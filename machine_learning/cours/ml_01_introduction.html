1. C'est quoi le Machine Learning ?

🎯Définition en une phrase : Le Machine Learning (apprentissage automatique), c'est donner à un ordinateur la capacité d'apprendre à partir de données, sans être explicitement programmé pour chaque cas.

Programmation classique vs Machine Learning

💡En résumé :

  • Programmation classique : L'humain dit à la machine QUOI faire (si X → fais Y)
  • Machine Learning : L'humain donne des EXEMPLES et la machine apprend COMMENT faire toute seule

2. Analogie du quotidien

👶 Analogie : Comment un enfant apprend les fruits

Programmation classique : Tu donnes à l'enfant un livre de règles :

  • "Si c'est rouge et rond → c'est une pomme"
  • "Si c'est jaune et courbé → c'est une banane"
  • "Si c'est orange et rond → c'est une orange"

Problème : Et si l'enfant voit une pomme verte ? Il ne sait plus. Les règles sont trop rigides.


Machine Learning : Tu montres à l'enfant 1000 photos de fruits avec le nom :

  • Photo de pomme rouge → "pomme"
  • Photo de pomme verte → "pomme"
  • Photo de banane → "banane"

Après avoir vu suffisamment d'exemples, l'enfant peut reconnaître un fruit qu'il n'a jamais vu, même si c'est une pomme d'une couleur bizarre.

C'est exactement ça le ML ! On montre des milliers d'exemples, et la machine apprend à généraliser.

📧 Analogie : Le filtre anti-spam

Programmation classique :

  • Si le mail contient "viagra" → spam
  • Si le mail contient "vous avez gagné" → spam
  • Mais les spammeurs changent les mots : "v1agra", "V.I.A.G.R.A"... Les règles ne suivent plus.

Machine Learning :

  • On donne au ML 1 million de mails étiquetés "spam" ou "pas spam"
  • Le ML apprend des patterns invisibles que même un humain ne verrait pas
  • Il détecte les nouveaux spams automatiquement, même avec des mots qu'il n'a jamais vus

🎵 Analogie : Spotify et les recommandations

Spotify ne te demande pas "quel genre de musique tu aimes ?" (programmation classique).

Spotify observe ce que tu écoutes, ce que tu skips, à quelle heure, etc. Puis le ML découvre tout seul que :

  • "Cette personne aime le rock alternatif le matin et le jazz le soir"
  • "Les gens qui écoutent A écoutent aussi B"

Personne n'a programmé ces règles. Le ML les a découvertes dans les données.

3. IA vs ML vs Deep Learning

Ces 3 termes sont souvent confondus. Voici la différence :

ConceptDéfinitionExemples
Intelligence Artificielle Tout programme qui essaie de faire des choses "intelligentes". C'est le concept le plus large. GPS, correcteur orthographique, thermostat intelligent, robots
Machine Learning Un sous-ensemble de l'IA. Le programme apprend à partir d'exemples/données au lieu de suivre des règles écrites à la main. Filtre spam, recommandations Netflix, détection de fraude
Deep Learning Un sous-ensemble du ML. Utilise des réseaux de neurones avec beaucoup de couches. Très puissant pour les images, le texte et l'audio. ChatGPT, reconnaissance faciale, voitures autonomes, DALL-E

📌Retiens : Tout Deep Learning est du ML. Tout ML est de l'IA. Mais tout IA n'est pas du ML (un GPS utilise un algorithme classique, pas du ML).

4. Les 3 types d'apprentissage

5. Apprentissage Supervisé - En détail

👨‍🏫Principe : On donne à la machine des exemples AVEC les bonnes réponses (appelées labels). Elle apprend la relation entre les données (features) et les réponses (labels).

Les 2 sous-types

🏫 Analogie : L'école

Entraînement : C'est comme étudier pour un examen. Le prof te donne des exercices AVEC les corrections. Tu apprends les patterns.

Test : Le jour de l'examen, tu reçois des questions que tu n'as jamais vues. Si tu as bien appris, tu sais répondre.

Le ML fait pareil : il étudie les exemples (entraînement), puis on lui donne de nouvelles données qu'il n'a jamais vues (test). S'il a bien appris, il prédit correctement.

6. Apprentissage Non Supervisé - En détail

🔍Principe : On donne à la machine des données SANS les réponses. Elle doit trouver des structures cachées toute seule.

🧩 Analogie : Trier des Legos sans notice

Imagine une boîte de 1000 pièces de Lego mélangées. Personne ne te dit quoi faire.

Instinctivement, tu vas probablement les trier par couleur, ou par taille, ou par forme.

C'est exactement ce que fait l'apprentissage non supervisé : il trouve des groupes naturels dans les données, sans qu'on lui dise quels groupes chercher.

Applications concrètes

ApplicationCe que le ML trouve
Segmentation clients"Ces 3 groupes de clients ont des comportements différents"
Détection d'anomalies"Cette transaction ne ressemble à aucune autre → fraude ?"
Compression d'images"Ces pixels peuvent être résumés en moins de dimensions"
Recommandations"Ces films sont similaires, les gens qui aiment l'un aiment l'autre"

7. Apprentissage par Renforcement - En détail

🎮Principe : Un agent (le programme) interagit avec un environnement. Il reçoit des récompenses quand il fait bien et des pénalités quand il fait mal. Il apprend par essai-erreur.

🐕 Analogie : Dresser un chien

Agent : le chien

Environnement : le parc

Action : le chien s'assoit

Récompense : il reçoit une friandise (+ récompense)

Pénalité : il saute sur quelqu'un → pas de friandise (- récompense)

Après beaucoup de tentatives, le chien apprend quelles actions donnent des friandises.

C'est exactement comme ça qu'AlphaGo a appris à jouer aux échecs : en jouant des millions de parties contre lui-même, en étant "récompensé" quand il gagne.

10. Histoire du Machine Learning

1943
Le premier neurone artificiel - McCulloch et Pitts créent un modèle mathématique du neurone. L'idée que les machines peuvent "penser" naît.
1950
Alan Turing - "Les machines peuvent-elles penser ?" Le fameux Test de Turing. Début de l'IA comme discipline.
1957
Le Perceptron - Frank Rosenblatt crée le premier réseau de neurones qui apprend. Enthousiasme énorme.
1969
Premier "hiver de l'IA" - Minsky démontre les limites du Perceptron. Les financements s'arrêtent. Période sombre.
1986
Backpropagation - Rumelhart popularise la rétropropagation. Les réseaux de neurones peuvent enfin apprendre des choses complexes.
1997
Deep Blue bat Kasparov - Un ordinateur IBM bat le champion du monde d'échecs. Le monde prend conscience.
2006
Deep Learning - Geoffrey Hinton montre que les réseaux profonds (beaucoup de couches) fonctionnent. Révolution.
2012
AlexNet - Un réseau de neurones profond gagne ImageNet avec une avance énorme. Le Deep Learning explose.
2016
AlphaGo bat Lee Sedol - Un programme de DeepMind bat le champion du monde de Go (jeu bien plus complexe que les échecs). Renforcement Learning.
2017
"Attention Is All You Need" - Google publie le paper sur les Transformers. C'est la base de GPT, BERT, et de tout le NLP moderne.
2022
ChatGPT - OpenAI lance ChatGPT. L'IA générative devient grand public. Plus de 100 millions d'utilisateurs en 2 mois.
2024-26
Agents IA - Les LLMs deviennent des agents autonomes. RAG, LangGraph, multi-agents. L'IA ne fait plus que répondre, elle agit.

12. Mythes vs Réalité

MytheRéalité
"Le ML comprend comme un humain" Non. Le ML trouve des corrélations statistiques dans les données. Il ne comprend pas le sens. Il ne pense pas.
"Plus de données = toujours mieux" Pas toujours. La qualité des données est plus importante que la quantité. 1000 données propres > 1 million de données sales.
"Le ML peut tout résoudre" Non. Le ML ne peut résoudre que des problèmes où il existe des patterns dans les données. Pas de données = pas de ML.
"Les algorithmes sont magiques" Non. 80% du travail en ML c'est préparer les données. Les algorithmes sont la partie facile.
"L'IA va remplacer tout le monde" L'IA est un outil. Elle augmente les capacités humaines mais ne remplace pas la créativité, l'empathie et le jugement.
"Il faut un doctorat en maths" Utile pour la recherche, mais pour utiliser le ML, il faut surtout comprendre les concepts et savoir utiliser les outils.

⚠️La vérité que personne ne dit : En pratique, le ML c'est 80% de nettoyage de données, 10% de choix de modèle, et 10% d'entraînement. La partie "sexy" (les algorithmes) n'est que la pointe de l'iceberg.

14. ML et RAG : le lien

🔗Le RAG utilise le ML partout ! Voici comment les concepts ML s'appliquent au RAG :

Concept MLUtilisation dans le RAG
Embeddings (Deep Learning) Transformer le texte en vecteurs pour la recherche sémantique. C'est un modèle de ML !
Cosine Similarity Comparer la similarité entre la question et les documents. C'est une métrique ML !
LLM (Transformers) Le modèle qui génère la réponse. C'est un réseau de neurones profond !
Classification Router les questions (ce sujet → cette base de données). C'est de la classification !
Évaluation Mesurer la qualité du RAG (RAGAS). Ce sont des métriques ML !
Clustering Regrouper les documents similaires pour organiser la base vectorielle.

💡Conclusion : Comprendre le ML te permet de comprendre POURQUOI le RAG fonctionne, pas juste COMMENT l'utiliser. C'est la différence entre un utilisateur et un expert.
