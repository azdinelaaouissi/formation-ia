📈 La Régression

Cours 4/12 - Prédire un nombre (prix, température, ventes...)

1. C'est quoi la régression ?

La régression = prédire un nombre continu. Contrairement à la classification (chat ou chien ?), ici on prédit une VALEUR (combien ?).

QuestionRéponseC'est de la...

Quel sera le prix de cette maison ?247 500 €Régression
Ce patient a-t-il le diabète ?Oui / NonClassification

2. Régression linéaire simple

Idée : Tracer la meilleure droite qui passe au milieu des points de données. Cette droite permet ensuite de prédire de nouvelles valeurs.

📏 Analogie : La règle sur le nuage de points

Imagine que tu as plein de points sur une feuille (les données). Tu prends une règle et tu essaies de la poser de façon à ce qu'elle passe le plus près possible de tous les points.

La position de la règle = le modèle. Tu peux maintenant prolonger la règle pour prédire des valeurs que tu n'as pas encore.

3. Comment la droite est trouvée

Résumé : La droite "optimale" est celle qui minimise la somme des carrés des erreurs. C'est pour ça qu'on appelle cette méthode les "Moindres Carrés" (Least Squares).

4. Régression linéaire multiple

Simple : 1 feature → 1 prédiction (surface → prix). C'est une droite.

Multiple : Plusieurs features → 1 prédiction (surface + chambres + âge → prix). C'est un plan (ou un hyperplan en dimension supérieure).

5. Régression polynomiale

Problème : Parfois les données ne suivent pas une droite. Elles suivent une courbe. La régression polynomiale utilise une courbe au lieu d'une droite.

Attention : Plus le degré du polynôme est élevé, plus la courbe est flexible. MAIS trop flexible = overfitting (elle colle trop aux données et ne généralise pas).

6. Overfitting vs Underfitting

C'est le concept le plus important en ML !

📚 Analogie : L'étudiant

  • Underfitting = l'étudiant n'a pas révisé. Il ne connaît rien → mauvaises notes
  • Good fit = l'étudiant a compris les concepts. Il sait répondre même aux questions nouvelles → bonnes notes
  • Overfitting = l'étudiant a mémorisé les réponses par coeur. Si la question change un peu, il est perdu → bonnes notes en entraînement, mauvaises à l'examen

7. Régularisation (contre l'overfitting)

La régularisation = une technique pour empêcher le modèle d'overfitter. Elle pénalise les modèles trop complexes.

TechniqueComment ça marcheAnalogie
Ridge (L2) Réduit les paramètres mais ne les met jamais à zéro. Tous les features sont gardés mais avec moins d'impact. Comme baisser le volume de tous les instruments d'un orchestre
Lasso (L1) Peut mettre certains paramètres à ZÉRO. Élimine les features inutiles. Comme virer les musiciens qui jouent faux
ElasticNet Mélange de Ridge et Lasso. Le meilleur des deux mondes. Baisser le volume ET virer les pires

8. Les métriques de régression

Comment mesurer si notre modèle est bon ?

MétriqueSignificationIdéalAnalogie
MAE
(Mean Absolute Error)		 Erreur moyenne en valeur absolue. "En moyenne, le modèle se trompe de X." Le plus bas possible En moyenne tu arrives 5 min en retard
MSE
(Mean Squared Error)		 Erreur moyenne au carré. Pénalise plus les grosses erreurs. Le plus bas possible Les gros retards comptent beaucoup plus
RMSE
(Root MSE)		 Racine de MSE. Même unité que la donnée (€, °C...) Le plus bas possible Le plus utilisé car interprétable
R²
(R-squared)		 % de la variance expliquée. 1 = parfait. 0 = nul. Le plus proche de 1 0.85 = le modèle explique 85% des variations

9. Quand utiliser quoi ?

SituationModèle recommandé
Relation simple et linéaireRégression linéaire simple
Plusieurs featuresRégression linéaire multiple
Relation courbeRégression polynomiale
Beaucoup de features, risque d'overfittingRidge ou Lasso

Données complexes, non linéairesRandom Forest, XGBoost (cours suivants)

11. Les erreurs courantes

ErreurPourquoi c'est graveSolution
Ne pas normaliser les donnéesLes features avec de grands nombres dominentNormaliser/Standardiser avant
Ignorer les outliersUn outlier peut tirer toute la droiteDétecter et traiter les outliers
Tester sur les données d'entraînementR² sera artificiellement haut (triche)Toujours séparer train/test
Trop de featuresOverfitting, le modèle mémorise le bruitSélection de features, régularisation
Supposer que tout est linéaireLe monde réel est rarement parfaitement linéaireEssayer polynomiale ou autres modèles
