TD Pratique - Déployer un Agent en Production

Agents IA - TD 5/5 : Encapsuler un agent LangGraph dans une API FastAPI avec streaming SSE, LangSmith tracing, error handling et Docker

1. Récapitulatif : Déploiement, Observabilité, Monitoring

Construire un agent qui fonctionne en local est une chose. Le déployer en production en est une tout autre. Ce TD couvre les aspects essentiels du passage en production d'un agent IA.

Le déploiement en production d'un agent IA implique de le rendre accessible via une API, de le monitorer en continu (tracing, métriques, alertes), de gérer les erreurs gracieusement (retry, fallback, timeout), et d'optimiser les coûts (rate limiting, caching). C'est la discipline du MLOps / LLMOps.

Les piliers d'un agent en production

Pilier Description Outils
API Exposer l'agent via des endpoints REST et streaming FastAPI, SSE
Observabilité Tracer chaque appel LLM, outil, et décision LangSmith, OpenTelemetry
Résilience Gérer les erreurs, timeouts, et dégradation Retry, Fallback, Circuit Breaker
Performance Optimiser la latence et le débit Streaming, Cache, Async
Coût Contrôler la consommation de tokens et d'API calls Rate Limiting, Budget Alerts
Conteneurisation Packager pour un déploiement reproductible Docker, docker-compose
En entreprise, un agent qui fonctionne en notebook Jupyter est un prototype. Pour qu'il ait de la valeur, il doit être déployé, monitoré et maintenu. Ce TD vous donne les bases pour faire ce saut.
Ne déployez JAMAIS un agent avec des clés API en dur dans le code. Utilisez toujours des variables d'environnement ou un gestionnaire de secrets (AWS Secrets Manager, HashiCorp Vault, etc.).

Args: query: Requête de recherche. """ try: response = tavily.search(query=query, max_results=3, include_answer=True) results = [] if response.get("answer"): results.append(f"Réponse: {response['answer']}") for r in response["results"]: results.append(f"{r['title']}: {r['content'][:200]}") return "\n".join(results) except Exception as e: return f"Erreur recherche: {e}" @tool def calculator(expression: str) -> str: """Calcul mathématique sécurisé. Args: expression: Expression math.""" import ast, operator ops = {ast.Add: operator.add, ast.Sub: operator.sub, ast.Mult: operator.mul, ast.Div: operator.truediv, ast.Pow: operator.pow} def _eval(node): if isinstance(node, ast.Constant): return node.value if isinstance(node, ast.BinOp): return ops[type(node.op)](_eval(node.left), _eval(node.right)) if isinstance(node, ast.UnaryOp) and isinstance(node.op, ast.USub): return -_eval(node.operand) raise ValueError("Expression non supportée") try: return str(_eval(ast.parse(expression, mode="eval").body)) except Exception as e: return f"Erreur calcul: {e}" @tool def get_current_time(query: str = "now") -> str: """Obtenir la date et l'heure actuelles. Args: query: Inutilisé.""" now = datetime.now() return f"Date: {now.strftime('%d/%m/%Y %H:%M:%S')}" ALL_TOOLS = [web_search, calculator, get_current_time]
Étape 3.4 : Créer l'agent LangGraph.
# app/agent.py from langchain_openai import ChatOpenAI from langgraph.prebuilt import create_react_agent from langchain_core.messages import HumanMessage from app.config import settings from app.tools import ALL_TOOLS SYSTEM_PROMPT = """Tu es un assistant IA professionnel avec accès à des outils. Réponds en français de manière claire et structurée. Utilise les outils quand nécessaire pour fournir des informations précises.""" # Créer le modèle llm = ChatOpenAI( model=settings.model_name, temperature=settings.temperature, max_tokens=settings.max_tokens, api_key=settings.openai_api_key, ) # Créer l'agent agent = create_react_agent( model=llm, tools=ALL_TOOLS, state_modifier=SYSTEM_PROMPT, ) def invoke_agent(question: str) -> str: """Invoquer l'agent de manière synchrone.""" inputs = {"messages": [HumanMessage(content=question)]} result = agent.invoke(inputs) return result["messages"][-1].content async def stream_agent(question: str): """Invoquer l'agent avec streaming.""" inputs = {"messages": [HumanMessage(content=question)]} async for event in agent.astream_events(inputs, version="v2"): kind = event["event"] if kind == "on_chat_model_stream": content = event["data"]["chunk"].content if content: yield content
Notre agent est construit avec des composants séparés et proprement structurés. La fonction stream_agent utilise astream_events pour le streaming en temps réel vers le client.

4. Exercice : Wrapper FastAPI avec endpoints REST et streaming SSE

Encapsulons notre agent dans une API FastAPI avec un endpoint synchrone et un endpoint streaming SSE (Server-Sent Events).

Étape 4.1 : Créer l'application FastAPI avec les modèles de requête/réponse.
# app/main.py import time import uuid import asyncio import logging from fastapi import FastAPI, HTTPException from fastapi.responses import StreamingResponse from fastapi.middleware.cors import CORSMiddleware from pydantic import BaseModel, Field from typing import Optional from app.agent import invoke_agent, stream_agent from app.config import settings # Logger logging.basicConfig(level=logging.INFO) logger = logging.getLogger("agent-api") # App FastAPI app = FastAPI( title="Agent IA API", description="API pour interagir avec un agent IA ReAct", version="1.0.0", ) # CORS app.add_middleware( CORSMiddleware, allow_origins=["*"], allow_methods=["*"], allow_headers=["*"], ) # Modèles Pydantic class ChatRequest(BaseModel): question: str = Field(..., min_length=1, max_length=2000) session_id: Optional[str] = None class ChatResponse(BaseModel): answer: str session_id: str processing_time: float tokens_used: Optional[int] = None
Étape 4.2 : Créer les endpoints synchrone et streaming.
# Endpoint synchrone @app.post("/chat", response_model=ChatResponse) async def chat(request: ChatRequest): """Endpoint synchrone : attend la réponse complète avant de répondre.""" session_id = request.session_id or str(uuid.uuid4()) start = time.time() logger.info(f"[{session_id}] Question: {request.question[:80]}...") try: # Timeout avec asyncio answer = await asyncio.wait_for( asyncio.to_thread(invoke_agent, request.question), timeout=settings.agent_timeout ) elapsed = time.time() - start logger.info(f"[{session_id}] Réponse en {elapsed:.1f}s") return ChatResponse( answer=answer, session_id=session_id, processing_time=round(elapsed, 2), ) except asyncio.TimeoutError: logger.error(f"[{session_id}] Timeout après {settings.agent_timeout}s") raise HTTPException( status_code=504, detail="L'agent a mis trop de temps à répondre." ) except Exception as e: logger.error(f"[{session_id}] Erreur: {str(e)}") raise HTTPException(status_code=500, detail=str(e)) # Endpoint streaming SSE @app.post("/chat/stream") async def chat_stream(request: ChatRequest): """Endpoint streaming : envoie la réponse token par token via SSE.""" session_id = request.session_id or str(uuid.uuid4()) logger.info(f"[{session_id}] Stream: {request.question[:80]}...") async def event_generator(): try: async for chunk in stream_agent(request.question): # Format SSE yield f"data: {chunk}\n\n" yield "data: [DONE]\n\n" except Exception as e: yield f"data: [ERROR] {str(e)}\n\n" return StreamingResponse( event_generator(), media_type="text/event-stream", headers={ "Cache-Control": "no-cache", "Connection": "keep-alive", } ) # Health check @app.get("/health") async def health(): return { "status": "healthy", "model": settings.model_name, "tools": ["web_search", "calculator", "get_current_time"], }

6. Exercice : Error Handling Robuste (Retry, Fallback, Timeout)

En production, les erreurs sont inévitables : API indisponible, timeout, rate limit atteint. Notre agent doit les gérer gracieusement.

Étape 6.1 : Implémenter un décorateur de retry avec backoff exponentiel.
# app/resilience.py import time import asyncio import logging from functools import wraps from typing import Callable, Type, Tuple logger = logging.getLogger("agent-resilience") def retry_with_backoff( max_retries: int = 3, base_delay: float = 1.0, max_delay: float = 30.0, exceptions: Tuple[Type[Exception], ...] = (Exception,), ): """Décorateur de retry avec backoff exponentiel.""" def decorator(func: Callable): @wraps(func) def wrapper(*args, **kwargs): last_exception = None for attempt in range(max_retries + 1): try: return func(*args, **kwargs) except exceptions as e: last_exception = e if attempt < max_retries: delay = min( base_delay * (2 ** attempt), max_delay ) logger.warning( f"Retry {attempt+1}/{max_retries} " f"après {delay:.1f}s - Erreur: {e}" ) time.sleep(delay) else: logger.error( f"Max retries atteint ({max_retries}). " f"Dernière erreur: {e}" ) raise last_exception return wrapper return decorator

# Fallback : si GPT-4o échoue, basculer sur GPT-4o-mini from langchain_openai import ChatOpenAI from langchain_core.runnables import RunnableWithFallbacks # Modèle principal primary_llm = ChatOpenAI( model="gpt-4o", temperature=0, max_retries=2, request_timeout=30, ) # Modèle de fallback (moins cher, plus rapide) fallback_llm = ChatOpenAI( model="gpt-4o-mini", temperature=0, max_retries=2, request_timeout=15, ) # LLM avec fallback automatique llm_with_fallback = primary_llm.with_fallbacks([fallback_llm]) print("LLM avec fallback configuré :") print(" Primary: gpt-4o (timeout: 30s)") print(" Fallback: gpt-4o-mini (timeout: 15s)")

Étape 6.3 : Wrapper complet avec tous les mécanismes de résilience.
import traceback class AgentError(Exception): """Erreur personnalisée de l'agent.""" def __init__(self, message: str, error_type: str, retry_count: int = 0): self.message = message self.error_type = error_type self.retry_count = retry_count super().__init__(message) @retry_with_backoff(max_retries=2, base_delay=1.0) def safe_invoke_agent(question: str, session_id: str) -> dict: """Invoquer l'agent avec tous les mécanismes de sécurité.""" start = time.time() try: # Validation de l'entrée if len(question) > 2000: raise AgentError("Question trop longue (max 2000 car.)", "validation") if not question.strip(): raise AgentError("Question vide", "validation") # Invoquer l'agent answer = invoke_agent(question) elapsed = time.time() - start return { "answer": answer, "processing_time": round(elapsed, 2), "status": "success", "model_used": settings.model_name, } except AgentError: raise # Pas de retry pour les erreurs de validation except Exception as e: logger.error(f"[{session_id}] Erreur agent: {traceback.format_exc()}") raise
Ne jamais renvoyer des stack traces brutes au client. Loguez les erreurs détaillées côté serveur et renvoyez un message d'erreur générique au client. Les stack traces peuvent révéler des informations sensibles sur l'architecture.

7. Exercice : Rate Limiting et Gestion des Coûts

En production, il faut contrôler le nombre de requêtes et les coûts associés pour éviter les factures surprises.

Étape 7.1 : Implémenter un rate limiter simple en mémoire.
# app/middleware.py import time from collections import defaultdict from fastapi import Request, HTTPException from starlette.middleware.base import BaseHTTPMiddleware class RateLimiter: """Rate limiter simple basé sur un token bucket.""" def __init__(self, max_requests: int = 20, window_seconds: int = 60): self.max_requests = max_requests self.window_seconds = window_seconds self.requests = defaultdict(list) # IP -> [timestamps] def is_allowed(self, client_ip: str) -> bool: """Vérifie si la requête est autorisée.""" now = time.time() window_start = now - self.window_seconds # Nettoyer les anciennes requêtes self.requests[client_ip] = [ t for t in self.requests[client_ip] if t > window_start ] # Vérifier la limite if len(self.requests[client_ip]) >= self.max_requests: return False self.requests[client_ip].append(now) return True def get_remaining(self, client_ip: str) -> int: """Retourne le nombre de requêtes restantes.""" now = time.time() window_start = now - self.window_seconds recent = [t for t in self.requests[client_ip] if t > window_start] return max(0, self.max_requests - len(recent)) # Instancier le rate limiter rate_limiter = RateLimiter( max_requests=settings.rate_limit_per_minute, window_seconds=60 ) class RateLimitMiddleware(BaseHTTPMiddleware): async def dispatch(self, request: Request, call_next): # Ignorer le health check if request.url.path == "/health": return await call_next(request) client_ip = request.client.host if not rate_limiter.is_allowed(client_ip): raise HTTPException( status_code=429, detail="Trop de requêtes. Réessayez dans une minute.", headers={"Retry-After": "60"} ) response = await call_next(request) # Ajouter les headers de rate limit remaining = rate_limiter.get_remaining(client_ip) response.headers["X-RateLimit-Limit"] = str(settings.rate_limit_per_minute) response.headers["X-RateLimit-Remaining"] = str(remaining) return response

Étape 7.3 : Implémenter un cost tracker pour surveiller les dépenses.
# app/cost_tracker.py import json import logging from datetime import datetime, timedelta from collections import defaultdict logger = logging.getLogger("cost-tracker") # Coûts par 1M tokens (mars 2025) TOKEN_COSTS = { "gpt-4o": {"input": 2.50, "output": 10.00}, "gpt-4o-mini": {"input": 0.15, "output": 0.60}, } class CostTracker: """Suit les coûts d'utilisation des API LLM.""" def __init__(self, daily_budget: float = 10.0): self.daily_budget = daily_budget self.daily_costs = defaultdict(float) # date -> cost self.total_requests = 0 self.total_tokens = {"input": 0, "output": 0} def track_usage(self, model: str, input_tokens: int, output_tokens: int): """Enregistre un usage et calcule le coût.""" costs = TOKEN_COSTS.get(model, TOKEN_COSTS["gpt-4o"]) cost = ( (input_tokens / 1_000_000) * costs["input"] + (output_tokens / 1_000_000) * costs["output"] ) today = datetime.now().strftime("%Y-%m-%d") self.daily_costs[today] += cost self.total_requests += 1 self.total_tokens["input"] += input_tokens self.total_tokens["output"] += output_tokens # Alerte si on approche du budget daily_total = self.daily_costs[today] if daily_total > self.daily_budget * 0.8: logger.warning( f"ALERTE BUDGET: {daily_total:.4f}$ / {self.daily_budget}$ " f"({daily_total/self.daily_budget*100:.0f}%)" ) return cost def is_budget_exceeded(self) -> bool: """Vérifie si le budget quotidien est dépassé.""" today = datetime.now().strftime("%Y-%m-%d") return self.daily_costs[today] >= self.daily_budget def get_stats(self) -> dict: """Retourne les statistiques d'utilisation.""" today = datetime.now().strftime("%Y-%m-%d") return { "daily_cost": round(self.daily_costs[today], 4), "daily_budget": self.daily_budget, "total_requests": self.total_requests, "total_input_tokens": self.total_tokens["input"], "total_output_tokens": self.total_tokens["output"], } cost_tracker = CostTracker(daily_budget=10.0)
Le cost tracker surveille en temps réel les dépenses API. S'il détecte que 80% du budget quotidien est atteint, il émet une alerte. À 100%, l'agent peut être automatiquement mis en pause.

9. Points clés à retenir + Checklist Production

Résumé du TD - Agent en Production

  • FastAPI est le framework idéal pour encapsuler un agent IA : async natif, SSE streaming, validation Pydantic, documentation auto (Swagger).
  • Le streaming SSE améliore drastiquement l'UX : l'utilisateur voit la réponse se construire en temps réel au lieu d'attendre.
  • LangSmith est indispensable pour l'observabilité : chaque appel LLM, chaque outil, chaque décision est tracée et visualisable.
  • Le retry avec backoff exponentiel gère les erreurs transitoires (réseau, rate limit API). Le fallback model gère les pannes persistantes.
  • Le rate limiting protège à la fois l'API et le budget. Un rate limiter en mémoire suffit pour un service unique ; utilisez Redis pour un cluster.
  • Le cost tracking est essentiel : sans surveillance, un agent en production peut générer des factures énormes en cas de boucle ou d'abus.
  • Docker garantit la reproductibilité du déploiement. Le multi-stage build et le healthcheck sont des bonnes pratiques essentielles.
  • Les secrets (clés API) ne doivent JAMAIS être dans le code ou l'image Docker. Utilisez des variables d'environnement ou un gestionnaire de secrets.
