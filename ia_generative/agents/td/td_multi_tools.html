TD Pratique - Agent avec Outils Multiples

Agents IA - TD 2/5 : Concevoir et intégrer 5+ outils spécialisés dans un agent intelligent avec routing automatique

1. Récapitulatif : Tools, Function Calling et @tool

Dans le TD précédent, nous avons construit un agent ReAct avec 3 outils simples. Maintenant, nous allons créer un agent robuste avec 5 outils spécialisés et un système de routing intelligent.

Un Tool (outil) dans le contexte des agents IA est une fonction exécutable que le LLM peut appeler pour interagir avec le monde extérieur. Chaque outil possède un nom, une description (utilisée par le LLM pour décider quand l'utiliser), un schéma d'entrée, et une fonction d'exécution.

Function Calling vs Parsing Texte

Il existe deux approches pour connecter un LLM à des outils :

Approche Mécanisme Avantages Inconvénients
Parsing texte (TD 1) Le LLM écrit "Action: tool_name" et on parse avec regex Fonctionne avec tout LLM Fragile, pas de validation de schéma
Function Calling Le LLM retourne un JSON structuré avec le nom de l'outil et les arguments typés Fiable, typé, appels parallèles possibles Nécessite un LLM compatible (GPT-4, Claude, etc.)

Le décorateur @tool de LangChain

LangChain fournit le décorateur @tool qui transforme automatiquement une fonction Python en outil compatible avec le function calling :

from langchain_core.tools import tool @tool def mon_outil(param1: str, param2: int = 10) -> str: """Description de l'outil utilisée par le LLM pour décider quand l'appeler. Args: param1: Description du premier paramètre. param2: Description du second paramètre (défaut: 10). """ return f"Résultat avec {param1} et {param2}"
La docstring est CRUCIALE : c'est elle que le LLM lit pour décider quand et comment utiliser l'outil. Une mauvaise description mène à un mauvais routing. Soyez précis, concis, et donnez des exemples si nécessaire.

Les 5 outils que nous allons construire

  1. Web Search Tool : Recherche d'informations actuelles via Tavily
  2. Calculator Tool : Évaluation sécurisée d'expressions mathématiques
  3. Python Executor Tool : Exécution de code Python dans un sandbox
  4. File Reader Tool : Lecture et analyse de fichiers documents
  5. API Caller Tool : Appels REST API génériques

2. Architecture : Agent Hub avec Tool Spokes

Notre agent fonctionne comme un hub central qui dispatche les requêtes vers les outils appropriés. Le LLM joue le rôle de routeur intelligent.

Le LLM agit comme un routeur intelligent : il analyse la question, détermine quels outils sont nécessaires, et les appelle dans le bon ordre. Avec le function calling, il peut même appeler plusieurs outils en parallèle dans une seule itération.

4. Exercice : Calculator Tool (Safe Math Evaluation)

Notre calculatrice doit pouvoir gérer des expressions mathématiques complexes tout en restant sécurisée contre l'injection de code.

Étape 4.1 : Implémenter une calculatrice avancée avec support des fonctions mathématiques.
import ast import math import operator # Opérateurs binaires autorisés SAFE_OPERATORS = { ast.Add: operator.add, ast.Sub: operator.sub, ast.Mult: operator.mul, ast.Div: operator.truediv, ast.FloorDiv: operator.floordiv, ast.Pow: operator.pow, ast.Mod: operator.mod, } # Opérateurs unaires autorisés SAFE_UNARY = { ast.USub: operator.neg, ast.UAdd: operator.pos, } # Fonctions mathématiques autorisées SAFE_FUNCTIONS = { "sqrt": math.sqrt, "abs": abs, "round": round, "log": math.log, "log10": math.log10, "sin": math.sin, "cos": math.cos, "tan": math.tan, "pi": math.pi, "e": math.e, } def safe_eval_node(node): """Évalue récursivement un noeud AST de manière sécurisée.""" if isinstance(node, ast.Constant): if isinstance(node.value, (int, float)): return node.value raise ValueError(f"Type non autorisé: {type(node.value)}") elif isinstance(node, ast.BinOp): left = safe_eval_node(node.left) right = safe_eval_node(node.right) op = SAFE_OPERATORS.get(type(node.op)) if op is None: raise ValueError(f"Opérateur non autorisé") return op(left, right) elif isinstance(node, ast.UnaryOp): operand = safe_eval_node(node.operand) op = SAFE_UNARY.get(type(node.op)) if op is None: raise ValueError(f"Opérateur unaire non autorisé") return op(operand) elif isinstance(node, ast.Call): # Appel de fonction (sqrt, log, etc.) func_name = node.func.id if isinstance(node.func, ast.Name) else None if func_name not in SAFE_FUNCTIONS: raise ValueError(f"Fonction non autorisée: {func_name}") func = SAFE_FUNCTIONS[func_name] if callable(func): args = [safe_eval_node(a) for a in node.args] return func(*args) return func # Pour les constantes comme pi, e elif isinstance(node, ast.Name): # Variable (pi, e) if node.id in SAFE_FUNCTIONS: return SAFE_FUNCTIONS[node.id] raise ValueError(f"Variable non autorisée: {node.id}") raise ValueError(f"Expression non autorisée: {type(node)}") @tool def calculator(expression: str) -> str: """Effectuer un calcul mathématique de manière sécurisée. Supporte : +, -, *, /, **, %, sqrt(), log(), sin(), cos(), tan(), pi, e. Exemples: '25 * 4 + 10', 'sqrt(144)', '2 ** 10', 'pi * 5 ** 2'. Args: expression: L'expression mathématique à évaluer. """ try: tree = ast.parse(expression, mode="eval") result = safe_eval_node(tree.body) # Formater le résultat if isinstance(result, float) and result == int(result): result = int(result) return f"{expression} = {result}" except Exception as e: return f"Erreur de calcul pour '{expression}': {str(e)}"
Étape 4.2 : Tester la calculatrice avec différentes expressions.
# Tests variés tests = [ "25 * 4 + 10", "sqrt(144)", "2 ** 10", "pi * 5 ** 2", "log10(1000)", "sin(pi / 2)", ] for expr in tests: result = calculator.invoke({"expression": expr}) print(result)
25 * 4 + 10 = 110 sqrt(144) = 12 2 ** 10 = 1024 pi * 5 ** 2 = 78.53981633974483 log10(1000) = 3 sin(pi / 2) = 1.0
Notre calculatrice refuse automatiquement les appels dangereux comme __import__('os').system('rm -rf /'). Seules les opérations mathématiques pures sont autorisées grâce à la validation AST.

5. Exercice : Python Executor Tool (Sandboxed)

Parfois, la calculatrice ne suffit pas et l'agent a besoin d'exécuter du code Python arbitraire. Nous allons créer un environnement d'exécution sécurisé.

Étape 5.1 : Implémenter un exécuteur Python sandboxé.
import sys import io import signal import traceback class TimeoutError(Exception): pass def timeout_handler(signum, frame): raise TimeoutError("Exécution interrompue: timeout de 10 secondes dépassé") @tool def python_executor(code: str) -> str: """Exécuter du code Python dans un environnement sécurisé. Utilise cet outil pour des calculs complexes, du traitement de données, ou des opérations qui ne peuvent pas être faites avec la calculatrice simple. Le code est exécuté dans un sandbox avec un timeout de 10 secondes. Les modules autorisés: math, statistics, collections, itertools, json, re. Args: code: Le code Python à exécuter. Utilise print() pour afficher les résultats. """ # Modules autorisés dans le sandbox safe_builtins = { "print": print, "range": range, "len": len, "int": int, "float": float, "str": str, "list": list, "dict": dict, "tuple": tuple, "set": set, "sorted": sorted, "enumerate": enumerate, "zip": zip, "map": map, "filter": filter, "sum": sum, "min": min, "max": max, "abs": abs, "round": round, "isinstance": isinstance, "True": True, "False": False, "None": None, } # Modules autorisés pour import safe_modules = { "math": math, "statistics": __import__("statistics"), "collections": __import__("collections"), "itertools": __import__("itertools"), "json": __import__("json"), "re": __import__("re"), } # Capturer stdout old_stdout = sys.stdout sys.stdout = captured = io.StringIO() try: # Préparer l'environnement d'exécution exec_globals = {"__builtins__": safe_builtins} exec_globals.update(safe_modules) # Exécuter avec timeout (Linux/Mac seulement) try: signal.signal(signal.SIGALRM, timeout_handler) signal.alarm(10) # 10 secondes max except (AttributeError, ValueError): pass # Windows n'a pas SIGALRM exec(code, exec_globals) try: signal.alarm(0) # Désactiver le timeout except (AttributeError, ValueError): pass output = captured.getvalue() return output if output else "Code exécuté avec succès (pas de sortie print)." except Exception as e: return f"Erreur d'exécution:\n{traceback.format_exc()}" finally: sys.stdout = old_stdout
Étape 5.2 : Tester le Python Executor avec du code complexe.
# Test : calcul statistique code_test = """ données = [23, 45, 67, 89, 12, 34, 56, 78, 90, 11] moyenne = sum(données) / len(données) données_triées = sorted(données) n = len(données_triées) if n % 2 == 0: médiane = (données_triées[n//2 - 1] + données_triées[n//2]) / 2 else: médiane = données_triées[n//2] print(f"Données: {données}") print(f"Moyenne: {moyenne}") print(f"Médiane: {médiane}") print(f"Min: {min(données)}, Max: {max(données)}") print(f"Étendue: {max(données) - min(données)}") """ result = python_executor.invoke({"code": code_test}) print(result)
Données: [23, 45, 67, 89, 12, 34, 56, 78, 90, 11] Moyenne: 50.5 Médiane: 50.5 Min: 11, Max: 90 Étendue: 79
Le Python Executor permet à l'agent de résoudre des problèmes arbitrairement complexes. Il est sandboxé : pas d'accès au système de fichiers, pas d'import de modules dangereux, et un timeout de 10 secondes.

6. Exercice : File Reader Tool

Cet outil permet à l'agent de lire et analyser des fichiers texte, CSV, JSON, etc. C'est essentiel pour les tâches d'analyse de données.

Étape 6.1 : Implémenter le File Reader avec support multi-format.
import csv import json from pathlib import Path # Répertoire autorisé pour la lecture de fichiers ALLOWED_DIR = Path("./workspace").resolve() @tool def file_reader(file_path: str, max_lines: int = 50) -> str: """Lire le contenu d'un fichier texte, CSV ou JSON. Utilise cet outil quand tu dois analyser le contenu d'un fichier. Les fichiers doivent être dans le répertoire workspace/. Args: file_path: Chemin du fichier relatif au workspace (ex: 'data.csv'). max_lines: Nombre maximum de lignes à lire (défaut: 50). """ try: # Sécurité : vérifier que le chemin est dans le répertoire autorisé full_path = (ALLOWED_DIR / file_path).resolve() if not str(full_path).startswith(str(ALLOWED_DIR)): return "Erreur: Accès refusé. Le fichier doit être dans workspace/" if not full_path.exists(): return f"Erreur: Fichier '{file_path}' non trouvé dans workspace/" suffix = full_path.suffix.lower() # Lecture CSV if suffix == ".csv": with open(full_path, "r", encoding="utf-8") as f: reader = csv.DictReader(f) rows = [] for i, row in enumerate(reader): if i >= max_lines: break rows.append(dict(row)) header = f"Fichier CSV: {file_path}\n" header += f"Colonnes: {list(rows[0].keys()) if rows else 'vide'}\n" header += f"Lignes affichées: {len(rows)}\n\n" for row in rows: header += " | ".join(f"{k}: {v}" for k, v in row.items()) + "\n" return header # Lecture JSON elif suffix == ".json": with open(full_path, "r", encoding="utf-8") as f: data = json.load(f) formatted = json.dumps(data, indent=2, ensure_ascii=False) lines = formatted.split("\n") if len(lines) > max_lines: formatted = "\n".join(lines[:max_lines]) + "\n... (tronqué)" return f"Fichier JSON: {file_path}\n\n{formatted}" # Lecture texte générique else: with open(full_path, "r", encoding="utf-8") as f: lines = f.readlines()[:max_lines] content = "".join(lines) return f"Fichier: {file_path} ({len(lines)} lignes)\n\n{content}" except Exception as e: return f"Erreur lors de la lecture: {str(e)}"
La sécurité est primordiale pour les outils d'accès fichier. Notre implémentation vérifie que le chemin demandé reste dans le répertoire workspace/ autorisé, empêchant les attaques par traversée de répertoire (../../../etc/passwd).

7. Exercice : API Caller Tool (REST API)

Cet outil générique permet à l'agent d'appeler n'importe quelle API REST publique. C'est un outil puissant pour intégrer des services externes.

Étape 7.1 : Implémenter l'API Caller avec support GET et POST.
import requests from urllib.parse import urlparse # Domaines autorisés (whitelist de sécurité) ALLOWED_DOMAINS = [ "api.open-meteo.com", "jsonplaceholder.typicode.com", "api.exchangerate-api.com", "api.github.com", "restcountries.com", ] @tool def api_caller( url: str, method: str = "GET", body: str = "" ) -> str: """Effectuer un appel API REST vers un service externe. Domaines autorisés: open-meteo.com, jsonplaceholder.typicode.com, exchangerate-api.com, api.github.com, restcountries.com. Args: url: L'URL complète de l'API à appeler. method: Méthode HTTP (GET ou POST, défaut: GET). body: Corps de la requête en JSON pour les POST (optionnel). """ try: # Vérification du domaine parsed = urlparse(url) if parsed.hostname not in ALLOWED_DOMAINS: return ( f"Erreur: Domaine '{parsed.hostname}' non autorisé.\n" f"Domaines autorisés: {', '.join(ALLOWED_DOMAINS)}" ) # Exécuter la requête headers = {"Content-Type": "application/json"} if method.upper() == "GET": resp = requests.get(url, headers=headers, timeout=10) elif method.upper() == "POST": resp = requests.post( url, headers=headers, data=body if body else None, timeout=10 ) else: return f"Erreur: Méthode '{method}' non supportée. Utiliser GET ou POST." # Formater la réponse if resp.status_code == 200: try: data = resp.json() formatted = json.dumps(data, indent=2, ensure_ascii=False) # Tronquer si trop long if len(formatted) > 2000: formatted = formatted[:2000] + "\n... (tronqué)" return f"Status: {resp.status_code}\n\n{formatted}" except: return f"Status: {resp.status_code}\n\n{resp.text[:2000]}" else: return f"Erreur HTTP {resp.status_code}: {resp.text[:500]}" except requests.Timeout: return "Erreur: Timeout (l'API n'a pas répondu en 10 secondes)" except Exception as e: return f"Erreur lors de l'appel API: {str(e)}"

8. Exercice : Assembler l'agent complet avec routing intelligent

Assemblons maintenant tous nos outils dans un agent LangGraph complet avec gestion d'erreurs robuste.

Étape 8.1 : Créer l'agent multi-tools avec LangGraph.
from langchain_openai import ChatOpenAI from langgraph.prebuilt import create_react_agent from langchain_core.messages import HumanMessage, SystemMessage # Rassembler tous les outils all_tools = [web_search, calculator, python_executor, file_reader, api_caller] # System prompt spécialisé pour le routing SYSTEM_PROMPT = """Tu es un assistant IA expert avec accès à 5 outils spécialisés. ## Stratégie de sélection d'outils : 1. **web_search** : Pour les informations actuelles, actualités, faits récents 2. **calculator** : Pour les calculs mathématiques simples (expressions) 3. **python_executor** : Pour les calculs complexes, statistiques, traitement de données 4. **file_reader** : Pour lire et analyser des fichiers dans le workspace/ 5. **api_caller** : Pour obtenir des données en temps réel (météo, devises, etc.) ## Règles : - Utilise le bon outil pour chaque sous-tâche - Tu peux combiner plusieurs outils dans une même réponse - Vérifie toujours les résultats avant de les présenter - Réponds en français de manière structurée et claire """ # Créer le modèle et l'agent llm = ChatOpenAI(model="gpt-4o", temperature=0) agent = create_react_agent( model=llm, tools=all_tools, state_modifier=SYSTEM_PROMPT, ) print(f"Agent créé avec {len(all_tools)} outils :") for t in all_tools: print(f" - {t.name}: {t.description[:60]}...")

Étape 8.2 : Ajouter un wrapper avec error handling et logging.
import time def run_agent(question: str, verbose: bool = True) -> str: """Exécuter l'agent avec logging et error handling.""" start = time.time() try: inputs = { "messages": [HumanMessage(content=question)] } if verbose: print(f"\n{'='*60}") print(f"Question: {question}") print(f"{'='*60}") # Stream pour voir les étapes final_response = "" tool_calls_count = 0 for event in agent.stream(inputs, stream_mode="values"): last_msg = event["messages"][-1] if hasattr(last_msg, "tool_calls") and last_msg.tool_calls: for tc in last_msg.tool_calls: tool_calls_count += 1 if verbose: print(f" [{tool_calls_count}] {tc['name']}({str(tc['args'])[:80]}...)") if last_msg.__class__.__name__ == "AIMessage" and last_msg.content: if not (hasattr(last_msg, "tool_calls") and last_msg.tool_calls): final_response = last_msg.content elapsed = time.time() - start if verbose: print(f"\n--- Résultat ({elapsed:.1f}s, {tool_calls_count} appels outils) ---") print(final_response) return final_response except Exception as e: error_msg = f"Erreur de l'agent: {str(e)}" if verbose: print(f"\nERREUR: {error_msg}") return error_msg

9. Points clés à retenir

Résumé du TD - Agent Multi-Tools

  • Le décorateur @tool transforme une fonction Python en outil compatible function calling. La docstring est la clé : c'est elle que le LLM utilise pour décider quand et comment appeler l'outil.
  • Chaque outil doit être spécialisé : un seul rôle bien défini. Si un outil fait trop de choses, le LLM aura du mal à l'utiliser correctement.
  • La sécurité est non-négociable : sandboxer l'exécution de code, valider les chemins de fichiers, whitelister les domaines API, limiter les timeouts.
  • Le function calling natif (GPT-4, Claude) est bien plus fiable que le parsing texte. Il permet aussi les appels d'outils parallèles.
  • Le system prompt de routing guide le LLM dans le choix des outils. Décrivez quand utiliser chaque outil et dans quel ordre.
  • Le wrapper avec error handling est essentiel en production : logging, timing, gestion des exceptions, retry logic.
  • Testez chaque outil individuellement avant de les assembler. Un outil défaillant peut faire échouer tout l'agent.
  • LangGraph gère automatiquement la boucle ReAct, le parsing, et le routing. Utilisez create_react_agent pour un démarrage rapide.

Checklist de création d'un outil

  1. Nom clair et descriptif (snake_case)
  2. Docstring détaillée avec exemples d'utilisation
  3. Types explicites pour tous les paramètres
  4. Gestion d'erreurs complète (try/except)
  5. Validation des entrées (sécurité)
  6. Timeout pour les opérations externes
  7. Tests unitaires indépendants
