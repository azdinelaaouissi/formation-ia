TD Pratique - Construire un Agent ReAct from Scratch

Agents IA - TD 1/5 : Implémenter le pattern Thought / Action / Observation en Python puis migrer vers LangGraph

1. Récapitulatif du pattern ReAct

Avant de plonger dans le code, rappelons les concepts fondamentaux du pattern ReAct (Reasoning + Acting), introduit par Yao et al. en 2022. Ce paradigme permet à un LLM de résoudre des tâches complexes en alternant entre raisonnement et action.

Le pattern ReAct est un framework d'agent où le modèle de langage alterne entre trois phases : Thought (raisonnement interne sur la situation), Action (appel à un outil externe), et Observation (lecture du résultat de l'outil). Cette boucle se répète jusqu'à ce que l'agent puisse fournir une réponse finale.

Pourquoi ReAct est révolutionnaire

Avant ReAct, deux approches existaient séparément :

  • Chain-of-Thought (CoT) : Le modèle raisonne étape par étape mais ne peut pas agir sur le monde extérieur. Il est limité à ses connaissances internes.
  • Action-only agents : Le modèle exécute des actions sans expliciter son raisonnement, rendant le debug difficile et les erreurs fréquentes.

ReAct fusionne les deux approches : le modèle pense à voix haute puis agit, ce qui améliore considérablement la fiabilité et la traçabilité.

Le cycle Thought / Action / Observation

Phase Rôle Exemple
Thought Le modèle réfléchit à ce qu'il doit faire ensuite "Je dois chercher la population actuelle de la France pour répondre"
Action Le modèle choisit un outil et formule l'appel Action: search("population France 2024")
Observation Le résultat de l'outil est injecté dans le contexte Observation: "La France compte 68,4 millions d'habitants..."
Le pattern ReAct est la base de la plupart des agents modernes. Comprendre son fonctionnement interne est essentiel avant d'utiliser des frameworks comme LangGraph qui l'abstraient. Dans ce TD, nous allons d'abord le coder from scratch, puis migrer vers LangGraph.

Quand le cycle s'arrête

L'agent termine sa boucle dans deux cas :

  1. Réponse finale : L'agent estime avoir assez d'informations et produit un "Final Answer".
  2. Limite d'itérations : On fixe un nombre maximum de tours (typiquement 5-10) pour éviter les boucles infinies.
Sans limite d'itérations, un agent peut tourner en boucle indéfiniment, consommant des tokens (et donc de l'argent) sans jamais converger. Toujours définir un max_iterations.

2. Architecture de notre Agent ReAct

Voici l'architecture complète de l'agent que nous allons construire dans ce TD. Nous commencerons par une implémentation Python pure, puis nous migrerons vers LangGraph.

Observez la boucle : Thought, Action, Observation se répètent. Le LLM décide à chaque itération s'il a assez d'information pour répondre ou s'il doit exécuter une nouvelle action. C'est ce qu'on appelle le raisonnement agentic.

Étape 4.3 : Implémenter l'outil calculatrice sécurisé.
import ast import operator # Opérateurs autorisés pour la sécurité SAFE_OPERATORS = { ast.Add: operator.add, ast.Sub: operator.sub, ast.Mult: operator.mul, ast.Div: operator.truediv, ast.Pow: operator.pow, ast.Mod: operator.mod, ast.USub: operator.neg, } def safe_eval(node): """Évalue une expression mathématique de manière sécurisée.""" if isinstance(node, ast.Constant): if isinstance(node.value, (int, float)): return node.value raise ValueError(f"Type non autorisé: {type(node.value)}") elif isinstance(node, ast.BinOp): left = safe_eval(node.left) right = safe_eval(node.right) op = SAFE_OPERATORS.get(type(node.op)) if op is None: raise ValueError(f"Opérateur non autorisé: {type(node.op)}") return op(left, right) elif isinstance(node, ast.UnaryOp): operand = safe_eval(node.operand) op = SAFE_OPERATORS.get(type(node.op)) if op is None: raise ValueError(f"Opérateur unaire non autorisé") return op(operand) else: raise ValueError(f"Noeud AST non autorisé: {type(node)}") def calculate(expression: str) -> str: """Calculatrice sécurisée utilisant l'AST Python.""" try: tree = ast.parse(expression, mode="eval") result = safe_eval(tree.body) return f"Résultat de {expression} = {result}" except Exception as e: return f"Erreur de calcul: {str(e)}"
N'utilisez JAMAIS eval() directement sur une entrée utilisateur. C'est une faille de sécurité critique. Notre implémentation utilise l'AST Python pour n'autoriser que les opérations mathématiques sûres.

5. Exercice : Construire le prompt ReAct

Le prompt est le coeur de l'agent ReAct. Il doit instruire le LLM sur le format Thought/Action/Observation et lui présenter les outils disponibles.

Étape 5.1 : Créer le system prompt ReAct. Ce prompt doit être très précis sur le format attendu.
def build_system_prompt() -> str: """Construit le system prompt ReAct avec la liste des outils.""" # Générer la description des outils dynamiquement tools_desc = "\n".join( f" - {name}: {desc}" for name, desc in TOOL_DESCRIPTIONS.items() ) return f"""Tu es un assistant IA qui résout les problèmes étape par étape en utilisant le format ReAct (Reasoning + Acting). ## Outils disponibles : {tools_desc} ## Format STRICT à respecter : Pour chaque étape de raisonnement, utilise EXACTEMENT ce format : Thought: [ton raisonnement sur ce que tu dois faire] Action: [nom_outil] Action Input: [entrée pour l'outil] Après avoir reçu le résultat de l'outil (Observation), tu peux : - Continuer avec un nouveau Thought/Action si tu as besoin de plus d'info - Donner ta réponse finale avec ce format : Thought: J'ai maintenant assez d'informations pour répondre. Final Answer: [ta réponse complète et détaillée] ## Règles importantes : 1. Toujours commencer par un Thought 2. Un seul Action par étape 3. Attendre l'Observation avant le prochain Thought 4. Ne jamais inventer de données - utiliser les outils 5. Répondre en français """
La qualité du prompt ReAct détermine directement la qualité de l'agent. Chaque mot compte. Notez que nous imposons un format STRICT pour faciliter le parsing automatique des réponses.
Étape 5.2 : Créer la fonction de formatage de l'historique de conversation.
def format_history(question: str, steps: list) -> list: """Formate l'historique pour l'API OpenAI.""" messages = [ {"role": "system", "content": build_system_prompt()}, {"role": "user", "content": question}, ] # Ajouter chaque étape précédente for step in steps: # La réponse du LLM (Thought + Action) messages.append({ "role": "assistant", "content": step["llm_output"] }) # L'observation (résultat de l'outil) if "observation" in step: messages.append({ "role": "user", "content": f"Observation: {step['observation']}" }) return messages

6. Exercice : Implémenter la boucle agent

C'est le coeur de notre implémentation : la boucle qui orchestre le cycle Thought/Action/Observation jusqu'à obtenir une réponse finale.

Étape 6.1 : Créer le parser qui extrait Action et Action Input de la réponse du LLM.
def parse_llm_output(output: str) -> dict: """Parse la sortie du LLM pour extraire l'action ou la réponse finale.""" # Vérifier s'il y a une réponse finale if "Final Answer:" in output: final = output.split("Final Answer:")[1].strip() return {"type": "final", "answer": final} # Extraire Action et Action Input avec regex action_match = re.search( r"Action:\s*(\w+)", output ) input_match = re.search( r"Action Input:\s*(.+)", output ) if action_match and input_match: return { "type": "action", "tool": action_match.group(1), "input": input_match.group(1).strip() } # Format non reconnu return {"type": "error", "message": "Format de sortie non reconnu"} # Tests du parser test1 = """Thought: Je dois chercher l'info. Action: search Action Input: population France 2024""" test2 = """Thought: J'ai assez d'informations. Final Answer: La population de la France est de 68 millions.""" print(parse_llm_output(test1)) print(parse_llm_output(test2))
{'type': 'action', 'tool': 'search', 'input': 'population France 2024'} {'type': 'final', 'answer': 'La population de la France est de 68 millions.'}
Étape 6.2 : Implémenter la boucle principale de l'agent ReAct.
def react_agent(question: str, max_iterations: int = 5, verbose: bool = True) -> str: """ Agent ReAct complet. Args: question: La question de l'utilisateur max_iterations: Nombre max de cycles Thought/Action/Observation verbose: Afficher les étapes intermédiaires Returns: La réponse finale de l'agent """ steps = [] if verbose: print(f"\n{'='*60}") print(f"Question : {question}") print(f"{'='*60}\n") for i in range(max_iterations): if verbose: print(f"--- Itération {i+1}/{max_iterations} ---") # 1. Appeler le LLM messages = format_history(question, steps) response = client.chat.completions.create( model="gpt-4o", messages=messages, temperature=0, max_tokens=1000 ) llm_output = response.choices[0].message.content if verbose: print(f"\nLLM Output:\n{llm_output}\n") # 2. Parser la sortie parsed = parse_llm_output(llm_output) # 3. Vérifier si c'est une réponse finale if parsed["type"] == "final": if verbose: print(f"\n{'='*60}") print(f"RÉPONSE FINALE (après {i+1} itérations):") print(parsed["answer"]) print(f"{'='*60}") return parsed["answer"] # 4. Exécuter l'outil if parsed["type"] == "action": tool_name = parsed["tool"] tool_input = parsed["input"] if verbose: print(f"→ Exécution : {tool_name}({tool_input})") observation = execute_tool(tool_name, tool_input) if verbose: print(f"← Observation : {observation[:200]}...") # 5. Sauvegarder l'étape steps.append({ "llm_output": llm_output, "observation": observation }) else: # Erreur de parsing steps.append({ "llm_output": llm_output, "observation": "Erreur: format non reconnu. Réessaie avec le format Thought/Action/Action Input." }) return "Limite d'itérations atteinte sans réponse finale."
Notre agent ReAct est maintenant complet. Il suit le cycle Thought/Action/Observation, parse automatiquement les réponses du LLM, exécute les outils, et accumule le contexte à chaque itération.

7. Exercice : Migrer vers LangGraph

Maintenant que nous comprenons le fonctionnement interne d'un agent ReAct, migrons vers LangGraph qui gère automatiquement la boucle, le parsing et l'exécution des outils.

Étape 7.1 : Définir les outils avec le décorateur @tool de LangChain.
from langchain_core.tools import tool from langchain_openai import ChatOpenAI from langgraph.prebuilt import create_react_agent @tool def web_search(query: str) -> str: """Rechercher des informations actuelles sur internet. Args: query: La requête de recherche en langage naturel. """ return search(query) @tool def calculator(expression: str) -> str: """Effectuer un calcul mathématique. Args: expression: L'expression mathématique à évaluer (ex: '25 * 4 + 10'). """ return calculate(expression) @tool def current_datetime(query: str = "now") -> str: """Obtenir la date et l'heure actuelles. Args: query: Inutilisé, mettre 'now'. """ return get_datetime(query) tools = [web_search, calculator, current_datetime] print(f"Outils enregistrés : {[t.name for t in tools]}")
Outils enregistrés : ['web_search', 'calculator', 'current_datetime']

Étape 7.4 : Construire manuellement le même agent avec StateGraph pour comprendre l'architecture LangGraph.
from typing import Annotated, TypedDict from langgraph.graph import StateGraph, END from langgraph.graph.message import add_messages from langgraph.prebuilt import ToolNode # 1. Définir le State class AgentState(TypedDict): messages: Annotated[list, add_messages] # 2. Node : appeler le LLM llm_with_tools = llm.bind_tools(tools) def call_model(state: AgentState): response = llm_with_tools.invoke(state["messages"]) return {"messages": [response]} # 3. Node : exécuter les outils tool_node = ToolNode(tools) # 4. Condition : continuer ou terminer ? def should_continue(state: AgentState) -> str: last_message = state["messages"][-1] if last_message.tool_calls: return "tools" return END # 5. Construire le graphe graph = StateGraph(AgentState) graph.add_node("agent", call_model) graph.add_node("tools", tool_node) graph.set_entry_point("agent") graph.add_conditional_edges("agent", should_continue) graph.add_edge("tools", "agent") # 6. Compiler react_graph = graph.compile() print("Graphe ReAct compilé avec succès !")
Cette version explicite montre exactement ce que create_react_agent fait sous le capot : un noeud agent, un noeud outils, et une arête conditionnelle qui décide si on continue ou si on termine.

9. Points clés à retenir

Résumé du TD - Agent ReAct from Scratch

  • Le pattern ReAct alterne entre Thought (raisonnement), Action (appel d'outil), et Observation (résultat). Ce cycle se répète jusqu'à obtention d'une réponse finale.
  • Le prompt est crucial : il doit imposer un format strict et décrire précisément chaque outil disponible. Un mauvais prompt donne un mauvais agent.
  • Le parsing regex permet d'extraire les actions du texte libre du LLM. En production, préférer le function calling natif.
  • La boucle agent est simple : appeler le LLM, parser, exécuter l'outil, réinjecter le résultat, recommencer.
  • Toujours limiter les itérations avec un max_iterations pour éviter les boucles infinies et la surconsommation de tokens.
  • LangGraph simplifie énormément l'implémentation : create_react_agent fait en 3 lignes ce que nous avons codé en 100. Il ajoute aussi le support des appels parallèles.
  • Sécurité des outils : ne jamais utiliser eval() directement. Toujours valider et sandboxer les entrées utilisateur.
  • Le debug est plus facile avec le mode verbose qui affiche chaque étape du raisonnement. En production, utilisez LangSmith pour le tracing.

Architecture à retenir

L'agent ReAct suit une architecture en boucle conditionnelle :

  1. Entrée : Question utilisateur + contexte des étapes précédentes
  2. LLM : Produit un Thought + Action (ou Final Answer)
  3. Router : Si Action -> exécuter l'outil ; Si Final Answer -> retourner la réponse
  4. Outil : Exécute l'action et retourne une Observation
  5. Boucle : L'Observation est ajoutée au contexte et on retourne à l'étape 2
