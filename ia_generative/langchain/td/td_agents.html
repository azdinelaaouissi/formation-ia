TD Pratique - Agents et Tools avec LangChain

LangChain v0.3 -- Creer des agents autonomes qui raisonnent et utilisent des outils

1. Recapitulatif : Agents, Tools et le Pattern ReAct

Un Agent est un systeme qui utilise un LLM comme moteur de raisonnement pour decider dynamiquement quelles actions effectuer, dans quel ordre, et avec quels parametres. Contrairement aux chaines lineaires, un agent peut adapter son comportement en fonction des resultats intermediaires.

Les trois composants fondamentaux d'un agent LangChain :

LLM (Cerveau) : Le modele de langage qui raisonne, planifie et decide. Il recoit le contexte, les outils disponibles et l'historique pour choisir la prochaine action.
Tools (Outils) : Les capacites de l'agent. Chaque outil est une fonction que l'agent peut appeler : calculatrice, recherche web, base de donnees, API externe, etc.
AgentExecutor (Moteur) : L'orchestrateur qui execute la boucle Think-Act-Observe et gere les erreurs, les iterations et la memoire.

Le Pattern ReAct (Reasoning + Acting)

Le pattern ReAct est le paradigme le plus utilise pour les agents LLM. Il combine :

  • Reasoning (Raisonnement) : Le LLM reflechit a ce qu'il doit faire et pourquoi
  • Acting (Action) : Le LLM choisit un outil et formule les parametres
  • Observation : Le resultat de l'outil est renvoye au LLM pour la prochaine iteration

Ce cycle se repete jusqu'a ce que l'agent ait suffisamment d'information pour formuler une reponse finale.

En LangChain v0.3, l'approche recommandee pour les nouveaux projets complexes est LangGraph. Cependant, create_react_agent avec AgentExecutor reste parfaitement valide et plus simple pour apprendre les concepts fondamentaux des agents. C'est ce que nous utiliserons dans ce TD.

Exercice 1 - Creer un Tool simple avec @tool

Le decorateur @tool est la maniere la plus simple de creer un outil dans LangChain. La docstring de la fonction devient automatiquement la description de l'outil, utilisee par le LLM pour decider quand l'utiliser.

Regles pour creer un bon outil

RegleExplication
Docstring claireLe LLM lit la description pour choisir l'outil. Soyez precis.
Type hintsLes types definissent le schema d'entree de l'outil.
Retour stringL'outil doit retourner une chaine lisible par le LLM.
Gestion d'erreursRetournez un message d'erreur plutot que de lever une exception.
# Exercice 1 : Creer un outil calculatrice avec @tool from langchain_core.tools import tool @tool def calculer(expression: str) -> str: """Calcule le resultat d'une expression mathematique. Utilise cet outil quand tu as besoin de faire un calcul. L'expression doit etre une expression Python valide. Args: expression: Expression mathematique a evaluer (ex: '2 + 3 * 4') """ try: # Evaluer de maniere securisee resultat = eval(expression, {"__builtins__": {}}, {}) return f"Le resultat de {expression} est {resultat}" except Exception as e: return f"Erreur de calcul : {str(e)}" # Inspecter l'outil cree print(f"Nom : {calculer.name}") print(f"Description : {calculer.description}") print(f"Schema : {calculer.args}") # Tester l'outil directement print(f"\nTest 1 : {calculer.invoke('42 * 17')}") print(f"Test 2 : {calculer.invoke('2 ** 10')}") print(f"Test 3 : {calculer.invoke('100 / 3')}") print(f"Test 4 (erreur) : {calculer.invoke('abc + 1')}")
Nom : calculer Description : Calcule le resultat d'une expression mathematique. Utilise cet outil quand tu as besoin de faire un calcul. L'expression doit etre une expression Python valide. Schema : {'expression': {'title': 'Expression', 'type': 'string'}} Test 1 : Le resultat de 42 * 17 est 714 Test 2 : Le resultat de 2 ** 10 est 1024 Test 3 : Le resultat de 100 / 3 est 33.333333333333336 Test 4 (erreur) : Erreur de calcul : name 'abc' is not defined
# Creer un deuxieme outil : convertisseur de temperature @tool def convertir_temperature(valeur: float, de_unite: str, vers_unite: str) -> str: """Convertit une temperature entre Celsius, Fahrenheit et Kelvin. Args: valeur: La temperature a convertir de_unite: Unite source ('celsius', 'fahrenheit' ou 'kelvin') vers_unite: Unite cible ('celsius', 'fahrenheit' ou 'kelvin') """ # D'abord convertir en Celsius if de_unite == "celsius": celsius = valeur elif de_unite == "fahrenheit": celsius = (valeur - 32) * 5 / 9 elif de_unite == "kelvin": celsius = valeur - 273.15 else: return f"Unite inconnue : {de_unite}" # Puis convertir vers l'unite cible if vers_unite == "celsius": resultat = celsius elif vers_unite == "fahrenheit": resultat = celsius * 9 / 5 + 32 elif vers_unite == "kelvin": resultat = celsius + 273.15 else: return f"Unite inconnue : {vers_unite}" return f"{valeur} {de_unite} = {resultat:.2f} {vers_unite}" # Tester print(convertir_temperature.invoke({ "valeur": 100, "de_unite": "celsius", "vers_unite": "fahrenheit" })) print(convertir_temperature.invoke({ "valeur": 0, "de_unite": "kelvin", "vers_unite": "celsius" }))
100 celsius = 212.00 fahrenheit 0 kelvin = -273.15 celsius
Le decorateur @tool transforme automatiquement la fonction en un objet StructuredTool avec un nom, une description et un schema d'arguments deduits de la signature et de la docstring. Les outils avec plusieurs parametres necessitent un dictionnaire lors de l'appel via invoke().

Exercice 2 - StructuredTool.from_function()

Pour un controle plus fin sur la configuration de l'outil (nom personnalise, schema Pydantic, return_direct), utilisez StructuredTool.from_function().

# Exercice 2 : Creer un outil structure avec validation Pydantic from langchain_core.tools import StructuredTool from pydantic import BaseModel, Field # Definir le schema d'entree avec Pydantic class RechercheInput(BaseModel): query: str = Field(description="Le terme ou la phrase a rechercher") max_results: int = Field( default=3, description="Nombre maximum de resultats a retourner", ge=1, le=10 ) # Definir la fonction def recherche_base_connaissances(query: str, max_results: int = 3) -> str: """Recherche dans la base de connaissances interne de l'entreprise.""" # Simulation d'une base de connaissances base = { "python": ["Python 3.12 est la derniere version stable", "Python supporte le typage statique avec mypy", "Python est interprete et multi-paradigme"], "ia": ["L'IA generative cree du contenu nouveau", "Le RAG ameliore la factualite des LLM", "Les agents IA peuvent utiliser des outils"], "langchain": ["LangChain v0.3 est la version actuelle", "LangChain supporte les agents ReAct", "LangGraph est recommande pour les workflows complexes"], } resultats = [] for cle, docs in base.items(): if cle in query.lower(): resultats.extend(docs[:max_results]) if not resultats: return f"Aucun resultat trouve pour '{query}'" return "\n".join([f"- {r}" for r in resultats[:max_results]]) # Creer l'outil avec StructuredTool.from_function outil_recherche = StructuredTool.from_function( func=recherche_base_connaissances, name="RechercheInterne", description="Recherche dans la base de connaissances interne. " "Utilise quand l'utilisateur pose une question technique.", args_schema=RechercheInput, return_direct=False # Si True, le resultat est retourne sans passer par le LLM ) # Inspecter print(f"Nom : {outil_recherche.name}") print(f"Description : {outil_recherche.description}") print(f"Schema args : {outil_recherche.args}") # Tester print(f"\nRecherche 'python' :") print(outil_recherche.invoke({"query": "python", "max_results": 2})) print(f"\nRecherche 'langchain' :") print(outil_recherche.invoke({"query": "langchain"})) print(f"\nRecherche inconnue :") print(outil_recherche.invoke({"query": "cuisine"}))
Nom : RechercheInterne Description : Recherche dans la base de connaissances interne. Utilise quand l'utilisateur pose une question technique. Schema args : {'query': {'description': "Le terme ou la phrase a rechercher", 'title': 'Query', 'type': 'string'}, 'max_results': {'default': 3, 'description': 'Nombre maximum de resultats a retourner', 'ge': 1, 'le': 10, 'title': 'Max Results', 'type': 'integer'}} Recherche 'python' : - Python 3.12 est la derniere version stable - Python supporte le typage statique avec mypy Recherche 'langchain' : - LangChain v0.3 est la version actuelle - LangChain supporte les agents ReAct - LangGraph est recommande pour les workflows complexes Recherche inconnue : Aucun resultat trouve pour 'cuisine'
Le parametre return_direct=True est utile quand l'outil retourne deja une reponse formatee pour l'utilisateur. Dans ce cas, l'agent ne reformule pas le resultat. Utilisez-le avec parcimonie car il empeche l'agent de synthetiser les resultats.

Exercice 4 - Agent ReAct avec create_react_agent

C'est le coeur de ce TD : creer un agent qui raisonne et utilise des outils de maniere autonome avec le pattern ReAct.

# Exercice 4 : Agent ReAct complet avec verbose import os os.environ["OPENAI_API_KEY"] = "votre-cle-api" from langchain_openai import ChatOpenAI from langchain.agents import create_react_agent, AgentExecutor from langchain import hub from langchain_core.tools import tool # Definir les outils @tool def calculer(expression: str) -> str: """Calcule une expression mathematique. Utilise pour les operations arithmetiques.""" try: return str(eval(expression, {"__builtins__": {}}, {})) except: return "Erreur de calcul" @tool def obtenir_meteo(ville: str) -> str: """Retourne la meteo actuelle pour une ville donnee.""" # Simulation meteos = { "paris": "Paris : 18C, nuageux, vent 15 km/h", "lyon": "Lyon : 22C, ensoleille, vent 8 km/h", "marseille": "Marseille : 25C, grand soleil, vent 20 km/h", } return meteos.get(ville.lower(), f"Meteo non disponible pour {ville}") tools = [calculer, obtenir_meteo] # Initialiser le LLM llm = ChatOpenAI(model="gpt-4o-mini", temperature=0) # Charger le prompt ReAct depuis le hub LangChain prompt = hub.pull("hwchase17/react") # Creer l'agent ReAct agent = create_react_agent(llm, tools, prompt) # Creer l'executeur avec verbose pour voir le raisonnement agent_executor = AgentExecutor( agent=agent, tools=tools, verbose=True, # Affiche les etapes de raisonnement max_iterations=5, handle_parsing_errors=True ) # Executer l'agent resultat = agent_executor.invoke({ "input": "Quelle est la meteo a Paris ? " "Et si la temperature est en Celsius, combien ca fait en Fahrenheit ?" })
> Entering new AgentExecutor chain... Thought: L'utilisateur veut la meteo a Paris et une conversion de temperature. Je dois d'abord obtenir la meteo, puis faire un calcul de conversion. Action: obtenir_meteo Action Input: paris Observation: Paris : 18C, nuageux, vent 15 km/h Thought: J'ai la temperature : 18C. Je dois maintenant convertir en Fahrenheit. La formule est : F = C * 9/5 + 32 Action: calculer Action Input: 18 * 9/5 + 32 Observation: 64.4 Thought: J'ai maintenant toutes les informations pour repondre. Final Answer: La meteo a Paris est de 18 degres Celsius, nuageux avec un vent de 15 km/h. En Fahrenheit, cela correspond a 64.4 degres F. > Finished chain.
L'agent a effectue 2 appels d'outils de maniere autonome. Il a d'abord obtenu la meteo, extrait la temperature, puis utilise la calculatrice pour la conversion. Le mode verbose=True montre chaque etape du raisonnement ReAct.
Le prompt hwchase17/react est le prompt ReAct officiel heberge sur le LangChain Hub. Il instruit le LLM a suivre le format Thought/Action/Action Input/Observation. Vous pouvez aussi creer votre propre prompt personnalise.

Exercice 5 - Agent Multi-outils

Un agent devient vraiment puissant quand il dispose de plusieurs outils et doit choisir le bon en fonction de la question.

# Exercice 5 : Agent avec multiple outils et logique de selection from langchain_openai import ChatOpenAI from langchain.agents import create_react_agent, AgentExecutor from langchain import hub from langchain_core.tools import tool from langchain_community.tools import DuckDuckGoSearchRun # Outil 1 : Calculatrice avancee @tool def calculer(expression: str) -> str: """Calcule des expressions mathematiques. Utilise pour tout calcul numerique. Supporte +, -, *, /, **, sqrt, etc. Exemples: '2+3', '100/7', '2**10'""" import math try: return str(eval(expression, {"__builtins__": {}, "math": math}, {})) except Exception as e: return f"Erreur : {e}" # Outil 2 : Recherche web outil_recherche = DuckDuckGoSearchRun() # Outil 3 : Base de connaissances interne @tool def base_connaissances(sujet: str) -> str: """Recherche dans la documentation interne de l'entreprise. Utilise pour les questions sur les politiques, procedures ou produits internes. Exercice 6 - Agent avec Memoire

Par defaut, un agent ne se souvient pas des echanges precedents. En ajoutant une memoire, l'agent peut maintenir une conversation multi-tours coherente tout en utilisant ses outils.

# Exercice 6 : Agent avec memoire conversationnelle from langchain_openai import ChatOpenAI from langchain.agents import create_react_agent, AgentExecutor from langchain import hub from langchain_core.tools import tool from langchain.memory import ConversationBufferMemory from langchain_core.prompts import PromptTemplate # Outils @tool def calculer(expression: str) -> str: """Calcule une expression mathematique.""" try: return str(eval(expression, {"__builtins__": {}}, {})) except: return "Erreur" @tool def recherche_produit(nom: str) -> str: """Recherche le prix d'un produit dans le catalogue.""" catalogue = { "laptop": "Laptop Pro X1 : 1299 EUR", "ecran": "Ecran 27 4K : 449 EUR", "clavier": "Clavier mecanique : 129 EUR", "souris": "Souris ergonomique : 79 EUR", } for cle, val in catalogue.items(): if cle in nom.lower(): return val return f"Produit '{nom}' non trouve." tools = [calculer, recherche_produit] llm = ChatOpenAI(model="gpt-4o-mini", temperature=0) # Prompt ReAct adapte pour la memoire (avec chat_history) template = """Reponds au mieux en utilisant les outils disponibles. Tu as acces aux outils suivants : {tools} Utilise le format suivant : Question: la question de l'utilisateur Thought: reflechis a ce que tu dois faire Action: le nom de l'outil a utiliser [{tool_names}] Action Input: l'entree de l'outil Observation: le resultat de l'outil ... (ce cycle peut se repeter) Thought: j'ai maintenant la reponse finale Final Answer: la reponse finale Historique de conversation : {chat_history} Question: {input} {agent_scratchpad}""" prompt = PromptTemplate( template=template, input_variables=["input", "chat_history", "agent_scratchpad"], partial_variables={ "tools": "\n".join([f"{t.name}: {t.description}" for t in tools]), "tool_names": ", ".join([t.name for t in tools]) } ) # Creer la memoire memory = ConversationBufferMemory( memory_key="chat_history", return_messages=False ) # Creer l'agent avec memoire agent = create_react_agent(llm, tools, prompt) agent_executor = AgentExecutor( agent=agent, tools=tools, memory=memory, verbose=True, handle_parsing_errors=True, max_iterations=5 ) # Conversation multi-tours print("=== Tour 1 ===") r1 = agent_executor.invoke({"input": "Combien coute le laptop ?"}) print(f"Agent : {r1['output']}\n") print("=== Tour 2 ===") r2 = agent_executor.invoke({"input": "Et l'ecran ?"}) print(f"Agent : {r2['output']}\n") print("=== Tour 3 ===") r3 = agent_executor.invoke({"input": "Quel serait le total des deux produits ?"}) print(f"Agent : {r3['output']}")
=== Tour 1 === > Entering new AgentExecutor chain... Thought: Je dois chercher le prix du laptop. Action: recherche_produit Action Input: laptop Observation: Laptop Pro X1 : 1299 EUR Final Answer: Le Laptop Pro X1 coute 1299 EUR. > Finished chain. Agent : Le Laptop Pro X1 coute 1299 EUR. === Tour 2 === > Entering new AgentExecutor chain... Thought: L'utilisateur demande "et l'ecran", c'est un suivi de conversation. Action: recherche_produit Action Input: ecran Observation: Ecran 27 4K : 449 EUR Final Answer: L'Ecran 27 pouces 4K coute 449 EUR. > Finished chain. Agent : L'Ecran 27 pouces 4K coute 449 EUR. === Tour 3 === > Entering new AgentExecutor chain... Thought: L'utilisateur veut le total. D'apres l'historique : laptop = 1299 EUR, ecran = 449 EUR. Je dois additionner. Action: calculer Action Input: 1299 + 449 Observation: 1748 Final Answer: Le total du Laptop Pro X1 (1299 EUR) et de l'Ecran 4K (449 EUR) est de 1748 EUR. > Finished chain. Agent : Le total du Laptop Pro X1 (1299 EUR) et de l'Ecran 4K (449 EUR) est de 1748 EUR.
Au tour 3, l'agent se souvient que le laptop coute 1299 EUR et l'ecran 449 EUR grace a la memoire conversationnelle. Il combine cette information avec un appel a la calculatrice pour donner le total. Sans memoire, il n'aurait pas pu repondre a cette question.

Exercice 7 - Gestion d'erreurs et robustesse

En production, les agents peuvent rencontrer des erreurs : outil indisponible, boucle infinie, format de sortie incorrect. LangChain fournit des mecanismes pour gerer ces situations.

# Exercice 7 : Gestion des erreurs et parametres de robustesse from langchain_openai import ChatOpenAI from langchain.agents import create_react_agent, AgentExecutor from langchain import hub from langchain_core.tools import tool # Outil qui peut echouer @tool def api_instable(query: str) -> str: """Interroge une API externe (peut echouer parfois).""" import random if random.random() < 0.5: raise Exception("Erreur 503 : Service temporairement indisponible") return f"Resultat de l'API pour '{query}': donnees disponibles" @tool def calculer(expression: str) -> str: """Calcule une expression mathematique.""" try: return str(eval(expression, {"__builtins__": {}}, {})) except: return "Erreur de calcul" tools = [api_instable, calculer] llm = ChatOpenAI(model="gpt-4o-mini", temperature=0) prompt = hub.pull("hwchase17/react") agent = create_react_agent(llm, tools, prompt) # === Configuration robuste === agent_executor = AgentExecutor( agent=agent, tools=tools, verbose=True, # 1. Limite d'iterations (evite les boucles infinies) max_iterations=5, # 2. Gestion des erreurs de parsing handle_parsing_errors=True, # OU avec un message personnalise : # handle_parsing_errors="Format incorrect. Utilise le format Thought/Action/..." # 3. Timeout par iteration (en secondes) max_execution_time=30, # 4. Message si max_iterations atteint early_stopping_method="generate", # "force" ou "generate" ) # Test avec une question qui pourrait causer une erreur print("=== Test avec API instable ===") try: resultat = agent_executor.invoke({ "input": "Interroge l'API pour obtenir des donnees sur la France" }) print(f"Reponse : {resultat['output']}") except Exception as e: print(f"Erreur capturee : {e}")
=== Test avec API instable === > Entering new AgentExecutor chain... Thought: Je dois interroger l'API pour des donnees sur la France. Action: api_instable Action Input: France Observation: Erreur 503 : Service temporairement indisponible Thought: L'API a echoue. Je vais reessayer. Action: api_instable Action Input: France Observation: Resultat de l'API pour 'France': donnees disponibles Thought: J'ai obtenu les donnees. Final Answer: Les donnees sur la France sont disponibles via l'API. > Finished chain. Reponse : Les donnees sur la France sont disponibles via l'API.

Fixez toujours max_iterations a une valeur raisonnable (3 a 10). Sans cette limite, un agent peut entrer dans une boucle infinie qui consomme des tokens (et donc du budget API) sans fin. En production, ajoutez aussi max_execution_time comme filet de securite.

Experimentation - Impact du jeu d'outils

L'ensemble d'outils disponibles influence directement le comportement de l'agent. La meme question produit des reponses differentes selon les outils accessibles.

# Experimentation : Meme question, outils differents from langchain_openai import ChatOpenAI from langchain.agents import create_react_agent, AgentExecutor from langchain import hub from langchain_core.tools import tool from langchain_community.tools import WikipediaQueryRun, DuckDuckGoSearchRun from langchain_community.utilities import WikipediaAPIWrapper question = "Quelle est la population de la France et combien cela represente par km2 ?" llm = ChatOpenAI(model="gpt-4o-mini", temperature=0) prompt = hub.pull("hwchase17/react") # --- Configuration A : Calculatrice seule --- @tool def calc(expr: str) -> str: """Calcule une expression mathematique.""" try: return str(eval(expr, {"__builtins__": {}}, {})) except: return "Erreur" agent_a = AgentExecutor( agent=create_react_agent(llm, [calc], prompt), tools=[calc], verbose=False, max_iterations=5, handle_parsing_errors=True ) # --- Configuration B : Wikipedia + Calculatrice --- wiki = WikipediaQueryRun(api_wrapper=WikipediaAPIWrapper( top_k_results=1, doc_content_chars_max=500, lang="fr" )) agent_b = AgentExecutor( agent=create_react_agent(llm, [calc, wiki], prompt), tools=[calc, wiki], verbose=False, max_iterations=5, handle_parsing_errors=True ) # --- Configuration C : Search + Wikipedia + Calculatrice --- ddg = DuckDuckGoSearchRun() agent_c = AgentExecutor( agent=create_react_agent(llm, [calc, wiki, ddg], prompt), tools=[calc, wiki, ddg], verbose=False, max_iterations=5, handle_parsing_errors=True ) # Comparer les reponses print(f"Question : {question}\n") print("=== A (Calculatrice seule) ===") print(agent_a.invoke({"input": question})["output"]) print("\n=== B (Wikipedia + Calculatrice) ===") print(agent_b.invoke({"input": question})["output"]) print("\n=== C (Search + Wikipedia + Calculatrice) ===") print(agent_c.invoke({"input": question})["output"])
Question : Quelle est la population de la France et combien cela represente par km2 ? === A (Calculatrice seule) === La population de la France est d'environ 68 millions d'habitants. La superficie est d'environ 551 695 km2. Densite : 68000000 / 551695 = environ 123 habitants par km2. (Note : ces chiffres proviennent de mes connaissances, pas d'un calcul verifie) === B (Wikipedia + Calculatrice) === Selon Wikipedia, la France compte environ 68,04 millions d'habitants pour une superficie de 643 801 km2 (avec l'outre-mer). La densite est de : 68040000 / 643801 = 105.7 habitants par km2. === C (Search + Wikipedia + Calculatrice) === D'apres les donnees les plus recentes (2024), la population de la France est estimee a 68,4 millions d'habitants. Avec une superficie metropolitaine de 551 695 km2, la densite est de 68400000 / 551695 = 123.97 habitants par km2.
Configuration A utilise les connaissances internes du LLM (potentiellement obsoletes). Configuration B consulte Wikipedia pour des donnees plus fiables. Configuration C cherche les donnees les plus recentes sur le web. Plus l'agent a d'outils pertinents, plus ses reponses sont precises et a jour.

Erreurs Courantes

Description d'outil vague : Si la description est floue ("Un outil utile"), le LLM ne saura pas quand l'utiliser. Soyez precis : "Calcule des expressions mathematiques. Accepte des expressions Python comme '2+3' ou 'math.sqrt(16)'."
Pas de handle_parsing_errors : Sans ce parametre, une sortie mal formatee du LLM fait planter tout l'agent. Mettez toujours handle_parsing_errors=True en production.
max_iterations trop haut ou absent : Un agent avec max_iterations=100 peut consommer des centaines de milliers de tokens en boucle. Commencez a 5, augmentez si necessaire. Chaque iteration = un appel au LLM = un cout.
Outils qui levent des exceptions : Si un outil fait un raise Exception() non gere, l'agent s'arrete. Encapsulez toujours vos outils dans un try/except et retournez un message d'erreur lisible au lieu de lever une exception.
Trop d'outils : Avec 20+ outils, le LLM a du mal a choisir le bon (le prompt devient trop long). Regroupez les outils par domaine et limitez a 5-8 outils par agent. Utilisez des sous-agents specialises pour des workflows complexes.
Oublier la memoire pour les conversations : Sans memoire, l'agent perd tout contexte entre les tours. Si l'utilisateur dit "Et pour l'autre produit ?", l'agent ne sait pas de quoi il parle. Ajoutez ConversationBufferMemory pour tout agent conversationnel.

Agent minimal complet en 15 lignes

from langchain_openai import ChatOpenAI from langchain.agents import create_react_agent, AgentExecutor from langchain import hub from langchain_core.tools import tool @tool def calculer(expr: str) -> str: """Calcule une expression mathematique.""" return str(eval(expr, {"__builtins__": {}}, {})) llm = ChatOpenAI(model="gpt-4o-mini", temperature=0) prompt = hub.pull("hwchase17/react") agent = create_react_agent(llm, [calculer], prompt) executor = AgentExecutor(agent=agent, tools=[calculer], verbose=True, max_iterations=5, handle_parsing_errors=True) resultat = executor.invoke({"input": "Combien font 2^32 ?"}) print(resultat["output"]) # 4294967296
Ce TD vous a donne les bases pour creer des agents LangChain avec le pattern ReAct. Pour aller plus loin, explorez LangGraph pour des workflows plus complexes avec des graphes d'etats, des branches conditionnelles et une gestion fine des erreurs. LangGraph est la recommandation officielle de LangChain pour les agents de production.
