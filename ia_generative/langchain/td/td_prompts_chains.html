TD Pratique - Prompts et Chains avec LangChain

LangChain v0.3 -- LCEL (LangChain Expression Language) -- PromptTemplates, Chains et Output Parsers

1. Recapitulatif

Un Prompt est un modele de texte parametrable envoye a un LLM. Une Chain (chaine) est une sequence d'operations ou chaque composant transforme les donnees et les passe au suivant. En LangChain v0.3, on construit des chains grace a LCEL (LangChain Expression Language), une syntaxe declarative qui utilise l'operateur pipe | pour composer des Runnables.

Les trois piliers fondamentaux de LCEL :

PromptTemplate

Formate les variables d'entree en un prompt lisible par le modele. Il existe PromptTemplate pour les modeles texte et ChatPromptTemplate pour les modeles de chat (messages systeme + utilisateur).

ChatModel (LLM)

Le modele de langage qui recoit le prompt formate et genere une reponse. Exemples : ChatOpenAI, ChatAnthropic, ChatOllama.

OutputParser

Transforme la reponse brute du modele en un format exploitable : texte brut (StrOutputParser), JSON (JsonOutputParser), ou objet Pydantic (PydanticOutputParser).

En LCEL, une chain s'ecrit simplement :

# La syntaxe fondamentale LCEL chain = prompt | model | output_parser
Chaque composant LCEL implemente l'interface Runnable, ce qui garantit les methodes invoke(), stream(), batch() et ainvoke() de facon uniforme. C'est la grande force de LCEL par rapport aux anciennes LLMChain desormais obsoletes.

2. Principe : Architecture d'une Chain LCEL

L'operateur | (pipe) est surcharge par LangChain. Il ne s'agit pas du pipe Unix ou du OR binaire Python. Quand vous ecrivez a | b, LangChain cree un RunnableSequence ou la sortie de a devient l'entree de b. C'est le coeur de la composition LCEL.

Exercice 1 : PromptTemplate basique -- Prompt de traduction

Objectif : Creer un prompt parametrable qui demande une traduction dans une langue donnee.

Etape 1 : Installer les dependances necessaires.
# Installation (a executer dans le terminal) # pip install langchain-core langchain-openai python-dotenv import os from dotenv import load_dotenv load_dotenv() # Charge OPENAI_API_KEY depuis .env
Etape 2 : Creer un PromptTemplate avec des variables.
from langchain_core.prompts import PromptTemplate # PromptTemplate avec deux variables : langue et texte prompt = PromptTemplate( template="Traduis le texte suivant en {langue}.\n\nTexte : {texte}\n\nTraduction :", input_variables=["langue", "texte"] ) # Formater le prompt avec des valeurs concretes prompt_formate = prompt.format( langue="espagnol", texte="Bonjour, comment allez-vous aujourd'hui ?" ) print(prompt_formate)
Traduis le texte suivant en espagnol. Texte : Bonjour, comment allez-vous aujourd'hui ? Traduction :
Etape 3 : Utiliser la methode raccourcie from_template.
# Syntaxe raccourcie -- pas besoin de specifier input_variables prompt_v2 = PromptTemplate.from_template( "Donne-moi {nombre} faits interessants sur {sujet}." ) print(prompt_v2.input_variables) # Les variables sont detectees automatiquement print(prompt_v2.format(nombre=3, sujet="les octopodes"))
['nombre', 'sujet'] Donne-moi 3 faits interessants sur les octopodes.
Le PromptTemplate est un objet reutilisable. Vous pouvez l'invoquer avec differentes valeurs sans re-ecrire le prompt a chaque fois. C'est la base de la modularite LCEL.

Exercice 2 : ChatPromptTemplate -- Messages systeme et utilisateur

Objectif : Construire un prompt structure avec un message systeme qui definit le role du LLM et un message utilisateur parametrable.

Le ChatPromptTemplate est concu pour les modeles de chat (GPT-4, Claude, etc.) qui distinguent les roles : system (instructions), human (utilisateur) et ai (assistant). C'est le format recommande en LangChain v0.3.
from langchain_core.prompts import ChatPromptTemplate # Creer un ChatPromptTemplate avec messages structures chat_prompt = ChatPromptTemplate.from_messages([ ("system", "Tu es un expert en {domaine}. Reponds de maniere concise et pedagogique."), ("human", "Explique-moi {concept} en maximum {nb_phrases} phrases.") ]) # Formatter le prompt messages = chat_prompt.format_messages( domaine="intelligence artificielle", concept="le mecanisme d'attention", nb_phrases=3 ) for msg in messages: print(f"[{msg.type}] {msg.content}")
[system] Tu es un expert en intelligence artificielle. Reponds de maniere concise et pedagogique. [human] Explique-moi le mecanisme d'attention en maximum 3 phrases.

Variante : ajout d'un placeholder pour l'historique

from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder # Prompt avec un emplacement reserve pour des messages dynamiques chat_prompt_avec_historique = ChatPromptTemplate.from_messages([ ("system", "Tu es un assistant pedagogique specialise en {domaine}."), MessagesPlaceholder(variable_name="historique"), ("human", "{question}") ]) print(chat_prompt_avec_historique.input_variables)
['domaine', 'historique', 'question']
Le MessagesPlaceholder est essentiel pour integrer de la memoire dans une chain. Il sera reutilise dans le TD suivant sur la Memory. Retenez son existence des maintenant !

Exercice 3 : Premiere chain LCEL -- prompt | model | parser

Objectif : Assembler les composants en une chain fonctionnelle et l'invoquer.

Etape 1 : Configurer le modele.
from langchain_openai import ChatOpenAI from langchain_core.prompts import ChatPromptTemplate from langchain_core.output_parsers import StrOutputParser # Initialiser le modele model = ChatOpenAI( model="gpt-4o-mini", temperature=0.7 )
Etape 2 : Creer le prompt et le parser.
# Prompt structure pour un chatbot culinaire prompt = ChatPromptTemplate.from_messages([ ("system", "Tu es un chef cuisinier francais. Donne des recettes simples et savoureuses."), ("human", "Propose-moi une recette avec {ingredient} pour {nb_personnes} personnes.") ]) # Parser qui extrait le texte brut de la reponse parser = StrOutputParser()
Etape 3 : Composer la chain avec l'operateur pipe.
# Composition LCEL : prompt -> model -> parser chain = prompt | model | parser # Invoquer la chain avec des parametres resultat = chain.invoke({ "ingredient": "des champignons", "nb_personnes": 4 }) print(resultat)
Voici une recette simple de risotto aux champignons pour 4 personnes : Ingredients : - 300g de riz arborio - 250g de champignons de Paris - 1 oignon - 1L de bouillon de legumes chaud - 10cl de vin blanc sec - 50g de parmesan rape - 30g de beurre - Sel, poivre, persil frais Instructions : 1. Emincer l'oignon et les champignons. Faire revenir l'oignon dans le beurre. 2. Ajouter les champignons, cuire 5 minutes. 3. Ajouter le riz, nacrer 2 minutes, puis deglacez au vin blanc. 4. Ajouter le bouillon louche par louche en remuant. 5. Cuire 18 minutes, incorporer le parmesan. Servir aussitot !
L'operateur | cree en coulisses un RunnableSequence. Quand vous appelez chain.invoke(data), voici ce qui se passe :

1. prompt.invoke(data) → retourne un ChatPromptValue (les messages formates)
2. model.invoke(messages) → retourne un AIMessage (reponse du LLM)
3. parser.invoke(ai_message) → retourne une str (le contenu textuel)

Chaque etape recoit la sortie de la precedente. C'est le pattern fondamental de LCEL.

Bonus : Streaming de la reponse

# Streamer la reponse token par token for chunk in chain.stream({"ingredient": "du saumon", "nb_personnes": 2}): print(chunk, end="", flush=True)
Grace a stream(), les tokens s'affichent un par un au lieu d'attendre la reponse complete. Cela fonctionne sur toute la chain grace a l'interface Runnable uniforme.

Exercice 4 : Sortie structuree avec JsonOutputParser

Objectif : Forcer le modele a repondre en JSON et parser automatiquement la sortie en dictionnaire Python.

from langchain_core.output_parsers import JsonOutputParser from langchain_core.prompts import ChatPromptTemplate from langchain_openai import ChatOpenAI # Creer le parser JSON json_parser = JsonOutputParser() # Le prompt inclut les instructions de formatage du parser prompt = ChatPromptTemplate.from_messages([ ("system", """Tu es un analyste de films. Pour chaque film, fournis une fiche technique en JSON. {format_instructions}"""), ("human", "Analyse le film : {titre_film}") ]) # Injecter les instructions de formatage dans le prompt prompt = prompt.partial( format_instructions=json_parser.get_format_instructions() ) model = ChatOpenAI(model="gpt-4o-mini", temperature=0) # Chain complete chain = prompt | model | json_parser # Invoquer resultat = chain.invoke({"titre_film": "Inception"}) print(type(resultat)) print(resultat)
<class 'dict'> { "titre": "Inception", "realisateur": "Christopher Nolan", "annee": 2010, "genre": ["Science-fiction", "Thriller", "Action"], "acteurs_principaux": ["Leonardo DiCaprio", "Joseph Gordon-Levitt", "Ellen Page"], "note_sur_10": 8.8, "resume": "Un voleur specialise dans l'extraction d'idees par les reves accepte une mission inverse : implanter une idee dans l'esprit d'un heritier." }

Variante avancee : PydanticOutputParser avec schema type

from langchain_core.output_parsers import PydanticOutputParser from pydantic import BaseModel, Field from typing import List # Definir un schema Pydantic class FicheFilm(BaseModel): titre: str = Field(description="Titre du film") realisateur: str = Field(description="Nom du realisateur") annee: int = Field(description="Annee de sortie") genres: List[str] = Field(description="Liste des genres") note: float = Field(description="Note sur 10") # Le parser sait generer les instructions a partir du schema pydantic_parser = PydanticOutputParser(pydantic_object=FicheFilm) prompt = ChatPromptTemplate.from_messages([ ("system", "Analyse le film demande.\n{format_instructions}"), ("human", "{titre_film}") ]) prompt = prompt.partial( format_instructions=pydantic_parser.get_format_instructions() ) chain = prompt | model | pydantic_parser fiche = chain.invoke({"titre_film": "Le Seigneur des Anneaux"}) print(type(fiche)) # <class 'FicheFilm'> print(fiche.titre, "-", fiche.annee)
<class 'FicheFilm'> Le Seigneur des Anneaux : La Communaute de l'Anneau - 2001
Le PydanticOutputParser valide la sortie du LLM contre le schema. Si un champ manque ou a le mauvais type, une erreur explicite est levee. C'est ideal pour des pipelines de production.

Exercice 5 : RunnablePassthrough et RunnableParallel

Objectif : Gerer des entrees multiples et executer des branches en parallele dans une chain LCEL.

RunnablePassthrough transmet les donnees d'entree sans modification (comme une identite). RunnableParallel execute plusieurs Runnables en parallele sur la meme entree et retourne un dictionnaire de resultats.

Exemple 1 : RunnablePassthrough pour conserver l'entree originale

from langchain_core.runnables import RunnablePassthrough, RunnableParallel from langchain_core.prompts import ChatPromptTemplate from langchain_core.output_parsers import StrOutputParser from langchain_openai import ChatOpenAI model = ChatOpenAI(model="gpt-4o-mini", temperature=0) # Prompt qui attend "question" et "contexte" prompt = ChatPromptTemplate.from_messages([ ("system", "Reponds a la question en te basant sur le contexte fourni."), ("human", "Contexte : {contexte}\n\nQuestion : {question}") ]) # RunnablePassthrough transmet la question telle quelle # tandis qu'une autre branche pourrait enrichir le contexte chain = ( { "contexte": lambda x: "LangChain est un framework Python pour construire des applications LLM.", "question": RunnablePassthrough() } | prompt | model | StrOutputParser() ) # Ici, l'entree est directement la question (une string) resultat = chain.invoke("Qu'est-ce que LangChain ?") print(resultat)
LangChain est un framework Python concu pour faciliter la construction d'applications basees sur des modeles de langage (LLM). Il fournit des abstractions pour les prompts, les chains, la memoire et les outils, permettant de creer des pipelines d'IA complexes de maniere modulaire.

Exemple 2 : RunnableParallel pour des analyses multiples

# Analyser un texte sous trois angles differents en parallele texte_prompt = ChatPromptTemplate.from_template( "Analyse le sentiment du texte suivant : {texte}" ) resume_prompt = ChatPromptTemplate.from_template( "Resume ce texte en une phrase : {texte}" ) mots_cles_prompt = ChatPromptTemplate.from_template( "Extrais 5 mots-cles de ce texte : {texte}" ) parser = StrOutputParser() # Trois chains independantes chain_sentiment = texte_prompt | model | parser chain_resume = resume_prompt | model | parser chain_mots_cles = mots_cles_prompt | model | parser # Execution parallele avec RunnableParallel analyse_parallele = RunnableParallel( sentiment=chain_sentiment, resume=chain_resume, mots_cles=chain_mots_cles ) # Toutes les analyses sont executees en meme temps ! resultats = analyse_parallele.invoke({ "texte": """L'equipe de France a remporte une victoire eclatante hier soir au Stade de France. Les joueurs ont fait preuve d'une determination sans faille, malgre un debut de match difficile. Le public etait en delire apres le but decisif a la 89e minute.""" }) for cle, valeur in resultats.items(): print(f"\n--- {cle.upper()} ---") print(valeur)
--- SENTIMENT --- Le sentiment de ce texte est tres positif. Il exprime l'enthousiasme, la fierte et la joie liee a une victoire sportive. --- RESUME --- L'equipe de France a remporte une victoire spectaculaire au Stade de France grace a un but decisif a la 89e minute, apres un debut de match complique. --- MOTS_CLES --- 1. Victoire 2. Equipe de France 3. Stade de France 4. Determination 5. But decisif
RunnableParallel est extremement utile en RAG (Retrieval-Augmented Generation) ou vous devez souvent preparer plusieurs inputs simultanement (recuperer des documents, formatter la question, etc.) avant de les passer au modele. Notez que la syntaxe {"cle": chain} en debut de pipeline est automatiquement convertie en RunnableParallel par LCEL.

Exercice 6 : RunnableLambda -- Transformations personnalisees

Objectif : Integrer des fonctions Python personnalisees dans une chain LCEL grace a RunnableLambda.

RunnableLambda transforme n'importe quelle fonction Python (synchrone ou asynchrone) en un composant Runnable compatible avec l'operateur pipe |. Cela permet d'ajouter du pre-traitement, de la validation ou des transformations intermediaires.
from langchain_core.runnables import RunnableLambda from langchain_core.prompts import ChatPromptTemplate from langchain_core.output_parsers import StrOutputParser from langchain_openai import ChatOpenAI model = ChatOpenAI(model="gpt-4o-mini", temperature=0.3) # Fonction de pre-traitement : nettoyer et enrichir l'entree def preprocesser_texte(entree: dict) -> dict: """Nettoie le texte et ajoute des metadonnees.""" texte = entree["texte"].strip().lower() nb_mots = len(texte.split()) return { "texte_original": entree["texte"], "texte_nettoye": texte, "nb_mots": nb_mots, "est_court": nb_mots < 50 } # Fonction de post-traitement : formater la sortie def formater_sortie(reponse: str) -> str: """Ajoute un encadrement a la reponse.""" separateur = "=" * 50 return f"{separateur}\nANALYSE AUTOMATIQUE\n{separateur}\n{reponse}\n{separateur}" # Prompt qui utilise les donnees enrichies prompt = ChatPromptTemplate.from_messages([ ("system", "Tu es un analyste linguistique."), ("human", """Analyse ce texte ({nb_mots} mots, texte {'court' if est_court else 'long'}) : Texte : {texte_nettoye} Fournis : le ton, le theme principal, et une suggestion d'amelioration.""") ]) # Chain avec RunnableLambda en pre et post-traitement chain = ( RunnableLambda(preprocesser_texte) | prompt | model | StrOutputParser() | RunnableLambda(formater_sortie) ) resultat = chain.invoke({ "texte": " L'IA transforme notre monde. Les possibilites sont infinies. " }) print(resultat)
================================================== ANALYSE AUTOMATIQUE ================================================== Ton : Enthousiaste et affirmatif. Le texte adopte un ton optimiste face aux avancees technologiques. Theme principal : L'impact transformateur de l'intelligence artificielle sur la societe. Suggestion d'amelioration : Le texte est tres court et pourrait beneficier d'exemples concrets pour illustrer les "possibilites infinies" mentionnees. ==================================================

Astuce : Utiliser le decorateur @chain

from langchain_core.runnables import chain as lc_chain # Le decorateur @chain transforme une fonction en Runnable @lc_chain def analyser_et_traduire(entree: dict): """Analyse un texte puis le traduit.""" # Premiere chain : analyser analyse_prompt = ChatPromptTemplate.from_template( "Analyse le sentiment : {texte}" ) analyse = (analyse_prompt | model | StrOutputParser()).invoke(entree) # Deuxieme chain : traduire l'analyse traduction_prompt = ChatPromptTemplate.from_template( "Traduis en anglais : {texte_a_traduire}" ) traduction = (traduction_prompt | model | StrOutputParser()).invoke( {"texte_a_traduire": analyse} ) return {"analyse_fr": analyse, "analyse_en": traduction} # Utilisable comme un Runnable classique resultat = analyser_et_traduire.invoke({"texte": "Quelle belle journee !"}) print(resultat)
Le decorateur @chain est un raccourci elegant pour creer des RunnableLambda complexes. Il permet d'ecrire du code Python classique tout en restant compatible avec l'ecosysteme LCEL.

Exercice 7 : Pipeline complexe multi-etapes

Objectif : Combiner tous les concepts vus precedemment dans un pipeline de traitement complet qui analyse un article, en extrait des donnees structurees, et produit un rapport formate.

from langchain_core.runnables import ( RunnableLambda, RunnableParallel, RunnablePassthrough ) from langchain_core.prompts import ChatPromptTemplate from langchain_core.output_parsers import StrOutputParser, JsonOutputParser from langchain_openai import ChatOpenAI from datetime import datetime model = ChatOpenAI(model="gpt-4o-mini", temperature=0) parser = StrOutputParser() # ============================================ # ETAPE 1 : Pre-traitement (RunnableLambda) # ============================================ def pretraiter(entree: dict) -> dict: article = entree["article"] return { "article": article, "nb_mots": len(article.split()), "date_analyse": datetime.now().strftime("%Y-%m-%d %H:%M"), "langue": entree.get("langue", "francais") } # ============================================ # ETAPE 2 : Analyses paralleles # ============================================ sentiment_prompt = ChatPromptTemplate.from_template( "Analyse le sentiment de ce texte en un mot (positif/negatif/neutre) " "suivi d'un score de -1 a 1 : {article}" ) mots_cles_prompt = ChatPromptTemplate.from_messages([ ("system", "Extrais des mots-cles. Reponds UNIQUEMENT en JSON : " '{{"mots_cles": ["mot1", "mot2", ...]}}'), ("human", "{article}") ]) resume_prompt = ChatPromptTemplate.from_template( "Resume ce texte en exactement 2 phrases : {article}" ) analyse_parallele = RunnableParallel( sentiment=sentiment_prompt | model | parser, mots_cles=mots_cles_prompt | model | JsonOutputParser(), resume=resume_prompt | model | parser, metadonnees=RunnablePassthrough() # Conserver les metadonnees ) # ============================================ # ETAPE 3 : Synthese finale # ============================================ synthese_prompt = ChatPromptTemplate.from_template( """A partir de ces analyses, redige une conclusion en 3 lignes : - Sentiment : {sentiment} - Mots-cles : {mots_cles} - Resume : {resume}""" ) # ============================================ # ETAPE 4 : Formatage (RunnableLambda) # ============================================ def formater_rapport(donnees: dict) -> str: meta = donnees["metadonnees"] return f""" ======================================== RAPPORT D'ANALYSE AUTOMATIQUE Date : {meta['date_analyse']} Mots : {meta['nb_mots']} ======================================== SENTIMENT : {donnees['sentiment']} MOTS-CLES : {', '.join(donnees['mots_cles']['mots_cles'])} RESUME : {donnees['resume']} CONCLUSION : {donnees.get('conclusion', 'N/A')} ========================================""" # ============================================ # ASSEMBLAGE DU PIPELINE COMPLET # ============================================ def ajouter_conclusion(donnees: dict) -> dict: # Generer la conclusion a partir des analyses conclusion_chain = synthese_prompt | model | parser conclusion = conclusion_chain.invoke({ "sentiment": donnees["sentiment"], "mots_cles": str(donnees["mots_cles"]), "resume": donnees["resume"] }) donnees["conclusion"] = conclusion return donnees pipeline = ( RunnableLambda(pretraiter) # Etape 1 : pre-traitement | analyse_parallele # Etape 2 : analyses paralleles | RunnableLambda(ajouter_conclusion) # Etape 3 : synthese | RunnableLambda(formater_rapport) # Etape 4 : formatage ) # Lancer le pipeline complet rapport = pipeline.invoke({ "article": """Les avancees recentes en intelligence artificielle generative soulèvent des questions fondamentales sur l'avenir du travail. Si certains experts predisent une revolution positive, d'autres mettent en garde contre les risques de deplacements d'emplois massifs. Le debat reste ouvert et les gouvernements tardent a reguler.""" }) print(rapport)
======================================== RAPPORT D'ANALYSE AUTOMATIQUE Date : 2026-03-03 14:30 Mots : 47 ======================================== SENTIMENT : Neutre (0.1) - Le texte presente une perspective equilibree avec des elements positifs et negatifs. MOTS-CLES : intelligence artificielle, travail, revolution, regulation, emplois RESUME : Les avancees en IA generative posent des questions sur l'avenir professionnel, avec des avis partages entre optimisme et inquietude. Les gouvernements peinent a mettre en place une regulation adequate face a ces transformations rapides. CONCLUSION : Ce texte adopte une position nuancee sur l'IA generative, soulignant a la fois les opportunites et les risques. Les mots-cles revelent une preoccupation centree sur l'emploi et la regulation. L'absence de consensus parmi les experts appelle a un debat public approfondi et a une action politique rapide. ========================================
Ce pipeline illustre la puissance de LCEL : chaque etape est un Runnable independant, testable et reutilisable. Vous pouvez remplacer n'importe quel composant (par exemple, changer le modele, ajouter une branche parallele) sans modifier le reste du pipeline. C'est l'essence de la composabilite LCEL.

Erreurs Courantes

Oublier les accolades dans le template. Ecrire "Traduis en langue" au lieu de "Traduis en {langue}". Sans accolades, la variable ne sera jamais substituee et le LLM recevra le texte brut.
Passer une string au lieu d'un dict a invoke(). Si votre chain attend {"question": "..."}, ne passez pas juste "...". Exception typique : KeyError: 'question'. Seule exception : si votre premiere etape est un RunnablePassthrough qui accepte une string.
Utiliser LLMChain au lieu de LCEL. LLMChain est obsolete depuis LangChain v0.3. Utilisez toujours prompt | model | parser. Les anciennes chains ne supportent ni le streaming natif, ni le batch, ni l'execution asynchrone.
Ne pas appeler .partial() pour les format_instructions. Quand vous utilisez un JsonOutputParser ou PydanticOutputParser, il faut injecter format_instructions dans le prompt avec prompt.partial(). Sinon, le modele ne saura pas quel format JSON produire.
Confondre format() et invoke(). prompt.format() retourne une string (utile pour le debug). prompt.invoke() retourne un PromptValue (necessaire dans une chain LCEL). En chain, c'est toujours invoke() qui est appele automatiquement.
Oublier le StrOutputParser() en fin de chain. Sans parser, le resultat est un objet AIMessage, pas une string. Pour obtenir le texte directement, ajoutez toujours | StrOutputParser() comme derniere etape de votre chain.

Resume : LCEL Cheatsheet

Composant Import Role
PromptTemplate langchain_core.prompts Template de prompt simple avec variables
ChatPromptTemplate langchain_core.prompts Template avec messages structures (system/human/ai)
MessagesPlaceholder langchain_core.prompts Emplacement pour messages dynamiques (historique)
ChatOpenAI langchain_openai Modele de chat OpenAI (gpt-4o, gpt-4o-mini)
StrOutputParser langchain_core.output_parsers Extrait le texte brut de la reponse
JsonOutputParser langchain_core.output_parsers Parse la sortie JSON en dictionnaire Python
PydanticOutputParser langchain_core.output_parsers Valide la sortie contre un schema Pydantic
RunnablePassthrough langchain_core.runnables Passe les donnees sans modification
RunnableParallel langchain_core.runnables Execute plusieurs Runnables en parallele
RunnableLambda langchain_core.runnables Transforme une fonction Python en Runnable

Syntaxe LCEL essentielle

# Chain simple chain = prompt | model | parser # Chain avec entrees paralleles chain = {"a": runnable1, "b": runnable2} | prompt | model | parser # Chain avec transformation personnalisee chain = RunnableLambda(fn) | prompt | model | parser # Methodes disponibles sur toute chain chain.invoke(data) # Execution synchrone chain.stream(data) # Streaming token par token chain.batch([d1, d2]) # Execution en lot await chain.ainvoke(data) # Execution asynchrone

Points cles a retenir

  • LCEL remplace LLMChain : ne plus utiliser les anciennes classes
  • L'operateur | cree un RunnableSequence
  • Tout est Runnable : invoke, stream, batch, ainvoke disponibles partout
  • ChatPromptTemplate est prefere pour les modeles de chat modernes
  • Toujours finir par un parser pour obtenir le format voulu
  • RunnableParallel pour les traitements simultanement, RunnableLambda pour le code custom
