TD Pratique - Documents, Embeddings et VectorStores

LangChain v0.3 -- Pipeline complet : du texte brut a la recherche semantique

1. Recapitulatif du Pipeline

Un VectorStore est une base de donnees specialisee qui stocke des documents sous forme de vecteurs numeriques (embeddings) et permet la recherche par similarite semantique plutot que par mots-cles exacts.

Le pipeline RAG complet suit cinq etapes fondamentales :

Etape 1 - Load : Charger les documents bruts (texte, PDF, CSV, web...) avec des DocumentLoaders.
Etape 2 - Split : Decouper les documents en morceaux (chunks) de taille geree avec des TextSplitters.
Etape 3 - Embed : Convertir chaque chunk en vecteur numerique avec un modele d'Embeddings.
Etape 4 - Store : Stocker les vecteurs dans un VectorStore (FAISS, Chroma, Pinecone...).
Etape 5 - Search : Rechercher les documents les plus pertinents par similarite cosinus.
Ce pipeline est le coeur de tout systeme RAG (Retrieval-Augmented Generation). Chaque etape influence la qualite des resultats finaux. Un mauvais decoupage peut ruiner une excellente recherche.

Exercice 3 - Decoupage avec RecursiveCharacterTextSplitter

Un document entier est souvent trop long pour etre traite par un LLM. Le RecursiveCharacterTextSplitter decoupe intelligemment le texte en preservant la coherence semantique.

Le RecursiveCharacterTextSplitter decoupe le texte en essayant recursivement differents separateurs dans l'ordre : ['\n\n', '\n', ' ', '']. Il tente d'abord de couper aux paragraphes, puis aux lignes, puis aux mots.

Parametres cles du TextSplitter

ParametreDefautDescription
chunk_size1000Taille maximale de chaque chunk (en caracteres)
chunk_overlap200Chevauchement entre chunks consecutifs
separators['\n\n', '\n', ' ', '']Ordre des separateurs a essayer
length_functionlenFonction pour mesurer la taille

FAISS utilise la distance L2 (euclidienne) par defaut. Un score plus bas signifie que les documents sont plus similaires. C'est l'inverse de la similarite cosinus ou un score plus haut est meilleur. Gardez cela en tete lors de l'interpretation des resultats.

Exercice 7 - Comparer les Types de Recherche

LangChain supporte plusieurs strategies de recherche. Chacune a ses avantages selon le contexte.

Similarity : retourne les K documents les plus proches. MMR (Maximal Marginal Relevance) : retourne des documents pertinents ET diversifies. Score Threshold : retourne uniquement les documents au-dessus d'un seuil de similarite.
# Exercice 7 : Comparer similarity, MMR et score_threshold from langchain_community.vectorstores import FAISS from langchain_community.embeddings import HuggingFaceEmbeddings from langchain_core.documents import Document embeddings = HuggingFaceEmbeddings( model_name="sentence-transformers/all-MiniLM-L6-v2" ) # Documents volontairement proches et redondants docs = [ Document(page_content="Python est un langage de programmation tres populaire pour l'IA."), Document(page_content="Python est le langage prefere des data scientists et chercheurs en IA."), Document(page_content="Python offre des bibliotheques comme TensorFlow et PyTorch pour le ML."), Document(page_content="JavaScript est utilise pour le developpement web front-end et back-end."), Document(page_content="R est un langage specialise dans les statistiques et la visualisation."), Document(page_content="Java est utilise en entreprise pour des applications robustes et scalables."), ] vectorstore = FAISS.from_documents(docs, embeddings) query = "Quel langage utiliser pour l'intelligence artificielle ?" # --- 1. Recherche par similarite classique --- print("=== SIMILARITY SEARCH (k=3) ===") results_sim = vectorstore.similarity_search(query, k=3) for i, doc in enumerate(results_sim): print(f" {i+1}. {doc.page_content}") # --- 2. Recherche MMR (diversite) --- print("\n=== MMR SEARCH (k=3, fetch_k=6, lambda_mult=0.5) ===") results_mmr = vectorstore.max_marginal_relevance_search( query, k=3, fetch_k=6, lambda_mult=0.5 ) for i, doc in enumerate(results_mmr): print(f" {i+1}. {doc.page_content}") # --- 3. Recherche avec seuil de score --- print("\n=== SCORE THRESHOLD (threshold=0.5) ===") retriever = vectorstore.as_retriever( search_type="similarity_score_threshold", search_kwargs={"score_threshold": 0.5} ) results_threshold = retriever.invoke(query) for i, doc in enumerate(results_threshold): print(f" {i+1}. {doc.page_content}") if not results_threshold: print(" Aucun document au-dessus du seuil.")
=== SIMILARITY SEARCH (k=3) === 1. Python est un langage de programmation tres populaire pour l'IA. 2. Python est le langage prefere des data scientists et chercheurs en IA. 3. Python offre des bibliotheques comme TensorFlow et PyTorch pour le ML. === MMR SEARCH (k=3, fetch_k=6, lambda_mult=0.5) === 1. Python est un langage de programmation tres populaire pour l'IA. 2. R est un langage specialise dans les statistiques et la visualisation. 3. Python offre des bibliotheques comme TensorFlow et PyTorch pour le ML. === SCORE THRESHOLD (threshold=0.5) === 1. Python est un langage de programmation tres populaire pour l'IA. 2. Python est le langage prefere des data scientists et chercheurs en IA.
La recherche par similarite retourne 3 resultats Python (redondants). MMR introduit de la diversite en incluant R. Le seuil de score ne retourne que les documents tres pertinents. Choisissez votre strategie selon le besoin.

lambda_mult0.0 = diversite maximale, 1.0 = pertinence maximale (defaut: 0.5)

Exercice 8 - Filtrage par Metadonnees

Les metadonnees permettent de filtrer les resultats de recherche sans affecter la recherche semantique elle-meme. C'est essentiel pour restreindre les resultats a une source, une date ou une categorie specifique.

Le filtrage par metadonnees est un outil puissant en production. Par exemple, dans un chatbot de support client, vous pouvez filtrer par version de produit pour ne retourner que la documentation pertinente. Combinez toujours filtrage et recherche semantique pour des resultats precis.

Erreurs Courantes

chunk_size trop grand : Si vos chunks font 5000+ caracteres, la recherche semantique sera imprecise car trop de sujets seront melanges dans un seul vecteur. Visez 200 a 1000 caracteres selon le contenu.
chunk_overlap a zero : Sans chevauchement, les informations a la frontiere de deux chunks sont perdues. Utilisez toujours un overlap de 10 a 20% du chunk_size (ex: chunk_size=500, overlap=50 a 100).
Oublier de persister le VectorStore : Avec FAISS, si vous ne faites pas vectorstore.save_local("dossier"), toutes les donnees sont perdues a la fermeture du programme. Avec Chroma, specifiez toujours un persist_directory.
Melanger les modeles d'embeddings : Si vous indexez avec all-MiniLM-L6-v2 (384 dimensions) et cherchez avec text-embedding-3-small (1536 dimensions), les resultats seront absurdes. Utilisez toujours le meme modele pour l'indexation et la recherche.
Ne pas gerer l'encodage : Lors du chargement de fichiers avec TextLoader, specifiez toujours encoding="utf-8" pour eviter les erreurs avec les caracteres accentues en francais.
Ignorer les metadonnees : Ne pas ajouter de metadonnees rend impossible le filtrage et la tracabilite. Ajoutez au minimum source, date, et un identifiant unique a chaque document.
