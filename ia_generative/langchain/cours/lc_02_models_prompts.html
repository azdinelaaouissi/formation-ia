Models et PromptTemplates

Les Fondations de LangChain -- Cours 2/2

1. ChatModels : l'interface unifiée vers les LLM

Un ChatModel dans LangChain est un wrapper qui encapsule l'API d'un fournisseur de LLM derrière une interface standardisée. Il prend en entrée une liste de messages (système, humain, IA) et retourne un objet AIMessage contenant la réponse du modèle.

Les ChatModels sont la brique fondamentale de toute application LangChain. Ils héritent de l'interface Runnable, ce qui signifie qu'ils supportent nativement invoke(), stream(), batch(), ainsi que leurs variantes asynchrones.

1.1 ChatOpenAI en détail

from langchain_openai import ChatOpenAI # Instanciation avec tous les paramètres courants llm = ChatOpenAI( model="gpt-4o-mini", # Nom du modèle temperature=0.7, # Créativité : 0 (déterministe) à 2 (très créatif) max_tokens=1000, # Nombre max de tokens en sortie top_p=1.0, # Nucleus sampling (alternative à temperature) frequency_penalty=0.0, # Pénalité de fréquence (-2 à 2) presence_penalty=0.0, # Pénalité de présence (-2 à 2) timeout=60, # Timeout en secondes max_retries=2, # Tentatives en cas d'erreur réseau api_key="sk-...", # Clé API (ou OPENAI_API_KEY env var) streaming=True, # Activer le streaming par défaut ) # Vérification des paramètres print(f"Modèle : {llm.model_name}") print(f"Température : {llm.temperature}") print(f"Max tokens : {llm.max_tokens}")
Modèle : gpt-4o-mini Température : 0.7 Max tokens : 1000

Paramètres clés de ChatOpenAI

Paramètre Type Défaut Description
model str "gpt-4o-mini" Identifiant du modèle OpenAI à utiliser
temperature float 0.7 Contrôle la créativité. 0 = déterministe, 1+ = plus aléatoire
max_tokens int None Limite le nombre de tokens en sortie. Converti en max_completion_tokens depuis sept. 2024
top_p float 1.0 Nucleus sampling. Alternative à temperature (ne pas combiner les deux)
timeout int None Délai d'attente maximal en secondes
max_retries int 2 Nombre de tentatives automatiques en cas d'erreur réseau
streaming bool False Active le streaming par défaut pour toutes les invocations
stream_usage bool False Inclut les métadonnées d'usage dans les chunks de streaming

1.2 ChatAnthropic en détail

from langchain_anthropic import ChatAnthropic # Anthropic a une API légèrement différente llm = ChatAnthropic( model="claude-sonnet-4-20250514", # Modèle Claude temperature=0.7, # Même concept que OpenAI max_tokens=1024, # OBLIGATOIRE pour Anthropic ! top_p=1.0, timeout=60, max_retries=2, api_key="sk-ant-...", # Ou ANTHROPIC_API_KEY env var ) # Utilisation identique à ChatOpenAI from langchain_core.messages import HumanMessage response = llm.invoke([HumanMessage(content="Qu'est-ce que le deep learning ?")]) print(response.content)
Le deep learning (apprentissage profond) est un sous-domaine du machine learning qui utilise des réseaux de neurones artificiels avec de nombreuses couches cachées. Ces architectures profondes sont capables d'apprendre des représentations hiérarchiques des données, ce qui les rend particulièrement efficaces pour des tâches comme la reconnaissance d'images, le traitement du langage naturel et la génération de contenu.
Contrairement à OpenAI, le paramètre max_tokens est obligatoire pour ChatAnthropic. Si vous l'omettez, vous obtiendrez une erreur. Utilisez au minimum 1024 pour la plupart des cas d'usage.

1.4 Méthodes communes des ChatModels

from langchain_openai import ChatOpenAI from langchain_core.messages import HumanMessage llm = ChatOpenAI(model="gpt-4o-mini", temperature=0) messages = [HumanMessage(content="Qu'est-ce qu'une API ?")] # --- invoke : appel unique synchrone --- response = llm.invoke(messages) print("invoke:", response.content[:80], "...") # --- stream : réponse token par token --- print("\nstream:", end=" ") for chunk in llm.stream(messages): print(chunk.content, end="", flush=True) print() # --- batch : traitement par lot (plusieurs requêtes en parallèle) --- questions = [ [HumanMessage(content="Capital de la France ?")], [HumanMessage(content="Capital de l'Allemagne ?")], [HumanMessage(content="Capital de l'Italie ?")], ] responses = llm.batch(questions) for r in responses: print(f"batch: {r.content}") # --- ainvoke : appel asynchrone (dans un contexte async) --- import asyncio async def main(): response = await llm.ainvoke(messages) print(f"ainvoke: {response.content[:80]}...") asyncio.run(main())
invoke: Une API (Application Programming Interface) est une interface de programmation ... stream: Une API (Application Programming Interface) est une interface... batch: La capitale de la France est Paris. batch: La capitale de l'Allemagne est Berlin. batch: La capitale de l'Italie est Rome. ainvoke: Une API (Application Programming Interface) est une interface de programmation ...

2. Les Messages : la structure des conversations

Dans LangChain, une conversation est modélisée comme une liste de messages typés. Chaque message a un rôle (système, humain, IA, outil) et un contenu. Cette structure reflète le format utilisé par les APIs de chat des LLM modernes.

2.1 Les quatre types de messages

2.2 Code complet avec messages

from langchain_openai import ChatOpenAI from langchain_core.messages import ( SystemMessage, HumanMessage, AIMessage, ToolMessage, ) llm = ChatOpenAI(model="gpt-4o-mini", temperature=0) # Conversation structurée avec des messages typés messages = [ # Message système : définit le comportement de l'IA SystemMessage(content="""Tu es un professeur de Python expert. Tu réponds toujours en français. Tu donnes des exemples de code concrets. Tu expliques comme si tu parlais à un débutant."""), # Premier échange HumanMessage(content="Qu'est-ce qu'une liste en Python ?"), AIMessage(content="Une liste en Python est une collection ordonnée et modifiable d'éléments. On la crée avec des crochets : ma_liste = [1, 2, 3]"), # Deuxième échange (le modèle voit tout l'historique) HumanMessage(content="Comment ajouter un élément ?"), ] # Le modèle comprend le contexte grâce à l'historique response = llm.invoke(messages) print(response.content)
Pour ajouter un élément à une liste en Python, vous avez plusieurs méthodes : 1. **append()** - Ajoute un élément à la fin : ```python ma_liste = [1, 2, 3] ma_liste.append(4) print(ma_liste) # [1, 2, 3, 4] ``` 2. **insert()** - Ajoute un élément à une position spécifique : ```python ma_liste.insert(0, 0) # Ajoute 0 au début print(ma_liste) # [0, 1, 2, 3, 4] ``` 3. **extend()** - Ajoute plusieurs éléments d'un coup : ```python ma_liste.extend([5, 6]) print(ma_liste) # [0, 1, 2, 3, 4, 5, 6] ```
Le modèle répond "à une liste en Python" alors que la deuxième question ne mentionne ni "liste" ni "Python". C'est parce qu'il a accès à tout l'historique de conversation. C'est ainsi que fonctionne la "mémoire" dans les applications LLM : on renvoie les messages précédents.

2.3 Syntaxe raccourcie avec des tuples

LangChain accepte aussi des tuples (role, content) pour une syntaxe plus concise :

from langchain_openai import ChatOpenAI llm = ChatOpenAI(model="gpt-4o-mini") # Syntaxe avec tuples (role, content) - plus concise messages = [ ("system", "Tu es un historien spécialisé dans l'Antiquité. Réponds en français."), ("human", "Qui était Jules César ?"), ("ai", "Jules César était un général et homme d'État romain né en 100 av. J.-C."), ("human", "Comment est-il mort ?"), ] response = llm.invoke(messages) print(response.content)
Jules César a été assassiné le 15 mars 44 av. J.-C. (les Ides de mars) par un groupe de sénateurs romains conspirateurs, dont Brutus et Cassius. Il a été poignardé 23 fois dans le théâtre de Pompée, lors d'une séance du Sénat.
Les rôles valides pour les tuples sont : "system", "human" (ou "user"), "ai" (ou "assistant"), et "tool". Cette syntaxe est pratique pour le prototypage rapide, mais préférez les classes SystemMessage/HumanMessage dans du code de production pour plus de clarté.

3. PromptTemplate : les templates basiques

Un PromptTemplate est un objet qui permet de créer des prompts réutilisables avec des variables. Il prend un template avec des placeholders {variable} et les remplace par des valeurs concrètes lors de l'invocation. C'est le composant essentiel pour séparer la logique du prompt de la logique applicative.

3.1 Création basique avec from_template

from langchain_core.prompts import PromptTemplate # Méthode 1 : from_template (détecte les variables automatiquement) prompt = PromptTemplate.from_template( "Donne-moi {nombre} faits intéressants sur {sujet}." ) # Vérifier les variables détectées print("Variables :", prompt.input_variables) # Formater le prompt result = prompt.invoke({ "nombre": "5", "sujet": "la Tour Eiffel" }) print("Prompt formaté :", result) print("Type :", type(result))
Variables : ['nombre', 'sujet'] Prompt formaté : text='Donne-moi 5 faits intéressants sur la Tour Eiffel.' Type : <class 'langchain_core.prompt_values.StringPromptValue'>

3.2 Création explicite avec le constructeur

from langchain_core.prompts import PromptTemplate # Méthode 2 : Constructeur explicite (contrôle total) prompt = PromptTemplate( template="Traduis le texte suivant en {langue} :\n\n{texte}\n\nTraduction :", input_variables=["langue", "texte"], ) result = prompt.format( langue="anglais", texte="Bonjour, comment allez-vous aujourd'hui ?" ) print(result)
Traduis le texte suivant en anglais : Bonjour, comment allez-vous aujourd'hui ? Traduction :

3.3 Template multilignes complexes

from langchain_core.prompts import PromptTemplate from langchain_openai import ChatOpenAI from langchain_core.output_parsers import StrOutputParser # Template complexe pour l'analyse de code prompt = PromptTemplate.from_template("""Tu es un expert en {langage}. Analyse le code suivant et fournis : 1. Une explication de ce que fait le code 2. Les bugs potentiels 3. Des suggestions d'amélioration Code à analyser : ```{langage} {code} ``` Analyse détaillée :""") llm = ChatOpenAI(model="gpt-4o-mini", temperature=0) parser = StrOutputParser() chain = prompt | llm | parser result = chain.invoke({ "langage": "Python", "code": """def fibonacci(n): if n <= 1: return n return fibonacci(n-1) + fibonacci(n-2)""" }) print(result)
**1. Explication :** Cette fonction calcule le n-ième nombre de Fibonacci de manière récursive. Elle utilise le cas de base (n <= 1) et l'appel récursif pour calculer la somme des deux termes précédents. **2. Bugs potentiels :** - Pas de validation de l'entrée (n négatif, n non entier) - Complexité exponentielle O(2^n) : très lent pour n > 35 **3. Suggestions d'amélioration :** - Utiliser la mémoïsation avec @functools.lru_cache - Ou une approche itérative pour O(n) en temps et O(1) en espace

4. ChatPromptTemplate : templates pour les modèles de chat

ChatPromptTemplate est une version spécialisée de PromptTemplate conçue pour les modèles de chat. Au lieu de produire une simple chaîne de texte, il génère une liste de messages typés (System, Human, AI). C'est le template recommandé pour tous les ChatModels modernes.

4.1 Création avec from_messages

from langchain_core.prompts import ChatPromptTemplate # ChatPromptTemplate avec des messages structurés prompt = ChatPromptTemplate.from_messages([ # Message système (contexte global, personnalité) ("system", "Tu es un assistant spécialisé en {domaine}. Réponds en français, de manière {style}."), # Message humain (la requête avec variables) ("human", "{question}"), ]) # Vérifier les variables print("Variables :", prompt.input_variables) # Formater messages = prompt.invoke({ "domaine": "la cybersécurité", "style": "concise et technique", "question": "Qu'est-ce qu'une attaque XSS ?" }) # Afficher les messages générés for msg in messages.messages: print(f"[{msg.type}] {msg.content}")
[system] Tu es un assistant spécialisé en la cybersécurité. Réponds en français, de manière concise et technique. [human] Qu'est-ce qu'une attaque XSS ?

4.2 Chaîne complète avec ChatPromptTemplate

from langchain_core.prompts import ChatPromptTemplate from langchain_openai import ChatOpenAI from langchain_core.output_parsers import StrOutputParser # Template pour un assistant de rédaction prompt = ChatPromptTemplate.from_messages([ ("system", """Tu es un rédacteur professionnel. Ton style d'écriture est : {style}. Tu écris toujours en français. Tu structures tes textes avec des paragraphes clairs."""), ("human", "Rédige un {type_texte} sur le sujet suivant : {sujet}"), ]) llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.8) parser = StrOutputParser() # Chaîne LCEL chain = prompt | llm | parser # Invocation result = chain.invoke({ "style": "journalistique, engagé", "type_texte": "article court (200 mots)", "sujet": "L'impact de l'IA sur l'éducation", }) print(result)
L'intelligence artificielle bouleverse l'éducation à une vitesse vertigineuse. Des salles de classe aux amphithéâtres universitaires, les outils alimentés par l'IA transforment la manière dont nous apprenons et enseignons. Les tuteurs IA personnalisés s'adaptent au rythme de chaque élève, identifiant ses lacunes et proposant des exercices ciblés. Les enseignants, libérés des tâches répétitives de correction, peuvent enfin se consacrer à ce qui compte vraiment : l'accompagnement humain. Mais ne soyons pas naïfs. Cette révolution soulève des questions cruciales. Comment garantir l'équité d'accès ? Comment prévenir la dépendance des étudiants à ces outils ? L'esprit critique, la créativité, l'argumentation -- ces compétences fondamentales ne s'enseignent pas par algorithme. L'IA dans l'éducation n'est ni une menace ni une panacée. C'est un outil puissant qui, bien encadré, peut démocratiser le savoir. À condition de ne jamais oublier que l'éducation est, avant tout, une aventure humaine.

4.3 Template avec message IA (pre-remplissage)

from langchain_core.prompts import ChatPromptTemplate # On peut inclure un message AI pour "amorcer" la réponse prompt = ChatPromptTemplate.from_messages([ ("system", "Tu es un assistant qui génère des données JSON valides."), ("human", "Génère un objet JSON pour un {objet} avec les champs : {champs}"), ("ai", "```json\n{{"), # Amorce la réponse en JSON ]) # Le modèle continuera naturellement avec du JSON valide print(prompt.input_variables) # ['objet', 'champs']
L'astuce du pré-remplissage du message IA est très puissante. En commençant la réponse de l'IA par ```json\n{, vous forcez le modèle à continuer en JSON. Cette technique fonctionne particulièrement bien avec Claude (Anthropic), qui supporte nativement les "prefilled responses".

5. MessagesPlaceholder : injecter l'historique

MessagesPlaceholder est un composant spécial qui réserve une place dans le template pour une liste de messages dynamiques. Il est principalement utilisé pour injecter l'historique de conversation (mémoire) dans le prompt.

5.1 Utilisation basique

from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder from langchain_core.messages import HumanMessage, AIMessage # Template avec un placeholder pour l'historique prompt = ChatPromptTemplate.from_messages([ ("system", "Tu es un assistant utile. Réponds en français."), # Placeholder : ici seront injectés les messages précédents MessagesPlaceholder(variable_name="historique"), ("human", "{question}"), ]) print("Variables :", prompt.input_variables) # ['historique', 'question'] # Simuler un historique de conversation historique = [ HumanMessage(content="Je m'appelle Alice."), AIMessage(content="Bonjour Alice ! Comment puis-je vous aider ?"), HumanMessage(content="J'apprends Python."), AIMessage(content="Excellent choix ! Python est un langage très accessible."), ] # Formater le prompt avec l'historique messages = prompt.invoke({ "historique": historique, "question": "Quel est mon prénom ?", }) # Vérifier les messages générés for msg in messages.messages: print(f"[{msg.type}] {msg.content[:60]}...")
Variables : ['historique', 'question'] [system] Tu es un assistant utile. Réponds en français.... [human] Je m'appelle Alice.... [ai] Bonjour Alice ! Comment puis-je vous aider ?... [human] J'apprends Python.... [ai] Excellent choix ! Python est un langage très accessible.... [human] Quel est mon prénom ?...

5.2 MessagesPlaceholder avec optional

from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder # Le placeholder peut être optionnel (pas d'historique au premier tour) prompt = ChatPromptTemplate.from_messages([ ("system", "Tu es un assistant technique."), MessagesPlaceholder(variable_name="historique", optional=True), ("human", "{question}"), ]) # Premier appel : pas d'historique messages_1 = prompt.invoke({ "question": "Qu'est-ce que Docker ?", # Pas besoin de fournir 'historique' grâce à optional=True }) print(f"Nombre de messages (1er appel) : {len(messages_1.messages)}") # Deuxième appel : avec historique messages_2 = prompt.invoke({ "historique": [ ("human", "Qu'est-ce que Docker ?"), ("ai", "Docker est une plateforme de conteneurisation..."), ], "question": "Comment l'installer ?", }) print(f"Nombre de messages (2e appel) : {len(messages_2.messages)}")
Nombre de messages (1er appel) : 2 Nombre de messages (2e appel) : 4
Avec optional=True, le placeholder est ignoré si la variable n'est pas fournie. C'est idéal pour les chatbots où le premier message n'a pas d'historique. Sans cette option, omettre la variable historique provoquerait une erreur.

5.3 Chaîne complète avec mémoire manuelle

from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder from langchain_openai import ChatOpenAI from langchain_core.output_parsers import StrOutputParser from langchain_core.messages import HumanMessage, AIMessage # Prompt avec placeholder pour l'historique prompt = ChatPromptTemplate.from_messages([ ("system", "Tu es un assistant mathématique pédagogue."), MessagesPlaceholder(variable_name="history", optional=True), ("human", "{input}"), ]) llm = ChatOpenAI(model="gpt-4o-mini", temperature=0) chain = prompt | llm | StrOutputParser() # Simuler une conversation multi-tours history = [] # Tour 1 question1 = "Qu'est-ce qu'une dérivée ?" response1 = chain.invoke({"input": question1, "history": history}) print(f"Q1: {question1}") print(f"R1: {response1[:100]}...\n") # Ajouter à l'historique history.append(HumanMessage(content=question1)) history.append(AIMessage(content=response1)) # Tour 2 (le modèle voit le tour 1) question2 = "Donne-moi un exemple concret." response2 = chain.invoke({"input": question2, "history": history}) print(f"Q2: {question2}") print(f"R2: {response2[:100]}...")

Cette gestion manuelle de l'historique fonctionne parfaitement pour des cas simples. Pour des chatbots en production, utilisez les abstractions de mémoire de LangChain ou, mieux encore, LangGraph qui gère l'état de manière plus robuste avec un graphe de conversation.

6. OutputParsers : structurer les sorties du LLM

Les OutputParsers transforment la sortie brute du LLM (texte libre) en un format structuré utilisable par votre application. Ils sont le dernier maillon de la chaîne LCEL prompt | model | parser.

6.1 StrOutputParser

Le parser le plus simple : il extrait la propriété .content de l'AIMessage pour retourner une chaîne de caractères brute.

from langchain_core.output_parsers import StrOutputParser from langchain_openai import ChatOpenAI from langchain_core.prompts import ChatPromptTemplate prompt = ChatPromptTemplate.from_template( "Résume en une phrase : {texte}" ) llm = ChatOpenAI(model="gpt-4o-mini") parser = StrOutputParser() chain = prompt | llm | parser result = chain.invoke({ "texte": "LangChain est un framework open source pour créer des applications avec des LLM. Il fournit des abstractions pour les prompts, les chaînes, la mémoire et les agents." }) print(result) print(f"Type : {type(result)}") # str
LangChain est un framework open source fournissant des abstractions pour construire des applications LLM avec prompts, chaînes, mémoire et agents. Type : <class 'str'>

6.2 JsonOutputParser

Parse la sortie du LLM en tant que dictionnaire Python. Il peut fonctionner avec ou sans schéma Pydantic.

from langchain_core.output_parsers import JsonOutputParser from langchain_openai import ChatOpenAI from langchain_core.prompts import ChatPromptTemplate # JsonOutputParser sans schéma (JSON libre) parser = JsonOutputParser() prompt = ChatPromptTemplate.from_messages([ ("system", "Tu génères toujours des réponses en JSON valide."), ("human", """Génère les informations suivantes pour la ville {ville} : - nom - pays - population (approximative) - langue_principale - continent {format_instructions}"""), ]) llm = ChatOpenAI(model="gpt-4o-mini", temperature=0) chain = prompt | llm | parser result = chain.invoke({ "ville": "Tokyo", "format_instructions": parser.get_format_instructions(), }) print(f"Type : {type(result)}") # dict print(f"Nom : {result['nom']}") print(f"Population : {result['population']}") import json print("\nJSON complet :") print(json.dumps(result, indent=2, ensure_ascii=False))
Type : <class 'dict'> Nom : Tokyo Population : 14000000 JSON complet : { "nom": "Tokyo", "pays": "Japon", "population": 14000000, "langue_principale": "japonais", "continent": "Asie" }

6.3 PydanticOutputParser

Le parser le plus robuste : il valide la sortie contre un schéma Pydantic, garantissant le typage et la structure.

from langchain_core.output_parsers import PydanticOutputParser from langchain_openai import ChatOpenAI from langchain_core.prompts import ChatPromptTemplate from pydantic import BaseModel, Field from typing import List # 1. Définir le schéma Pydantic class Recette(BaseModel): """Schéma d'une recette de cuisine.""" nom: str = Field(description="Le nom de la recette") temps_preparation: str = Field(description="Le temps de préparation") difficulte: str = Field(description="Niveau de difficulté (facile, moyen, difficile)") ingredients: List[str] = Field(description="Liste des ingrédients") etapes: List[str] = Field(description="Les étapes de préparation") calories: int = Field(description="Nombre approximatif de calories par portion") # 2. Créer le parser parser = PydanticOutputParser(pydantic_object=Recette) # 3. Créer le prompt avec les instructions de format prompt = ChatPromptTemplate.from_messages([ ("system", "Tu es un chef cuisinier expert. Génère des recettes détaillées."), ("human", """Donne-moi une recette pour : {plat} {format_instructions}"""), ]) llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.5) # 4. Chaîne complète chain = prompt | llm | parser # 5. Invocation recette = chain.invoke({ "plat": "crêpes bretonnes", "format_instructions": parser.get_format_instructions(), }) # Le résultat est un objet Pydantic typé ! print(f"Type : {type(recette).__name__}") print(f"Nom : {recette.nom}") print(f"Difficulté : {recette.difficulte}") print(f"Temps : {recette.temps_preparation}") print(f"Calories : {recette.calories} kcal") print(f"Ingrédients : {len(recette.ingredients)}") for i, etape in enumerate(recette.etapes, 1): print(f" Étape {i}: {etape[:60]}...")
Type : Recette Nom : Crêpes Bretonnes Traditionnelles Difficulté : facile Temps : 30 minutes (+ 1h de repos) Calories : 180 kcal Ingrédients : 7 Étape 1: Mélanger la farine et le sel dans un grand saladier. Creuse... Étape 2: Ajouter les oeufs un à un en mélangeant bien après chaque... Étape 3: Verser progressivement le lait tout en fouettant pour évit... Étape 4: Laisser reposer la pâte au réfrigérateur pendant au moins... Étape 5: Chauffer une poêle antiadhésive (ou une billig) à feu moy...

Avantage du PydanticOutputParser

Le résultat est un objet Python typé, pas un simple dictionnaire. Cela signifie :

  • Autocomplétion dans votre IDE (recette.nom, recette.ingredients)
  • Validation automatique : si le LLM ne fournit pas un champ obligatoire, une erreur est levée
  • Typage vérifiable : recette.calories est garanti être un int
  • Sérialisation facile : recette.model_dump() retourne un dictionnaire

6.4 Alternative moderne : with_structured_output

Depuis LangChain v0.3, les ChatModels offrent une méthode with_structured_output() qui est souvent plus simple que d'utiliser un PydanticOutputParser :

from langchain_openai import ChatOpenAI from pydantic import BaseModel, Field from typing import List class Personne(BaseModel): """Informations sur une personne.""" nom: str = Field(description="Le nom complet") age: int = Field(description="L'âge estimé") profession: str = Field(description="La profession") competences: List[str] = Field(description="Liste des compétences principales") # with_structured_output utilise le function calling du LLM llm = ChatOpenAI(model="gpt-4o-mini", temperature=0) structured_llm = llm.with_structured_output(Personne) # Pas besoin de format_instructions ni de parser ! result = structured_llm.invoke( "Décris un développeur Python senior nommé Jean Dupont, 35 ans." ) print(f"Type : {type(result).__name__}") print(f"Nom : {result.nom}") print(f"Age : {result.age}") print(f"Profession : {result.profession}") print(f"Compétences : {result.competences}")
Type : Personne Nom : Jean Dupont Age : 35 Profession : Développeur Python Senior Compétences : ['Python', 'Django', 'FastAPI', 'PostgreSQL', 'Docker', 'CI/CD']
La méthode with_structured_output() utilise le "function calling" / "tool use" natif du LLM, ce qui est plus fiable que de demander au modèle de générer du JSON brut. Cependant, tous les modèles ne supportent pas cette fonctionnalité. Vérifiez la documentation de votre provider.

7. Assembler le tout : chaînes complètes

Maintenant que nous maîtrisons chaque composant individuellement, combinons-les dans des chaînes LCEL plus complexes et réalistes.

7.1 Chaîne simple : traducteur automatique

from langchain_core.prompts import ChatPromptTemplate from langchain_openai import ChatOpenAI from langchain_core.output_parsers import StrOutputParser prompt = ChatPromptTemplate.from_messages([ ("system", """Tu es un traducteur professionnel. Tu traduis fidèlement de {langue_source} vers {langue_cible}. Tu conserves le ton et le style du texte original. Tu retournes uniquement la traduction, sans commentaire."""), ("human", "{texte}"), ]) chain = prompt | ChatOpenAI(model="gpt-4o-mini", temperature=0.3) | StrOutputParser() result = chain.invoke({ "langue_source": "français", "langue_cible": "anglais", "texte": "L'intelligence artificielle transforme notre manière de travailler et de communiquer.", }) print(result)
Artificial intelligence is transforming the way we work and communicate.

7.2 Chaîne séquentielle : analyse puis résumé

from langchain_core.prompts import ChatPromptTemplate from langchain_openai import ChatOpenAI from langchain_core.output_parsers import StrOutputParser from langchain_core.runnables import RunnablePassthrough llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.5) parser = StrOutputParser() # Chaîne 1 : Analyser un code prompt_analyse = ChatPromptTemplate.from_messages([ ("system", "Tu es un expert en revue de code. Analyse le code fourni en détail."), ("human", "Analyse ce code Python :\n```python\n{code}\n```"), ]) # Chaîne 2 : Résumer l'analyse prompt_resume = ChatPromptTemplate.from_messages([ ("system", "Tu résumes des analyses techniques en 3 bullet points maximum."), ("human", "Résume cette analyse :\n{analyse}"), ]) # Composition séquentielle chain_analyse = prompt_analyse | llm | parser chain_resume = prompt_resume | llm | parser # Pipeline complet chain_complete = ( {"analyse": chain_analyse} # Le résultat de chain_analyse est injecté dans "analyse" | chain_resume ) result = chain_complete.invoke({ "code": """def binary_search(arr, target): left, right = 0, len(arr) - 1 while left <= right: mid = (left + right) // 2 if arr[mid] == target: return mid elif arr[mid] < target: left = mid + 1 else: right = mid - 1 return -1""" }) print(result)
- **Algorithme correct** : implémentation classique et efficace de la recherche binaire avec une complexité O(log n) - **Manque de robustesse** : pas de validation des entrées (tableau vide, tableau non trié, type de target incorrect) - **Amélioration possible** : ajouter des type hints, une docstring et une gestion des cas limites pour une meilleure maintenabilité

7.3 Chaîne parallèle puis fusion

from langchain_core.prompts import ChatPromptTemplate from langchain_openai import ChatOpenAI from langchain_core.output_parsers import StrOutputParser from langchain_core.runnables import RunnableParallel llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.7) parser = StrOutputParser() # Trois perspectives différentes sur un même sujet prompt_technique = ChatPromptTemplate.from_template( "Donne une analyse TECHNIQUE (3 lignes) de : {sujet}" ) prompt_business = ChatPromptTemplate.from_template( "Donne une analyse BUSINESS (3 lignes) de : {sujet}" ) prompt_ethique = ChatPromptTemplate.from_template( "Donne une analyse ÉTHIQUE (3 lignes) de : {sujet}" ) # Exécution en parallèle analyses = RunnableParallel( technique=prompt_technique | llm | parser, business=prompt_business | llm | parser, ethique=prompt_ethique | llm | parser, ) # Fusion des résultats prompt_synthese = ChatPromptTemplate.from_template( """Fais une synthèse en 5 lignes combinant ces 3 analyses : Analyse technique : {technique} Analyse business : {business} Analyse éthique : {ethique}""" ) chain = analyses | prompt_synthese | llm | parser result = chain.invoke({"sujet": "la reconnaissance faciale"}) print(result)
La reconnaissance faciale repose sur des algorithmes de deep learning (CNN) capables d'identifier des visages avec une précision supérieure à 99%, ce qui en fait une technologie mature. Sur le plan business, le marché devrait atteindre 12 milliards de dollars d'ici 2028, porté par la sécurité, le commerce et l'authentification. Cependant, les biais algorithmiques persistants (notamment envers les minorités) et les risques de surveillance de masse soulèvent de graves questions éthiques. L'enjeu est de trouver l'équilibre entre innovation technologique, opportunité commerciale et respect des libertés individuelles. Une régulation solide et transparente est indispensable pour encadrer le déploiement de cette technologie.
Ce pattern "analyse parallèle puis synthèse" est extrêmement utile en production. Les trois branches s'exécutent en parallèle (réduction du temps de réponse), puis une dernière étape fusionne les résultats. C'est une forme de "raisonnement multi-perspective" qui produit des réponses plus nuancées qu'un seul prompt.

8. Few-shot Prompts : apprendre par l'exemple

Le few-shot prompting consiste à fournir au LLM quelques exemples (entrée/sortie attendue) dans le prompt pour guider sa réponse. LangChain propose FewShotPromptTemplate et FewShotChatMessagePromptTemplate pour structurer cette approche.

8.1 Few-shot basique avec ChatPromptTemplate

from langchain_core.prompts import ChatPromptTemplate from langchain_openai import ChatOpenAI from langchain_core.output_parsers import StrOutputParser # Few-shot : exemples directement dans les messages prompt = ChatPromptTemplate.from_messages([ ("system", "Tu es un classificateur de sentiments. Réponds uniquement par : positif, négatif, ou neutre."), # Exemples (few-shot) ("human", "Ce film était absolument magnifique !"), ("ai", "positif"), ("human", "Le service client est catastrophique."), ("ai", "négatif"), ("human", "La réunion est prévue à 14h."), ("ai", "neutre"), # La vraie question ("human", "{texte}"), ]) chain = prompt | ChatOpenAI(model="gpt-4o-mini", temperature=0) | StrOutputParser() # Tests textes = [ "J'adore cette nouvelle fonctionnalité !", "Ce produit est une vraie déception.", "Le colis sera livré demain matin.", "Pas mal, mais il y a mieux sur le marché.", ] for texte in textes: sentiment = chain.invoke({"texte": texte}) print(f"'{texte}' → {sentiment}")
'J'adore cette nouvelle fonctionnalité !' → positif 'Ce produit est une vraie déception.' → négatif 'Le colis sera livré demain matin.' → neutre 'Pas mal, mais il y a mieux sur le marché.' → négatif

8.2 FewShotChatMessagePromptTemplate

Pour une gestion plus flexible des exemples, utilisez le template dédié :

from langchain_core.prompts import ( ChatPromptTemplate, FewShotChatMessagePromptTemplate, ) from langchain_openai import ChatOpenAI from langchain_core.output_parsers import StrOutputParser # Définir les exemples comme une liste de dicts examples = [ {"input": "2 + 2", "output": "4"}, {"input": "la capitale de la France", "output": "Paris"}, {"input": "le contraire de grand", "output": "petit"}, ] # Template pour chaque exemple example_prompt = ChatPromptTemplate.from_messages([ ("human", "{input}"), ("ai", "{output}"), ]) # Few-shot template few_shot_prompt = FewShotChatMessagePromptTemplate( example_prompt=example_prompt, examples=examples, ) # Template final final_prompt = ChatPromptTemplate.from_messages([ ("system", "Tu réponds de manière ultra-concise (1-3 mots maximum)."), few_shot_prompt, ("human", "{input}"), ]) chain = final_prompt | ChatOpenAI(model="gpt-4o-mini", temperature=0) | StrOutputParser() # Le modèle a appris le format de réponse concise print(chain.invoke({"input": "la couleur du ciel"})) print(chain.invoke({"input": "10 * 5"})) print(chain.invoke({"input": "le synonyme de content"}))
bleu 50 heureux

8.3 Sélection dynamique des exemples

from langchain_core.prompts import ChatPromptTemplate, FewShotChatMessagePromptTemplate from langchain_core.example_selectors import LengthBasedExampleSelector from langchain_core.prompts import PromptTemplate # Beaucoup d'exemples disponibles examples = [ {"input": "Bonjour", "output": "Hello"}, {"input": "Merci beaucoup", "output": "Thank you very much"}, {"input": "Au revoir", "output": "Goodbye"}, {"input": "Comment allez-vous ?", "output": "How are you?"}, {"input": "S'il vous plaît", "output": "Please"}, {"input": "Bonne nuit", "output": "Good night"}, ] # Sélecteur basé sur la longueur (garde les exemples courts) example_selector = LengthBasedExampleSelector( examples=examples, example_prompt=PromptTemplate.from_template("{input} → {output}"), max_length=50, # Limite de mots pour les exemples sélectionnés ) # Le sélecteur choisit automatiquement les exemples selected = example_selector.select_examples({"input": "Bonsoir"}) print(f"Exemples sélectionnés : {len(selected)}") for ex in selected: print(f" {ex['input']} → {ex['output']}")
Exemples sélectionnés : 4 Bonjour → Hello Au revoir → Goodbye S'il vous plaît → Please Bonne nuit → Good night
En production, le SemanticSimilarityExampleSelector est bien plus puissant : il utilise les embeddings pour sélectionner les exemples les plus similaires à la requête de l'utilisateur. Cela nécessite un vector store, que nous verrons dans le cours sur le RAG.

9. Partial Prompts : variables préremplies

Les partial prompts permettent de préremplir certaines variables d'un template avant de le passer dans une chaîne. C'est utile quand certaines variables sont connues à l'avance (comme la date du jour) et d'autres ne le sont qu'au moment de l'invocation.

9.1 Partial avec des valeurs statiques

from langchain_core.prompts import PromptTemplate # Template avec 3 variables prompt = PromptTemplate.from_template( "Tu es un assistant {role}. La date est {date}. Question : {question}" ) print("Variables avant partial :", prompt.input_variables) # Préremplir certaines variables partial_prompt = prompt.partial( role="médical", date="2 mars 2026", ) print("Variables après partial :", partial_prompt.input_variables) # Il ne reste qu'une seule variable à fournir result = partial_prompt.format(question="Qu'est-ce que la grippe ?") print("\nPrompt final :", result)
Variables avant partial : ['date', 'question', 'role'] Variables après partial : ['question'] Prompt final : Tu es un assistant médical. La date est 2 mars 2026. Question : Qu'est-ce que la grippe ?

9.2 Partial avec des fonctions

Vous pouvez aussi utiliser des fonctions qui seront exécutées au moment du formatage :

from langchain_core.prompts import PromptTemplate from datetime import datetime # Fonction qui retourne la date/heure actuelle def get_datetime(): return datetime.now().strftime("%d/%m/%Y %H:%M") # Template avec partial_variables (fonctions) prompt = PromptTemplate( template="Date actuelle : {datetime}\n\nQuestion : {question}\nRéponse :", input_variables=["question"], partial_variables={"datetime": get_datetime}, # Fonction appelée au formatage ) # La date est calculée dynamiquement result = prompt.format(question="Quel jour sommes-nous ?") print(result)
Date actuelle : 02/03/2026 14:30 Question : Quel jour sommes-nous ? Réponse :

9.3 Partial dans une chaîne LCEL

from langchain_core.prompts import ChatPromptTemplate from langchain_openai import ChatOpenAI from langchain_core.output_parsers import StrOutputParser from datetime import datetime # Template avec variable pré-remplie prompt = ChatPromptTemplate.from_messages([ ("system", "Tu es un assistant. Nous sommes le {date}. Réponds en {langue}."), ("human", "{question}"), ]) # Partial : fixer la langue et la date prompt_fr = prompt.partial( langue="français", date=datetime.now().strftime("%d %B %Y"), ) prompt_en = prompt.partial( langue="anglais", date=datetime.now().strftime("%B %d, %Y"), ) llm = ChatOpenAI(model="gpt-4o-mini") parser = StrOutputParser() # Deux chaînes partant du même template chain_fr = prompt_fr | llm | parser chain_en = prompt_en | llm | parser # Même question, deux langues question = "Quel temps fait-il généralement en mars ?" print("FR:", chain_fr.invoke({"question": question})[:100], "...") print("EN:", chain_en.invoke({"question": question})[:100], "...")
FR: En mars, le temps est généralement variable selon les régions. En France, c'est la transition entre ... EN: In March, the weather is typically transitional between winter and spring. In many temperate region...
Les partial prompts sont très utiles pour créer des "variantes" d'un même template. Vous définissez un template générique une seule fois, puis créez des versions spécialisées avec .partial(). Cela évite la duplication de code et facilite la maintenance.

Guide de décision : quel parser choisir ?

Vous avez besoin de texte libre ? → Utilisez StrOutputParser. C'est le défaut pour les chatbots, résumés, traductions.
Vous avez besoin de données structurées simples ? → Utilisez JsonOutputParser. Idéal pour extraire des informations en format clé-valeur.
Vous avez besoin de validation et typage fort ? → Utilisez PydanticOutputParser ou .with_structured_output(). Recommandé pour la production.
Votre modèle supporte le function calling ? → Préférez .with_structured_output(). Plus fiable car utilise les capacités natives du LLM.
