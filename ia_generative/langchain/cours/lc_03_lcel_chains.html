LCEL et Chains - Composer des Pipelines LLM

Cours 3 - LangChain Expression Language : la colonne vertebrale de LangChain v0.3+

1. Qu'est-ce que LCEL ?

LCEL (LangChain Expression Language) est le langage declaratif de composition de LangChain. Il permet de construire des pipelines complexes en connectant des composants via l'operateur pipe |. Chaque composant est un Runnable : un objet qui accepte une entree, la transforme, et produit une sortie.

Avant LCEL, LangChain utilisait des classes monolithiques comme LLMChain, SequentialChain, SimpleSequentialChain, etc. Ces "legacy chains" presentaient plusieurs problemes :

  • Chaque type de chain avait sa propre API, rendant l'apprentissage difficile
  • Impossible de streamer facilement les tokens du LLM
  • Pas de support natif pour le batch ou l'execution asynchrone
  • La composition de chains etait rigide et peu intuitive
  • Le debugging etait complexe car le flux de donnees etait opaque
Depuis LangChain v0.3, les legacy chains (LLMChain, SequentialChain, etc.) sont officiellement depreciees. LCEL est devenu le standard unique pour composer des pipelines. Tout nouveau projet doit utiliser LCEL.

LCEL resout tous ces problemes grace a une interface unifiee : l'interface Runnable. Chaque composant (prompt, modele, parser, retriever, fonction personnalisee) implemente cette interface, ce qui garantit que tous les composants sont interchangeables et composables.

1.1 L'operateur pipe |

L'operateur | est surcharge via la methode Python __or__. Quand vous ecrivez A | B, cela cree un RunnableSequence qui execute d'abord A, puis passe le resultat a B.

# La syntaxe LCEL la plus basique from langchain_core.prompts import ChatPromptTemplate from langchain_openai import ChatOpenAI from langchain_core.output_parsers import StrOutputParser # Chaque composant est un Runnable prompt = ChatPromptTemplate.from_template("Traduis en anglais : {texte}") model = ChatOpenAI(model="gpt-4o") parser = StrOutputParser() # L'operateur | les connecte en sequence chain = prompt | model | parser # Equivalent explicite : # chain = RunnableSequence(first=prompt, middle=[model], last=parser)
L'operateur | cree un RunnableSequence ou les donnees circulent de gauche a droite. La sortie de chaque composant devient l'entree du suivant.

2. Le concept de Runnable

Un Runnable est tout objet qui implemente l'interface Runnable de langchain_core. Cette interface garantit que chaque composant possede les methodes invoke(), stream(), batch() et leurs equivalents asynchrones (ainvoke(), astream(), abatch()).

L'interface Runnable est le contrat fondamental de LCEL. Voici les methodes principales :

MethodeDescriptionSignature
invoke(input)Transforme une seule entree en sortieInput -> Output
stream(input)Produit la sortie en streaming (token par token)Input -> Iterator[Output]
batch(inputs)Transforme une liste d'entrees en paralleleList[Input] -> List[Output]
ainvoke(input)Version asynchrone de invokeInput -> Awaitable[Output]
astream(input)Version asynchrone de streamInput -> AsyncIterator[Output]
abatch(inputs)Version asynchrone de batchList[Input] -> Awaitable[List[Output]]

LangChain fournit plusieurs types de Runnables specialises :

Les types de Runnables

  • RunnableSequence -- Execute des Runnables en serie (cree par l'operateur |)
  • RunnableParallel -- Execute des Runnables en parallele (cree par un dict)
  • RunnablePassthrough -- Passe l'entree telle quelle au composant suivant
  • RunnableLambda -- Transforme une fonction Python en Runnable
  • RunnableBranch -- Routage conditionnel (if/else)
# Imports de tous les Runnables principaux from langchain_core.runnables import ( RunnableSequence, RunnableParallel, RunnablePassthrough, RunnableLambda, RunnableBranch, )

3. Creer une chain simple

La chain la plus courante dans LangChain suit le pattern Prompt -> Model -> Parser. C'est le remplacement direct de l'ancien LLMChain.

3.1 Etape par etape

Etape 1 : Creer le template de prompt avec des variables
from langchain_core.prompts import ChatPromptTemplate # Template avec deux variables : {langue} et {texte} prompt = ChatPromptTemplate.from_messages([ ("system", "Tu es un traducteur expert. Traduis le texte en {langue}."), ("human", "{texte}"), ])
Etape 2 : Configurer le modele LLM
from langchain_openai import ChatOpenAI # Le modele est un Runnable : il accepte des messages et retourne un AIMessage model = ChatOpenAI( model="gpt-4o", temperature=0.3, )
Etape 3 : Ajouter un parser pour extraire le texte brut
from langchain_core.output_parsers import StrOutputParser # StrOutputParser extrait le contenu textuel de l'AIMessage parser = StrOutputParser()
Etape 4 : Composer la chain avec l'operateur |
# Composition LCEL : prompt | model | parser chain = prompt | model | parser # Invoquer la chain resultat = chain.invoke({ "langue": "anglais", "texte": "Bonjour, comment allez-vous aujourd'hui ?" }) print(resultat)
Hello, how are you today?

3.2 Que se passe-t-il en interne ?

Flux de donnees detaille :

  1. invoke({"langue": "anglais", "texte": "Bonjour..."}) -- Le dict est passe au prompt
  2. Le Prompt interpole les variables et produit un PromptValue contenant les messages formates
  3. Le Model recoit le PromptValue, appelle l'API OpenAI, et retourne un AIMessage
  4. Le Parser recoit l'AIMessage et extrait .content sous forme de str
Chaque composant effectue une transformation de type. Le prompt transforme un dict en PromptValue, le modele transforme un PromptValue en AIMessage, et le parser transforme un AIMessage en str. La compatibilite des types entre composants adjacents est essentielle.

3.3 Avec un JsonOutputParser

from langchain_core.output_parsers import JsonOutputParser prompt = ChatPromptTemplate.from_template( """Analyse le sentiment du texte suivant. Reponds en JSON avec les cles "sentiment" et "score". Texte : {texte}""" ) chain = prompt | model | JsonOutputParser() resultat = chain.invoke({"texte": "J'adore ce nouveau restaurant !"}) print(resultat) print(type(resultat))
{'sentiment': 'positif', 'score': 0.95} <class 'dict'>

4. RunnablePassthrough

RunnablePassthrough est un Runnable qui passe son entree telle quelle a la sortie, sans aucune modification. Il est crucial dans les patterns ou l'on a besoin de transmettre des donnees originales tout en ajoutant des donnees supplementaires.

4.1 Utilisation basique

from langchain_core.runnables import RunnablePassthrough # RunnablePassthrough transmet l'entree sans changement passthrough = RunnablePassthrough() resultat = passthrough.invoke("Bonjour") print(resultat)
Bonjour

4.2 Avec assign() : enrichir les donnees

La methode RunnablePassthrough.assign() est extremement utile : elle passe les donnees d'entree telles quelles tout en y ajoutant de nouvelles cles calculees.

from langchain_core.runnables import RunnablePassthrough # assign() garde l'entree ET ajoute de nouvelles cles chain = RunnablePassthrough.assign( longueur=lambda x: len(x["texte"]), majuscule=lambda x: x["texte"].upper(), ) resultat = chain.invoke({"texte": "bonjour"}) print(resultat)
{'texte': 'bonjour', 'longueur': 7, 'majuscule': 'BONJOUR'}
assign() conserve toutes les cles existantes du dictionnaire d'entree et y ajoute les nouvelles cles calculees. C'est comme un "merge" de l'entree avec des valeurs supplementaires.

4.3 Pattern RAG avec RunnablePassthrough

from langchain_core.runnables import RunnablePassthrough from langchain_core.prompts import ChatPromptTemplate from langchain_openai import ChatOpenAI from langchain_core.output_parsers import StrOutputParser # Simulons un retriever def fake_retriever(query): return "Paris est la capitale de la France. Elle compte 2.1 millions d'habitants." prompt = ChatPromptTemplate.from_template( """Reponds a la question en utilisant le contexte. Contexte : {context} Question : {question}""" ) # RunnablePassthrough transmet la question directement # tandis que le retriever genere le contexte chain = ( {"context": lambda x: fake_retriever(x["question"]), "question": lambda x: x["question"]} | prompt | ChatOpenAI(model="gpt-4o") | StrOutputParser() ) resultat = chain.invoke({"question": "Quelle est la capitale de la France ?"}) print(resultat)
D'apres le contexte fourni, Paris est la capitale de la France. Elle compte 2.1 millions d'habitants.
Dans un vrai pipeline RAG, on utilise RunnablePassthrough pour transmettre la question de l'utilisateur directement au prompt, tandis qu'un retriever genere le contexte a partir d'une base vectorielle. Le pattern typique est : {"context": retriever, "question": RunnablePassthrough()}.

5. RunnableParallel

RunnableParallel execute plusieurs Runnables en parallele sur la meme entree. Chaque Runnable recoit la meme entree, et les resultats sont collectes dans un dictionnaire. C'est l'un des deux primitives de composition fondamentales de LCEL, avec RunnableSequence.

5.1 Creation implicite avec un dict

La facon la plus courante de creer un RunnableParallel est d'utiliser un dictionnaire Python dans une chaine LCEL. LangChain convertit automatiquement tout dictionnaire en RunnableParallel.

from langchain_core.runnables import RunnableParallel # Creation implicite : un dict dans une chain LCEL # est automatiquement converti en RunnableParallel parallel = { "majuscule": lambda x: x.upper(), "longueur": lambda x: len(x), "inverse": lambda x: x[::-1], } # Creation explicite (equivalent) parallel_explicit = RunnableParallel( majuscule=lambda x: x.upper(), longueur=lambda x: len(x), inverse=lambda x: x[::-1], ) resultat = parallel_explicit.invoke("bonjour") print(resultat)
{'majuscule': 'BONJOUR', 'longueur': 7, 'inverse': 'ruojnob'}

5.2 Chains paralleles avec LLM

from langchain_core.prompts import ChatPromptTemplate from langchain_openai import ChatOpenAI from langchain_core.output_parsers import StrOutputParser from langchain_core.runnables import RunnableParallel model = ChatOpenAI(model="gpt-4o", temperature=0.7) parser = StrOutputParser() # Deux chains independantes chain_resume = ( ChatPromptTemplate.from_template("Resume en 1 phrase : {texte}") | model | parser ) chain_mots_cles = ( ChatPromptTemplate.from_template("Extrais 5 mots-cles de : {texte}") | model | parser ) chain_sentiment = ( ChatPromptTemplate.from_template("Quel est le sentiment de : {texte}") | model | parser ) # Execution parallele des trois analyses chain_analyse = RunnableParallel( resume=chain_resume, mots_cles=chain_mots_cles, sentiment=chain_sentiment, ) resultat = chain_analyse.invoke({ "texte": "L'intelligence artificielle transforme radicalement le monde du travail..." }) for cle, valeur in resultat.items(): print(f"{cle}: {valeur}\n")
resume: L'IA revolutionne le monde professionnel en automatisant des taches et en creant de nouvelles opportunites. mots_cles: intelligence artificielle, travail, transformation, automatisation, technologie sentiment: Le sentiment est globalement neutre avec une tendance positive, evoquant le progres technologique.
Les trois appels LLM sont executes en parallele (via asyncio), ce qui reduit considerablement le temps total. Si chaque appel prend 2 secondes, le total sera d'environ 2 secondes au lieu de 6 secondes en sequentiel.

6. RunnableLambda

RunnableLambda permet de transformer n'importe quelle fonction Python en un Runnable compatible LCEL. C'est le moyen d'integrer de la logique metier personnalisee dans vos pipelines.

6.1 Creation a partir d'une fonction

from langchain_core.runnables import RunnableLambda # Methode 1 : avec le constructeur def nettoyer_texte(texte: str) -> str: """Supprime les espaces superflus et met en minuscule.""" return " ".join(texte.split()).lower().strip() runnable_nettoyer = RunnableLambda(nettoyer_texte) # Methode 2 : les lambdas sont auto-converties dans LCEL # mais RunnableLambda est prefere pour la lisibilite runnable_longueur = RunnableLambda(lambda x: len(x)) # Utilisation resultat = runnable_nettoyer.invoke(" Bonjour le MONDE ") print(resultat)
bonjour le monde

6.2 Dans une chain LCEL

from langchain_core.runnables import RunnableLambda from langchain_core.prompts import ChatPromptTemplate from langchain_openai import ChatOpenAI from langchain_core.output_parsers import StrOutputParser # Fonction de pre-traitement def preparer_input(data: dict) -> dict: """Enrichit les donnees avant le prompt.""" texte = data["texte"] return { "texte": texte, "nb_mots": len(texte.split()), "langue_detectee": "francais" if any(c in texte for c in "eaiou") else "inconnu", } # Fonction de post-traitement def formater_sortie(texte: str) -> dict: """Formate la sortie en structure riche.""" return { "reponse": texte, "nb_caracteres": len(texte), "statut": "succes", } prompt = ChatPromptTemplate.from_template( "Analyse ce texte ({nb_mots} mots, langue: {langue_detectee}):\n{texte}" ) chain = ( RunnableLambda(preparer_input) # Pre-traitement | prompt # Formatage du prompt | ChatOpenAI(model="gpt-4o") # Appel LLM | StrOutputParser() # Extraction du texte | RunnableLambda(formater_sortie) # Post-traitement ) resultat = chain.invoke({"texte": "L'IA change le monde"}) print(resultat)
{'reponse': 'Ce texte exprime une observation sur l impact transformateur...', 'nb_caracteres': 156, 'statut': 'succes'}

6.3 RunnableLambda avec fonctions asynchrones

import asyncio from langchain_core.runnables import RunnableLambda # RunnableLambda supporte les fonctions async async def appel_api_externe(query: str) -> str: # Simuler un appel API asynchrone await asyncio.sleep(0.1) return f"Resultat API pour: {query}" runnable_api = RunnableLambda(appel_api_externe) # Utilisation asynchrone resultat = await runnable_api.ainvoke("recherche IA") print(resultat)
Resultat API pour: recherche IA
Quand vous passez une fonction asynchrone a RunnableLambda, elle est automatiquement utilisee pour ainvoke(), astream() et abatch(). La version synchrone leve une erreur si vous essayez d'appeler invoke() sur une fonction purement async. Privilegiez les fonctions async pour les operations I/O (appels API, lecture de fichiers, etc.).

7. RunnableBranch - Logique conditionnelle

RunnableBranch permet d'implementer une logique de routage conditionnel dans une chain LCEL. Il evalue une serie de conditions et execute le Runnable associe a la premiere condition qui retourne True. Si aucune condition n'est satisfaite, le branch par defaut est execute.

7.1 Syntaxe

from langchain_core.runnables import RunnableBranch # Structure : liste de tuples (condition, runnable) + default branch = RunnableBranch( (condition_1, runnable_si_vrai_1), # Si condition_1(input) est True (condition_2, runnable_si_vrai_2), # Sinon si condition_2(input) est True runnable_par_defaut, # Sinon (default, toujours en dernier) )

7.2 Exemple : routage de questions

from langchain_core.runnables import RunnableBranch, RunnableLambda from langchain_core.prompts import ChatPromptTemplate from langchain_openai import ChatOpenAI from langchain_core.output_parsers import StrOutputParser model = ChatOpenAI(model="gpt-4o") parser = StrOutputParser() # Trois prompts specialises selon le type de question prompt_science = ChatPromptTemplate.from_template( "Tu es un expert en sciences. Reponds a : {question}" ) prompt_histoire = ChatPromptTemplate.from_template( "Tu es un historien. Reponds a : {question}" ) prompt_general = ChatPromptTemplate.from_template( "Reponds a cette question : {question}" ) # Chains specialisees chain_science = prompt_science | model | parser chain_histoire = prompt_histoire | model | parser chain_general = prompt_general | model | parser # Routage conditionnel def est_science(x): mots = ["physique", "chimie", "biologie", "atome", "molecule", "ADN"] return any(mot in x["question"].lower() for mot in mots) def est_histoire(x): mots = ["histoire", "guerre", "revolution", "roi", "siecle"] return any(mot in x["question"].lower() for mot in mots) branch = RunnableBranch( (est_science, chain_science), (est_histoire, chain_histoire), chain_general, # Default : question generale ) # Test avec differents types de questions print(branch.invoke({"question": "Comment fonctionne l'ADN ?"})) print("---") print(branch.invoke({"question": "Raconte la Revolution francaise"})) print("---") print(branch.invoke({"question": "Quel temps fait-il ?"}))
[Expert sciences] L'ADN (acide desoxyribonucleique) est une molecule en double helice... --- [Historien] La Revolution francaise de 1789 est un evenement majeur... --- [General] Je ne dispose pas d'informations meteo en temps reel...

7.3 Alternative : routage avec une fonction classifieur

from langchain_core.runnables import RunnableLambda # Approche alternative : utiliser un LLM pour classifier classifieur_prompt = ChatPromptTemplate.from_template( """Classifie cette question dans une categorie. Reponds UNIQUEMENT par : science, histoire, ou general. Question : {question}""" ) classifieur = classifieur_prompt | model | parser # Routeur base sur la classification du LLM def router(info: dict) -> str: categorie = info["categorie"].strip().lower() if categorie == "science": return chain_science.invoke(info) elif categorie == "histoire": return chain_histoire.invoke(info) else: return chain_general.invoke(info) # Chain complete : classifie puis route chain_complete = ( RunnablePassthrough.assign( categorie=classifieur ) | RunnableLambda(router) )
L'approche avec un LLM classifieur est plus robuste que les mots-cles mais plus couteuse (un appel LLM supplementaire). En production, on peut mettre en cache les classifications frequentes ou utiliser un petit modele rapide pour le routage.

8. Chains complexes

La puissance de LCEL reside dans la capacite a combiner tous les types de Runnables pour creer des pipelines sophistiques. Voici un exemple complet de pipeline RAG (Retrieval-Augmented Generation).

8.2 Code complet de la chain RAG

from langchain_core.prompts import ChatPromptTemplate from langchain_core.output_parsers import StrOutputParser from langchain_core.runnables import RunnablePassthrough, RunnableLambda from langchain_openai import ChatOpenAI, OpenAIEmbeddings from langchain_community.vectorstores import FAISS from langchain_core.documents import Document # 1. Creer la base de documents docs = [ Document(page_content="LangChain est un framework pour les applications LLM."), Document(page_content="LCEL est le langage de composition de LangChain."), Document(page_content="Les Runnables sont les briques de base de LCEL."), Document(page_content="RAG combine la recherche de documents avec la generation."), ] # 2. Creer le vector store et le retriever embeddings = OpenAIEmbeddings(model="text-embedding-3-small") vectorstore = FAISS.from_documents(docs, embeddings) retriever = vectorstore.as_retriever(search_kwargs={"k": 2}) # 3. Fonction pour formater les documents def format_docs(docs): return "\n\n".join(doc.page_content for doc in docs) # 4. Template du prompt prompt = ChatPromptTemplate.from_template( """Reponds a la question en utilisant uniquement le contexte fourni. Contexte : {context} Question : {question} Reponse :""" ) # 5. Assembler la chain RAG complete rag_chain = ( { "context": retriever | RunnableLambda(format_docs), "question": RunnablePassthrough(), } | prompt | ChatOpenAI(model="gpt-4o", temperature=0) | StrOutputParser() ) # 6. Invoquer reponse = rag_chain.invoke("Qu'est-ce que LCEL ?") print(reponse)
LCEL est le langage de composition de LangChain. Les Runnables sont les briques de base de LCEL, et il est utilise pour construire des applications LLM.

8.3 Chain multi-etapes avec enrichissement

from langchain_core.runnables import ( RunnablePassthrough, RunnableLambda, RunnableParallel ) from langchain_core.prompts import ChatPromptTemplate from langchain_openai import ChatOpenAI from langchain_core.output_parsers import StrOutputParser model = ChatOpenAI(model="gpt-4o") parser = StrOutputParser() # Etape 1 : Generer un resume prompt_resume = ChatPromptTemplate.from_template( "Resume ce texte en 2 phrases : {texte}" ) chain_resume = prompt_resume | model | parser # Etape 2 : Generer des questions a partir du resume prompt_questions = ChatPromptTemplate.from_template( "Genere 3 questions sur ce resume : {resume}" ) chain_questions = prompt_questions | model | parser # Etape 3 : Evaluer la qualite prompt_eval = ChatPromptTemplate.from_template( "Evalue la qualite de ce resume (note sur 10) : {resume}" ) chain_eval = prompt_eval | model | parser # Pipeline complet : texte -> resume -> (questions + evaluation) en parallele pipeline = ( chain_resume # texte -> resume | RunnablePassthrough.assign(resume=lambda x: x) # garder le resume | RunnableParallel( # execution parallele resume=lambda x: x["resume"], questions=lambda x: chain_questions.invoke({"resume": x["resume"]}), evaluation=lambda x: chain_eval.invoke({"resume": x["resume"]}), ) )
Ce pipeline demonstre la composition avancee : une etape sequentielle (resume) suivie d'une etape parallele (questions + evaluation). Le resultat final est un dictionnaire avec les trois cles.

9. Streaming et Batch

L'un des principaux avantages de LCEL est le support natif du streaming et du batch pour tous les Runnables.

9.1 Streaming : .stream()

Le streaming permet de recevoir les tokens du LLM au fur et a mesure de leur generation, au lieu d'attendre la reponse complete. C'est essentiel pour les interfaces utilisateur reactives.

from langchain_core.prompts import ChatPromptTemplate from langchain_openai import ChatOpenAI from langchain_core.output_parsers import StrOutputParser chain = ( ChatPromptTemplate.from_template("Explique {sujet} en 3 points.") | ChatOpenAI(model="gpt-4o") | StrOutputParser() ) # stream() retourne un iterateur de chunks for chunk in chain.stream({"sujet": "LCEL"}): print(chunk, end="", flush=True)
1. LCEL (LangChain Expression Language) est... 2. Il utilise l'operateur pipe | pour... 3. Chaque composant implemente...

9.2 Batch : .batch()

# batch() traite plusieurs entrees en parallele resultats = chain.batch([ {"sujet": "Python"}, {"sujet": "Machine Learning"}, {"sujet": "Docker"}, ], config={"max_concurrency": 3}) for i, res in enumerate(resultats): print(f"--- Resultat {i+1} ---") print(res[:100], "...") # Afficher les 100 premiers caracteres
--- Resultat 1 --- 1. Python est un langage de programmation interprete, de haut niveau... --- Resultat 2 --- 1. Le Machine Learning est une branche de l'intelligence artificielle... --- Resultat 3 --- 1. Docker est une plateforme de conteneurisation qui permet...
Le parametre max_concurrency controle le nombre maximum d'executions simultanees. C'est important pour eviter de depasser les limites de taux (rate limits) des API LLM. La valeur par defaut est generalement suffisante, mais ajustez-la selon votre forfait API.

9.3 Asynchrone : .ainvoke() et .astream()

import asyncio async def main(): # ainvoke : version asynchrone de invoke resultat = await chain.ainvoke({"sujet": "l'IA generative"}) print(resultat) # astream : streaming asynchrone async for chunk in chain.astream({"sujet": "les reseaux de neurones"}): print(chunk, end="", flush=True) # abatch : batch asynchrone resultats = await chain.abatch([ {"sujet": "GPT-4"}, {"sujet": "BERT"}, ]) asyncio.run(main())
Les methodes asynchrones (ainvoke, astream, abatch) sont recommandees dans les applications web (FastAPI, Django async) et les serveurs a haute concurrence. Elles evitent de bloquer le thread principal pendant les appels API.

10. Fallbacks - Gestion des erreurs

Les fallbacks permettent de definir des Runnables alternatifs qui seront utilises si le Runnable principal echoue. C'est essentiel en production pour gerer les pannes d'API, les depassements de limites de taux, etc.

10.1 .with_fallbacks()

from langchain_openai import ChatOpenAI from langchain_anthropic import ChatAnthropic # Modele principal : GPT-4o model_principal = ChatOpenAI(model="gpt-4o") # Modele de secours : Claude model_secours = ChatAnthropic(model="claude-sonnet-4-20250514") # Modele avec fallback : si GPT-4o echoue, utilise Claude model_resilient = model_principal.with_fallbacks([model_secours]) # Utilisation dans une chain chain = ( ChatPromptTemplate.from_template("Explique {concept}") | model_resilient | StrOutputParser() ) # Si OpenAI est en panne, la chain bascule automatiquement sur Anthropic resultat = chain.invoke({"concept": "les transformers"})

10.2 Fallback avec retry integre

# with_retry : relancer automatiquement en cas d'erreur transitoire model_avec_retry = ChatOpenAI(model="gpt-4o").with_retry( stop_after_attempt=3, # Maximum 3 tentatives wait_exponential_jitter=True, # Attente exponentielle avec jitter ) # Combiner retry ET fallback model_robuste = ( ChatOpenAI(model="gpt-4o") .with_retry(stop_after_attempt=2) .with_fallbacks([ ChatOpenAI(model="gpt-4o-mini").with_retry(stop_after_attempt=2), ChatAnthropic(model="claude-sonnet-4-20250514"), ]) ) # Ordre d'execution : # 1. GPT-4o (avec 2 retries) # 2. Si echoue -> GPT-4o-mini (avec 2 retries) # 3. Si echoue -> Claude (sans retry)

Strategies de fallback en production

StrategieDescriptionQuand l'utiliser
Retry seul.with_retry()Erreurs temporaires (timeout, rate limit)
Fallback seul.with_fallbacks()Panne complete d'un fournisseur
Retry + FallbackLes deux combinesProduction haute disponibilite
Degradation gracieuseFallback vers un modele moins cherMaitriser les couts en cas de surcharge

11. Legacy Chains vs LCEL - Guide de migration

Les legacy chains (LLMChain, SequentialChain, TransformChain, etc.) sont officiellement depreciees depuis LangChain v0.3. Tout nouveau code doit utiliser LCEL. Voici comment migrer.

11.1 LLMChain -> LCEL

# ============================================ # ANCIEN CODE (deprecated) - NE PAS UTILISER # ============================================ from langchain.chains import LLMChain # DEPRECATED ! chain_old = LLMChain( llm=ChatOpenAI(model="gpt-4o"), prompt=ChatPromptTemplate.from_template("Traduis en anglais : {texte}"), ) resultat = chain_old.invoke({"texte": "Bonjour"}) # Retourne: {"texte": "Bonjour", "text": "Hello"} # ============================================ # NOUVEAU CODE (LCEL) - A UTILISER # ============================================ chain_new = ( ChatPromptTemplate.from_template("Traduis en anglais : {texte}") | ChatOpenAI(model="gpt-4o") | StrOutputParser() ) resultat = chain_new.invoke({"texte": "Bonjour"}) # Retourne: "Hello"

11.2 SequentialChain -> LCEL pipe

# ============================================ # ANCIEN (SequentialChain) - DEPRECATED # ============================================ from langchain.chains import SequentialChain # DEPRECATED ! chain1 = LLMChain(llm=model, prompt=prompt1, output_key="resume") chain2 = LLMChain(llm=model, prompt=prompt2, output_key="traduction") seq = SequentialChain(chains=[chain1, chain2], input_variables=["texte"]) # ============================================ # NOUVEAU (LCEL) - Beaucoup plus simple ! # ============================================ chain_resume = prompt1 | model | parser chain_traduction = ( {"resume": chain_resume} | prompt2 | model | parser )
