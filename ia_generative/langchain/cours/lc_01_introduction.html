Introduction à LangChain

Le Framework pour Applications LLM -- Cours 1/2

1. Qu'est-ce que LangChain ?

LangChain est un framework open-source en Python (et JavaScript/TypeScript) conçu pour simplifier la création d'applications alimentées par des modèles de langage (LLM). Il fournit des abstractions standardisées pour connecter les LLM à des sources de données externes, des outils, de la mémoire conversationnelle, et bien plus encore.

Les grands modèles de langage comme GPT-4, Claude, ou Llama sont incroyablement puissants pour comprendre et générer du texte. Cependant, utiliser un LLM seul dans une application réelle pose de nombreux défis. LangChain agit comme un pont entre le LLM brut et votre application métier.

Imaginez LangChain comme un chef d'orchestre : il coordonne le LLM, vos données, vos outils externes et la logique de votre application pour produire un résultat cohérent et utile.

LangChain a été créé par Harrison Chase en octobre 2022. Depuis, le projet a connu une croissance fulgurante et est devenu le framework de référence pour les applications LLM, avec plus de 90 000 étoiles sur GitHub. La version 0.3, sortie en 2024, a apporté des changements majeurs dans l'architecture.

Les cas d'usage typiques

  • Chatbots intelligents avec mémoire conversationnelle et accès à des bases de connaissances
  • RAG (Retrieval Augmented Generation) : interroger vos propres documents (PDF, bases de données)
  • Agents autonomes capables d'utiliser des outils (recherche web, calcul, APIs)
  • Extraction de données structurées à partir de texte non structuré
  • Résumé automatique de documents longs
  • Pipelines de transformation de données textuelles

2. Pourquoi LangChain ?

Un LLM seul, aussi puissant soit-il, présente des limitations fondamentales pour les applications en production. LangChain résout ces problèmes en fournissant une couche d'abstraction standardisée.

Les limites d'un LLM seul

Problème 1 : Pas d'accès aux données récentes

Un LLM a été entraîné sur des données jusqu'à une date de coupure. Il ne connaît pas vos documents internes, vos bases de données, ni les événements récents.

Solution LangChain : Les Retrievers et le pattern RAG permettent d'injecter des données contextuelles dans le prompt avant l'appel au LLM.

Problème 2 : Pas de mémoire conversationnelle

Chaque appel API à un LLM est indépendant. Le modèle ne se souvient pas des messages précédents sauf si on les renvoie explicitement.

Solution LangChain : Les composants de Memory gèrent automatiquement l'historique de conversation et l'injectent dans chaque requête.

Problème 3 : Pas d'interaction avec le monde extérieur

Un LLM ne peut pas faire de calculs précis, interroger une API, lire un fichier ou exécuter du code.

Solution LangChain : Les Tools et Agents permettent au LLM de décider quels outils utiliser et d'exécuter des actions concrètes.

Avant / Après LangChain

Aspect Sans LangChain Avec LangChain
Appel au LLM Code HTTP brut, gestion manuelle des tokens Interface unifiée ChatOpenAI, ChatAnthropic
Prompts Concaténation de chaînes, f-strings fragiles ChatPromptTemplate avec variables typées
Chaînage Code procédural imbriqué, difficile à maintenir Opérateur | (pipe) LCEL, composition déclarative
Mémoire Gestion manuelle d'un historique de messages Composants de mémoire intégrés
Outils Intégration manuelle de chaque API Écosystème de tools prêts à l'emploi
Changement de LLM Réécriture complète du code Changer une seule ligne (même interface)
Streaming Implémentation complexe avec SSE/WebSocket Supporté nativement par tous les composants
Observabilité Logging manuel, debugging pénible LangSmith intégré pour le tracing
L'un des avantages les plus sous-estimés de LangChain est la portabilité entre providers. Le même code fonctionne avec OpenAI, Anthropic, Google, ou un modèle local Ollama. Cela vous protège du vendor lock-in et facilite les tests comparatifs entre modèles.

3. L'écosystème LangChain

LangChain n'est pas un simple package monolithique. Depuis la version 0.2, le projet a été découpé en plusieurs packages spécialisés pour offrir modularité et flexibilité. Comprendre cet écosystème est essentiel avant de commencer.

Détail de chaque package

Package Rôle Installation
langchain-core Abstractions fondamentales : Runnable, LCEL, Messages, PromptTemplates, OutputParsers. C'est le socle de tout l'écosystème. pip install langchain-core
langchain Chaînes cognitives de haut niveau, agents, retrievers, memory. Dépend de langchain-core. pip install langchain
langchain-openai Intégrations OpenAI : ChatOpenAI, OpenAIEmbeddings. Package séparé depuis v0.2. pip install langchain-openai
langchain-anthropic Intégration Anthropic : ChatAnthropic pour les modèles Claude. pip install langchain-anthropic
langchain-ollama Intégration Ollama : ChatOllama pour exécuter des modèles localement (Llama, Mistral, etc.). pip install langchain-ollama
langchain-community Plus de 700 intégrations tierces (bases vectorielles, APIs, etc.). pip install langchain-community
LangGraph Bibliothèque pour construire des agents stateful et des workflows multi-acteurs. Recommandé pour les agents depuis v0.2. pip install langgraph
LangSmith Plateforme d'observabilité, debugging, évaluation et déploiement d'agents. Service cloud avec SDK. pip install langsmith
Depuis LangChain v0.3, les anciens imports depuis langchain.chat_models sont dépréciés. Utilisez toujours les packages spécifiques : from langchain_openai import ChatOpenAI au lieu de from langchain.chat_models import ChatOpenAI. Ne pas suivre cette règle provoquera des avertissements de dépréciation.
La règle simple : installez langchain-core + le package d'intégration de votre provider (ex: langchain-openai). N'installez langchain complet que si vous avez besoin des chaînes de haut niveau, agents ou retrievers. Cette approche modulaire réduit les dépendances et les conflits.

4. Installation et Configuration

4.1 Créer un environnement virtuel

Il est fortement recommandé de travailler dans un environnement virtuel Python pour isoler les dépendances de votre projet.

# Créer un environnement virtuel python -m venv langchain-env # Activer l'environnement (Linux/macOS) source langchain-env/bin/activate # Activer l'environnement (Windows) langchain-env\Scripts\activate

4.2 Installer les packages LangChain

# Installation minimale avec OpenAI pip install langchain-core langchain-openai # Installation complète pour ce cours pip install langchain langchain-openai langchain-anthropic langchain-ollama # Avec LangSmith pour le debugging (optionnel mais recommandé) pip install langsmith # Pour gérer les variables d'environnement pip install python-dotenv

4.3 Configurer les clés API

Les LLM commerciaux (OpenAI, Anthropic) nécessitent des clés API. La bonne pratique est de les stocker dans un fichier .env à la racine de votre projet.

# Fichier .env (à la racine de votre projet) OPENAI_API_KEY="sk-proj-votre-cle-openai-ici" ANTHROPIC_API_KEY="sk-ant-votre-cle-anthropic-ici" # LangSmith (optionnel - pour le tracing et debugging) LANGSMITH_API_KEY="ls__votre-cle-langsmith" LANGSMITH_TRACING="true"
Ne commitez JAMAIS votre fichier .env dans Git ! Ajoutez-le à votre .gitignore. Si vos clés API sont exposées, révoquez-les immédiatement depuis le tableau de bord du provider.

4.4 Charger les clés dans votre code

import os from dotenv import load_dotenv # Charge les variables depuis le fichier .env load_dotenv() # Vérification que les clés sont chargées print("OpenAI key chargée :", "Oui" if os.getenv("OPENAI_API_KEY") else "Non") print("Anthropic key chargée :", "Oui" if os.getenv("ANTHROPIC_API_KEY") else "Non")
OpenAI key chargée : Oui Anthropic key chargée : Oui

4.5 Vérifier l'installation

import langchain_core import langchain_openai print("langchain-core version :", langchain_core.__version__) # Test rapide d'instanciation from langchain_openai import ChatOpenAI llm = ChatOpenAI(model="gpt-4o-mini", temperature=0) print("ChatOpenAI instancié avec succès !") print("Modèle :", llm.model_name)

Le modèle gpt-4o-mini est recommandé pour l'apprentissage : il est peu coûteux, rapide, et suffisamment capable pour la plupart des exercices. En production, vous pourrez passer à gpt-4o ou claude-3.5-sonnet sans changer votre code grâce à l'abstraction de LangChain.

5. Architecture conceptuelle

LangChain s'articule autour de plusieurs composants fondamentaux qui peuvent être combinés librement. Comprendre chaque composant et son rôle est la clé pour maîtriser le framework.

Description de chaque composant

Models (Modèles) : Wrappers autour des APIs de LLM. Fournissent une interface unifiée pour appeler GPT-4, Claude, Llama, etc. Supportent l'invocation synchrone, asynchrone, le streaming et le batch.
Prompts (Templates de prompts) : Permettent de créer des prompts réutilisables avec des variables. Gèrent les messages système, humain, et IA de manière structurée.
Output Parsers : Transforment la sortie brute du LLM en un format structuré (string, JSON, objet Pydantic). Essentiels pour intégrer les réponses dans votre application.
LCEL / Chains : Le LangChain Expression Language permet de chaîner les composants avec l'opérateur |. C'est la manière recommandée de composer des pipelines depuis v0.2.
Memory : Gère l'historique des conversations pour permettre des échanges multi-tours cohérents. Plusieurs stratégies : buffer complet, résumé, fenêtre glissante.
Retrievers : Recherchent des documents pertinents dans une base de connaissances (vector store) pour alimenter le prompt du LLM. C'est le coeur du pattern RAG.
Agents : Systèmes où le LLM décide dynamiquement quelles actions exécuter et dans quel ordre. LangGraph est maintenant recommandé pour construire des agents complexes.
Tools (Outils) : Fonctions que le LLM peut invoquer : recherche web, calculs, requêtes SQL, appels API, etc. Décorés avec @tool pour être utilisables par les agents.

6. Premier exemple : De basique à structuré

Construisons notre premier programme LangChain étape par étape, en allant du plus simple au plus structuré. Cette progression illustre la philosophie du framework.

6.1 Appel direct au LLM

Le niveau le plus basique : instancier un modèle et lui envoyer un message.

from langchain_openai import ChatOpenAI from langchain_core.messages import HumanMessage # Instanciation du modèle llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.7) # Appel direct avec un message humain response = llm.invoke([ HumanMessage(content="Explique-moi ce qu'est Python en 2 phrases.") ]) print(response.content) print(f"\nType de la réponse : {type(response)}") print(f"Tokens utilisés : {response.usage_metadata}")
Python est un langage de programmation interprété, de haut niveau, connu pour sa syntaxe claire et lisible. Il est utilisé dans de nombreux domaines comme le développement web, la science des données, l'intelligence artificielle et l'automatisation. Type de la réponse : <class 'langchain_core.messages.ai.AIMessage'> Tokens utilisés : {'input_tokens': 22, 'output_tokens': 48, 'total_tokens': 70}
La méthode invoke() est la manière standard d'appeler un composant LangChain. Elle prend une entrée et retourne une sortie de manière synchrone. Tous les composants LangChain (modèles, prompts, parsers, chaînes) implémentent cette même interface Runnable.

6.2 Avec un PromptTemplate

Ajoutons un template de prompt pour rendre notre requête réutilisable et paramétrable.

from langchain_openai import ChatOpenAI from langchain_core.prompts import ChatPromptTemplate # Définir un template réutilisable prompt = ChatPromptTemplate.from_messages([ ("system", "Tu es un professeur d'informatique expert. Réponds en français, de manière claire et pédagogique."), ("human", "Explique-moi ce qu'est {sujet} en {nb_phrases} phrases.") ]) # Instancier le modèle llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.7) # Formater le prompt avec des variables messages = prompt.invoke({ "sujet": "le machine learning", "nb_phrases": "3" }) # Appeler le modèle avec le prompt formaté response = llm.invoke(messages) print(response.content)
Le machine learning, ou apprentissage automatique, est une branche de l'intelligence artificielle qui permet aux ordinateurs d'apprendre à partir de données sans être explicitement programmés. Les algorithmes identifient des patterns dans les données d'entraînement pour faire des prédictions ou prendre des décisions sur de nouvelles données. On distingue trois approches principales : l'apprentissage supervisé, non supervisé et par renforcement.

6.3 Avec une chaîne LCEL

Maintenant, combinons tout avec l'opérateur pipe | pour créer une chaîne complète et élégante.

from langchain_openai import ChatOpenAI from langchain_core.prompts import ChatPromptTemplate from langchain_core.output_parsers import StrOutputParser # 1. Prompt template prompt = ChatPromptTemplate.from_messages([ ("system", "Tu es un professeur d'informatique expert. Réponds en français."), ("human", "Explique-moi ce qu'est {sujet} en {nb_phrases} phrases.") ]) # 2. Modèle llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.7) # 3. Parser (extrait le texte brut de la réponse) parser = StrOutputParser() # 4. Chaîne complète avec l'opérateur | (LCEL) chain = prompt | llm | parser # 5. Invoquer la chaîne result = chain.invoke({ "sujet": "les réseaux de neurones", "nb_phrases": "3" }) print(result) print(f"\nType du résultat : {type(result)}") # str, pas AIMessage
Les réseaux de neurones sont des modèles informatiques inspirés du fonctionnement du cerveau humain, composés de couches de neurones artificiels interconnectés. Chaque neurone reçoit des entrées, applique une transformation mathématique (fonction d'activation) et transmet le résultat à la couche suivante. Grâce à un processus d'entraînement appelé rétropropagation, le réseau ajuste ses poids pour minimiser l'erreur entre ses prédictions et les résultats attendus. Type du résultat : <class 'str'>

La progression clé

Remarquez la progression :

  1. Étape 1 : Appel direct -- simple mais pas réutilisable
  2. Étape 2 : Avec template -- réutilisable mais verbeux (2 appels séparés)
  3. Étape 3 : Chaîne LCEL -- une seule ligne déclare tout le pipeline, un seul invoke()

La chaîne LCEL prompt | llm | parser est la forme idiomatique de LangChain v0.3. Elle sera utilisée dans tout le reste du cours.

Le StrOutputParser peut sembler superflu ici, mais il joue un rôle important : il extrait le .content de l'objet AIMessage retourné par le modèle pour vous donner directement une chaîne de caractères. Sans lui, vous obtenez un objet AIMessage complet avec des métadonnées.

7. LCEL (LangChain Expression Language)

LCEL (LangChain Expression Language) est une syntaxe déclarative pour composer des composants LangChain en chaînes (pipelines). Il utilise l'opérateur | (pipe) de Python pour connecter des composants, où la sortie de chaque composant devient l'entrée du suivant.

7.1 L'interface Runnable

Tous les composants LangChain implémentent l'interface Runnable, définie dans langchain-core. Cette interface garantit que chaque composant possède les mêmes méthodes :

Méthode Description Retour
invoke(input) Appel synchrone unique Résultat complet
ainvoke(input) Appel asynchrone unique Résultat complet (await)
stream(input) Streaming synchrone Iterator de chunks
astream(input) Streaming asynchrone AsyncIterator de chunks
batch(inputs) Traitement par lot synchrone Liste de résultats
abatch(inputs) Traitement par lot asynchrone Liste de résultats (await)
Grâce à cette interface uniforme, une chaîne LCEL hérite automatiquement de toutes ces méthodes. Si chaque composant supporte le streaming, la chaîne entière supporte le streaming. C'est la puissance de la composition.

7.2 L'opérateur pipe

L'opérateur | en Python est normalement utilisé pour le "OU binaire" (bitwise OR). LangChain le surcharge via les méthodes spéciales __or__ et __ror__ de la classe Runnable. Quand vous écrivez a | b, Python crée un RunnableSequence qui exécute a puis b.

from langchain_openai import ChatOpenAI from langchain_core.prompts import ChatPromptTemplate from langchain_core.output_parsers import StrOutputParser # Chaque composant est un Runnable prompt = ChatPromptTemplate.from_template( "Donne-moi 3 faits intéressants sur {sujet}." ) llm = ChatOpenAI(model="gpt-4o-mini") parser = StrOutputParser() # L'opérateur | crée un RunnableSequence chain = prompt | llm | parser # Vérifier le type print(f"Type de chain : {type(chain).__name__}") # Invoquer (synchrone) result = chain.invoke({"sujet": "la Lune"}) print(result) # Streaming for chunk in chain.stream({"sujet": "Mars"}): print(chunk, end="", flush=True)
Type de chain : RunnableSequence 1. La Lune s'éloigne de la Terre d'environ 3,8 cm par an... 2. Il n'y a pas de vent sur la Lune, donc les empreintes... 3. La face cachée de la Lune n'est jamais visible depuis la Terre...

7.3 Pourquoi LCEL remplace les anciennes Chains

Avant LCEL (LangChain v0.1), on utilisait des classes comme LLMChain, SequentialChain, etc. Ces classes sont maintenant dépréciées.

Ancien style (déprécié) Nouveau style (LCEL)
LLMChain(llm=llm, prompt=prompt) prompt | llm | parser
SequentialChain(chains=[c1, c2]) chain1 | chain2
chain.run("input") chain.invoke({"key": "input"})
Pas de streaming natif chain.stream({"key": "input"})
Pas d'async natif await chain.ainvoke({"key": "input"})
Si vous trouvez du code utilisant LLMChain, ConversationChain ou la méthode .run(), c'est du code obsolète datant de LangChain v0.1. Convertissez-le en LCEL. Les anciennes classes seront supprimées dans une version future.

7.4 RunnableParallel : exécution en parallèle

LCEL permet aussi l'exécution parallèle avec un dictionnaire :

from langchain_core.runnables import RunnableParallel from langchain_openai import ChatOpenAI from langchain_core.prompts import ChatPromptTemplate from langchain_core.output_parsers import StrOutputParser llm = ChatOpenAI(model="gpt-4o-mini") parser = StrOutputParser() # Deux prompts différents prompt_avantages = ChatPromptTemplate.from_template( "Liste 3 avantages de {sujet}." ) prompt_inconvenients = ChatPromptTemplate.from_template( "Liste 3 inconvénients de {sujet}." ) # Exécution en parallèle avec un dictionnaire chain = RunnableParallel( avantages=prompt_avantages | llm | parser, inconvenients=prompt_inconvenients | llm | parser ) result = chain.invoke({"sujet": "le cloud computing"}) print("--- Avantages ---") print(result["avantages"]) print("\n--- Inconvénients ---") print(result["inconvenients"])
--- Avantages --- 1. Évolutivité : le cloud permet d'ajuster les ressources à la demande... 2. Réduction des coûts : pas besoin d'investir dans une infrastructure physique... 3. Accessibilité : les données et applications sont accessibles partout... --- Inconvénients --- 1. Dépendance au réseau : une connexion Internet est indispensable... 2. Sécurité : les données sensibles sont hébergées chez un tiers... 3. Coûts récurrents : les frais d'abonnement peuvent s'accumuler...
Le RunnableParallel exécute réellement les deux branches en parallèle (en async sous le capot), ce qui réduit le temps de réponse. C'est un avantage majeur de LCEL par rapport aux anciennes chaînes séquentielles.

8. Les Providers (Fournisseurs de LLM)

L'un des grands avantages de LangChain est son abstraction des providers. Le même code fonctionne avec n'importe quel LLM, en changeant simplement l'import et l'instanciation. Voici les quatre providers principaux.

8.1 OpenAI (ChatOpenAI)

from langchain_openai import ChatOpenAI from langchain_core.messages import HumanMessage # Installation : pip install langchain-openai # Nécessite : OPENAI_API_KEY dans l'environnement llm = ChatOpenAI( model="gpt-4o-mini", # ou "gpt-4o", "gpt-4-turbo" temperature=0.7, # créativité (0=déterministe, 1=créatif) max_tokens=500, # limite de tokens en sortie timeout=30, # timeout en secondes max_retries=2, # nombre de tentatives en cas d'erreur ) response = llm.invoke([HumanMessage(content="Bonjour, qui es-tu ?")]) print(response.content)
Bonjour ! Je suis un assistant IA créé par OpenAI. Je suis ici pour vous aider avec vos questions et tâches. Comment puis-je vous aider aujourd'hui ?

8.2 Anthropic (ChatAnthropic)

from langchain_anthropic import ChatAnthropic from langchain_core.messages import HumanMessage # Installation : pip install langchain-anthropic # Nécessite : ANTHROPIC_API_KEY dans l'environnement llm = ChatAnthropic( model="claude-sonnet-4-20250514", # ou "claude-3-haiku-20240307" temperature=0.7, max_tokens=1024, # obligatoire pour Anthropic ) response = llm.invoke([HumanMessage(content="Bonjour, qui es-tu ?")]) print(response.content)
Bonjour ! Je suis Claude, un assistant IA créé par Anthropic. Je suis conçu pour être utile, inoffensif et honnête. Comment puis-je vous aider ?

8.3 Ollama (ChatOllama - Modèles locaux)

from langchain_ollama import ChatOllama from langchain_core.messages import HumanMessage # Installation : pip install langchain-ollama # Prérequis : Ollama installé et un modèle téléchargé # Terminal : ollama pull llama3.2 llm = ChatOllama( model="llama3.2", # ou "mistral", "deepseek-r1:8b" temperature=0.7, base_url="http://localhost:11434", # URL du serveur Ollama ) response = llm.invoke([HumanMessage(content="Bonjour, qui es-tu ?")]) print(response.content)
Bonjour ! Je suis Llama, un modèle de langage open source développé par Meta. Je fonctionne localement sur votre machine, sans envoyer de données à un serveur externe.

8.4 HuggingFace

from langchain_community.chat_models import ChatHuggingFace from langchain_community.llms import HuggingFaceEndpoint # Installation : pip install langchain-community langchain-huggingface # Nécessite : HUGGINGFACEHUB_API_TOKEN dans l'environnement llm = HuggingFaceEndpoint( repo_id="mistralai/Mistral-7B-Instruct-v0.3", temperature=0.7, max_new_tokens=512, ) chat_model = ChatHuggingFace(llm=llm) response = chat_model.invoke("Bonjour, qui es-tu ?") print(response.content)

Portabilité du code

Grâce à l'interface Runnable commune, vous pouvez changer de provider en modifiant une seule ligne :

# Changer de provider sans modifier le reste du code # llm = ChatOpenAI(model="gpt-4o-mini") # llm = ChatAnthropic(model="claude-sonnet-4-20250514", max_tokens=1024) llm = ChatOllama(model="llama3.2") # Le reste du code reste identique ! chain = prompt | llm | parser result = chain.invoke({"sujet": "Python"})
Pour les étudiants avec un budget limité, je recommande fortement Ollama. Installez Ollama sur votre machine, téléchargez un modèle comme llama3.2 ou mistral, et vous aurez un LLM local gratuit et illimité. Les performances sont suffisantes pour l'apprentissage et les exercices.
