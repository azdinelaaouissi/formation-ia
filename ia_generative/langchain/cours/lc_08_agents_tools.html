Agents et Tools - LLMs qui Agissent

Cours 8 - LangChain : Quand les modèles de langage prennent des décisions et utilisent des outils

1. Qu'est-ce qu'un Agent ?

Un Agent est un système intelligent piloté par un LLM qui peut décider dynamiquement quelles actions entreprendre et quels outils utiliser pour répondre à une requête. Contrairement à une chaîne (chain) qui suit un parcours fixe et prédéterminé, un agent raisonne, planifie et adapte son comportement en fonction de la situation.

Chain vs Agent : la différence fondamentale

Caractéristique Chain (Chaîne) Agent
Parcours Fixe, défini par le développeur Dynamique, décidé par le LLM
Étapes Toujours les mêmes, dans le même ordre Variables selon la question
Outils Utilisés séquentiellement, tous appelés Sélectionnés dynamiquement selon le besoin
Boucle Pas de boucle (exécution linéaire) Boucle : raisonne → agit → observe → répète
Complexité Simple, prévisible Plus complexe, plus flexible
Exemple Récupérer docs → Générer réponse Analyser la question → Chercher sur Google → Calculer → Vérifier → Répondre
Les agents représentent le niveau le plus avancé de LangChain. Ils transforment le LLM d'un simple générateur de texte en un système décisionnel autonome capable d'interagir avec le monde extérieur via des outils.

2. Tools (Outils)

Un Tool (outil) est une fonction que l'agent peut appeler pour accomplir une action spécifique. Chaque tool a un nom, une description (que le LLM lit pour décider quand l'utiliser), et une fonction à exécuter. La qualité de la description est cruciale : c'est elle qui guide le LLM dans ses choix.

2.1 Le décorateur @tool

La manière la plus simple de créer un outil est d'utiliser le décorateur @tool :

from langchain_core.tools import tool # Outil simple : calculatrice @tool def calculer(expression: str) -> str: """Calcule une expression mathématique. Utilisez cet outil quand vous devez faire un calcul numérique. L'expression doit être une expression Python valide. Exemple : '2 + 3 * 4' retourne '14'""" try: resultat = eval(expression) return str(resultat) except Exception as e: return f"Erreur de calcul : {e}" # Vérifier les propriétés de l'outil print(f"Nom : {calculer.name}") print(f"Description : {calculer.description}") print(f"Schéma des arguments : {calculer.args}") # Tester l'outil directement resultat = calculer.invoke("15 * 24 + 7") print(f"Résultat : {resultat}")
Nom : calculer Description : Calcule une expression mathématique. Utilisez cet outil quand vous devez faire un calcul numérique. L'expression doit être une expression Python valide. Exemple : '2 + 3 * 4' retourne '14' Schéma des arguments : {'expression': {'title': 'Expression', 'type': 'string'}} Résultat : 367

2.2 Outil avec plusieurs paramètres

from langchain_core.tools import tool from pydantic import BaseModel, Field # Schéma des arguments avec Pydantic class RechercheArgs(BaseModel): query: str = Field(description="La requête de recherche") max_results: int = Field(default=5, description="Nombre maximum de résultats") @tool(args_schema=RechercheArgs) def rechercher_base(query: str, max_results: int = 5) -> str: """Recherche dans la base de connaissances interne. Utilisez cet outil pour trouver des informations dans nos documents.""" # Simulation de recherche return f"Résultats pour '{query}' (max {max_results}) : [Doc1, Doc2, Doc3]" # Tester avec des arguments nommés result = rechercher_base.invoke({"query": "LangChain agents", "max_results": 3}) print(result)
Résultats pour 'LangChain agents' (max 3) : [Doc1, Doc2, Doc3]

2.3 StructuredTool.from_function()

Pour plus de contrôle sur la configuration de l'outil, utilisez StructuredTool.from_function() :

from langchain_core.tools import StructuredTool # Fonction normale (sans décorateur) def obtenir_meteo(ville: str, unite: str = "celsius") -> str: """Obtient la météo pour une ville donnée.""" # Simulation meteo = {"Paris": 18, "Lyon": 22, "Marseille": 25} temp = meteo.get(ville, 15) if unite == "fahrenheit": temp = temp * 9/5 + 32 return f"Météo à {ville} : {temp}°{unite[0].upper()}" # Créer l'outil avec StructuredTool outil_meteo = StructuredTool.from_function( func=obtenir_meteo, name="meteo", description="""Obtient la température actuelle pour une ville française. Utilisez cet outil quand l'utilisateur demande la météo ou la température. Paramètres : - ville : nom de la ville (ex: 'Paris', 'Lyon') - unite : 'celsius' ou 'fahrenheit' (défaut: 'celsius')""", return_direct=False # Si True, retourne directement sans passer par le LLM ) print(f"Nom : {outil_meteo.name}") print(f"Args : {outil_meteo.args}") result = outil_meteo.invoke({"ville": "Paris", "unite": "celsius"}) print(result)
Nom : meteo Args : {'ville': {'title': 'Ville', 'type': 'string'}, 'unite': {'default': 'celsius', 'title': 'Unite', 'type': 'string'}} Météo à Paris : 18°C

2.4 Créer un outil de requête base de données

import json @tool def requete_utilisateurs(filtre: str) -> str: """Interroge la base de données des utilisateurs. Utilisez cet outil pour chercher des informations sur les utilisateurs. Le filtre peut être : 'tous', 'actifs', 'premium', ou un nom d'utilisateur.""" # Base simulée utilisateurs = [ {"nom": "Alice", "statut": "premium", "actif": True}, {"nom": "Bob", "statut": "gratuit", "actif": True}, {"nom": "Charlie", "statut": "premium", "actif": False}, ] if filtre == "tous": return json.dumps(utilisateurs, ensure_ascii=False) elif filtre == "actifs": return json.dumps([u for u in utilisateurs if u["actif"]], ensure_ascii=False) elif filtre == "premium": return json.dumps([u for u in utilisateurs if u["statut"] == "premium"], ensure_ascii=False) else: user = [u for u in utilisateurs if u["nom"].lower() == filtre.lower()] return json.dumps(user, ensure_ascii=False) if user else "Utilisateur non trouvé" result = requete_utilisateurs.invoke("premium") print(result)
[{"nom": "Alice", "statut": "premium", "actif": true}, {"nom": "Charlie", "statut": "premium", "actif": false}]
La description du tool est critique. Le LLM décide quel outil utiliser uniquement en lisant les descriptions. Une description vague ou imprécise mènera à des mauvais choix d'outils. Soyez explicite sur le quand et le comment utiliser l'outil.

4. Tool Calling (Appel d'outils par le LLM)

Le Tool Calling est le mécanisme par lequel un LLM génère une sortie structurée indiquant quel outil appeler et avec quels arguments. Les LLMs modernes (GPT-4, Claude, Gemini) supportent nativement cette fonctionnalité.

4.1 bind_tools() : lier des outils à un LLM

from langchain_openai import ChatOpenAI from langchain_core.tools import tool @tool def addition(a: int, b: int) -> int: """Additionne deux nombres entiers.""" return a + b @tool def multiplication(a: int, b: int) -> int: """Multiplie deux nombres entiers.""" return a * b # Créer le LLM avec les outils liés llm = ChatOpenAI(model="gpt-4o-mini") llm_with_tools = llm.bind_tools([addition, multiplication]) # Le LLM décide quel outil utiliser response = llm_with_tools.invoke("Combien font 15 fois 23 ?") # Inspecter la réponse print(f"Contenu texte : {response.content}") print(f"Appels d'outils : {response.tool_calls}")
Contenu texte : Appels d'outils : [{'name': 'multiplication', 'args': {'a': 15, 'b': 23}, 'id': 'call_abc123', 'type': 'tool_call'}]
Le LLM n'a pas répondu en texte libre. Il a généré un appel structuré à l'outil multiplication avec les arguments a=15 et b=23. C'est l'agent qui exécutera réellement l'outil et renverra le résultat au LLM.

4.2 Le paramètre tool_choice

# Forcer l'utilisation d'un outil spécifique llm_force = llm.bind_tools( [addition, multiplication], tool_choice="addition" # Force l'utilisation de 'addition' ) # Même pour une multiplication, le LLM utilisera 'addition' response_force = llm_force.invoke("Multiplie 5 par 3") print(f"Outil forcé : {response_force.tool_calls[0]['name']}") # Autres options pour tool_choice : # tool_choice="auto" - Le LLM décide (défaut) # tool_choice="none" - Pas d'outil # tool_choice="any" - Le LLM doit utiliser au moins un outil # tool_choice="addition" - Force un outil spécifique
Outil forcé : addition

4.3 Exécuter les appels d'outils manuellement

from langchain_core.messages import HumanMessage, ToolMessage # Étape 1 : Le LLM décide d'utiliser un outil messages = [HumanMessage(content="Combien font 42 + 58 ?")] ai_response = llm_with_tools.invoke(messages) messages.append(ai_response) print("Étape 1 - LLM décide :") print(f" Outil : {ai_response.tool_calls[0]['name']}") print(f" Args : {ai_response.tool_calls[0]['args']}") # Étape 2 : Exécuter l'outil tool_call = ai_response.tool_calls[0] if tool_call["name"] == "addition": result = addition.invoke(tool_call["args"]) elif tool_call["name"] == "multiplication": result = multiplication.invoke(tool_call["args"]) print(f"\nÉtape 2 - Résultat de l'outil : {result}") # Étape 3 : Renvoyer le résultat au LLM messages.append(ToolMessage( content=str(result), tool_call_id=tool_call["id"] )) final_response = llm_with_tools.invoke(messages) print(f"\nÉtape 3 - Réponse finale : {final_response.content}")
Étape 1 - LLM décide : Outil : addition Args : {'a': 42, 'b': 58} Étape 2 - Résultat de l'outil : 100 Étape 3 - Réponse finale : 42 + 58 = 100
Ce processus en 3 étapes (LLM décide → outil exécute → LLM formule la réponse) est exactement ce que l'AgentExecutor automatise pour vous. Comprendre ce mécanisme est essentiel pour déboguer les agents.

5. create_react_agent - Le pattern ReAct

Le pattern ReAct (Reason + Act) est une approche où l'agent alterne entre des phases de raisonnement (Thought), d'action (Action) et d'observation (Observation). Cette boucle se répète jusqu'à ce que l'agent ait suffisamment d'informations pour formuler une réponse finale.

5.1 Implémentation avec LangChain (legacy)

from langchain.agents import create_react_agent, AgentExecutor from langchain_openai import ChatOpenAI from langchain_core.prompts import PromptTemplate from langchain_core.tools import tool # Définir les outils @tool def rechercher_web(query: str) -> str: """Recherche des informations sur le web. Utilisez cet outil quand vous avez besoin d'informations actuelles.""" return f"Résultats pour '{query}' : LangChain v0.3 est sorti en 2024 avec le support LangGraph intégré." @tool def calculer(expression: str) -> str: """Calcule une expression mathématique Python.""" return str(eval(expression)) tools = [rechercher_web, calculer] # Prompt ReAct standard react_prompt = PromptTemplate.from_template("""Répondez du mieux possible aux questions suivantes. Vous avez accès aux outils suivants : {tools} Utilisez le format suivant : Question: la question à laquelle répondre Thought: vous devez toujours réfléchir à ce qu'il faut faire Action: l'action à entreprendre, doit être l'un de [{tool_names}] Action Input: l'entrée de l'action Observation: le résultat de l'action ... (ce cycle Thought/Action/Action Input/Observation peut se répéter) Thought: Je connais maintenant la réponse finale Final Answer: la réponse finale à la question originale Commencez ! Question: {input} Thought: {agent_scratchpad}""") # Créer l'agent ReAct llm = ChatOpenAI(model="gpt-4o-mini", temperature=0) agent = create_react_agent(llm, tools, react_prompt) # Créer l'exécuteur agent_executor = AgentExecutor( agent=agent, tools=tools, verbose=True, # Voir le raisonnement max_iterations=5, handle_parsing_errors=True ) # Exécuter result = agent_executor.invoke({ "input": "Quelle est la dernière version de LangChain et combien font 2024 * 3 ?" })
> Entering new AgentExecutor chain... Thought: Je dois chercher la dernière version de LangChain puis faire un calcul. Action: rechercher_web Action Input: dernière version LangChain Observation: Résultats pour 'dernière version LangChain' : LangChain v0.3 est sorti en 2024 avec le support LangGraph intégré. Thought: J'ai trouvé la version. Maintenant je dois calculer 2024 * 3. Action: calculer Action Input: 2024 * 3 Observation: 6072 Thought: Je connais maintenant la réponse finale. Final Answer: La dernière version de LangChain est la v0.3, sortie en 2024, avec le support intégré de LangGraph. Et 2024 x 3 = 6072. > Finished chain.
L'agent a utilisé deux outils de manière séquentielle : d'abord la recherche web pour trouver la version, puis la calculatrice pour le calcul. Le raisonnement en chaîne (Thought → Action → Observation) est clairement visible.

6. create_tool_calling_agent - Approche moderne

Le create_tool_calling_agent est l'approche moderne recommandée pour créer des agents dans LangChain. Contrairement au ReAct qui repose sur un prompt textuel, cette approche utilise les capacités natives d'appel d'outils du LLM (function calling / tool calling). Elle est plus fiable, plus rapide, et peut effectuer plusieurs appels d'outils en parallèle.

create_react_agent vs create_tool_calling_agent

Aspect create_react_agent create_tool_calling_agent
Mécanisme Prompt textuel (Thought/Action/Observation) Tool calling natif du LLM (JSON structuré)
Appels parallèles Non (séquentiel uniquement) Oui (plusieurs outils en un tour)
Fiabilité Peut mal parser le format texte Très fiable (JSON structuré)
Compatibilité Tout LLM (même basiques) LLMs supportant le tool calling (GPT-4, Claude, Gemini)
Performance Plus lent (parsing texte) Plus rapide (moins de tokens)
from langchain.agents import create_tool_calling_agent, AgentExecutor from langchain_openai import ChatOpenAI from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder from langchain_core.tools import tool # Outils @tool def addition(a: int, b: int) -> int: """Additionne deux nombres entiers.""" return a + b @tool def multiplication(a: int, b: int) -> int: """Multiplie deux nombres entiers.""" return a * b @tool def rechercher(query: str) -> str: """Recherche des informations sur internet.""" return f"Info trouvée : La population de la France est d'environ 68 millions." tools = [addition, multiplication, rechercher] # Prompt pour tool calling agent (IMPORTANT : inclure agent_scratchpad) prompt = ChatPromptTemplate.from_messages([ ("system", """Vous êtes un assistant intelligent et serviable. Répondez en français. Utilisez les outils disponibles quand c'est pertinent. Si vous n'avez pas besoin d'outil, répondez directement."""), ("human", "{input}"), MessagesPlaceholder(variable_name="agent_scratchpad"), ]) # Créer l'agent llm = ChatOpenAI(model="gpt-4o-mini", temperature=0) agent = create_tool_calling_agent(llm, tools, prompt) # Exécuteur executor = AgentExecutor( agent=agent, tools=tools, verbose=True ) # Test : l'agent décide dynamiquement quels outils utiliser result = executor.invoke({ "input": "Calcule 25 + 37, puis multiplie le résultat par 4" }) print(f"\nRéponse : {result['output']}")
> Entering new AgentExecutor chain... Invoking: `addition` with `{'a': 25, 'b': 37}` 62 Invoking: `multiplication` with `{'a': 62, 'b': 4}` 248 25 + 37 = 62, puis 62 x 4 = 248. > Finished chain. Réponse : 25 + 37 = 62, puis 62 x 4 = 248.
En 2025-2026, préférez create_tool_calling_agent à create_react_agent pour tous les projets qui utilisent des LLMs modernes. L'ancien pattern ReAct reste utile pour des LLMs qui ne supportent pas le tool calling natif ou pour des besoins pédagogiques.

7. AgentExecutor - Exécuter les agents

L'AgentExecutor est le runtime qui gère l'exécution de l'agent. Il orchestre la boucle : appeler le LLM, exécuter les outils, gérer les erreurs, limiter les itérations, et retourner le résultat final. Sans AgentExecutor, l'agent ne peut pas fonctionner.

Paramètres clés de l'AgentExecutor

Paramètre Type Défaut Description
agent Agent - L'agent à exécuter (obligatoire)
tools List[Tool] - Liste des outils disponibles (obligatoire)
verbose bool False Afficher le raisonnement de l'agent
max_iterations int 15 Nombre maximum de cycles Think-Act-Observe
max_execution_time float None Timeout en secondes
handle_parsing_errors bool/str/callable False Gérer les erreurs de parsing du LLM
early_stopping_method str "force" "force" ou "generate" (comment arrêter)
return_intermediate_steps bool False Retourner les étapes intermédiaires
# Configuration avancée de l'AgentExecutor executor_avance = AgentExecutor( agent=agent, tools=tools, verbose=True, max_iterations=10, # Maximum 10 cycles max_execution_time=30.0, # Timeout de 30 secondes handle_parsing_errors=True, # Gérer les erreurs gracieusement early_stopping_method="generate", # Générer une réponse même si max atteint return_intermediate_steps=True, # Garder les étapes ) result = executor_avance.invoke({"input": "Quelle est la population de la France ?"}) # Accéder aux différentes parties du résultat print(f"Réponse finale : {result['output']}") print(f"\nÉtapes intermédiaires :") for step in result["intermediate_steps"]: action, observation = step print(f" Outil : {action.tool}") print(f" Entrée : {action.tool_input}") print(f" Observation : {observation}") print()
> Entering new AgentExecutor chain... Invoking: `rechercher` with `{'query': 'population France actuelle'}` Info trouvée : La population de la France est d'environ 68 millions. La population de la France est d'environ 68 millions d'habitants. > Finished chain. Réponse finale : La population de la France est d'environ 68 millions d'habitants. Étapes intermédiaires : Outil : rechercher Entrée : {'query': 'population France actuelle'} Observation : Info trouvée : La population de la France est d'environ 68 millions.

7.1 Gestion des erreurs de parsing

# Différentes stratégies pour handle_parsing_errors # Stratégie 1 : Booléen simple (message par défaut) executor1 = AgentExecutor( agent=agent, tools=tools, handle_parsing_errors=True # Message générique ) # Stratégie 2 : Message personnalisé executor2 = AgentExecutor( agent=agent, tools=tools, handle_parsing_errors="Désolé, une erreur interne s'est produite. Veuillez reformuler." ) # Stratégie 3 : Fonction personnalisée def gerer_erreur(error): return f"L'agent a rencontré une erreur : {str(error)[:100]}. Essayez différemment." executor3 = AgentExecutor( agent=agent, tools=tools, handle_parsing_errors=gerer_erreur )
Toujours définir max_iterations en production ! Un agent sans limite peut entrer dans une boucle infinie (et coûter cher en appels API). Une valeur de 5-10 est généralement suffisante.

8. Agent avec mémoire conversationnelle

Par défaut, chaque appel à un agent est indépendant : l'agent ne se souvient pas des conversations précédentes. En ajoutant une mémoire, l'agent peut maintenir le contexte d'une conversation et référencer des informations mentionnées précédemment.
from langchain.agents import create_tool_calling_agent, AgentExecutor from langchain_openai import ChatOpenAI from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder from langchain_core.tools import tool from langchain_community.chat_message_histories import ChatMessageHistory from langchain_core.runnables.history import RunnableWithMessageHistory @tool def calculer(expression: str) -> str: """Calcule une expression mathématique Python.""" return str(eval(expression)) @tool def rechercher(query: str) -> str: """Recherche des informations.""" return f"Résultat pour '{query}' : La Tour Eiffel mesure 330 mètres." tools = [calculer, rechercher] # Prompt avec historique de conversation prompt = ChatPromptTemplate.from_messages([ ("system", "Vous êtes un assistant intelligent. Répondez en français."), MessagesPlaceholder(variable_name="chat_history"), # Historique ici ("human", "{input}"), MessagesPlaceholder(variable_name="agent_scratchpad"), ]) llm = ChatOpenAI(model="gpt-4o-mini", temperature=0) agent = create_tool_calling_agent(llm, tools, prompt) executor = AgentExecutor(agent=agent, tools=tools, verbose=True) # Stockage de l'historique par session store = {} def get_session_history(session_id: str): if session_id not in store: store[session_id] = ChatMessageHistory() return store[session_id] # Wrapper avec mémoire agent_with_memory = RunnableWithMessageHistory( executor, get_session_history, input_messages_key="input", history_messages_key="chat_history", ) # Conversation avec mémoire config = {"configurable": {"session_id": "session_001"}} # Tour 1 r1 = agent_with_memory.invoke( {"input": "Quelle est la hauteur de la Tour Eiffel ?"}, config=config ) print(f"Tour 1 : {r1['output']}") # Tour 2 : référence au tour précédent r2 = agent_with_memory.invoke( {"input": "Et si on la multiplie par 2, combien ça fait ?"}, config=config ) print(f"Tour 2 : {r2['output']}")
Tour 1 : La Tour Eiffel mesure 330 mètres de hauteur. Tour 2 : 330 x 2 = 660 mètres. Si on multiplie la hauteur de la Tour Eiffel (330 m) par 2, on obtient 660 mètres.
Au tour 2, l'agent comprend que "la" réfère à la Tour Eiffel et que le nombre à multiplier est 330. Sans mémoire, il n'aurait aucun contexte et ne pourrait pas répondre correctement.

9. Agents multi-outils : exemple complet

Construisons un agent complet avec plusieurs outils complémentaires qui doit raisonner sur la sélection d'outils :

from langchain.agents import create_tool_calling_agent, AgentExecutor from langchain_openai import ChatOpenAI from langchain_core.prompts import ChatPromptTemplate, MessagesPlaceholder from langchain_core.tools import tool from datetime import datetime import json # === OUTIL 1 : Calculatrice === @tool def calculatrice(expression: str) -> str: """Calcule une expression mathématique. Accepte toute expression Python valide (math, statistiques). Exemples : '2**10', 'sum([1,2,3])/3', 'round(3.14159, 2)'""" import math try: result = eval(expression, {"__builtins__": {}, "math": math, "sum": sum, "round": round, "len": len, "max": max, "min": min, "abs": abs}) return f"Résultat : {result}" except Exception as e: return f"Erreur : {e}" # === OUTIL 2 : Recherche web === @tool def recherche_web(query: str) -> str: """Recherche des informations actuelles sur internet. Utilisez cet outil pour des faits, actualités, définitions.""" # Simulation infos = { "python": "Python 3.12 est la dernière version stable. 10. LangGraph Agents - L'avenir des agents

LangGraph est le framework de nouvelle génération de LangChain pour construire des agents. Il représente les agents sous forme de graphes orientés, où chaque nœud est une étape (appel LLM, exécution d'outil, décision) et chaque arête définit le flux. LangGraph offre un contrôle beaucoup plus fin que l'AgentExecutor classique.

Pourquoi LangGraph remplace AgentExecutor ?

  • Contrôle granulaire : chaque étape du graphe est configurable individuellement.
  • Persistance d'état : sauvegarde automatique de l'état entre les étapes (checkpointing).
  • Human-in-the-loop : interruption facile pour validation humaine avant certaines actions.
  • Branchement conditionnel : le flux peut diverger selon des conditions complexes.
  • Streaming : streaming en temps réel des résultats de chaque nœud.
  • Sous-graphes : composition d'agents complexes à partir de sous-agents.
  • Support MCP : intégration native avec le Model Context Protocol.

10.1 Agent ReAct avec LangGraph

# pip install langgraph from langgraph.prebuilt import create_react_agent from langchain_openai import ChatOpenAI from langchain_core.tools import tool @tool def calculer(expression: str) -> str: """Calcule une expression mathématique.""" return str(eval(expression)) @tool def rechercher(query: str) -> str: """Recherche des informations.""" return f"Résultat : LangGraph 1.0 a été publié en 2025." # LangGraph simplifie la création d'agents llm = ChatOpenAI(model="gpt-4o-mini") # En UNE ligne : création d'un agent ReAct complet graph = create_react_agent( model=llm, tools=[calculer, rechercher], ) # Exécuter l'agent result = graph.invoke({ "messages": [{"role": "user", "content": "Quand est sorti LangGraph 1.0 ?"}] }) # Récupérer la réponse for msg in result["messages"]: print(f"[{msg.__class__.__name__}] {msg.content[:100]}")

10.2 Agent LangGraph avec persistance

from langgraph.prebuilt import create_react_agent from langgraph.checkpoint.memory import MemorySaver from langchain_openai import ChatOpenAI # Checkpointer pour la persistance d'état checkpointer = MemorySaver() llm = ChatOpenAI(model="gpt-4o-mini") graph = create_react_agent( model=llm, tools=[calculer, rechercher], checkpointer=checkpointer # Ajoute la persistance ) # Configuration avec identifiant de thread config = {"configurable": {"thread_id": "thread-123"}} # Tour 1 r1 = graph.invoke( {"messages": [{"role": "user", "content": "Calcule 256 * 3"}]}, config=config ) print("Tour 1 :", r1["messages"][-1].content) # Tour 2 : se souvient du résultat précédent r2 = graph.invoke( {"messages": [{"role": "user", "content": "Ajoute 100 au résultat précédent"}]}, config=config ) print("Tour 2 :", r2["messages"][-1].content)
Tour 1 : 256 * 3 = 768 Tour 2 : 768 + 100 = 868
LangGraph est maintenant le framework recommandé par LangChain pour tout nouvel agent en production. L'AgentExecutor classique reste disponible pour les cas simples et l'apprentissage, mais pour des agents robustes avec persistance, streaming et contrôle fin, LangGraph est le choix à privilégier. C'est un sujet suffisamment vaste pour mériter un cours entièrement dédié.

11.3 Quand utiliser un Agent vs une Chain ?

Guide de décision

Situation Recommandation Pourquoi
Pipeline fixe et prévisible Chain (LCEL) Plus rapide, plus prévisible, moins de tokens
RAG classique (recherche + génération) Chain (LCEL) Le parcours est toujours le même
L'action dépend de la question Agent Le LLM doit décider quoi faire
Accès à des APIs externes variées Agent Le LLM choisit quelle API appeler
Questions multi-étapes complexes Agent La prochaine étape dépend du résultat courant
Agents sophistiqués en production LangGraph Contrôle fin, persistance, streaming

Résumé des points clés

  • Un Agent est un LLM qui raisonne et décide dynamiquement quels outils utiliser.
  • Les Tools sont des fonctions que l'agent peut appeler. La description est critique pour guider le LLM.
  • Le décorateur @tool est le moyen le plus rapide de créer un outil. Le docstring devient la description.
  • bind_tools() lie des outils à un LLM pour le tool calling natif.
  • Le pattern ReAct (Thought → Action → Observation) est le modèle de raisonnement des agents.
  • create_tool_calling_agent est plus fiable et rapide que create_react_agent pour les LLMs modernes.
  • L'AgentExecutor orchestre l'exécution. Toujours définir max_iterations en production.
  • Ajoutez une mémoire via RunnableWithMessageHistory pour les conversations multi-tours.
  • Les agents multi-outils peuvent résoudre des questions complexes nécessitant plusieurs outils.
  • LangGraph est l'avenir des agents LangChain : graphes orientés, persistance, streaming, human-in-the-loop.
