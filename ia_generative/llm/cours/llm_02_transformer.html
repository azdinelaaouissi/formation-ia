Architecture Transformer - Le Coeur des LLMs

Cours 2/3 - Comprendre l'architecture qui a revolutionne le NLP et l'IA

1. Pourquoi le Transformer ?

Les problemes des architectures precedentes

Avant le Transformer (2017), le traitement du langage naturel (NLP) reposait principalement sur les RNN (Recurrent Neural Networks) et les LSTM (Long Short-Term Memory). Ces architectures avaient des limites fondamentales :

La revolution "Attention is All You Need"

En juin 2017, une equipe de Google publie le papier "Attention is All You Need" (Vaswani et al.). Le titre est provocateur : ils proposent une architecture qui utilise uniquement le mecanisme d'attention, sans aucune recurrence (RNN) ni convolution (CNN).

Ce papier est sans doute le plus influent de l'histoire de l'IA moderne. Toutes les avancees depuis (GPT, BERT, Claude, Gemini, Llama, Mistral...) sont basees sur cette architecture.

Le titre "Attention is All You Need" est un jeu de mots avec la chanson des Beatles "All You Need Is Love". L'idee cle est que le mecanisme d'attention seul, sans recurrence, est suffisant pour capturer toutes les relations entre les mots d'une phrase.

2. Architecture globale du Transformer

Le Transformer original est compose de deux parties principales : un Encoder (qui comprend le texte d'entree) et un Decoder (qui genere le texte de sortie).

Ne paniquez pas devant la complexite du schema ! Nous allons detailler chaque composant un par un dans les sections suivantes. L'important pour l'instant est de comprendre la structure globale : des donnees entrent, passent par plusieurs couches de "Self-Attention" et "Feed-Forward", et des probabilites de mots sortent.

3. Self-Attention : le mecanisme cle

Le Self-Attention (auto-attention) est le mecanisme central du Transformer. Il permet a chaque mot d'une phrase de "regarder" tous les autres mots pour comprendre le contexte. Chaque mot obtient un score d'importance pour chaque autre mot.

L'analogie de la bibliotheque

Comprendre Q, K, V avec une analogie

Imaginez que vous etes dans une bibliotheque :

  • Query (Q) = Votre question : "Je cherche un livre sur la physique quantique"
  • Key (K) = Le titre de chaque livre : "Mecanique quantique", "Cuisine francaise", "Physique moderne"...
  • Value (V) = Le contenu de chaque livre : Le texte reel a l'interieur

Le mecanisme d'attention compare votre question (Q) avec les titres (K) de tous les livres pour determiner lesquels sont pertinents. Puis il recupere le contenu (V) des livres les plus pertinents, en donnant plus de poids a ceux dont le titre correspond le mieux.

Comment ca marche, etape par etape

Etape 1 - Creer Q, K, V : Pour chaque mot (token), on calcule trois vecteurs en multipliant l'embedding du mot par trois matrices de poids appris (W_Q, W_K, W_V). Ces matrices sont apprises pendant l'entrainement.
Etape 2 - Calculer les scores d'attention : On multiplie la Query de chaque mot par la Key de tous les autres mots (produit scalaire). Un score eleve signifie que les deux mots sont "pertinents" l'un pour l'autre.
Etape 3 - Normaliser (diviser par sqrt(d_k)) : On divise les scores par la racine carree de la dimension des vecteurs Key. Cela evite que les scores soient trop grands, ce qui rendrait le softmax trop "pointu".
Etape 4 - Appliquer Softmax : On transforme les scores en probabilites (entre 0 et 1, somme = 1). Les mots les plus pertinents obtiennent des poids proches de 1, les autres proches de 0.
Etape 5 - Multiplier par V : On fait la somme ponderee des Values, en utilisant les poids d'attention. Le resultat est une nouvelle representation de chaque mot, enrichie par le contexte.

La formule

Attention(Q, K, V) = softmax( Q . KT / sqrt(dk) ) . V

Ou :

  • Q = matrice des Queries (toutes les questions)
  • K = matrice des Keys (tous les titres)
  • V = matrice des Values (tous les contenus)
  • d_k = dimension des vecteurs Key (par exemple 64)
  • Q . K^T = produit scalaire entre chaque Q et chaque K

Visualisation des poids d'attention

Dans cet exemple, le mot "chat" fait fortement attention a "souris" (0.50) et "mange" (0.25). C'est logique : pour comprendre ce qu'est le "chat" dans cette phrase, il est crucial de savoir qu'il "mange" une "souris". Le modele apprend ces associations pendant l'entrainement.

Exemple de code simplifie

# Implementation simplifiee de Self-Attention en PyTorch import torch import torch.nn.functional as F # Dimensions d_model = 512 # Dimension de l'embedding d_k = 64 # Dimension des vecteurs Q, K, V seq_len = 5 # "Le chat mange la souris" = 5 tokens # Matrices de projection (apprises pendant l'entrainement) W_Q = torch.randn(d_model, d_k) W_K = torch.randn(d_model, d_k) W_V = torch.randn(d_model, d_k) # Embeddings d'entree (5 mots, chacun un vecteur de 512 dimensions) X = torch.randn(seq_len, d_model) # Etape 1 : Calculer Q, K, V Q = X @ W_Q # (5, 64) - les questions K = X @ W_K # (5, 64) - les cles V = X @ W_V # (5, 64) - les valeurs # Etape 2 : Scores d'attention = Q . K^T / sqrt(d_k) scores = Q @ K.T / (d_k ** 0.5) # (5, 5) - score entre chaque paire de mots # Etape 3 : Softmax pour obtenir les poids (entre 0 et 1) attention_weights = F.softmax(scores, dim=-1) # (5, 5) # Etape 4 : Somme ponderee des Values output = attention_weights @ V # (5, 64) - nouvelles representations print("Poids d'attention (5x5) :") print(attention_weights.round(decimals=2))
Poids d'attention (5x5) : tensor([[0.18, 0.22, 0.20, 0.19, 0.21], [0.05, 0.15, 0.25, 0.05, 0.50], [0.10, 0.30, 0.20, 0.10, 0.30], [0.15, 0.10, 0.15, 0.20, 0.40], [0.08, 0.35, 0.25, 0.12, 0.20]]) (Chaque ligne = a quel mot ce token "fait attention")

4. Multi-Head Attention

Le Multi-Head Attention consiste a executer le mecanisme d'attention en parallele plusieurs fois (avec des matrices de poids differentes), puis a combiner les resultats. Chaque "tete" (head) apprend a se concentrer sur un aspect different du langage.

Analogie : un comite d'experts

Imaginez qu'au lieu d'un seul lecteur analysant une phrase, vous avez 8 experts specialises :

  • Expert 1 (syntaxe) : "chat" est le sujet de "mange"
  • Expert 2 (semantique) : "chat" et "souris" sont des animaux lies (predateur-proie)
  • Expert 3 (coreference) : "la" fait reference a "souris"
  • Expert 4 (proximite) : "mange" est entre "chat" et "souris"
  • ... et ainsi de suite

Chaque expert regarde la phrase sous un angle different, puis on combine leurs analyses pour obtenir une comprehension complete.

L'utilisation de plusieurs tetes permet au modele de capturer simultanement differents types de relations entre les mots : syntaxiques (sujet-verbe), semantiques (synonymes, antonymes), positionnelles (proximite), et bien d'autres patterns que le modele decouvre pendant l'entrainement.

5. Positional Encoding : l'ordre des mots

Le probleme

Le mecanisme d'attention traite tous les mots en parallele sans notion d'ordre. Pour l'attention seule, "Le chat mange la souris" et "La souris mange le chat" sont identiques ! Or, l'ordre des mots change completement le sens.

La solution : encoder la position

Le Positional Encoding ajoute un vecteur de position a chaque embedding de mot. Ce vecteur encode la position du mot dans la phrase (1er mot, 2eme mot, etc.).

Comment ca marche : fonctions sinus et cosinus

Le papier original utilise des fonctions sinusoidales pour encoder les positions :

  • PE(pos, 2i) = sin(pos / 10000^(2i/d_model))
  • PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))

Ou pos est la position du mot et i est la dimension.

L'avantage : le modele peut generaliser a des sequences plus longues que celles vues pendant l'entrainement, et les positions relatives sont facilement calculables.

Les modeles recents comme GPT-4 et Llama utilisent des RoPE (Rotary Positional Embeddings) au lieu des sinusoides du papier original. RoPE encode les positions relatives plutot qu'absolues, ce qui permet une meilleure generalisation aux longues sequences. Mais le principe reste le meme : informer le modele de l'ordre des tokens.

6. Feed-Forward Networks

Apres le mecanisme d'attention, chaque token passe par un reseau Feed-Forward (FFN), un petit reseau de neurones a deux couches qui transforme independamment la representation de chaque token. C'est la ou se fait une grande partie du "raisonnement" du modele.

Structure du FFN

Le Feed-Forward est simple : deux couches lineaires avec une activation non-lineaire entre les deux.

FFN(x) = max(0, x . W1 + b1) . W2 + b2

  • W1 : projette de d_model vers d_ff (par exemple 512 vers 2048, soit 4x plus grand)
  • ReLU/GELU : activation non-lineaire
  • W2 : re-projette de d_ff vers d_model (2048 vers 512)

Analogie : le cerveau apres la lecture

Si l'attention est comme lire une phrase (comprendre le contexte), le FFN est comme reflechir a ce qu'on vient de lire. L'attention collecte l'information pertinente, puis le FFN la transforme et la raffine.

Des recherches recentes suggerent que les couches FFN fonctionnent comme une memoire cle-valeur : elles stockent des faits et des connaissances apprises pendant l'entrainement.

Dans les modeles modernes, le FFN represente environ 2/3 des parametres du modele (contre 1/3 pour l'attention). C'est la ou sont "stockees" les connaissances factuelles du LLM. Les modeles MoE (Mixture of Experts) comme GPT-4 et Mistral ont plusieurs FFN en parallele, et un mecanisme de routage decide quel "expert" (quel FFN) active pour chaque token.

7. Encoder vs Decoder : les variantes

Le Transformer original a deux parties (encoder + decoder), mais les modeles modernes n'utilisent souvent qu'une seule partie :

Architecture Exemples Direction Tache principale Entrainement
Encoder-only BERT, RoBERTa, DeBERTa Bidirectionnel Comprehension (classification, NER) MLM (Masked Language Model)
Decoder-only GPT-4, Claude, Llama, Mistral Unidirectionnel (gauche a droite) Generation de texte CLM (Causal Language Model)
Encoder-Decoder T5, BART, mBART, Flan-T5 Les deux Transformation (traduction, resume) Seq2Seq (denoising)
Ne confondez pas : BERT n'est PAS un LLM au sens generatif du terme. BERT est un modele encoder-only concu pour comprendre le texte, pas pour le generer. Quand on parle de "LLM" aujourd'hui, on fait generalement reference aux modeles decoder-only comme GPT, Claude et Llama.

8. De Transformer a LLM : comment GPT fonctionne

Comment passe-t-on de l'architecture Transformer a un LLM comme GPT ou Claude ? En 3 ingredients :

Ingredient 1 - Architecture decoder-only : GPT utilise uniquement la partie decoder du Transformer, avec un masque causal : chaque token ne peut "voir" que les tokens qui le precedent. Cela force le modele a predire le prochain token sans "tricher" en regardant le futur.
Ingredient 2 - Echelle massive : On empile des dizaines (voire centaines) de couches de Transformer, avec des milliards de parametres. GPT-3 : 96 couches, 175B parametres. Plus le modele est grand, plus il peut capturer des patterns complexes.
Ingredient 3 - Pre-entrainement sur Internet : On entraine le modele sur des trillions de tokens issus d'Internet. La tache est simple : predire le prochain mot. Mais en faisant cela sur une quantite astronomique de texte, le modele apprend la grammaire, les faits, le raisonnement, et meme le code.

Les 3 phases d'entrainement d'un LLM

Resume du cours

  • Le Transformer (2017) a remplace les RNN/LSTM grace a la parallelisation et l'attention
  • Le Self-Attention permet a chaque mot de "regarder" tous les autres avec des scores Q, K, V
  • Le Multi-Head Attention donne plusieurs perspectives simultanees
  • Le Positional Encoding encode l'ordre des mots (sin/cos ou RoPE)
  • Le Feed-Forward Network transforme et stocke les connaissances
  • Les LLMs modernes (GPT, Claude) sont des decoder-only Transformers massifs
  • L'entrainement passe par 3 phases : pre-entrainement, SFT, RLHF
