TD Pratique - Construire un Graphe Simple avec LangGraph

Premier TD LangGraph -- Comprendre les StateGraph, les noeuds, les aretes et le flux d'execution

1. Recapitulatif -- Concepts fondamentaux

LangGraph est un framework Python developpe par LangChain qui permet de construire des applications basees sur des LLM sous forme de graphes orientes. Chaque noeud represente une action (fonction), chaque arete definit le flux d'execution, et un etat partage (State) circule entre les noeuds.

Les 4 briques fondamentales

ConceptDescriptionCode
StateDictionnaire type partage entre tous les noeudsTypedDict
NodeFonction qui lit et modifie l'etatgraph.add_node()
EdgeConnexion directe entre deux noeudsgraph.add_edge()
Conditional EdgeArete qui choisit le noeud suivant selon une conditiongraph.add_conditional_edges()
Pensez a un graphe LangGraph comme un organigramme : chaque boite est un noeud, chaque fleche est une arete, et un dossier (l'etat) passe de boite en boite en se faisant enrichir a chaque etape.

3. Exercice 1 -- Definir un etat type (TypedDict)

L'etat est la structure de donnees centrale de tout graphe LangGraph. Il est defini avec TypedDict et circule entre chaque noeud. Chaque noeud recoit l'etat complet et retourne un dictionnaire partiel avec les cles a mettre a jour.

Regles de l'etat

  • Definir l'etat avec TypedDict pour le typage statique
  • Chaque noeud recoit l'etat entier en parametre
  • Chaque noeud retourne un dictionnaire partiel : seules les cles retournees sont mises a jour
  • Les cles non retournees conservent leur valeur precedente
# exercice1_state.py # Definir un etat type pour un graphe de traitement de texte from typing import TypedDict class GraphState(TypedDict): """Etat partage entre tous les noeuds du graphe.""" message: str # Le message d'entree de l'utilisateur sentiment: str # Le sentiment detecte (positif, negatif, neutre) reponse: str # La reponse generee nombre_mots: int # Le nombre de mots dans le message # Exemple d'etat initial etat_initial: GraphState = { "message": "Bonjour, je suis tres content !", "sentiment": "", "reponse": "", "nombre_mots": 0 } print("Etat initial :", etat_initial) print("Type :", type(etat_initial))
Etat initial : {'message': 'Bonjour, je suis tres content !', 'sentiment': '', 'reponse': '', 'nombre_mots': 0} Type : <class 'dict'>
Le TypedDict n'est pas un veritable objet -- c'est un dictionnaire Python standard avec des annotations de type. LangGraph l'utilise pour valider la structure de l'etat a la compilation du graphe.

4. Exercice 2 -- Creer un graphe a 2 noeuds (greet - respond)

Construisons notre premier graphe fonctionnel. Il comporte deux noeuds : greet qui analyse le message et respond qui genere une reponse.

Etape 1 : Definir l'etat
Etape 2 : Creer les fonctions noeuds
Etape 3 : Construire le graphe avec StateGraph
Etape 4 : Ajouter les aretes et compiler
Etape 5 : Executer et observer l'etat
# exercice2_graphe_2_noeuds.py from typing import TypedDict from langgraph.graph import StateGraph, START, END # --- Etape 1 : Definir l'etat --- class State(TypedDict): message: str greeting: str reponse: str # --- Etape 2 : Definir les fonctions noeuds --- def greet(state: State) -> dict: """Noeud 1 : Analyse le message et cree une salutation.""" msg = state["message"] nb_mots = len(msg.split()) greeting = f"Bonjour ! Vous avez envoye un message de {nb_mots} mots." print(f"[greet] Message recu : '{msg}'") print(f"[greet] Salutation generee : '{greeting}'") return {"greeting": greeting} def respond(state: State) -> dict: """Noeud 2 : Genere une reponse basee sur la salutation.""" greeting = state["greeting"] reponse = f"{greeting} Merci pour votre message, nous allons le traiter." print(f"[respond] Reponse finale : '{reponse}'") return {"reponse": reponse} # --- Etape 3 : Construire le graphe --- graph_builder = StateGraph(State) # Ajouter les noeuds graph_builder.add_node("greet", greet) graph_builder.add_node("respond", respond) # --- Etape 4 : Definir les aretes --- graph_builder.add_edge(START, "greet") # START -> greet graph_builder.add_edge("greet", "respond") # greet -> respond graph_builder.add_edge("respond", END) # respond -> END # Compiler le graphe graph = graph_builder.compile() # --- Etape 5 : Executer --- resultat = graph.invoke({ "message": "Bonjour, comment allez-vous aujourd'hui ?", "greeting": "", "reponse": "" }) print("\n--- Etat final ---") for cle, valeur in resultat.items(): print(f" {cle}: {valeur}")
[greet] Message recu : 'Bonjour, comment allez-vous aujourd'hui ?' [greet] Salutation generee : 'Bonjour ! Vous avez envoye un message de 5 mots.' [respond] Reponse finale : 'Bonjour ! Vous avez envoye un message de 5 mots. Merci pour votre message, nous allons le traiter.' --- Etat final --- message: Bonjour, comment allez-vous aujourd'hui ? greeting: Bonjour ! Vous avez envoye un message de 5 mots. reponse: Bonjour ! Vous avez envoye un message de 5 mots. Merci pour votre message, nous allons le traiter.
Le graphe s'execute de maniere sequentielle : START -> greet -> respond -> END. L'etat est enrichi a chaque etape. Le noeud greet ajoute la salutation, puis respond utilise cette salutation pour construire la reponse finale.

5. Exercice 3 -- Ajouter une arete conditionnelle (routage par sentiment)

Les aretes conditionnelles permettent de choisir dynamiquement le noeud suivant en fonction de l'etat. Ici, nous allons router vers differents noeuds selon le sentiment detecte dans le message.

# exercice3_conditional_edge.py from typing import TypedDict, Literal from langgraph.graph import StateGraph, START, END class State(TypedDict): message: str sentiment: str reponse: str # Mots-cles pour la detection de sentiment MOTS_POSITIFS = ["content", "heureux", "super", "genial", "merci", "bien", "excellent"] MOTS_NEGATIFS = ["triste", "decu", "mauvais", "probleme", "erreur", "nul", "horrible"] def analyser(state: State) -> dict: """Analyse le sentiment du message.""" msg = state["message"].lower() score = 0 for mot in msg.split(): if mot in MOTS_POSITIFS: score += 1 elif mot in MOTS_NEGATIFS: score -= 1 sentiment = "positif" if score >= 0 else "negatif" print(f"[analyser] Sentiment detecte : {sentiment} (score={score})") return {"sentiment": sentiment} def router(state: State) -> Literal["repondre_positif", "repondre_negatif"]: """Fonction de routage : retourne le nom du noeud suivant.""" if state["sentiment"] == "positif": return "repondre_positif" return "repondre_negatif" def repondre_positif(state: State) -> dict: reponse = "Merci pour votre message positif ! Nous sommes ravis." print(f"[repondre_positif] {reponse}") return {"reponse": reponse} def repondre_negatif(state: State) -> dict: reponse = "Nous sommes desoles. Un agent va vous contacter rapidement." print(f"[repondre_negatif] {reponse}") return {"reponse": reponse} # Construction du graphe builder = StateGraph(State) builder.add_node("analyser", analyser) builder.add_node("repondre_positif", repondre_positif) builder.add_node("repondre_negatif", repondre_negatif) # Aretes builder.add_edge(START, "analyser") builder.add_conditional_edges( "analyser", # Noeud source router, # Fonction de routage { # Mapping nom -> noeud "repondre_positif": "repondre_positif", "repondre_negatif": "repondre_negatif", } ) builder.add_edge("repondre_positif", END) builder.add_edge("repondre_negatif", END) graph = builder.compile() # --- Test avec message positif --- print("=== Test 1 : Message positif ===") r1 = graph.invoke({"message": "Merci, c'est super genial !", "sentiment": "", "reponse": ""}) print(f"Reponse : {r1['reponse']}\n") # --- Test avec message negatif --- print("=== Test 2 : Message negatif ===") r2 = graph.invoke({"message": "J'ai un probleme, c'est horrible", "sentiment": "", "reponse": ""}) print(f"Reponse : {r2['reponse']}")
=== Test 1 : Message positif === [analyser] Sentiment detecte : positif (score=2) [repondre_positif] Merci pour votre message positif ! Nous sommes ravis. Reponse : Merci pour votre message positif ! Nous sommes ravis. === Test 2 : Message negatif === [analyser] Sentiment detecte : negatif (score=-2) [repondre_negatif] Nous sommes desoles. Un agent va vous contacter rapidement. Reponse : Nous sommes desoles. Un agent va vous contacter rapidement.
La fonction router retourne une chaine de caracteres correspondant au nom du noeud suivant. Le dictionnaire passe a add_conditional_edges fait le mapping entre cette chaine et le noeud reel. Utilisez Literal pour le typage strict des valeurs de retour.

6. Exercice 4 -- Graphe complet : classify - route - respond

Construisons un graphe plus elabore a 3 niveaux : classification de l'intention, routage conditionnel, et reponse adaptee. Ce patron est la base de tout chatbot intelligent.

# exercice4_graphe_complet.py from typing import TypedDict, Literal from langgraph.graph import StateGraph, START, END class ServiceState(TypedDict): message: str categorie: str # "technique", "commercial", "general" priorite: str # "haute", "normale", "basse" reponse: str # --- Noeud 1 : Classifier --- def classifier(state: ServiceState) -> dict: """Classifie le message en categorie et priorite.""" msg = state["message"].lower() # Detection de categorie par mots-cles mots_techniques = ["bug", "erreur", "crash", "lent", "marche pas", "panne"] mots_commerciaux = ["prix", "tarif", "abonnement", "facture", "payer", "offre"] categorie = "general" for mot in mots_techniques: if mot in msg: categorie = "technique" break for mot in mots_commerciaux: if mot in msg: categorie = "commercial" break # Detection de priorite mots_urgents = ["urgent", "bloque", "impossible", "critique"] priorite = "haute" if any(m in msg for m in mots_urgents) else "normale" print(f"[classifier] Categorie: {categorie}, Priorite: {priorite}") return {"categorie": categorie, "priorite": priorite} # --- Fonction de routage --- def route_par_categorie(state: ServiceState) -> Literal["support_technique", "support_commercial", "support_general"]: mapping = { "technique": "support_technique", "commercial": "support_commercial", "general": "support_general", } return mapping[state["categorie"]] # --- Noeuds de reponse --- def support_technique(state: ServiceState) -> dict: priorite = state["priorite"] if priorite == "haute": rep = "[URGENT] Un technicien senior va prendre en charge votre probleme immediatement." else: rep = "Notre equipe technique va analyser votre probleme. Delai estime : 24h." print(f"[support_technique] {rep}") return {"reponse": rep} def support_commercial(state: ServiceState) -> dict: rep = "Un conseiller commercial va vous recontacter pour discuter de votre demande." print(f"[support_commercial] {rep}") return {"reponse": rep} def support_general(state: ServiceState) -> dict: rep = "Merci pour votre message. Notre equipe va vous repondre dans les plus brefs delais." print(f"[support_general] {rep}") return {"reponse": rep} # --- Construction du graphe --- builder = StateGraph(ServiceState) builder.add_node("classifier", classifier) builder.add_node("support_technique", support_technique) builder.add_node("support_commercial", support_commercial) builder.add_node("support_general", support_general) builder.add_edge(START, "classifier") builder.add_conditional_edges( "classifier", route_par_categorie, { "support_technique": "support_technique", "support_commercial": "support_commercial", "support_general": "support_general", } ) builder.add_edge("support_technique", END) builder.add_edge("support_commercial", END) builder.add_edge("support_general", END) graph = builder.compile() # --- Tests --- etat_vide = {"message": "", "categorie": "", "priorite": "", "reponse": ""} print("=== Test 1 : Probleme technique urgent ===") r1 = graph.invoke({**etat_vide, "message": "J'ai un bug urgent, l'appli crash !"}) print(f"-> {r1['reponse']}\n") print("=== Test 2 : Question commerciale ===") r2 = graph.invoke({**etat_vide, "message": "Quel est le tarif de l'abonnement pro ?"}) print(f"-> {r2['reponse']}\n") print("=== Test 3 : Question generale ===") r3 = graph.invoke({**etat_vide, "message": "Bonjour, quels sont vos horaires ?"}) print(f"-> {r3['reponse']}")
=== Test 1 : Probleme technique urgent === [classifier] Categorie: technique, Priorite: haute [support_technique] [URGENT] Un technicien senior va prendre en charge votre probleme immediatement. -> [URGENT] Un technicien senior va prendre en charge votre probleme immediatement. === Test 2 : Question commerciale === [classifier] Categorie: commercial, Priorite: normale [support_commercial] Un conseiller commercial va vous recontacter pour discuter de votre demande. -> Un conseiller commercial va vous recontacter pour discuter de votre demande. === Test 3 : Question generale === [classifier] Categorie: general, Priorite: normale [support_general] Merci pour votre message. Notre equipe va vous repondre dans les plus brefs delais. -> Merci pour votre message. Notre equipe va vous repondre dans les plus brefs delais.
Le graphe route automatiquement chaque message vers le bon service. La priorite est evaluee en meme temps que la categorie, montrant comment un seul noeud peut mettre a jour plusieurs cles de l'etat.

7. Exercice 5 -- Utiliser MessagesState pour un etat conversationnel

LangGraph fournit MessagesState, un etat preconstruit qui gere automatiquement une liste de messages avec un reducer add_messages integre. Au lieu d'ecraser la liste, chaque noeud ajoute ses messages a la suite.

MessagesState est un TypedDict avec une seule cle messages annotee avec le reducer add_messages. Ce reducer assure que les nouveaux messages sont ajoutes a la liste existante au lieu de la remplacer.
# exercice5_messages_state.py from langgraph.graph import StateGraph, START, END, MessagesState from langchain_core.messages import HumanMessage, AIMessage, SystemMessage def accueillir(state: MessagesState) -> dict: """Premier noeud : accueil du chatbot.""" dernier_msg = state["messages"][-1] print(f"[accueillir] Message recu : {dernier_msg.content}") reponse = AIMessage(content="Bonjour ! Je suis votre assistant. Comment puis-je vous aider ?") return {"messages": [reponse]} def traiter(state: MessagesState) -> dict: """Deuxieme noeud : traitement de la conversation.""" tous_messages = state["messages"] print(f"[traiter] Nombre total de messages : {len(tous_messages)}") for i, msg in enumerate(tous_messages): print(f" [{i}] {msg.__class__.__name__}: {msg.content}") resume = AIMessage(content=f"J'ai bien note votre demande. Conversation de {len(tous_messages)} messages.") return {"messages": [resume]} # Construction du graphe builder = StateGraph(MessagesState) builder.add_node("accueillir", accueillir) builder.add_node("traiter", traiter) builder.add_edge(START, "accueillir") builder.add_edge("accueillir", "traiter") builder.add_edge("traiter", END) graph = builder.compile() # Execution resultat = graph.invoke({ "messages": [ SystemMessage(content="Tu es un assistant serviable."), HumanMessage(content="Bonjour, je cherche de l'aide pour LangGraph."), ] }) print("\n--- Messages finaux ---") for msg in resultat["messages"]: print(f" {msg.__class__.__name__}: {msg.content}")
[accueillir] Message recu : Bonjour, je cherche de l'aide pour LangGraph. [traiter] Nombre total de messages : 3 [0] SystemMessage: Tu es un assistant serviable. [1] HumanMessage: Bonjour, je cherche de l'aide pour LangGraph. [2] AIMessage: Bonjour ! Je suis votre assistant. Comment puis-je vous aider ? --- Messages finaux --- SystemMessage: Tu es un assistant serviable. HumanMessage: Bonjour, je cherche de l'aide pour LangGraph. AIMessage: Bonjour ! Je suis votre assistant. Comment puis-je vous aider ? AIMessage: J'ai bien note votre demande. Conversation de 3 messages.
Remarquez que chaque noeud retourne {"messages": [nouveau_msg]} et le reducer add_messages les ajoute automatiquement a la liste. C'est le fondement de la memoire conversationnelle dans LangGraph. Sans ce reducer, le dernier retour ecraserait la liste entiere.

8. Exercice 6 -- Streamer l'execution du graphe

Le streaming permet d'observer l'execution noeud par noeud en temps reel. LangGraph propose deux modes principaux : values (etat complet apres chaque noeud) et updates (uniquement les modifications).

Modes de streaming

ModeDescriptionUtilisation
"values"Etat complet apres chaque noeudDebug, monitoring
"updates"Modifications uniquement (delta)Production, bandwith
# exercice6_streaming.py from typing import TypedDict from langgraph.graph import StateGraph, START, END class PipelineState(TypedDict): texte: str etape: str resultat: str def nettoyer(state: PipelineState) -> dict: """Noeud 1 : Nettoyage du texte.""" texte_propre = state["texte"].strip().lower() return {"texte": texte_propre, "etape": "nettoyage"} def analyser_longueur(state: PipelineState) -> dict: """Noeud 2 : Analyse de la longueur.""" nb = len(state["texte"].split()) return {"etape": "analyse", "resultat": f"Le texte contient {nb} mots"} def conclure(state: PipelineState) -> dict: """Noeud 3 : Conclusion.""" return {"etape": "conclusion", "resultat": f"[FINAL] {state['resultat']} -- Traitement termine."} # Construction builder = StateGraph(PipelineState) builder.add_node("nettoyer", nettoyer) builder.add_node("analyser_longueur", analyser_longueur) builder.add_node("conclure", conclure) builder.add_edge(START, "nettoyer") builder.add_edge("nettoyer", "analyser_longueur") builder.add_edge("analyser_longueur", "conclure") builder.add_edge("conclure", END) graph = builder.compile() # --- Streaming mode "values" : etat complet --- print("=== Streaming mode 'values' ===") for etat in graph.stream( {"texte": " Bonjour le Monde, Ceci Est un TEST ", "etape": "", "resultat": ""}, stream_mode="values" ): print(f" Etape='{etat.get('etape', '')}' | Texte='{etat.get('texte', '')[:40]}...' | Resultat='{etat.get('resultat', '')}'") print() # --- Streaming mode "updates" : deltas uniquement --- print("=== Streaming mode 'updates' ===") for update in graph.stream( {"texte": " Bonjour le Monde, Ceci Est un TEST ", "etape": "", "resultat": ""}, stream_mode="updates" ): for nom_noeud, delta in update.items(): print(f" Noeud '{nom_noeud}' -> {delta}")
=== Streaming mode 'values' === Etape='' | Texte=' Bonjour le Monde, Ceci Est un TEST ...' | Resultat='' Etape='nettoyage' | Texte='bonjour le monde, ceci est un test...' | Resultat='' Etape='analyse' | Texte='bonjour le monde, ceci est un test...' | Resultat='Le texte contient 7 mots' Etape='conclusion' | Texte='bonjour le monde, ceci est un test...' | Resultat='[FINAL] Le texte contient 7 mots -- Traitement termine.' === Streaming mode 'updates' === Noeud 'nettoyer' -> {'texte': 'bonjour le monde, ceci est un test', 'etape': 'nettoyage'} Noeud 'analyser_longueur' -> {'etape': 'analyse', 'resultat': 'Le texte contient 7 mots'} Noeud 'conclure' -> {'etape': 'conclusion', 'resultat': '[FINAL] Le texte contient 7 mots -- Traitement termine.'}
Le mode "values" emet aussi l'etat initial (avant le premier noeud). C'est pourquoi vous voyez 4 lignes pour 3 noeuds. Le mode "updates" n'emet que les deltas de chaque noeud.

9. Exercice 7 -- Mini-chatbot avec noeud LLM

Integrons maintenant un veritable LLM dans notre graphe. Ce mini-chatbot utilise ChatOpenAI pour generer ses reponses, avec un noeud de pre-traitement qui enrichit le contexte.

# exercice7_chatbot_llm.py import os from typing import Annotated from typing_extensions import TypedDict from langgraph.graph import StateGraph, START, END, MessagesState from langgraph.graph.message import add_messages from langchain_openai import ChatOpenAI from langchain_core.messages import HumanMessage, SystemMessage # Configuration du LLM os.environ["OPENAI_API_KEY"] = "votre-cle-api" llm = ChatOpenAI(model="gpt-4o-mini", temperature=0.7) # Etat personnalise qui herite du comportement MessagesState class ChatState(TypedDict): messages: Annotated[list, add_messages] contexte: str langue: str def preparer_contexte(state: ChatState) -> dict: """Noeud 1 : Prepare le contexte de la conversation.""" dernier_msg = state["messages"][-1].content # Detection simple de la langue mots_francais = ["bonjour", "salut", "merci", "comment", "je", "suis"] est_francais = any(m in dernier_msg.lower() for m in mots_francais) langue = "francais" if est_francais else "anglais" contexte = f"L'utilisateur parle en {langue}. Repondre dans cette langue." print(f"[preparer_contexte] Langue detectee : {langue}") return {"contexte": contexte, "langue": langue} def generer_reponse(state: ChatState) -> dict: """Noeud 2 : Genere la reponse avec le LLM.""" contexte = state["contexte"] messages = [ SystemMessage(content=f"Tu es un assistant pedagogique. {contexte}"), *state["messages"] ] print(f"[generer_reponse] Envoi de {len(messages)} messages au LLM...") reponse = llm.invoke(messages) print(f"[generer_reponse] Reponse generee : {reponse.content[:80]}...") return {"messages": [reponse]} # Construction du graphe builder = StateGraph(ChatState) builder.add_node("preparer_contexte", preparer_contexte) builder.add_node("generer_reponse", generer_reponse) builder.add_edge(START, "preparer_contexte") builder.add_edge("preparer_contexte", "generer_reponse") builder.add_edge("generer_reponse", END) graph = builder.compile() # --- Execution --- resultat = graph.invoke({ "messages": [HumanMessage(content="Bonjour, peux-tu m'expliquer ce qu'est un graphe en informatique ?")], "contexte": "", "langue": "", }) print("\n--- Conversation finale ---") for msg in resultat["messages"]: role = "Utilisateur" if isinstance(msg, HumanMessage) else "Assistant" print(f" {role}: {msg.content}")
[preparer_contexte] Langue detectee : francais [generer_reponse] Envoi de 3 messages au LLM... [generer_reponse] Reponse generee : Bonjour ! Un graphe en informatique est une structure de donnees co... --- Conversation finale --- Utilisateur: Bonjour, peux-tu m'expliquer ce qu'est un graphe en informatique ? Assistant: Bonjour ! Un graphe en informatique est une structure de donnees composee de noeuds (ou sommets) relies par des aretes (ou arcs). On distingue les graphes orientes (les aretes ont une direction) des graphes non-orientes. Les graphes sont utilises partout : reseaux sociaux, GPS, bases de donnees, et meme dans LangGraph pour orchestrer des agents IA !
Ce patron a deux noeuds (pre-traitement + LLM) est tres puissant. Le noeud de pre-traitement peut enrichir le contexte, detecter la langue, verifier les permissions, filtrer les contenus inappropries, etc. Le noeud LLM se concentre uniquement sur la generation.

10. Erreurs Courantes

Oublier START ou END -- Chaque graphe doit avoir au moins une arete depuis START et chaque branche doit mener a END. Sinon, le graphe ne compilera pas ou restera bloque.
Retourner l'etat entier au lieu d'un delta -- Un noeud doit retourner uniquement les cles qu'il modifie, pas une copie complete de l'etat. Retourner toutes les cles ecrase les valeurs des cles que vous ne vouliez pas modifier.
Mauvais nom de noeud dans add_edge -- Le nom passe a add_edge doit correspondre exactement au nom utilise dans add_node. Une faute de frappe provoque une erreur a la compilation.
Oublier de compiler le graphe -- Appeler builder.invoke() au lieu de graph = builder.compile() puis graph.invoke(). Le StateGraph (builder) n'est pas executable directement.
Fonction de routage qui retourne un nom invalide -- La valeur retournee par la fonction de routage doit correspondre exactement a une cle du dictionnaire path_map passe a add_conditional_edges. Sinon, LangGraph leve une ValueError.
Ne pas initialiser toutes les cles de l'etat -- Si votre TypedDict declare des cles obligatoires, assurez-vous de les fournir dans l'etat initial. Les cles manquantes provoqueront des KeyError dans les noeuds qui y accedent.

11. Resume -- Cheatsheet des Patrons

Patron 1 : Graphe lineaire

# START -> A -> B -> C -> END builder.add_edge(START, "A") builder.add_edge("A", "B") builder.add_edge("B", "C") builder.add_edge("C", END)

Patron 2 : Branchement conditionnel

# START -> classifier --(condition)--> A ou B -> END builder.add_edge(START, "classifier") builder.add_conditional_edges("classifier", router_fn, {"choix_a": "A", "choix_b": "B"}) builder.add_edge("A", END) builder.add_edge("B", END)

Patron 3 : MessagesState avec LLM

from langgraph.graph import MessagesState def chatbot(state: MessagesState) -> dict: reponse = llm.invoke(state["messages"]) return {"messages": [reponse]}

Patron 4 : Streaming

# Observer chaque etape en temps reel for event in graph.stream(etat_initial, stream_mode="updates"): print(event)
