TD Pratique - Construire un Agent Autonome avec LangGraph

Troisieme TD LangGraph -- Concevoir un agent autonome avec le pattern ReAct, outils et boucle de raisonnement

1. Recapitulatif -- Qu'est-ce qu'un Agent Autonome ?

Un agent autonome est un programme qui, a partir d'un objectif en langage naturel, est capable de raisonner sur les etapes necessaires, d'agir en appelant des outils externes, et d'observer les resultats pour decider de la suite. Contrairement a un chatbot qui repond a un seul message, l'agent boucle de facon autonome jusqu'a atteindre son objectif ou depasser une limite d'iterations.

Le pattern ReAct (Reasoning + Acting)

Le pattern ReAct combine le raisonnement (Reasoning) et l'action (Acting) en une boucle iterative. A chaque tour :

  1. Think (Penser) -- Le LLM analyse la situation et decide quoi faire
  2. Act (Agir) -- Le LLM appelle un outil ou produit une reponse finale
  3. Observe (Observer) -- Le resultat de l'outil est renvoye au LLM

Cette boucle se repete jusqu'a ce que le LLM decide qu'il a assez d'informations pour repondre, ou que la limite d'iterations est atteinte.

Imaginez l'agent autonome comme un detective qui resout une enquete. Il recoit une mission (la question de l'utilisateur), reflechit aux pistes a explorer (Think), interroge des temoins ou consulte des bases de donnees (Act), analyse les resultats (Observe), puis decide s'il a assez d'elements pour conclure ou s'il doit continuer a enqueter. Le detective ne s'arrete que quand il a la reponse -- ou quand le commissaire lui dit d'arreter (recursion_limit).

2. Principe de Fonctionnement -- La Boucle ReAct

Le coeur de l'agent autonome est un graphe cyclique. Contrairement au chatbot dont le graphe est un DAG (graphe oriente acyclique), l'agent possede une boucle qui lui permet de revenir a l'etape de reflexion apres chaque observation.

Les trois phases de la boucle

Phase 1 : Think (Raisonner) -- Le LLM recoit l'objectif et l'historique des actions precedentes. Il analyse la situation et decide quelle action entreprendre : appeler un outil specifique ou produire une reponse finale.
Phase 2 : Act (Agir) -- Si le LLM a decide d'appeler un outil, le noeud tools execute l'outil avec les parametres fournis par le LLM. Le resultat est enveloppe dans un ToolMessage et ajoute a l'historique.
Phase 3 : Observe (Observer) -- Le resultat de l'outil est renvoye au LLM qui l'analyse. Le LLM peut alors decider d'appeler un autre outil (retour a Act) ou de produire sa reponse finale (sortie vers END).

Diagramme de la boucle ReAct

La grande difference avec le chatbot du TD precedent est la fleche de retour (en vert pointille) qui va de tools vers agent. C'est cette fleche qui cree le cycle dans le graphe : l'agent peut appeler un outil, observer le resultat, puis decider d'appeler un autre outil. Sans le recursion_limit, cette boucle pourrait tourner indefiniment.

3. Les Donnees -- Outils, Schemas et ToolMessage

Les outils sont le bras arme de l'agent autonome. Sans outils, le LLM ne peut que raisonner. Avec des outils, il peut agir sur le monde exterieur : rechercher sur le web, interroger une base de donnees, executer du code, etc.

3.1 Definition des outils avec @tool

from langchain_core.tools import tool # Chaque outil est une fonction Python decoree avec @tool # La docstring est ESSENTIELLE : c'est elle que le LLM lit pour # decider quand et comment utiliser l'outil. @tool def rechercher_web(query: str) -> str: """Recherche des informations sur le web. Utilise cet outil quand tu as besoin d'informations actuelles ou de donnees que tu ne possedes pas dans ta memoire. Args: query: La requete de recherche en langage naturel. Returns: Les resultats de la recherche web.""" # Simulation d'une recherche web resultats = { "population france": "La population de la France est d'environ 68 millions d'habitants (2024).", "capitale japon": "La capitale du Japon est Tokyo, avec 14 millions d'habitants.", "python version": "La derniere version stable de Python est 3.12.1 (decembre 2024).", } for key, val in resultats.items(): if key in query.lower(): return val return f"Resultats de recherche pour '{query}': Aucun resultat specifique trouve." @tool def calculer_expression(expression: str) -> str: """Evalue une expression mathematique. Utilise cet outil pour tout calcul arithmetique, pourcentage ou conversion. Args: expression: L'expression mathematique a evaluer (ex: '15 * 24 + 30'). Returns: Le resultat du calcul.""" try: result = eval(expression) return f"Resultat : {expression} = {result}" except Exception as e: return f"Erreur de calcul : {str(e)}" @tool def lire_fichier(chemin: str) -> str: """Lit le contenu d'un fichier texte. Utilise cet outil pour acceder au contenu de fichiers locaux. Args: chemin: Le chemin du fichier a lire. Returns: Le contenu du fichier ou un message d'erreur.""" try: with open(chemin, "r") as f: return f.read() except FileNotFoundError: return f"Fichier non trouve : {chemin}" except Exception as e: return f"Erreur de lecture : {str(e)}" tools = [rechercher_web, calculer_expression, lire_fichier]
La docstring de l'outil est son mode d'emploi. Le LLM la lit pour decider quand utiliser l'outil et quels parametres lui passer. Une docstring vague produira des appels d'outils imprecis. Soyez aussi clair et detaille que possible.

4. Les Parametres -- Controle de l'Agent

4.1 recursion_limit (critique)

recursion_limit (int, defaut: 25)

Le parametre le plus important pour un agent autonome. Il definit le nombre maximum d'etapes (noeuds traverses) que le graphe peut executer avant de s'arreter de force. Sans ce garde-fou, un agent mal configure pourrait boucler indefiniment.

ValeurUsageRisque
10-15Taches simples (1-2 outils)L'agent pourrait manquer d'etapes
25Taches moderees (defaut)Bon equilibre securite/flexibilite
50-100Taches complexes (nombreux outils)Cout eleve si boucle inutile
# Definir la recursion_limit dans la configuration config = { "configurable": {"thread_id": "agent_1"}, "recursion_limit": 25 # Maximum 25 etapes } # Si la limite est atteinte, LangGraph leve : # GraphRecursionError: Recursion limit of 25 reached
Ne jamais deployer un agent sans recursion_limit. Un agent qui boucle indefiniment consomme des tokens (et donc de l'argent) sans produire de resultat utile. En production, commencez avec une limite basse (15-25) et augmentez si necessaire.

4.2 max_iterations (arret anticipe personnalise)

max_iterations (int, personnalise)

En complement de recursion_limit, vous pouvez implementer un compteur d'iterations dans l'etat de l'agent pour un controle plus fin.

class AgentStateWithCounter(TypedDict): messages: Annotated[list, add_messages] iteration_count: int # Compteur d'iterations max_iterations: int # Limite personnalisee def agent_with_limit(state: AgentStateWithCounter): # Verifier la limite avant de continuer if state["iteration_count"] >= state["max_iterations"]: return { "messages": [AIMessage(content="Limite d'iterations atteinte. Voici ma meilleure reponse...")] } response = llm_with_tools.invoke(state["messages"]) return { "messages": [response], "iteration_count": state["iteration_count"] + 1 }

4.3 Selection des outils (tool_choice)

tool_choice (str ou dict)

Controle la facon dont le LLM choisit les outils.

ValeurComportement
"auto"Le LLM decide librement (defaut, recommande pour les agents)
"required"Le LLM doit appeler au moins un outil a chaque tour
"none"Le LLM ne peut pas appeler d'outils
{"type": "function", "function": {"name": "..."}}Force l'appel d'un outil specifique
# Differentes strategies de selection d'outils llm_auto = ChatOpenAI(model="gpt-4o-mini").bind_tools(tools) # Auto (defaut) llm_required = ChatOpenAI(model="gpt-4o-mini").bind_tools(tools, tool_choice="required") # Force un outil llm_specific = ChatOpenAI(model="gpt-4o-mini").bind_tools(tools, tool_choice={ "type": "function", "function": {"name": "rechercher_web"} }) # Force un outil specifique

4.4 Early stopping (arret anticipe)

Strategies d'arret anticipe

L'agent peut s'arreter de plusieurs manieres :

  • Naturel : Le LLM decide de ne plus appeler d'outils (reponse finale)
  • recursion_limit : Le nombre maximum d'etapes est atteint
  • max_iterations : Un compteur personnalise dans l'etat atteint sa limite
  • Condition dans le routage : Une logique metier force l'arret
L'arret naturel est le comportement ideal : l'agent a reuni assez d'informations et produit sa reponse finale. Les autres mecanismes sont des filets de securite. Dans un bon agent, le recursion_limit ne devrait presque jamais etre atteint.

5. Les Metriques -- Evaluer la Performance de l'Agent

Un agent autonome se mesure differemment d'un chatbot simple. Voici les metriques essentielles :

Taux de completion des taches

Definition : Le pourcentage de taches que l'agent reussit a accomplir correctement sans atteindre le recursion_limit et en produisant une reponse exacte.

Comment mesurer : Preparer un jeu de test avec des taches et leurs reponses attendues. Comparer la sortie de l'agent avec la reponse de reference.

Cible : Superieur a 85% pour un agent en production.

Nombre d'etapes par tache

Definition : Le nombre moyen de tours de boucle (Think-Act-Observe) necessaires pour accomplir une tache.

Comment mesurer : Compter le nombre de ToolMessages dans l'historique final.

Cible : Le moins possible. Un agent efficace resout une tache en 2-4 etapes.

Efficacite d'utilisation des outils

Definition : Le ratio entre les appels d'outils utiles (qui apportent de l'information pertinente) et les appels inutiles (resultats non exploites ou redondants).

Comment mesurer : Analyser les ToolMessages : est-ce que chaque resultat contribue a la reponse finale ?

Cible : Superieur a 80%. Un agent qui appelle 5 outils mais n'en utilise que 2 dans sa reponse a une efficacite de 40%.

def analyser_performance_agent(result): """Analyse les metriques de performance apres l'execution d'un agent.""" messages = result["messages"] # Compter les types de messages human_msgs = [m for m in messages if m.type == "human"] ai_msgs = [m for m in messages if m.type == "ai"] tool_msgs = [m for m in messages if m.type == "tool"] tool_call_msgs = [m for m in ai_msgs if getattr(m, "tool_calls", [])] print(f"Messages humains : {len(human_msgs)}") print(f"Messages IA : {len(ai_msgs)}") print(f"Appels d'outils : {len(tool_call_msgs)}") print(f"Resultats d'outils : {len(tool_msgs)}") print(f"Nombre de boucles : {len(tool_msgs)}") # Outils utilises for tm in tool_msgs: print(f" - Outil '{tm.name}' : {tm.content[:60]}...") return { "total_steps": len(tool_msgs), "tools_used": [tm.name for tm in tool_msgs], "final_answer": ai_msgs[-1].content if ai_msgs else "Pas de reponse" }

Etape 2 : Definition des outils

# --- Outil 1 : Recherche web --- @tool def rechercher_web(query: str) -> str: """Recherche des informations actuelles sur internet. Utilise cet outil pour les questions factuelles, les donnees recentes ou toute information que tu ne possedes pas. Args: query: La requete de recherche.""" resultats = { "population france": "France : 68.4 millions d'habitants (INSEE, 2024).", "capitale japon": "Tokyo est la capitale du Japon, 14 millions d'habitants.", "pib france": "Le PIB de la France est de 2 780 milliards d'euros (2023).", "superficie france": "La France metropolitaine a une superficie de 551 695 km2.", "python version": "Python 3.12.1 est la derniere version stable (decembre 2024).", } for key, val in resultats.items(): if key in query.lower(): return val return f"Recherche '{query}' : information non trouvee dans la base simulee." # --- Outil 2 : Calculatrice --- @tool def calculer(expression: str) -> str: """Effectue un calcul mathematique precis. Utilise cet outil pour les operations arithmetiques, pourcentages, conversions et tout calcul numerique. Args: expression: L'expression mathematique (ex: '68.4 * 1000000 / 551695').""" try: result = eval(expression) if isinstance(result, float): return f"Resultat : {expression} = {result:.4f}" return f"Resultat : {expression} = {result}" except Exception as e: return f"Erreur : impossible d'evaluer '{expression}'. Raison : {str(e)}" # --- Outil 3 : Lecture de fichiers --- @tool def lire_fichier(chemin: str) -> str: """Lit le contenu d'un fichier texte local. Utilise cet outil pour acceder a des donnees stockees dans des fichiers. Args: chemin: Le chemin absolu ou relatif du fichier a lire.""" try: with open(chemin, "r", encoding="utf-8") as f: content = f.read() if len(content) > 2000: return content[:2000] + "\n... (fichier tronque)" return content except FileNotFoundError: return f"Erreur : fichier '{chemin}' introuvable." except Exception as e: return f"Erreur de lecture : {str(e)}" tools = [rechercher_web, calculer, lire_fichier]

Etape 3 : Creer le LLM, les noeuds et le routage

# --- Initialisation du LLM avec outils --- llm = ChatOpenAI(model="gpt-4o-mini", temperature=0) llm_with_tools = llm.bind_tools(tools) AGENT_SYSTEM_PROMPT = """Tu es un agent de recherche autonome et rigoureux. Tes regles : 1. Decompose les questions complexes en sous-questions 2. Utilise les outils disponibles pour obtenir des donnees factuelles 3. Verifie tes calculs avec l'outil calculer 4. Ne fais jamais d'approximations quand un outil peut donner une reponse precise 5. Synthetise toutes les informations collectees dans une reponse claire et structuree""" # --- Noeud agent : reflexion et decision --- def agent_node(state: AgentState): """Le cerveau de l'agent : raisonne et decide d'agir ou de repondre.""" messages = state["messages"] # Ajouter le system prompt s'il est absent if not messages or messages[0].type != "system": messages = [SystemMessage(content=AGENT_SYSTEM_PROMPT)] + messages response = llm_with_tools.invoke(messages) return {"messages": [response]} # --- Fonction de routage conditionnel --- def should_continue(state: AgentState) -> Literal["tools", "__end__"]: """Decide si l'agent doit appeler un outil ou produire la reponse finale.""" last_message = state["messages"][-1] # Si le dernier AIMessage contient des tool_calls, on continue if hasattr(last_message, "tool_calls") and last_message.tool_calls: return "tools" # Sinon, on a la reponse finale return "__end__"

Etape 4 : Assembler le graphe avec cycle

# --- Construction du graphe de l'agent --- graph_builder = StateGraph(AgentState) # Ajout des noeuds graph_builder.add_node("agent", agent_node) graph_builder.add_node("tools", ToolNode(tools=tools)) # Aretes d'entree graph_builder.add_edge(START, "agent") # Routage conditionnel : c'est ici que la boucle ReAct se cree graph_builder.add_conditional_edges( "agent", # Noeud source should_continue, # Fonction de decision { "tools": "tools", # Si tool_calls -> executer les outils "__end__": END # Sinon -> reponse finale } ) # LA BOUCLE : apres les outils, retour a l'agent pour analyser les resultats graph_builder.add_edge("tools", "agent") # Compilation avec memoire et recursion_limit memory = MemorySaver() agent = graph_builder.compile(checkpointer=memory) print("Agent autonome compile avec succes !")
Agent autonome compile avec succes !
La ligne graph_builder.add_edge("tools", "agent") est ce qui cree la boucle cyclique. C'est la difference fondamentale avec un chatbot simple. L'agent peut maintenant appeler des outils en boucle, tant que le LLM le juge necessaire.

Etape 5 : Fonction d'execution de l'agent

def executer_agent(tache: str, thread_id: str = "agent_default", verbose: bool = True): """Execute l'agent autonome sur une tache donnee.""" config = { "configurable": {"thread_id": thread_id}, "recursion_limit": 25 } if verbose: print(f"Tache : {tache}") print("=" * 60) result = agent.invoke( {"messages": [HumanMessage(content=tache)]}, config ) if verbose: # Afficher le deroulement etape par etape for msg in result["messages"]: if msg.type == "human": print(f"[HUMAIN] {msg.content}") elif msg.type == "ai" and getattr(msg, "tool_calls", []): for tc in msg.tool_calls: print(f"[THINK+ACT] Appel outil : {tc['name']}({tc['args']})") elif msg.type == "tool": print(f"[OBSERVE] Resultat de '{msg.name}' : {msg.content[:80]}...") elif msg.type == "ai": print(f"[REPONSE FINALE] {msg.content}") print("=" * 60) return result

8. Erreurs Courantes -- Les Six Pieges de l'Agent Autonome

Erreur 1 : Boucle infinie par absence de recursion_limit

Symptome : L'agent tourne en boucle indefiniment, consommant des tokens et des ressources sans jamais produire de reponse.

# FAUX -- pas de limite, risque de boucle infinie config = {"configurable": {"thread_id": "danger"}} agent.invoke({"messages": [HumanMessage(content="Recherche tout sur la France")]}, config) # Risque : l'agent appelle rechercher_web en boucle sans fin # CORRECT -- toujours definir une limite config = { "configurable": {"thread_id": "safe"}, "recursion_limit": 25 # Maximum 25 etapes } agent.invoke({"messages": [HumanMessage(content="Recherche tout sur la France")]}, config)

Conseil : En developpement, utilisez recursion_limit: 10 pour detecter rapidement les boucles infinies.

Erreur 2 : Schema d'outil incorrect ou docstring vague

Symptome : Le LLM passe de mauvais arguments a l'outil, ou ne l'utilise jamais car il ne comprend pas quand l'appeler.

# FAUX -- docstring inutile, pas d'indication sur les parametres @tool def mauvais_outil(x): """Fait quelque chose.""" # Trop vague ! return str(x) # CORRECT -- docstring precise avec Args et Returns @tool def bon_outil(query: str) -> str: """Recherche des informations sur le web. Utilise cet outil quand l'utilisateur demande des donnees factuelles. Args: query: La requete de recherche en langage naturel. Returns: Les resultats de la recherche.""" return f"Resultats pour '{query}'"

Erreur 3 : Oublier l'arete de retour tools -> agent

Symptome : L'agent appelle un outil mais ne peut pas analyser le resultat. Le graphe se termine immediatement apres l'execution de l'outil.

# FAUX -- les outils menent directement a END (pas de boucle) graph_builder.add_edge("tools", END) # L'agent appelle un outil puis s'arrete : pas de ReAct ! # CORRECT -- retour au noeud agent pour analyser le resultat graph_builder.add_edge("tools", "agent") # L'agent voit le ToolMessage et peut raisonner a nouveau

Erreur 4 : Ne pas gerer les erreurs d'outils

Symptome : Un outil leve une exception non capturee, ce qui plante tout le graphe et perd l'etat de la conversation.

# FAUX -- pas de gestion d'erreur @tool def outil_fragile(chemin: str) -> str: """Lit un fichier.""" with open(chemin) as f: return f.read() # Plante si le fichier n'existe pas ! # CORRECT -- gestion robuste des erreurs @tool def outil_robuste(chemin: str) -> str: """Lit un fichier.""" try: with open(chemin) as f: return f.read() except FileNotFoundError: return f"Erreur : fichier '{chemin}' introuvable." except Exception as e: return f"Erreur : {str(e)}"

L'agent peut comprendre un message d'erreur et adapter sa strategie. Une exception Python non capturee est irrecuperable.

Erreur 5 : Utiliser tool_choice="required" sans arret conditionnel

Symptome : L'agent est force d'appeler un outil a chaque tour, ce qui cree une boucle infinie car il ne peut jamais produire de reponse finale.

# FAUX -- le LLM doit toujours appeler un outil, jamais repondre llm_stuck = ChatOpenAI(model="gpt-4o-mini").bind_tools( tools, tool_choice="required" ) # Boucle infinie : tool -> agent -> tool -> agent -> ... # CORRECT -- utiliser "auto" pour que le LLM puisse choisir de conclure llm_free = ChatOpenAI(model="gpt-4o-mini").bind_tools(tools) # tool_choice="auto" est le defaut, le LLM peut repondre sans outil

Erreur 6 : Confondre le nombre d'appels d'outils et le recursion_limit

Symptome : Le recursion_limit est atteint trop tot car il compte toutes les etapes du graphe (noeuds traverses), pas seulement les appels d'outils.

# Attention : le recursion_limit compte les NOEUDS traverses # Une boucle ReAct complete = 2 noeuds (agent + tools) # Donc recursion_limit=10 permet environ 4-5 boucles, pas 10 ! # Calcul : START(1) + agent(1) + tools(1) + agent(1) + tools(1) + ... # recursion_limit=10 -> environ 4 appels d'outils maximum # Si vous voulez 10 appels d'outils : config = { "configurable": {"thread_id": "demo"}, "recursion_limit": 25 # Permet environ 12 appels d'outils }

Les trois regles d'or de l'agent autonome

1. Toujours definir un recursion_limit. Sans lui, un agent mal configure peut boucler indefiniment et consommer des ressources.

2. Ecrire des docstrings precises pour chaque outil. Le LLM les lit pour decider quand et comment utiliser les outils. Vague = inutile.

3. Gerer les erreurs dans chaque outil. Un outil qui leve une exception plante tout le graphe. Un outil qui retourne un message d'erreur permet a l'agent de s'adapter et de continuer.

Felicitations ! Vous avez maintenant les bases pour construire des agents autonomes avec LangGraph. La prochaine etape serait d'explorer les architectures multi-agents, ou plusieurs agents collaborent pour resoudre des taches encore plus complexes. Mais avant cela, assurez-vous de bien maitriser la boucle ReAct : c'est la brique fondamentale de tout systeme agentique.
