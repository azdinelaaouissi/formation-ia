1. C'est quoi LangGraph ?

🎯Définition simple : LangGraph est une bibliothèque Python qui permet de créer des applications IA sous forme de graphes. Au lieu d'écrire du code linéaire (étape 1 → étape 2 → étape 3), tu dessines un graphe avec des nœuds (actions) et des arêtes (connexions).

Pourquoi LangGraph ?

Avec LangChain classique, tu fais des chaînes linéaires :

# LangChain classique = linéaire
question → retriever → prompt → llm → réponse

Avec LangGraph, tu peux faire des boucles, des conditions, des retours en arrière :

# LangGraph = graphe avec boucles et conditions
question → analyser → ┬─ si simple → répondre directement
                      ├─ si complexe → chercher → répondre
                      └─ si pas sûr → reformuler → recommencer ↩

Analogie

🗺️Imagine un GPS :

  • LangChain = une ligne droite du point A au point B
  • LangGraph = un GPS intelligent qui peut prendre des détours, revenir en arrière si la route est bloquée, et choisir le meilleur chemin selon les conditions

Quand utiliser LangGraph ?

SituationOutil
Chaîne simple : question → réponseLangChain (LCEL)
RAG basique : chercher → répondreLangChain (LCEL)
Agent qui doit boucler et réfléchirLangGraph
Workflow avec conditions complexesLangGraph
Multi-agents qui collaborentLangGraph
Human-in-the-loop (intervention humaine)LangGraph

LangGraph repose sur 5 concepts :

4. State (État)

Le State est un dictionnaire typé qui contient toutes les données. Chaque nœud peut lire et modifier le State.

Méthode 1 : TypedDict (simple)

from typing import TypedDict

# Définir le State = les données de notre graphe
class MonState(TypedDict):
    question: str       # La question de l'utilisateur
    reponse: str        # La réponse générée
    etape: str          # L'étape actuelle

💡Explication : TypedDict c'est comme un dictionnaire Python classique, mais avec des types définis. Ça aide Python à vérifier que tu mets les bonnes données.

Méthode 2 : Avec Annotated et opérateur (pour les listes)

from typing import TypedDict, Annotated
from operator import add

class MonState(TypedDict):
    messages: Annotated[list, add]   # Les messages s'AJOUTENT au lieu de se remplacer
    question: str                     # Se remplace à chaque fois

⚠️Important :

  • Sans Annotated : la nouvelle valeur remplace l'ancienne
  • Avec Annotated[list, add] : la nouvelle valeur s'ajoute à l'ancienne (concaténation)

Méthode 3 : MessagesState (pour les chatbots)

from langgraph.graph import MessagesState

# MessagesState a déjà un champ "messages" avec Annotated[list, add]
# C'est un raccourci très pratique pour les chatbots
class MonState(MessagesState):
    nom_utilisateur: str  # Tu peux ajouter tes propres champs

Résumé des 3 méthodes

MéthodeQuand l'utiliser
TypedDictCas simple, pas de liste à accumuler
Annotated[list, add]Quand tu veux accumuler des données dans une liste
MessagesStatePour les chatbots (pré-configuré avec messages)

5. Nodes (Nœuds)

Un Node est simplement une fonction Python qui :

  1. Reçoit le State en entrée
  2. Fait quelque chose (appeler un LLM, chercher dans une base, etc.)
  3. Retourne les modifications au State
# Un nœud = une fonction Python simple
def analyser_question(state):
    """Ce nœud analyse la question de l'utilisateur"""
    question = state["question"]  # Lire le state

    # Faire quelque chose
    if len(question) > 50:
        difficulte = "complexe"
    else:
        difficulte = "simple"

    # Retourner les modifications (pas tout le state, juste les changements)
    return {"difficulte": difficulte}


# Un autre nœud
def generer_reponse(state):
    """Ce nœud génère une réponse"""
    question = state["question"]
    difficulte = state["difficulte"]

    reponse = f"Réponse à : {question} (niveau: {difficulte})"

    return {"reponse": reponse}

💡Règle d'or : Un nœud retourne seulement les champs modifiés, pas tout le state. LangGraph fusionne automatiquement.

Nœud avec un LLM

from langchain_ollama import ChatOllama

llm = ChatOllama(model="mistral")

def appeler_llm(state):
    """Nœud qui appelle un LLM"""
    question = state["question"]

    # Appeler le LLM
    reponse = llm.invoke(question)

    return {"reponse": reponse.content}

6. Edges (Arêtes)

Les Edges connectent les nœuds entre eux. Il y a 3 types :

Type 1 : Edge normal (direct)

# Après "node_a", aller toujours vers "node_b"
graph.add_edge("node_a", "node_b")

Type 2 : Edge depuis START

from langgraph.graph import START, END

# Le graphe commence par "node_a"
graph.add_edge(START, "node_a")

# Le graphe se termine après "node_c"
graph.add_edge("node_c", END)

Type 3 : Conditional Edge (conditionnel)

# Fonction qui décide où aller
def decider_route(state):
    if state["difficulte"] == "simple":
        return "reponse_directe"    # Nom du nœud cible
    else:
        return "recherche"          # Nom d'un autre nœud

# Après "analyser", appeler decider_route pour choisir
graph.add_conditional_edges(
    "analyser",           # Nœud source
    decider_route,         # Fonction de décision
    {                      # Mapping : retour → nœud cible
        "reponse_directe": "repondre",
        "recherche": "chercher"
    }
)

📌Résumé des edges :

TypeCodeUtilité
Normaladd_edge("A", "B")A → B toujours
STARTadd_edge(START, "A")Point d'entrée du graphe
ENDadd_edge("A", END)Point de sortie du graphe
Conditionneladd_conditional_edges(...)Choisir le nœud suivant selon une condition

7. Premier graphe simple

On assemble tout ! Voici un graphe complet étape par étape :

# ============================================
# ÉTAPE 1 : Imports
# ============================================
from typing import TypedDict
from langgraph.graph import StateGraph, START, END

# ============================================
# ÉTAPE 2 : Définir le State
# ============================================
class State(TypedDict):
    nom: str           # Nom de l'utilisateur
    message: str       # Le message à traiter
    resultat: str      # Le résultat final

# ============================================
# ÉTAPE 3 : Définir les nœuds (fonctions)
# ============================================
def saluer(state):
    """Nœud 1 : Créer un message de bienvenue"""
    nom = state["nom"]
    return {"message": f"Bonjour {nom} ! Bienvenue."}

def transformer(state):
    """Nœud 2 : Transformer le message en majuscules"""
    message = state["message"]
    return {"resultat": message.upper()}

# ============================================
# ÉTAPE 4 : Construire le graphe
# ============================================
graph = StateGraph(State)          # Créer le graphe avec notre State

graph.add_node("saluer", saluer)             # Ajouter nœud 1
graph.add_node("transformer", transformer)   # Ajouter nœud 2

graph.add_edge(START, "saluer")              # Commencer par "saluer"
graph.add_edge("saluer", "transformer")     # Puis aller à "transformer"
graph.add_edge("transformer", END)           # Terminer

# ============================================
# ÉTAPE 5 : Compiler et exécuter
# ============================================
app = graph.compile()              # Compiler le graphe

# Exécuter avec des données initiales
resultat = app.invoke({"nom": "Azdine"})

print(resultat)
# {'nom': 'Azdine', 'message': 'Bonjour Azdine ! Bienvenue.', 'resultat': 'BONJOUR AZDINE ! BIENVENUE.'}

8. Conditional Edges (Arêtes conditionnelles)

C'est la fonctionnalité la plus puissante de LangGraph. Ça permet de choisir dynamiquement quel nœud exécuter ensuite.

from typing import TypedDict
from langgraph.graph import StateGraph, START, END

# State avec un champ "langue"
class State(TypedDict):
    texte: str
    langue: str
    resultat: str

# Nœud 1 : Détecter la langue
def detecter_langue(state):
    texte = state["texte"]
    if "hello" in texte.lower():
        return {"langue": "anglais"}
    elif "bonjour" in texte.lower():
        return {"langue": "francais"}
    else:
        return {"langue": "inconnu"}

# Nœud 2a : Répondre en anglais
def repondre_anglais(state):
    return {"resultat": "Hi! How can I help you?"}

# Nœud 2b : Répondre en français
def repondre_francais(state):
    return {"resultat": "Salut ! Comment puis-je t'aider ?"}

# Nœud 2c : Langue inconnue
def repondre_inconnu(state):
    return {"resultat": "Désolé, je ne comprends pas cette langue."}

# ---- Fonction de routage ----
def router(state):
    """Décide quel nœud appeler selon la langue"""
    langue = state["langue"]
    if langue == "anglais":
        return "anglais"
    elif langue == "francais":
        return "francais"
    else:
        return "inconnu"

# ---- Construire le graphe ----
graph = StateGraph(State)

graph.add_node("detecter", detecter_langue)
graph.add_node("anglais", repondre_anglais)
graph.add_node("francais", repondre_francais)
graph.add_node("inconnu", repondre_inconnu)

graph.add_edge(START, "detecter")

# Edge conditionnel : après "detecter", appeler router()
graph.add_conditional_edges(
    "detecter",       # Nœud source
    router,            # Fonction qui retourne le nom de la route
    {                  # Mapping route → nœud cible
        "anglais": "anglais",
        "francais": "francais",
        "inconnu": "inconnu",
    }
)

# Tous les nœuds de réponse mènent à END
graph.add_edge("anglais", END)
graph.add_edge("francais", END)
graph.add_edge("inconnu", END)

app = graph.compile()

# Test
print(app.invoke({"texte": "Bonjour le monde"}))
# → {'texte': 'Bonjour le monde', 'langue': 'francais', 'resultat': 'Salut ! Comment puis-je t'aider ?'}

9. Cycles et boucles

C'est ce qui différencie vraiment LangGraph de LangChain : la capacité de boucler. Un nœud peut renvoyer vers un nœud précédent.

🔄Exemple concret : Un agent qui essaie de répondre. Si la réponse n'est pas bonne, il recommence. Maximum 3 essais.

from typing import TypedDict
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    nombre: int
    tentatives: int
    resultat: str

# Nœud : deviner un nombre
def deviner(state):
    import random
    tentatives = state.get("tentatives", 0) + 1
    guess = random.randint(1, 10)
    print(f"  Tentative {tentatives}: je devine {guess}")
    return {"nombre": guess, "tentatives": tentatives}

# Nœud : vérifier
def verifier(state):
    if state["nombre"] == 7:
        return {"resultat": "Trouvé !"}
    return {"resultat": "Pas trouvé"}

# Fonction de routage : continuer ou arrêter ?
def continuer_ou_stop(state):
    if state["resultat"] == "Trouvé !":
        return "fin"
    if state["tentatives"] >= 5:       # Maximum 5 tentatives
        return "fin"
    return "recommencer"              # Boucler !

# Construire le graphe
graph = StateGraph(State)
graph.add_node("deviner", deviner)
graph.add_node("verifier", verifier)

graph.add_edge(START, "deviner")
graph.add_edge("deviner", "verifier")

# BOUCLE : après "verifier", soit on recommence soit on finit
graph.add_conditional_edges(
    "verifier",
    continuer_ou_stop,
    {
        "recommencer": "deviner",   # ← Retour au nœud "deviner" (BOUCLE)
        "fin": END
    }
)

app = graph.compile()
resultat = app.invoke({"tentatives": 0})
print(resultat)

🚨Attention aux boucles infinies ! Mets TOUJOURS une condition d'arrêt (ex: max tentatives). Sinon ton programme ne s'arrêtera jamais.

10. Graphe avec LLM

Maintenant, connectons LangGraph avec un vrai LLM :

from typing import TypedDict
from langgraph.graph import StateGraph, START, END
from langchain_ollama import ChatOllama
from langchain_core.prompts import ChatPromptTemplate

# LLM local
llm = ChatOllama(model="mistral")

# State
class State(TypedDict):
    question: str
    categorie: str
    reponse: str

# Nœud 1 : Classifier la question
def classifier(state):
    prompt = ChatPromptTemplate.from_template(
        "Classifie cette question dans une catégorie parmi : "
        "science, histoire, cuisine, sport.\n"
        "Question: {question}\n"
        "Réponds UNIQUEMENT le mot de la catégorie."
    )
    chain = prompt | llm
    result = chain.invoke({"question": state["question"]})
    return {"categorie": result.content.strip().lower()}

# Nœud 2 : Répondre avec contexte
def repondre(state):
    prompt = ChatPromptTemplate.from_template(
        "Tu es un expert en {categorie}.\n"
        "Réponds à cette question de manière claire et simple :\n"
        "{question}"
    )
    chain = prompt | llm
    result = chain.invoke({
        "categorie": state["categorie"],
        "question": state["question"]
    })
    return {"reponse": result.content}

# Construire
graph = StateGraph(State)
graph.add_node("classifier", classifier)
graph.add_node("repondre", repondre)

graph.add_edge(START, "classifier")
graph.add_edge("classifier", "repondre")
graph.add_edge("repondre", END)

app = graph.compile()
result = app.invoke({"question": "Quelle est la vitesse de la lumière ?"})
print(f"Catégorie: {result['categorie']}")
print(f"Réponse: {result['reponse']}")

11. Agent ReAct avec outils

Le pattern ReAct (Reason + Act) : le LLM réfléchit, utilise un outil, observe le résultat, et boucle jusqu'à avoir la réponse finale.

from typing import TypedDict, Annotated
from operator import add
from langgraph.graph import StateGraph, START, END, MessagesState
from langgraph.prebuilt import ToolNode, tools_condition
from langchain_ollama import ChatOllama
from langchain_core.tools import tool

# ---- Définir des outils ----
@tool
def calculer(expression: str) -> str:
    """Calculer une expression mathématique"""
    try:
        return str(eval(expression))
    except:
        return "Erreur de calcul"

@tool
def meteo(ville: str) -> str:
    """Obtenir la météo d'une ville"""
    # Simulé pour l'exemple
    meteos = {"paris": "22°C, ensoleillé", "lyon": "18°C, nuageux"}
    return meteos.get(ville.lower(), "Ville inconnue")

# ---- LLM avec outils ----
tools = [calculer, meteo]
llm = ChatOllama(model="mistral").bind_tools(tools)

# ---- Nœud agent : appeler le LLM ----
def agent(state: MessagesState):
    result = llm.invoke(state["messages"])
    return {"messages": [result]}

# ---- Construire le graphe ReAct ----
graph = StateGraph(MessagesState)

graph.add_node("agent", agent)                  # Nœud 1 : le LLM réfléchit
graph.add_node("tools", ToolNode(tools))         # Nœud 2 : exécuter les outils

graph.add_edge(START, "agent")

# Après "agent" : si le LLM veut utiliser un outil → "tools"
#                 sinon → END (réponse finale)
graph.add_conditional_edges(
    "agent",
    tools_condition    # Fonction built-in qui vérifie si le LLM veut un outil
)

# Après avoir utilisé l'outil, revenir à l'agent (BOUCLE)
graph.add_edge("tools", "agent")

app = graph.compile()

# Utiliser l'agent
from langchain_core.messages import HumanMessage
result = app.invoke({"messages": [HumanMessage(content="Combien fait 25 * 48 ?")]})
print(result["messages"][-1].content)

💡ToolNode et tools_condition sont des composants pré-faits par LangGraph. Ils gèrent automatiquement l'exécution des outils et la vérification si le LLM veut en utiliser un.

12. Human-in-the-Loop

Human-in-the-Loop = mettre en pause le graphe pour demander une confirmation humaine avant de continuer.

from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import MemorySaver
from typing import TypedDict

class State(TypedDict):
    action: str
    confirmation: str
    resultat: str

def proposer_action(state):
    return {"action": "Supprimer le fichier temp.txt"}

def executer(state):
    return {"resultat": f"Action exécutée : {state['action']}"}

graph = StateGraph(State)
graph.add_node("proposer", proposer_action)
graph.add_node("executer", executer)

graph.add_edge(START, "proposer")
graph.add_edge("proposer", "executer")
graph.add_edge("executer", END)

# ---- La clé : interrupt_before ----
# Le graphe se met en PAUSE avant le nœud "executer"
checkpointer = MemorySaver()
app = graph.compile(
    checkpointer=checkpointer,
    interrupt_before=["executer"]    # ← PAUSE ICI
)

# Exécuter - le graphe s'arrête avant "executer"
config = {"configurable": {"thread_id": "1"}}
result = app.invoke({}, config)
print(f"Action proposée : {result['action']}")

# L'humain confirme...
reponse = input("Confirmer ? (oui/non) : ")

if reponse == "oui":
    # Reprendre l'exécution avec None = continuer
    result = app.invoke(None, config)
    print(result["resultat"])
else:
    print("Action annulée.")

📌Les 3 points clés :

  1. MemorySaver() = sauvegarde l'état du graphe en mémoire
  2. interrupt_before=["executer"] = pause avant ce nœud
  3. app.invoke(None, config) = reprendre l'exécution là où on s'est arrêté

13. Checkpointing (Mémoire persistante)

Le checkpointing permet de sauvegarder l'état du graphe pour le reprendre plus tard, même après un redémarrage.

En mémoire (RAM) - pour le développement

from langgraph.checkpoint.memory import MemorySaver

checkpointer = MemorySaver()
app = graph.compile(checkpointer=checkpointer)

Avec SQLite - pour la production

# pip install langgraph-checkpoint-sqlite
from langgraph.checkpoint.sqlite import SqliteSaver

checkpointer = SqliteSaver.from_conn_string("checkpoints.db")
app = graph.compile(checkpointer=checkpointer)

Conversations multiples avec thread_id

# Chaque conversation a son propre thread_id
config_user1 = {"configurable": {"thread_id": "user_alice"}}
config_user2 = {"configurable": {"thread_id": "user_bob"}}

# Alice et Bob ont chacun leur propre historique
app.invoke({"messages": ["Salut, je suis Alice"]}, config_user1)
app.invoke({"messages": ["Hello, I'm Bob"]}, config_user2)

# Alice continue sa conversation
app.invoke({"messages": ["Quel était mon nom ?"]}, config_user1)
# → "Vous êtes Alice"

💡Le thread_id est comme un numéro de dossier. Chaque utilisateur a son propre dossier avec son propre historique.

14. Sous-graphes

Un sous-graphe est un graphe à l'intérieur d'un autre graphe. C'est utile pour organiser des workflows complexes.

from typing import TypedDict
from langgraph.graph import StateGraph, START, END

# ---- Sous-graphe : traitement de texte ----
class TextState(TypedDict):
    texte: str

def nettoyer(state):
    return {"texte": state["texte"].strip().lower()}

def formater(state):
    return {"texte": state["texte"].capitalize()}

sub_graph = StateGraph(TextState)
sub_graph.add_node("nettoyer", nettoyer)
sub_graph.add_node("formater", formater)
sub_graph.add_edge(START, "nettoyer")
sub_graph.add_edge("nettoyer", "formater")
sub_graph.add_edge("formater", END)

# ---- Graphe principal ----
class MainState(TypedDict):
    texte: str
    resultat: str

def finaliser(state):
    return {"resultat": f"Texte traité : {state['texte']}"}

main_graph = StateGraph(MainState)
main_graph.add_node("traitement", sub_graph.compile())  # ← Sous-graphe comme nœud !
main_graph.add_node("finaliser", finaliser)

main_graph.add_edge(START, "traitement")
main_graph.add_edge("traitement", "finaliser")
main_graph.add_edge("finaliser", END)

app = main_graph.compile()
result = app.invoke({"texte": "  HELLO WORLD  "})
print(result)  # texte: "Hello world", resultat: "Texte traité : Hello world"

15. Streaming

Le streaming permet de voir le résultat en temps réel, nœud par nœud.

Stream des événements nœud par nœud

# stream() au lieu de invoke()
for event in app.stream({"question": "Bonjour"}):
    print(event)
    # {'classifier': {'categorie': 'general'}}

Stream token par token (LLM)

# astream_events() pour voir chaque token du LLM
async for event in app.astream_events(
    {"messages": ["Explique le RAG"]},
    version="v2"
):
    if event["event"] == "on_chat_model_stream":
        token = event["data"]["chunk"].content

16. Patterns avancés

Pattern 1 : Map-Reduce (traitement parallèle)

from langgraph.graph import StateGraph, START, END, Send
from typing import TypedDict, Annotated
from operator import add

class State(TypedDict):
    sujets: list[str]
    resumes: Annotated[list, add]    # Accumule les résumés

class ResumeState(TypedDict):
    sujet: str

def resumer(state: ResumeState):
    return {"resumes": [f"Résumé de {state['sujet']}"]}

def distribuer(state):
    """Envoyer chaque sujet en parallèle"""
    return [Send("resumer", {"sujet": s}) for s in state["sujets"]]

graph = StateGraph(State)
graph.add_node("resumer", resumer)
graph.add_conditional_edges(START, distribuer)
graph.add_edge("resumer", END)

app = graph.compile()
result = app.invoke({"sujets": ["Python", "RAG", "LangChain"]})
print(result["resumes"])
# ['Résumé de Python', 'Résumé de RAG', 'Résumé de LangChain']

Pattern 2 : Retry avec backoff

import time

class State(TypedDict):
    query: str
    result: str
    retries: int
    error: str

def appeler_api(state):
    retries = state.get("retries", 0)
    try:
        # Simuler un appel API
        if retries < 2:
            raise Exception("API indisponible")
        return {"result": "Succès !", "error": ""}
    except Exception as e:
        time.sleep(retries * 2)   # Attente exponentielle
        return {"error": str(e), "retries": retries + 1}

def verifier_erreur(state):
    if state.get("error") and state["retries"] < 3:
        return "retry"
    return "fin"

Pattern 3 : Multi-agents

# Plusieurs agents qui collaborent
class State(TypedDict):
    task: str
    research: str
    code: str
    review: str

def researcher(state):
    """Agent 1 : Chercheur"""
    return {"research": f"Recherche sur : {state['task']}"}

def coder(state):
    """Agent 2 : Développeur"""
    return {"code": f"Code basé sur : {state['research']}"}

def reviewer(state):
    """Agent 3 : Relecteur"""
    return {"review": f"Review de : {state['code']}"}

graph = StateGraph(State)
graph.add_node("researcher", researcher)
graph.add_node("coder", coder)
graph.add_node("reviewer", reviewer)

graph.add_edge(START, "researcher")
graph.add_edge("researcher", "coder")
graph.add_edge("coder", "reviewer")
graph.add_edge("reviewer", END)

TD1 - Premier graphe (15 min)

🎯Objectif : Créer un graphe qui prend un nom et un âge, et génère un message personnalisé.

Énoncé

Crée un graphe avec :

  1. State : nom (str), age (int), message (str), categorie (str)
  2. Nœud "classifier" : si age < 18 → categorie = "jeune", sinon "adulte"
  3. Nœud "generer" : crée un message adapté à la catégorie
  4. Flux : START → classifier → generer → END

Solution

from typing import TypedDict
from langgraph.graph import StateGraph, START, END

class State(TypedDict):
    nom: str
    age: int
    message: str
    categorie: str

def classifier(state):
    if state["age"] < 18:
        return {"categorie": "jeune"}
    return {"categorie": "adulte"}

def generer(state):
    if state["categorie"] == "jeune":
        msg = f"Salut {state['nom']} ! Tu as {state['age']} ans, profite bien !"
    else:
        msg = f"Bonjour {state['nom']}. Vous avez {state['age']} ans."
    return {"message": msg}

graph = StateGraph(State)
graph.add_node("classifier", classifier)
graph.add_node("generer", generer)
graph.add_edge(START, "classifier")
graph.add_edge("classifier", "generer")
graph.add_edge("generer", END)

app = graph.compile()

# Test 1
print(app.invoke({"nom": "Azdine", "age": 15}))
# → 'Salut Azdine ! Tu as 15 ans, profite bien !'

# Test 2
print(app.invoke({"nom": "Azdine", "age": 25}))
# → 'Bonjour Azdine. Vous avez 25 ans.'

TD2 - Chatbot avec mémoire (25 min)

🎯Objectif : Créer un chatbot qui se souvient de la conversation grâce au checkpointing.

Énoncé

  1. Utiliser MessagesState
  2. Un seul nœud "chatbot" qui appelle un LLM
  3. Utiliser MemorySaver pour la persistance
  4. Boucle interactive dans le terminal

Solution

from langgraph.graph import StateGraph, START, END, MessagesState
from langgraph.checkpoint.memory import MemorySaver
from langchain_ollama import ChatOllama
from langchain_core.messages import HumanMessage

# LLM
llm = ChatOllama(model="mistral")

# Nœud chatbot
def chatbot(state: MessagesState):
    reponse = llm.invoke(state["messages"])
    return {"messages": [reponse]}

# Graphe
graph = StateGraph(MessagesState)
graph.add_node("chatbot", chatbot)
graph.add_edge(START, "chatbot")
graph.add_edge("chatbot", END)

# Compiler avec mémoire
memory = MemorySaver()
app = graph.compile(checkpointer=memory)

# Configuration (thread_id = identifiant de conversation)
config = {"configurable": {"thread_id": "conversation_1"}}

# Boucle interactive
print("Chatbot LangGraph (tapez 'quit' pour quitter)")
while True:
    user_input = input("\nVous: ")
    if user_input.lower() == "quit":
        break

    result = app.invoke(
        {"messages": [HumanMessage(content=user_input)]},
        config
    )

    # Afficher la dernière réponse
    print(f"Bot: {result['messages'][-1].content}")

TD3 - Agent RAG avec LangGraph (40 min)

🎯Objectif : Créer un pipeline RAG sous forme de graphe avec routing conditionnel.

Énoncé

Un RAG qui :

  1. Reçoit une question
  2. Décide si la question nécessite une recherche dans les documents ou une réponse directe
  3. Si recherche → retriever → générer avec contexte
  4. Si directe → générer sans contexte
  5. Vérifie la qualité de la réponse, et si mauvaise → reformuler et réessayer

Solution

from typing import TypedDict
from langgraph.graph import StateGraph, START, END
from langchain_ollama import ChatOllama, OllamaEmbeddings
from langchain_community.vectorstores import Chroma
from langchain_core.prompts import ChatPromptTemplate
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain_community.document_loaders import TextLoader

# ---- Setup ----
llm = ChatOllama(model="mistral")
embeddings = OllamaEmbeddings(model="nomic-embed-text")

# Charger des documents
loader = TextLoader("data/mon_document.txt")
docs = loader.load()
splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
chunks = splitter.split_documents(docs)
vectorstore = Chroma.from_documents(chunks, embeddings)
retriever = vectorstore.as_retriever(search_kwargs={"k": 3})

# ---- State ----
class RAGState(TypedDict):
    question: str
    route: str           # "recherche" ou "directe"
    documents: list      # Documents trouvés
    reponse: str
    qualite: str         # "bonne" ou "mauvaise"
    tentatives: int

# ---- Nœuds ----
def router_question(state):
    """Décider si on a besoin de documents"""
    prompt = ChatPromptTemplate.from_template(
        "La question suivante nécessite-t-elle une recherche dans des documents ?\n"
        "Question: {question}\n"
        "Réponds UNIQUEMENT 'recherche' ou 'directe'."
    )
    chain = prompt | llm
    result = chain.invoke({"question": state["question"]})
    route = result.content.strip().lower()
    if "recherche" in route:
        return {"route": "recherche"}
    return {"route": "directe"}

def rechercher(state):
    """Chercher dans la base vectorielle"""
    docs = retriever.invoke(state["question"])
    return {"documents": [d.page_content for d in docs]}

def generer_avec_contexte(state):
    """Générer une réponse avec les documents"""
    contexte = "\n".join(state["documents"])
    prompt = ChatPromptTemplate.from_template(
        "Contexte:\n{contexte}\n\nQuestion: {question}\n\nRéponds en te basant sur le contexte."
    )
    chain = prompt | llm
    result = chain.invoke({"contexte": contexte, "question": state["question"]})
    tentatives = state.get("tentatives", 0) + 1
    return {"reponse": result.content, "tentatives": tentatives}

def generer_directe(state):
    """Générer une réponse directe sans documents"""
    result = llm.invoke(state["question"])
    return {"reponse": result.content, "tentatives": 1}

def verifier_qualite(state):
    """Vérifier si la réponse est pertinente"""
    prompt = ChatPromptTemplate.from_template(
        "Question: {question}\nRéponse: {reponse}\n\n"
        "Cette réponse est-elle pertinente ? Réponds UNIQUEMENT 'bonne' ou 'mauvaise'."
    )
    chain = prompt | llm
    result = chain.invoke({"question": state["question"], "reponse": state["reponse"]})
    return {"qualite": result.content.strip().lower()}

# ---- Fonctions de routage ----
def choisir_route(state):
    return state["route"]

def verifier_ou_finir(state):
    if "bonne" in state.get("qualite", ""):
        return "fin"
    if state.get("tentatives", 0) >= 2:
        return "fin"        # Max 2 tentatives
    return "retry"

# ---- Graphe ----
graph = StateGraph(RAGState)

graph.add_node("router", router_question)
graph.add_node("rechercher", rechercher)
graph.add_node("generer_contexte", generer_avec_contexte)
graph.add_node("generer_directe", generer_directe)
graph.add_node("verifier", verifier_qualite)

graph.add_edge(START, "router")

# Route conditionnelle
graph.add_conditional_edges("router", choisir_route, {
    "recherche": "rechercher",
    "directe": "generer_directe"
})

graph.add_edge("rechercher", "generer_contexte")
graph.add_edge("generer_contexte", "verifier")
graph.add_edge("generer_directe", "verifier")

# Vérification → retry ou fin
graph.add_conditional_edges("verifier", verifier_ou_finir, {
    "retry": "rechercher",    # BOUCLE : recommencer la recherche
    "fin": END
})

app = graph.compile()

# Test
result = app.invoke({"question": "Quels sont les bienfaits de la vitamine C ?"})
print(result["reponse"])

TD4 - Système Multi-Agents (45 min)

🎯Objectif : Créer un système avec 3 agents spécialisés : Chercheur, Rédacteur, Relecteur. Le Relecteur peut renvoyer au Rédacteur si la qualité est insuffisante.

Solution

from typing import TypedDict
from langgraph.graph import StateGraph, START, END
from langchain_ollama import ChatOllama
from langchain_core.prompts import ChatPromptTemplate

llm = ChatOllama(model="mistral")

class State(TypedDict):
    sujet: str
    recherche: str
    article: str
    feedback: str
    approuve: bool
    revisions: int

# ---- Agent 1 : Chercheur ----
def chercheur(state):
    prompt = ChatPromptTemplate.from_template(
        "Tu es un chercheur expert. Fais une recherche approfondie sur :\n{sujet}\n"
        "Donne les points clés, faits importants et sources."
    )
    chain = prompt | llm
    result = chain.invoke({"sujet": state["sujet"]})
    return {"recherche": result.content}

# ---- Agent 2 : Rédacteur ----
def redacteur(state):
    feedback = state.get("feedback", "Aucun feedback")
    prompt = ChatPromptTemplate.from_template(
        "Tu es un rédacteur. Écris un article basé sur cette recherche :\n"
        "{recherche}\n\n"
        "Feedback précédent à prendre en compte : {feedback}\n"
        "Écris un article clair et structuré."
    )
    chain = prompt | llm
    result = chain.invoke({
        "recherche": state["recherche"],
        "feedback": feedback
    })
    revisions = state.get("revisions", 0) + 1
    return {"article": result.content, "revisions": revisions}

# ---- Agent 3 : Relecteur ----
def relecteur(state):
    prompt = ChatPromptTemplate.from_template(
        "Tu es un relecteur exigeant. Évalue cet article :\n{article}\n\n"
        "L'article est-il complet et bien écrit ?\n"
        "Réponds avec :\n"
        "APPROUVE: oui ou non\n"
        "FEEDBACK: tes commentaires"
    )
    chain = prompt | llm
    result = chain.invoke({"article": state["article"]})

    content = result.content.lower()
    approuve = "approuve: oui" in content or "approuvé" in content
    return {"feedback": result.content, "approuve": approuve}

# ---- Routage ----
def decision_relecteur(state):
    if state["approuve"]:
        return "publier"
    if state["revisions"] >= 3:
        return "publier"      # Max 3 révisions
    return "reviser"

# ---- Graphe ----
graph = StateGraph(State)

graph.add_node("chercheur", chercheur)
graph.add_node("redacteur", redacteur)
graph.add_node("relecteur", relecteur)

graph.add_edge(START, "chercheur")
graph.add_edge("chercheur", "redacteur")
graph.add_edge("redacteur", "relecteur")

graph.add_conditional_edges("relecteur", decision_relecteur, {
    "reviser": "redacteur",   # BOUCLE → retour au rédacteur
    "publier": END
})

app = graph.compile()

# Test
result = app.invoke({"sujet": "Les bienfaits du sommeil"})
print(f"Article final :\n{result['article']}")
print(f"\nApprouvé : {result['approuve']}")
print(f"Nombre de révisions : {result['revisions']}")
