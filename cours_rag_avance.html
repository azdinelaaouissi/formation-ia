RAG Avance : Les Types & Workflows

Naive, Advanced, Modular, Agentic, Graph, Self-RAG, Corrective... Tous les types de RAG expliques avec leurs workflows detailles et les dernieres nouveautes.

1

L'Evolution du RAG

De 2020 a 2026

La timeline du RAG

Le RAG a evolue rapidement depuis sa creation par Facebook AI Research en 2020 :

2020 - Naissance du RAG

Facebook AI Research publie le paper original "Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks". Le concept de base est pose : chercher des documents pour augmenter les reponses d'un LLM.

2022-2023 - Explosion avec ChatGPT

L'arrivee de ChatGPT propulse le RAG. LangChain et LlamaIndex emergent. C'est l'ere du Naive RAG : simple mais efficace.

2024 - Maturation

Apparition de l'Advanced RAG (re-ranking, query rewriting) et du Modular RAG. Les entreprises l'adoptent en production. GraphRAG de Microsoft fait sensation.

2025 - L'ere Agentic

Le Agentic RAG emerge : des agents autonomes qui planifient, cherchent, et raisonnent. Self-RAG et Corrective RAG deviennent standards. Multimodal RAG (images, video) se developpe.

2026 - Context Engine

Le RAG evolue vers un "Context Engine" : moteur de contexte intelligent qui combine recherche semantique, graphes de connaissances, et raisonnement multi-etapes. Les systemes deviennent auto-ameliorants grace au reinforcement learning.

3

Naive RAG

Le modele de base - Point de depart
⚡

Naive RAG (RAG Basique)

Debutant   Le point de depart pour tous

Principe : Le Naive RAG suit le processus le plus simple : Indexer → Chercher → Generer. Pas d'optimisation, pas de re-ranking, pas de verification.


Comment ca marche :

  1. Les documents sont decoupes et indexes dans une base vectorielle
  2. La question de l'utilisateur est convertie en vecteur
  3. Les chunks les plus proches sont recuperes
  4. Question + chunks sont envoyes directement au LLM

Limites :

  • Precision limitee : les chunks recuperes ne sont pas toujours les plus pertinents
  • Pas de verification : aucun controle de qualite des resultats
  • Sensible au bruit : des chunks non pertinents polluent la reponse
  • Questions complexes : echoue sur les questions multi-hop
Simple Prototypage Precision moyenne
4

Advanced RAG

Optimisations pre et post-retrieval
🚀

Advanced RAG

Intermediaire   Le RAG optimise pour la production

Principe : L'Advanced RAG ajoute des etapes d'optimisation avant et apres la recherche pour ameliorer la qualite des resultats.


Optimisations PRE-Retrieval (avant la recherche)

  • Query Rewriting : Reformuler la question pour la rendre plus precise
  • Query Expansion : Generer des variantes de la question (HyDE, Multi-Query)
  • Query Routing : Diriger la question vers la bonne base de donnees
  • Step-back Prompting : Poser une question plus generale d'abord

Optimisations POST-Retrieval (apres la recherche)

  • Re-ranking : Reordonner les resultats par pertinence (Cross-Encoder)
  • Compression : Reduire les chunks au contenu pertinent uniquement
  • Filtrage : Supprimer les chunks avec un score trop bas
  • Fusion : Combiner les resultats de plusieurs recherches
Re-ranking Query Rewriting Production-ready
advanced_rag_reranking.py 
# Exemple : Advanced RAG avec Re-ranking
from langchain.retrievers import ContextualCompressionRetriever
from langchain.retrievers.document_compressors import CrossEncoderReranker
from langchain_community.cross_encoders import HuggingFaceCrossEncoder

# 1. Le modele de re-ranking
reranker = HuggingFaceCrossEncoder(model_name="cross-encoder/ms-marco-MiniLM-L-6-v2")
compressor = CrossEncoderReranker(model=reranker, top_n=3)

# 2. Retriever avec compression/re-ranking
advanced_retriever = ContextualCompressionRetriever(
    base_compressor=compressor,
    base_retriever=base_retriever  # Le retriever de base (ChromaDB)
)

# Resultat : les 3 chunks les plus pertinents apres re-ranking
docs = advanced_retriever.get_relevant_documents("Ma question")
5

Modular RAG

Architecture flexible a composants interchangeables
🔧

Modular RAG

Intermediaire   LEGO de l'IA - Composants modulaires

Principe : Le Modular RAG decompose le pipeline en modules independants et interchangeables. Chaque module a un role specifique et peut etre remplace ou ajoute selon les besoins.


Les modules disponibles :

Search

Recherche dans differentes sources (web, DB, API)

Memory

Memoire des conversations precedentes

Fusion

Combine les resultats de plusieurs sources

Routing

Dirige vers la bonne source/strategie

Predict

Predit si une recherche est necessaire

Task Adapter

Adapte le pipeline selon la tache

Flexible Multi-sources Interchangeable
Analogie

Le Modular RAG est comme un kit LEGO. Tu choisis les briques (modules) dont tu as besoin et tu les assembles comme tu veux. Tu peux ajouter un module de memoire, remplacer le moteur de recherche, ou ajouter un module de fusion - le tout sans reconstruire le systeme entier.

6

Agentic RAG

L'IA qui reflechit, planifie et agit
🤖

Agentic RAG

Avance   La revolution 2025 - L'agent autonome

Principe : L'Agentic RAG transforme le RAG en un agent intelligent capable de :

  • Planifier : Decomposer une question complexe en sous-etapes
  • Decider : Choisir quelle source interroger et quelle strategie utiliser
  • Agir : Executer des recherches, appeler des outils, des APIs
  • Evaluer : Verifier si la reponse est suffisante, sinon continuer
  • Iterer : Boucler jusqu'a obtenir une reponse satisfaisante

Difference cle avec les autres RAG :

Les RAG classiques font une seule recherche puis generent. L'Agentic RAG fait plusieurs boucles de recherche, evalue les resultats, et decide dynamiquement quoi faire ensuite. C'est comme un chercheur humain vs un robot qui suit des instructions.


Exemple concret :

Question complexe

"Comparer les effets secondaires du paracetamol et de l'ibuprofene chez les patients diabetiques de plus de 65 ans"

Naive RAG : Cherche tout en une seule fois → resultats mediocres.

Agentic RAG :

  1. Decompose : "effets paracetamol" + "effets ibuprofene" + "patients diabetiques 65+"
  2. Cherche chaque sous-question separement
  3. Evalue : "Ai-je assez d'info sur les diabetiques ?" → Non → Cherche encore
  4. Synthetise une reponse complete et precise
Agent autonome Multi-hop Raisonnement 2025

Graph RAG

Les graphes de connaissances au service du RAG
🔨

Graph RAG (Knowledge Graph RAG)

Avance   Popularise par Microsoft en 2024

Principe : Le Graph RAG combine la recherche vectorielle classique avec un graphe de connaissances (Knowledge Graph). Au lieu de juste chercher des textes similaires, il comprend les relations entre les entites.


C'est quoi un graphe de connaissances ?

Un graphe ou les noeuds sont des entites (personnes, concepts, medicaments...) et les aretes sont les relations entre eux.

knowledge_graph_exemple.txt 
# Exemple de graphe de connaissances medical :

  [Paracetamol] ---traite---> [Douleur]
  [Paracetamol] ---a_effet---> [Hepatotoxicite]
  [Paracetamol] ---interagit---> [Alcool]
  [Ibuprofene] ---traite---> [Inflammation]
  [Ibuprofene] ---appartient---> [AINS]
  [AINS] ---contre_indique---> [Ulcere gastrique]

# Question : "Le paracetamol est-il dangereux avec l'alcool ?"
# Le graphe suit le chemin :
# Paracetamol → interagit → Alcool
# Paracetamol → a_effet → Hepatotoxicite
# = Reponse precise et tracable !

Avantages par rapport au RAG vectoriel pur :

  • Relations : Comprend les liens entre concepts (cause/effet, hierarchie)
  • Precision jusqu'a 99% : En combinant vecteurs + graphe
  • Raisonnement : Peut suivre des chaines de relations
  • Questions globales : Repond aux questions de synthese ("Resume tous les risques")
Knowledge Graph Relations 99% precision Microsoft

Corrective RAG (CRAG)

L'auto-correction des resultats
⚠

Corrective RAG (CRAG)

Intermediaire+   Fiabilite maximale

Principe : Le CRAG ajoute un evaluateur de pertinence qui note la qualite des documents recuperes. Selon le score, il decide quoi faire :


CORRECT

Score eleve → Utiliser les documents tels quels pour generer la reponse

AMBIGU

Score moyen → Raffiner la recherche, reformuler, chercher dans d'autres sources

INCORRECT

Score bas → Rejeter les documents, chercher sur le web, ou dire "je ne sais pas"


Pourquoi c'est important :

Dans un RAG classique, si les documents recuperes sont mauvais, la reponse sera mauvaise. Le CRAG detecte le probleme avant de generer la reponse, evitant ainsi les hallucinations et les reponses incorrectes.

Auto-correction Evaluateur Fiabilite

Self-RAG

L'IA qui decide elle-meme si elle a besoin de chercher
🧠

Self-RAG (Self-Reflective RAG)

Intermediaire+   L'auto-reflexion intelligente

Principe : Le Self-RAG donne au LLM la capacite de reflechir sur lui-meme a travers des "tokens de reflexion" speciaux :


[Retrieve]

"Ai-je besoin de chercher des documents pour repondre ?" → Oui/Non

[ISREL]

"Ce document est-il pertinent pour ma question ?" → Pertinent/Non pertinent

[ISSUP]

"Ma reponse est-elle supportee par les documents ?" → Oui/Partiellement/Non

[ISUSE]

"Ma reponse est-elle utile pour l'utilisateur ?" → Score 1 a 5


Avantage majeur : Le Self-RAG ne fait une recherche que quand c'est necessaire. Pour une question simple ("Quelle est la capitale de la France ?"), il repond directement. Pour une question complexe, il cherche, verifie, et s'auto-corrige.

Self-reflection Tokens speciaux Anti-hallucination

Hybrid RAG

Le meilleur des deux mondes : semantique + mots-cles
🔀

Hybrid RAG (Recherche Hybride)

Intermediaire   Vectoriel + BM25

Principe : Combine deux types de recherche pour de meilleurs resultats :


Recherche Semantique (Dense)

Trouve les documents par sens. "Medecin" → trouve aussi "docteur", "praticien"

Recherche par Mots-Cles (Sparse/BM25)

Trouve les documents avec les mots exacts. Ideal pour les codes, noms propres, references


La Reciprocal Rank Fusion (RRF) combine les resultats des deux recherches en un classement unique optimise.

hybrid_search.py 
# Recherche hybride avec LangChain
from langchain.retrievers import EnsembleRetriever
from langchain_community.retrievers import BM25Retriever

# Retriever semantique (vecteurs)
vector_retriever = vectorstore.as_retriever(search_kwargs={"k": 5})

# Retriever BM25 (mots-cles)
bm25_retriever = BM25Retriever.from_documents(documents, k=5)

# Hybrid : combine les deux (50/50)
hybrid_retriever = EnsembleRetriever(
    retrievers=[vector_retriever, bm25_retriever],
    weights=[0.5, 0.5]  # Poids egal
)
Dense + Sparse BM25 RRF Fusion

Multimodal RAG

Au-dela du texte : images, audio, video
🎨

Multimodal RAG

Avance   Tendance forte 2025-2026

Principe : Le Multimodal RAG etend le RAG au-dela du texte pour inclure images, tableaux, graphiques, audio et meme video dans la base de connaissances.


Approches possibles :

  • Images → Texte : Extraire le texte des images (OCR, description par LLM multimodal)
  • Images → Vecteurs : Utiliser CLIP pour creer des embeddings d'images
  • Tableaux : Parser les tableaux PDF en donnees structurees
  • Audio : Transcrire avec Whisper puis indexer
  • Video : Extraire les frames + transcrire l'audio

Cas d'usage :

  • Documentation technique avec schemas et diagrammes
  • Dossiers medicaux (radiographies, scanners + notes)
  • E-commerce (recherche par image de produit)
  • Formation (cours video + transcriptions + slides)
Images Audio Video CLIP

Workflow : Naive RAG

Le flux complet etape par etape

Le pipeline lineaire (simple)

Le Naive RAG suit un flux lineaire sans boucle :

Q

1. Question de l'utilisateur

"Quels sont les effets secondaires du paracetamol ?"

E

2. Embedding de la question

Transformation en vecteur [0.23, -0.45, 0.89, ...]

R

3. Retrieval (Top-K)

Recherche des 4 chunks les plus similaires dans la base vectorielle

P

4. Prompt = Question + Contexte

"Voici le contexte : [chunks]. Question : Quels sont les effets secondaires ?"

G

5. Generation par le LLM

Le LLM genere la reponse finale basee sur le contexte

13

Workflow : Advanced RAG

Avec les optimisations pre et post-retrieval

Le pipeline optimise

L'Advanced RAG ajoute des etapes avant et apres la recherche :

Q

1. Question originale

"Les effets du paracetamol chez les personnes agees ?"

QR

2. PRE-RETRIEVAL : Query Rewriting

Reformulation : "Effets secondaires paracetamol geriatrie posologie senior"
Multi-query : 3 variantes de la question generees

R

3. Retrieval (Top-20)

On recupere PLUS de chunks (20) pour avoir un large pool de candidats

RR

4. POST-RETRIEVAL : Re-Ranking

Un Cross-Encoder reordonne les 20 chunks par pertinence reelle.
On garde les Top 5 les plus pertinents.

C

5. POST-RETRIEVAL : Compression

Les chunks sont compresses pour ne garder que les passages directement pertinents

G

6. Generation finale

Le LLM recoit un contexte propre, pertinent et bien ordonne

14

Workflow : Agentic RAG

Le flux avec boucles de decision

Le pipeline dynamique avec boucles

L'Agentic RAG est non-lineaire : l'agent decide a chaque etape ce qu'il fait ensuite.

Q

1. Question complexe

"Comparer l'efficacite du paracetamol vs ibuprofene pour la fievre chez l'enfant de 2 ans"

P

2. Planification (Agent)

L'agent decompose :
- Sous-question 1 : "Paracetamol fievre enfant 2 ans"
- Sous-question 2 : "Ibuprofene fievre enfant 2 ans"
- Sous-question 3 : "Etudes comparatives paracetamol vs ibuprofene pediatrie"

R1

3. Recherche itere 1 : Paracetamol

Cherche dans la base vectorielle → Evalue : "Suffisant ?" → Oui

R2

4. Recherche itere 2 : Ibuprofene

Cherche → Evalue : "Suffisant ?" → Non, pas assez de details → Recherche web

T

5. Utilisation d'outils

L'agent utilise un outil "web search" pour completer. Peut aussi appeler une API, une calculatrice, une base SQL...

E

6. Evaluation globale

"Ai-je assez d'info pour une comparaison complete ?" → Oui → Passer a la generation

G

7. Generation synthetique

Synthese de toutes les informations collectees en une reponse structuree et complete

Point cle

La difference fondamentale : le Naive/Advanced RAG fait un seul passage. L'Agentic RAG fait plusieurs boucles avec des decisions a chaque etape. C'est plus lent mais beaucoup plus precis pour les questions complexes.

Tendances 2025

Les nouvelles de l'annee en cours

Agentic RAG explose

2025 est l'annee de l'Agentic RAG. Les frameworks comme LangGraph, CrewAI et AutoGen permettent de creer des agents RAG sophistiques. Les agents peuvent planifier, iterer, et utiliser des outils.

GraphRAG se democratise

Apres le lancement par Microsoft en 2024, GraphRAG est devenu accessible. Des outils comme Neo4j + LangChain permettent de combiner graphes de connaissances et recherche vectorielle avec une precision jusqu'a 99%.

Semantic Chunking

Fini le chunking par taille fixe ! Le semantic chunking decoupe les documents par sens : chaque chunk contient une idee complete. Les "Contextual Headers" ajoutent le contexte du document a chaque chunk.

Evaluation avec RAGAS

RAGAS est devenu le standard pour evaluer les pipelines RAG. 4 metriques : Context Precision, Context Recall, Faithfulness, Answer Relevancy. Indispensable en production.

Long Context vs RAG

Les modeles avec contexte de 1M+ tokens (Gemini, Claude) posent la question : "A-t-on encore besoin du RAG ?" Reponse : OUI, car le RAG reste plus precis, moins cher, et plus scalable pour les gros volumes.

LLM Agnostic

Les systemes RAG modernes sont agnostiques au LLM : ils fonctionnent avec n'importe quel modele (OpenAI, Claude, Llama, Mistral). Cela permet de changer de modele sans reconstruire le pipeline.

Tendances 2026

L'avenir du RAG

Le RAG devient un "Context Engine"

En 2026, le RAG evolue vers un moteur de contexte intelligent qui ne se contente plus de chercher des documents, mais qui :

  • Memoire a long terme : Se souvient des conversations precedentes
  • Multimodal natif : Texte, images, audio, video dans un meme systeme
  • Auto-ameliorant : Utilise le reinforcement learning pour s'ameliorer automatiquement
  • Cross-lingual : Recherche dans plusieurs langues simultanément (MultiRAG, CrossRAG, tRAG)
  • Temps reel : Mise a jour en continu des connaissances

Multimodal Memory

Des systemes capables de comprendre et se souvenir simultanement de texte, images et video. Deja en phase de prototypage chez les grands acteurs.

Self-Improving RAG

Les systemes RAG qui s'ameliorent automatiquement grace au reinforcement learning et aux feedbacks utilisateurs, sans intervention humaine.

Evaluer un systeme RAG

Les metriques RAGAS

Les 4 metriques RAGAS

RAGAS (Retrieval Augmented Generation Assessment) est le framework standard pour evaluer un pipeline RAG :

1. Context Precision

"Les documents recuperes sont-ils vraiment pertinents pour la question ?"

Score : 0 a 1 (1 = tous les docs sont pertinents)

2. Context Recall

"A-t-on recupere TOUS les documents necessaires pour repondre ?"

Score : 0 a 1 (1 = rien n'a ete oublie)

3. Faithfulness

"La reponse est-elle fidele aux documents ? Pas d'hallucination ?"

Score : 0 a 1 (1 = 100% fidele au contexte)

4. Answer Relevancy

"La reponse est-elle utile et pertinente pour la question posee ?"

Score : 0 a 1 (1 = parfaitement utile)

evaluation_ragas.py 
# Evaluer votre RAG avec RAGAS
from ragas import evaluate
from ragas.metrics import (
    context_precision,
    context_recall,
    faithfulness,
    answer_relevancy
)

# Preparer les donnees de test
result = evaluate(
    dataset=test_dataset,  # Questions + reponses attendues
    metrics=[context_precision, context_recall, faithfulness, answer_relevancy]
)

print(result)
# {'context_precision': 0.87, 'context_recall': 0.92,
#  'faithfulness': 0.95, 'answer_relevancy': 0.89}

Quel RAG choisir ?

Guide de decision selon ton besoin

Arbre de decision

?

Tu debutes ou tu fais un prototype ?

→ Naive RAG Simple, rapide, ideal pour apprendre et tester.

?

Tu veux de la qualite en production ?

→ Advanced RAG + Re-ranking. Le standard pour la plupart des projets serieux.

?

Tu as plusieurs sources de donnees ?

→ Modular RAG Architecture flexible, multi-sources, routing.

?

Tes questions sont complexes / multi-hop ?

→ Agentic RAG L'agent decompose, cherche, evalue et itere.

?

Tes donnees ont des relations complexes ?

→ Graph RAG Ideal pour medical, juridique, entites liees.

?

La fiabilite est critique (medical, legal) ?

→ Corrective RAG + Self-RAG Auto-verification et auto-correction.

Conseil final

Commence TOUJOURS par un Naive RAG, puis ameliore progressivement. N'essaie pas de construire un Agentic RAG des le depart. Le chemin est : Naive → Advanced (re-ranking) → Hybrid → Modular → Agentic. Chaque etape resout un probleme specifique.
