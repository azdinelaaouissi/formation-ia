C'est quoi LangChain ?

Le framework #1 pour les applications LLM

Definition

LangChain est un framework Python (et JavaScript) open source qui permet de construire des applications utilisant des modeles de langage (LLM). Il fournit des briques reutilisables pour :

  • Connecter un LLM a vos donnees (RAG)
  • Creer des chatbots intelligents
  • Construire des agents autonomes
  • Chainer des operations complexes

Analogie : Les briques LEGO de l'IA

Imagine que chaque composant (LLM, base de donnees, prompt, etc.) est une brique LEGO. LangChain est le systeme d'assemblage qui permet de connecter ces briques entre elles pour construire n'importe quelle application IA.

Architecture de LangChain

Les composants et comment ils s'assemblent

Les 8 composants principaux

LangChain est organise en composants modulaires que tu peux assembler comme des briques :

1. LLMs / Chat Models

Les modeles d'IA (GPT-4, Claude, Llama, Mistral). C'est le "cerveau" qui genere du texte.

2. Prompt Templates

Des modeles de texte avec des variables. Permettent de structurer les instructions envoyees au LLM.

3. Chains (LCEL)

Enchainer plusieurs etapes : prompt → LLM → output parser. Le coeur de LangChain.

4. Document Loaders

Charger des donnees depuis n'importe quelle source : PDF, CSV, web, base de donnees, etc.

5. Text Splitters

Decouper les documents en chunks pour le RAG. RecursiveCharacterTextSplitter est le plus utilise.

6. Embeddings + Vector Stores

Transformer le texte en vecteurs et les stocker dans une base vectorielle (ChromaDB, FAISS).

7. Retrievers

Chercher les documents pertinents par similarite semantique dans la base vectorielle.

8. Tools & Agents

Des agents autonomes qui utilisent des outils (calculatrice, recherche web, API) pour resoudre des taches.

llm_examples.py
# === Option 1 : Ollama (gratuit, local) ===
from langchain_ollama import OllamaLLM

llm = OllamaLLM(model="llama3.2", temperature=0)
response = llm.invoke("Explique le RAG en 2 phrases")
print(response)  # String directement


# === Option 2 : OpenAI (payant, cloud) ===
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(
    model="gpt-4o-mini",
    api_key="sk-...",
    temperature=0
)
response = llm.invoke("Explique le RAG en 2 phrases")
print(response.content)  # .content car c'est un AIMessage


# === Option 3 : HuggingFace (gratuit, local) ===
from langchain_huggingface import HuggingFacePipeline

llm = HuggingFacePipeline.from_model_id(
    model_id="mistralai/Mistral-7B-Instruct-v0.3",
    task="text-generation",
    pipeline_kwargs={"max_new_tokens": 512}
)
invoke() est la methode universelle

Tous les composants LangChain utilisent .invoke() pour executer. C'est le standard depuis LangChain v0.2. Les anciennes methodes comme .predict() ou .run() sont deprecees.

Prompt Templates

Structurer les instructions pour le LLM

Pourquoi des templates ?

Au lieu d'ecrire le prompt a chaque fois, on cree un modele reutilisable avec des variables. C'est plus propre, plus maintenable, et plus facile a tester.

prompts.py
# === PromptTemplate (pour les LLMs simples) ===
from langchain.prompts import PromptTemplate

# Template avec 2 variables : {sujet} et {niveau}
template = PromptTemplate(
    input_variables=["sujet", "niveau"],
    template="""Explique {sujet} pour un {niveau}.
Donne 3 points cles avec des exemples."""
)

# Remplir les variables
prompt = template.format(sujet="le RAG", niveau="debutant")
print(prompt)
# "Explique le RAG pour un debutant.\nDonne 3 points cles..."


# === ChatPromptTemplate (pour les Chat Models) ===
from langchain.prompts import ChatPromptTemplate

template = ChatPromptTemplate.from_messages([
    ("system", "Tu es un expert en {domaine}. Reponds en francais."),
    ("human", "{question}")
])

messages = template.format_messages(
    domaine="nutrition",
    question="Quels sont les bienfaits de la vitamine D ?"
)


# === FewShotPromptTemplate (exemples dans le prompt) ===
from langchain.prompts import FewShotPromptTemplate

examples = [
    {"input": "chat", "output": "animal domestique felin"},
    {"input": "Python", "output": "langage de programmation"}
]

few_shot = FewShotPromptTemplate(
    examples=examples,
    example_prompt=PromptTemplate(
        input_variables=["input", "output"],
        template="Mot: {input}\nDefinition: {output}"
    ),
    prefix="Definis le mot suivant :",
    suffix="Mot: {input}\nDefinition:",
    input_variables=["input"]
)

Chains & LCEL

Enchainer les composants avec le pipe operator |

LCEL : LangChain Expression Language

LCEL est la facon moderne de creer des chains dans LangChain. On utilise l'operateur | (pipe) pour connecter les composants, comme dans un terminal Unix.

chains_lcel.py
from langchain.prompts import ChatPromptTemplate
from langchain_ollama import OllamaLLM
from langchain_core.output_parsers import StrOutputParser

# 1. Les composants
prompt = ChatPromptTemplate.from_messages([
    ("system", "Tu es un assistant en {domaine}."),
    ("human", "{question}")
])
llm = OllamaLLM(model="llama3.2")
parser = StrOutputParser()

# 2. La chain avec LCEL (operateur |)
chain = prompt | llm | parser
#       ^          ^       ^
#    Template   Modele   Parser
# Le | connecte : la sortie de prompt va dans llm,
# la sortie de llm va dans parser

# 3. Executer la chain
result = chain.invoke({
    "domaine": "nutrition",
    "question": "Quels aliments sont riches en fer ?"
})
print(result)  # String de la reponse


# === Chain avec streaming (affichage mot par mot) ===
for chunk in chain.stream({"domaine": "nutrition", "question": "Le fer ?"}):
    print(chunk, end="", flush=True)


# === Chain avec batch (plusieurs questions a la fois) ===
results = chain.batch([
    {"domaine": "nutrition", "question": "Le fer ?"},
    {"domaine": "nutrition", "question": "La vitamine C ?"},
])
3 methodes d'execution

invoke() : Une seule entree → une seule sortie.
stream() : Affiche la reponse mot par mot (streaming).
batch() : Plusieurs entrees → plusieurs sorties en parallele.

from langchain.text_splitter import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,                       # Taille max
    chunk_overlap=50,                     # Chevauchement
    separators=["\n\n", "\n", ". ", " "]  # Priorite de coupe

# === Pour du code source ===
from langchain.text_splitter import Language

splitter = RecursiveCharacterTextSplitter.from_language(
    language=Language.PYTHON,  # Coupe aux fonctions/classes
    chunk_size=1000,
    chunk_overlap=100
)

# === Par tokens (plus precis) ===
from langchain.text_splitter import TokenTextSplitter

splitter = TokenTextSplitter(
    chunk_size=256,     # 256 tokens (pas caracteres !)
    chunk_overlap=30

Retrievers

Chercher les documents pertinents
retrievers.py
# === Retriever basique ===
retriever = vectorstore.as_retriever(
    search_type="similarity",    # ou "mmr" (Max Marginal Relevance)
    search_kwargs={"k": 4}
)
docs = retriever.invoke("ma question")

# === MMR : diversifier les resultats ===
retriever = vectorstore.as_retriever(
    search_type="mmr",             # Max Marginal Relevance
    search_kwargs={
        "k": 4,
        "fetch_k": 20,            # Recuperer 20, garder les 4 plus divers
        "lambda_mult": 0.5       # 0 = max diversite, 1 = max similarite
    }
)

# === Filtrer par metadata ===
retriever = vectorstore.as_retriever(
    search_kwargs={
        "k": 3,
        "filter": {"source": "nutrition.txt"}  # Seulement ce fichier
    }
)

# === Multi-Query Retriever (genere des variantes) ===
from langchain.retrievers.multi_query import MultiQueryRetriever

multi_retriever = MultiQueryRetriever.from_llm(
    retriever=retriever,
    llm=llm   # Le LLM genere 3 variantes de la question
)

Memory

Se souvenir des conversations precedentes

Le probleme : les LLM n'ont pas de memoire

Par defaut, chaque appel au LLM est independant. Il ne se souvient pas de ce que tu as dit avant. La Memory de LangChain resout ce probleme.

memory.py
# === Memoire de conversation basique ===
from langchain.memory import ConversationBufferMemory

memory = ConversationBufferMemory()
memory.save_context(
    {"input": "Salut, je m'appelle Azdine"},
    {"output": "Bonjour Azdine ! Comment puis-je t'aider ?"}
)

# La prochaine question inclura l'historique
print(memory.load_memory_variables({}))
# {'history': 'Human: Salut...\nAI: Bonjour...'}


# === Memoire avec fenetre (garder les N derniers messages) ===
from langchain.memory import ConversationBufferWindowMemory

memory = ConversationBufferWindowMemory(k=5)
# Garde seulement les 5 derniers echanges


# === Memoire avec resume (pour les longues conversations) ===
from langchain.memory import ConversationSummaryMemory

memory = ConversationSummaryMemory(llm=llm)
# Le LLM resume automatiquement la conversation

Tools & Agents

Des agents qui utilisent des outils

C'est quoi un Agent ?

Un Agent est un LLM qui peut decider quels outils utiliser et dans quel ordre pour resoudre une tache. Au lieu de suivre un pipeline fixe, il raisonne et agit de maniere autonome.

agents.py
from langchain.tools import tool
from langchain.agents import create_tool_calling_agent, AgentExecutor
from langchain.prompts import ChatPromptTemplate

# 1. Definir des outils avec le decorateur @tool
@tool
def calculer(expression: str) -> str:
    """Calculer une expression mathematique. Ex: '2 + 3 * 4'"""
    return str(eval(expression))

@tool
def rechercher_docs(query: str) -> str:
    """Rechercher dans la base de documents."""
    docs = retriever.invoke(query)
    return "\n".join([d.page_content for d in docs])

# 2. Creer le prompt de l'agent
prompt = ChatPromptTemplate.from_messages([
    ("system", "Tu es un assistant. Utilise les outils disponibles."),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}")
])

# 3. Creer l'agent
agent = create_tool_calling_agent(
    llm=llm,
    tools=[calculer, rechercher_docs],
    prompt=prompt
)

# 4. L'executor gere la boucle outil → reflexion → outil
executor = AgentExecutor(agent=agent, tools=[calculer, rechercher_docs])

# 5. L'agent decide seul quoi faire !
result = executor.invoke({"input": "Combien font 25% de 180 ?"})
print(result["output"])  # "45" (l'agent a utilise 'calculer')
La docstring est cruciale

Le texte entre les """...""" de chaque outil est lu par l'agent pour comprendre quand et comment l'utiliser. Une mauvaise docstring = l'agent ne saura pas quand utiliser l'outil.

Patterns avances du pipe operator

from langchain_core.runnables import RunnablePassthrough, RunnableParallel

# === RAG Chain avec LCEL ===
rag_chain = (

    summary=prompt_resume | llm | parser,

)
# Les 2 branches s'executent en parallele !

# result["summary"], result["translation"]

from langchain_core.runnables import RunnableBranch

branch = RunnableBranch(

    (lambda x: "math" in x["question"], chain_math),     # Si "math" dans la question

TD1

TD 1 : Premier chatbot

Exercice pratique - 20 minutes
TD 1 - Niveau Debutant

Objectif : Creer un chatbot simple avec LangChain

Ce que tu vas faire :

  1. Creer un fichier chatbot.py
  2. Configurer un LLM (Ollama ou OpenAI)
  3. Creer un ChatPromptTemplate avec un role system
  4. Construire une chain LCEL : prompt | llm | parser
  5. Creer une boucle interactive (while True / input())

Bonus : Ajouter la ConversationBufferMemory pour que le chatbot se souvienne des messages precedents.

td1_chatbot.py - Solution
from langchain.prompts import ChatPromptTemplate, MessagesPlaceholder
from langchain_ollama import OllamaLLM
from langchain_core.output_parsers import StrOutputParser

# 1. Le LLM
llm = OllamaLLM(model="llama3.2", temperature=0.7)

# 2. Le prompt
prompt = ChatPromptTemplate.from_messages([
    ("system", "Tu es un assistant amical. Reponds en francais, de facon concise."),
    ("human", "{message}")
])

# 3. La chain LCEL
chain = prompt | llm | StrOutputParser()

# 4. Boucle interactive
print("Chatbot LangChain (tape 'quit' pour quitter)\n")

while True:
    user_input = input("Toi : ").strip()
    if user_input.lower() in ["quit", "q"]:
        break
    if not user_input:
        continue

    response = chain.invoke({"message": user_input})
    print(f"Bot : {response}\n")

from langchain_community.document_loaders import PyPDFLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain_huggingface import HuggingFaceEmbeddings
from langchain_community.vectorstores import Chroma

from langchain.prompts import ChatPromptTemplate

from langchain_core.runnables import RunnablePassthrough

# 1. Charger le PDF
docs = PyPDFLoader("mon_document.pdf").load()
print(f"{len(docs)} pages chargees")

# 2. Decouper
splitter = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50)
chunks = splitter.split_documents(docs)
print(f"{len(chunks)} chunks")

# 3. Embeddings + ChromaDB
embeddings = HuggingFaceEmbeddings(model_name="all-MiniLM-L6-v2")
vectorstore = Chroma.from_documents(chunks, embeddings, persist_directory="./td2_db")

# 4. Retriever
retriever = vectorstore.as_retriever(search_kwargs={"k": 3})

# 5. RAG Chain avec LCEL
prompt = ChatPromptTemplate.from_messages([
    ("system", """Reponds en te basant UNIQUEMENT sur ce contexte :
{context}
Si tu ne trouves pas la reponse, dis-le."""),
    ("human", "{question}")
])

def format_docs(docs):
    return "\n\n".join(d.page_content for d in docs)

rag_chain = (
    {"context": retriever | format_docs, "question": RunnablePassthrough()}

# 6. Tester
answer = rag_chain.invoke("De quoi parle ce document ?")
print(answer)

TD 3 : RAG complet avec sources

Exercice pratique - 45 minutes
TD 3 - Niveau Intermediaire+

Objectif : Un RAG qui cite ses sources

  1. Charger 3+ documents de sujets differents
  2. Implementer le pipeline complet (loader → splitter → embeddings → vectorstore)
  3. Ajouter un retriever MMR (diversite des resultats)
  4. Le prompt doit instruire le LLM de citer les sources
  5. Afficher la reponse + les fichiers sources utilises
  6. Bonus : Ajouter un seuil de score minimum (ignorer les chunks score < 0.5)
td3_rag_sources.py - Solution
# Pipeline complet avec sources
from langchain_community.document_loaders import DirectoryLoader, TextLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain_huggingface import HuggingFaceEmbeddings
from langchain_community.vectorstores import Chroma
from langchain_ollama import OllamaLLM
from langchain.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

# 1. Charger TOUS les .txt du dossier data/
loader = DirectoryLoader("./data/", glob="**/*.txt", loader_cls=TextLoader,
                          loader_kwargs={"encoding": "utf-8"})
docs = loader.load()

# 2. Chunking + Embeddings + ChromaDB
chunks = RecursiveCharacterTextSplitter(chunk_size=500, chunk_overlap=50).split_documents(docs)
embeddings = HuggingFaceEmbeddings(model_name="all-MiniLM-L6-v2")
vectorstore = Chroma.from_documents(chunks, embeddings, persist_directory="./td3_db")

# 3. Retriever MMR (diversifie les resultats)
retriever = vectorstore.as_retriever(
    search_type="mmr",
    search_kwargs={"k": 4, "fetch_k": 10}
)

# 4. Fonction qui retourne reponse + sources
def ask_with_sources(question):
    # Recuperer les chunks
    docs = retriever.invoke(question)
    context = "\n\n".join(d.page_content for d in docs)
    sources = set(d.metadata["source"] for d in docs)

    # Generer la reponse
    prompt = f"""Contexte:\n{context}\n\nQuestion: {question}\n
Reponds en citant les sources entre parentheses."""

    llm = OllamaLLM(model="llama3.2", temperature=0)
    answer = llm.invoke(prompt)

    return answer, sources

# 5. Tester
answer, sources = ask_with_sources("Combien d'eau boire par jour ?")
print(f"Reponse: {answer}")
print(f"Sources: {sources}")

from langchain.tools import tool
from langchain.agents import create_tool_calling_agent, AgentExecutor
from langchain.prompts import ChatPromptTemplate
from langchain_ollama import ChatOllama
from datetime import datetime

@tool
def rechercher_docs(query: str) -> str:
    """Rechercher des informations dans la base documentaire."""
    docs = retriever.invoke(query)
    return "\n".join(d.page_content for d in docs)

@tool
def calculer(expression: str) -> str:
    """Calculer une expression mathematique. Ex: '15 * 3 + 7'"""
    try:
        return str(eval(expression))
    except:
        return "Erreur de calcul"

@tool
def date_actuelle() -> str:
    """Retourner la date et l'heure actuelles."""
    return datetime.now().strftime("%d/%m/%Y %H:%M")

tools = [rechercher_docs, calculer, date_actuelle]

prompt = ChatPromptTemplate.from_messages([
    ("system", "Tu es un assistant intelligent. Utilise les outils pour repondre."),
    ("human", "{input}"),
    ("placeholder", "{agent_scratchpad}")
])

llm = ChatOllama(model="llama3.2")
agent = create_tool_calling_agent(llm, tools, prompt)
executor = AgentExecutor(agent=agent, tools=tools, verbose=True)

# Tests
print(executor.invoke({"input": "Quelle heure est-il ?"}))
print(executor.invoke({"input": "Combien font 25% de 340 ?"}))
print(executor.invoke({"input": "Quels aliments sont riches en fer ?"}))
