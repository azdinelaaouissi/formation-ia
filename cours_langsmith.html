Monitoring, tracing et évaluation de tes applications LLM

1. C'est quoi LangSmith ?

🎯Définition simple : LangSmith est une plateforme web créée par LangChain qui te permet de voir, tester et surveiller tout ce que fait ton application IA. C'est comme une caméra de surveillance pour ton code LLM.

Analogie

🏥Imagine un hôpital :

  • Ton application IA = le patient
  • Tracing = le scanner qui montre ce qui se passe à l'intérieur
  • Évaluation = les tests sanguins qui vérifient si tout va bien
  • Monitoring = le moniteur cardiaque qui surveille en continu
  • LangSmith = tout l'hôpital réuni en une seule plateforme

Les 4 piliers de LangSmith

PilierCe que ça faitQuand l'utiliser
TracingEnregistre chaque étape de ton applicationDéveloppement, debug
ÉvaluationTeste si tes réponses sont bonnesAvant le déploiement
MonitoringSurveille les performances en temps réelEn production
Prompt HubStocke et versione tes promptsCollaboration en équipe

2. Pourquoi LangSmith ?

Les problèmes sans LangSmith

😫Sans LangSmith, tu es aveugle :

  • "Pourquoi mon RAG donne une mauvaise réponse ?" → Tu ne sais pas quel chunk a été récupéré
  • "Pourquoi c'est si lent ?" → Tu ne sais pas quelle étape prend du temps
  • "Est-ce que ma version 2 est meilleure ?" → Tu n'as pas de métriques pour comparer
  • "Combien ça me coûte en tokens ?" → Tu ne comptes pas

Les solutions avec LangSmith

😎Avec LangSmith, tu vois tout :

  • Chaque appel LLM avec l'input et l'output exact
  • Le temps de chaque étape (en millisecondes)
  • Le nombre de tokens utilisés et le coût
  • Les documents récupérés par le retriever
  • Les erreurs avec le stacktrace complet
  • La comparaison entre différentes versions

6. Le décorateur @traceable

Pour tracer tes propres fonctions Python (pas juste les appels LangChain), utilise @traceable.

from langsmith import traceable

@traceable                         # ← Ajoute juste ce décorateur !
def nettoyer_texte(texte: str) -> str:
    """Nettoyer un texte brut"""
    texte = texte.strip()
    texte = texte.lower()
    texte = " ".join(texte.split())  # Enlever les espaces doubles
    return texte

@traceable(name="Mon RAG Pipeline")  # ← Nom personnalisé
def rag_pipeline(question: str) -> str:
    """Pipeline RAG complet"""
    # Étape 1 : Nettoyer
    question_propre = nettoyer_texte(question)

    # Étape 2 : Chercher (simulé)
    contexte = "Le RAG combine recherche et génération..."

    # Étape 3 : Générer
    reponse = f"Basé sur : {contexte}. Réponse à : {question_propre}"

    return reponse

# Appeler - tout est tracé dans LangSmith
result = rag_pipeline("  QU'EST-CE QUE LE RAG ?  ")
print(result)

Ce que tu vois dans LangSmith

# Trace hiérarchique :
📁 Mon RAG Pipeline (1.5s)
  ├── 📄 nettoyer_texte (0.01s)
  │     Input: "  QU'EST-CE QUE LE RAG ?  "
  │     Output: "qu'est-ce que le rag ?"
  └── 📄 (le reste de la pipeline)

💡Le tracing est hiérarchique : Si une fonction @traceable appelle une autre fonction @traceable, LangSmith montre l'arborescence parent → enfant. C'est super pour le debug !

Options de @traceable

OptionExempleEffet
name@traceable(name="Mon Step")Nom affiché dans LangSmith
run_type@traceable(run_type="retriever")Type : llm, retriever, tool, chain
metadata@traceable(metadata={"version": "2"})Données supplémentaires
tags@traceable(tags=["prod", "v2"])Tags pour filtrer

10. Évaluation (Evaluators)

L'évaluation permet de tester automatiquement si ton application donne de bonnes réponses.

Évaluation simple avec une fonction

from langsmith import Client, evaluate

client = Client()

# Ta fonction à tester (ton RAG par exemple)
def mon_rag(inputs: dict) -> dict:
    question = inputs["question"]
    # ... ton pipeline RAG ici ...
    reponse = "Le RAG combine recherche et génération"  # Simplifié
    return {"reponse": reponse}

# Évaluateur personnalisé
def contient_mots_cles(run, example) -> dict:
    """Vérifie si la réponse contient des mots-clés importants"""
    prediction = run.outputs["reponse"]                # Ce que ton RAG a répondu
    reference = example.outputs["reponse"]              # La réponse attendue

    # Vérifier si au moins 50% des mots importants sont présents
    mots_ref = set(reference.lower().split())
    mots_pred = set(prediction.lower().split())
    overlap = len(mots_ref & mots_pred) / len(mots_ref)

    return {
        "key": "mots_cles_score",     # Nom de la métrique
        "score": overlap,               # Score entre 0 et 1
        "comment": f"Overlap: {overlap:.0%}"
    }

# Lancer l'évaluation
results = evaluate(
    mon_rag,                                # Fonction à tester
    data="questions-rag-test",              # Nom du dataset
    evaluators=[contient_mots_cles],        # Liste d'évaluateurs
    experiment_prefix="test-rag-v1"         # Nom de l'expérience
)

print("Évaluation terminée ! Voir les résultats sur LangSmith.")

💡Comment ça marche :

  1. LangSmith prend chaque exemple du dataset
  2. Il appelle ta fonction mon_rag avec l'input
  3. Il compare l'output avec la réponse attendue via l'évaluateur
  4. Il enregistre les scores et les affiche dans l'interface

11. LLM-as-Judge (Le LLM évalue)

Au lieu d'écrire des règles manuelles, on demande à un LLM de juger la qualité des réponses.

from langsmith import evaluate
from langchain_ollama import ChatOllama
from langchain_core.prompts import ChatPromptTemplate

llm_juge = ChatOllama(model="mistral")

# Évaluateur LLM-as-Judge
def llm_evaluator(run, example) -> dict:
    """Un LLM évalue la qualité de la réponse"""
    question = example.inputs["question"]
    prediction = run.outputs["reponse"]
    reference = example.outputs["reponse"]

    prompt = ChatPromptTemplate.from_template(
        """Évalue la qualité de cette réponse.

Question: {question}
Réponse attendue: {reference}
Réponse à évaluer: {prediction}

Donne une note de 1 à 5 :
1 = Complètement faux
2 = Partiellement correct mais très incomplet
3 = Correct mais incomplet
4 = Bon, presque complet
5 = Excellent, complet et précis

Réponds UNIQUEMENT avec le chiffre."""
    )

    chain = prompt | llm_juge
    result = chain.invoke({
        "question": question,
        "reference": reference,
        "prediction": prediction
    })

    try:
        score = int(result.content.strip()) / 5  # Normaliser entre 0 et 1
    except:
        score = 0.5

    return {
        "key": "llm_qualite",
        "score": score,
        "comment": result.content.strip()
    }

# Lancer
results = evaluate(
    mon_rag,
    data="questions-rag-test",
    evaluators=[contient_mots_cles, llm_evaluator],  # Plusieurs évaluateurs !
    experiment_prefix="test-rag-v1-llm-judge"
)

Évaluateurs pré-construits par LangChain

12. Annotation humaine

Parfois, un humain doit vérifier les réponses. LangSmith permet de créer des queues d'annotation.

Créer une queue d'annotation

📌Dans l'interface LangSmith :

  1. Aller dans "Annotation Queues" → "New Queue"
  2. Donner un nom (ex: "review-rag-reponses")
  3. Sélectionner des traces à reviewer
  4. L'annotateur humain note chaque réponse (pouce haut/bas, score 1-5, commentaire)

Ajouter du feedback en Python

from langsmith import Client

client = Client()

# Ajouter un feedback sur une trace spécifique
client.create_feedback(
    run_id="id-de-la-trace",    # L'ID de la trace à évaluer
    key="qualite",               # Nom du critère
    score=0.8,                    # Score (0 à 1)
    comment="Bonne réponse mais manque un exemple"
)

# Feedback binaire (pouce haut/bas)
client.create_feedback(
    run_id="id-de-la-trace",
    key="user_satisfaction",
    score=1,                      # 1 = pouce haut, 0 = pouce bas
)

14. Prompt Hub

Le Prompt Hub permet de stocker, versionner et partager tes prompts.

Sauvegarder un prompt

from langchain import hub
from langchain_core.prompts import ChatPromptTemplate

# Créer un prompt
prompt = ChatPromptTemplate.from_template(
    "Tu es un assistant expert en {domaine}.\n"
    "Réponds à cette question de manière claire :\n"
    "{question}"
)

# Sauvegarder sur le Hub
hub.push("mon-username/rag-prompt-v1", prompt)
print("Prompt sauvegardé !")

Charger un prompt depuis le Hub

from langchain import hub

# Charger le prompt
prompt = hub.pull("mon-username/rag-prompt-v1")

# Charger une version spécifique
prompt_v2 = hub.pull("mon-username/rag-prompt-v1:v2")

# Utiliser
chain = prompt | llm
result = chain.invoke({"domaine": "nutrition", "question": "..."})

💡Avantages du Prompt Hub :

  • Versioning automatique (v1, v2, v3...)
  • Partage avec ton équipe
  • Modifier un prompt SANS changer le code
  • Tester différentes versions facilement

15. Comparaison de modèles

LangSmith permet de comparer les performances de différents modèles ou configurations.

from langsmith import evaluate
from langchain_ollama import ChatOllama

# ---- Configuration 1 : Mistral ----
def rag_mistral(inputs):
    llm = ChatOllama(model="mistral")
    result = llm.invoke(inputs["question"])
    return {"reponse": result.content}

# ---- Configuration 2 : Llama3 ----
def rag_llama(inputs):
    llm = ChatOllama(model="llama3")
    result = llm.invoke(inputs["question"])
    return {"reponse": result.content}

# Évaluer les deux avec le même dataset
results_mistral = evaluate(
    rag_mistral,
    data="questions-rag-test",
    evaluators=[contient_mots_cles, llm_evaluator],
    experiment_prefix="compare-mistral"
)

results_llama = evaluate(
    rag_llama,
    data="questions-rag-test",
    evaluators=[contient_mots_cles, llm_evaluator],
    experiment_prefix="compare-llama3"
)

# Aller sur LangSmith → Dataset → Comparer les deux expériences
# Tu verras côte à côte : scores, latence, coût

📌Dans l'interface : Va dans ton dataset → onglet "Experiments" → sélectionne les deux expériences → tu vois la comparaison détaillée question par question.

16. Production et bonnes pratiques

Checklist avant la production

✅Avant de mettre en production :

  1. Créer un dataset de test avec au moins 20 exemples
  2. Évaluer avec au minimum 2 évaluateurs (un automatique, un LLM-as-judge)
  3. Score moyen > 0.7 sur tous les évaluateurs
  4. Latence P99 < 10 secondes
  5. Taux d'erreur < 1% sur le dataset de test
  6. Configurer le monitoring dans un projet dédié "prod"
  7. Mettre en place des alertes sur les métriques critiques

Échantillonnage en production

import os

# En production, tu ne veux pas tracer 100% des requêtes
# (ça coûte des traces sur ton quota)

# Solution : échantillonner (ex: 10% des requêtes)
import random

if random.random() < 0.1:  # 10% de chance
    os.environ["LANGCHAIN_TRACING_V2"] = "true"
else:
    os.environ["LANGCHAIN_TRACING_V2"] = "false"

Séparer dev et prod

import os

environment = os.getenv("ENVIRONMENT", "dev")

if environment == "production":
    os.environ["LANGCHAIN_PROJECT"] = "mon-rag-prod"
else:
    os.environ["LANGCHAIN_PROJECT"] = "mon-rag-dev"

from langsmith import Client, evaluate
from langchain_ollama import ChatOllama

client = Client()
llm = ChatOllama(model="mistral")

# ============================================
# ÉTAPE 1 : Créer le dataset
# ============================================
dataset_name = "td2-evaluation-rag"

# Vérifier si le dataset existe déjà
try:
    dataset = client.read_dataset(dataset_name=dataset_name)
    print("Dataset existant trouvé.")
except:
    dataset = client.create_dataset(dataset_name=dataset_name)
    client.create_examples(
        inputs=[
            {"question": "Qu'est-ce que le RAG ?"},
            {"question": "Comment fonctionne ChromaDB ?"},
            {"question": "Qu'est-ce qu'un embedding ?"},
            {"question": "Explique le chunking"},
            {"question": "Qu'est-ce que FAISS ?"},
        ],
        outputs=[
            {"reponse": "RAG signifie Retrieval Augmented Generation. Il combine la recherche de documents pertinents avec la génération de texte par un LLM."},
            {"reponse": "ChromaDB est une base de données vectorielle open-source qui stocke des embeddings et permet de faire des recherches par similarité."},
            {"reponse": "Un embedding est une représentation numérique (vecteur) d'un texte qui capture son sens sémantique."},
            {"reponse": "Le chunking est le processus de découpage d'un document en morceaux plus petits pour faciliter l'indexation et la recherche."},
            {"reponse": "FAISS est une bibliothèque de Facebook pour la recherche efficace de similarité entre vecteurs."},
        ],
        dataset_id=dataset.id
    )
    print("Dataset créé avec 5 exemples.")

# ============================================
# ÉTAPE 2 : Fonction RAG à tester
# ============================================
def mon_rag(inputs: dict) -> dict:
    prompt = ChatPromptTemplate.from_template(
        "Réponds à cette question de manière précise et concise :\n{question}"
    )
    chain = prompt | llm
    result = chain.invoke({"question": inputs["question"]})
    return {"reponse": result.content}

# ÉTAPE 3 : Évaluateurs
# ============================================
def eval_longueur(run, example) -> dict:
    """La réponse fait-elle au moins 20 mots ?"""
    reponse = run.outputs["reponse"]
    nb_mots = len(reponse.split())
    return {
        "key": "longueur_suffisante",
        "score": 1.0 if nb_mots >= 20 else nb_mots / 20,
        "comment": f"{nb_mots} mots"
    }

def eval_pertinence(run, example) -> dict:
    """Un LLM évalue la pertinence"""
    prompt = ChatPromptTemplate.from_template(
        "Question: {q}\nRéponse attendue: {ref}\nRéponse donnée: {pred}\n\n"
        "Note de 1 à 5 la pertinence. Réponds JUSTE le chiffre."
    )
    chain = prompt | llm
    r = chain.invoke({
        "q": example.inputs["question"],
        "ref": example.outputs["reponse"],
        "pred": run.outputs["reponse"]
    })
    try:
        score = int(r.content.strip()[0]) / 5
    except:
        score = 0.5
    return {"key": "pertinence_llm", "score": score}

# ============================================
results = evaluate(
    mon_rag,
    data=dataset_name,
    evaluators=[eval_longueur, eval_pertinence],
    experiment_prefix="td2-rag-eval"
)
