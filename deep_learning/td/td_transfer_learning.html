TD Pratique - Transfer Learning avec ResNet

Fine-tuner un modele pre-entraine pour un dataset custom

Partie 1 : Recapitulatif theorique

Le Transfer Learning consiste a reutiliser un modele deja entraine sur un large dataset (ex: ImageNet, 1.2M images, 1000 classes) et a l'adapter a une nouvelle tache. Au lieu de partir de zero, on transfere les connaissances acquises par le modele source.

1.1 Pourquoi le Transfer Learning ?

Avantages du Transfer Learning

  • Moins de donnees necessaires : quelques centaines d'images suffisent au lieu de millions.
  • Entrainement plus rapide : le backbone est deja entraine, on ne modifie que les dernieres couches.
  • Meilleures performances : les features de bas niveau (bords, textures) sont universelles et transferables.
  • Moins de risque de sur-apprentissage : les poids pre-entraines servent de regularisation implicite.

1.2 Deux approches

Feature Extraction

On gele tout le backbone (les poids ne changent pas) et on entraine uniquement le nouveau classificateur.

Rapide, risque faible de sur-apprentissage.

Fine-Tuning

On degele progressivement les couches du backbone et on entraine tout ou partie du reseau.

Plus de potentiel, mais necessite plus de donnees et de soin.

1.3 Architecture ResNet

ResNet (Residual Network) introduit les connexions residuelles (skip connections) qui permettent d'entrainer des reseaux tres profonds sans probleme de vanishing gradient.

Bloc residuel

Sortie = F(x) + x

Ou F(x) represente les transformations apprises (convolutions) et x est l'entree directe (skip connection). Le reseau apprend ainsi le "residu" plutot que la transformation complete, ce qui facilite l'optimisation.

Partie 4 : Feature Extraction (backbone gele)

4.1 Geler le backbone

# Methode 1 : Feature extraction # Geler TOUS les parametres du backbone model_fe = models.resnet18(weights=models.ResNet18_Weights.IMAGENET1K_V1) for param in model_fe.parameters(): param.requires_grad = False # Remplacer la couche FC par un nouveau classificateur model_fe.fc = nn.Sequential( nn.Linear(num_features, 256), nn.ReLU(), nn.Dropout(0.3), nn.Linear(256, num_classes) ) model_fe = model_fe.to(device) # Verifier quels parametres sont entrainables trainable = sum(p.numel() for p in model_fe.parameters() if p.requires_grad) frozen = sum(p.numel() for p in model_fe.parameters() if not p.requires_grad) total = trainable + frozen print(f"Parametres totaux : {total:,}") print(f"Parametres geles : {frozen:,} ({frozen/total*100:.1f}%)") print(f"Parametres entrainables: {trainable:,} ({trainable/total*100:.1f}%)")
Parametres totaux : 11,309,898 Parametres geles : 11,176,512 (98.8%) Parametres entrainables: 133,386 (1.2%)
Seulement 1.2% des parametres sont entrainables ! L'entrainement sera donc tres rapide.

4.2 Fonctions utilitaires

def train_model(model, train_loader, val_loader, criterion, optimizer, num_epochs, device, scheduler=None): history = {'train_loss': [], 'train_acc': [], 'val_loss': [], 'val_acc': []} best_val_acc = 0.0 for epoch in range(num_epochs): # --- Entrainement --- model.train() running_loss, correct, total = 0.0, 0, 0 for images, labels in train_loader: images, labels = images.to(device), labels.to(device) optimizer.zero_grad() outputs = model(images) loss = criterion(outputs, labels) loss.backward() optimizer.step() running_loss += loss.item() * images.size(0) _, predicted = torch.max(outputs, 1) total += labels.size(0) correct += (predicted == labels).sum().item() train_loss = running_loss / total train_acc = correct / total # --- Validation --- model.eval() val_loss, val_correct, val_total = 0.0, 0, 0 with torch.no_grad(): for images, labels in val_loader: images, labels = images.to(device), labels.to(device) outputs = model(images) loss = criterion(outputs, labels) val_loss += loss.item() * images.size(0) _, predicted = torch.max(outputs, 1) val_total += labels.size(0) val_correct += (predicted == labels).sum().item() val_loss = val_loss / val_total val_acc = val_correct / val_total if scheduler: scheduler.step() history['train_loss'].append(train_loss) history['train_acc'].append(train_acc) history['val_loss'].append(val_loss) history['val_acc'].append(val_acc) if val_acc > best_val_acc: best_val_acc = val_acc if (epoch + 1) % 5 == 0: print(f" Epoch [{epoch+1}/{num_epochs}] " f"Train: {train_acc:.4f} | Val: {val_acc:.4f}") return history, best_val_acc

La feature extraction est rapide mais limitee. Le backbone gele produit des features generiques (apprises sur ImageNet), qui ne sont pas parfaitement adaptees a notre tache specifique. Le fine-tuning va ameliorer cela.

Partie 6 : Fine-Tuning complet

6.1 Degeler progressivement les couches

# Recharger un modele frais pour le fine-tuning model_ft = models.resnet18(weights=models.ResNet18_Weights.IMAGENET1K_V1) # Remplacer la couche FC model_ft.fc = nn.Sequential( nn.Linear(num_features, 256), nn.ReLU(), nn.Dropout(0.3), nn.Linear(256, num_classes) ) # Strategie : d'abord geler tout, puis degeler progressivement # Phase 1 : geler tout sauf FC (5 epochs) for param in model_ft.parameters(): param.requires_grad = False for param in model_ft.fc.parameters(): param.requires_grad = True model_ft = model_ft.to(device) print("Phase 1 : Entrainer uniquement le classificateur") optimizer_phase1 = optim.Adam(model_ft.fc.parameters(), lr=0.001) history_p1, _ = train_model(model_ft, train_loader, val_loader, criterion, optimizer_phase1, num_epochs=5, device=device) # Phase 2 : degeler layer4 + FC print("\nPhase 2 : Degeler layer4 + classificateur") for param in model_ft.layer4.parameters(): param.requires_grad = True trainable_p2 = sum(p.numel() for p in model_ft.parameters() if p.requires_grad) print(f"Parametres entrainables phase 2 : {trainable_p2:,}") optimizer_phase2 = optim.Adam( filter(lambda p: p.requires_grad, model_ft.parameters()), lr=0.0001 ) history_p2, _ = train_model(model_ft, train_loader, val_loader, criterion, optimizer_phase2, num_epochs=10, device=device) # Phase 3 : degeler tout le reseau print("\nPhase 3 : Fine-tuning complet (tout degele)") for param in model_ft.parameters(): param.requires_grad = True optimizer_phase3 = optim.Adam(model_ft.parameters(), lr=0.00001) history_p3, best_acc_ft = train_model(model_ft, train_loader, val_loader, criterion, optimizer_phase3, num_epochs=10, device=device) print(f"\nMeilleure accuracy fine-tuning : {best_acc_ft:.4f}")

Le learning rate doit diminuer a chaque phase de degelage. Les couches pre-entrainees sont deja proches de l'optimal : un lr trop grand detruirait les features apprises sur ImageNet.

Partie 7 : Learning Rate differentiel

7.1 Principe

Le learning rate differentiel consiste a attribuer des taux d'apprentissage differents aux differentes parties du reseau. Les couches profondes (proches de l'entree) recoivent un lr plus petit car leurs features sont plus generiques et mieux entraines. Les couches de tete recoivent un lr plus grand.

7.2 Implementation

# Recharger un modele frais model_dlr = models.resnet18(weights=models.ResNet18_Weights.IMAGENET1K_V1) model_dlr.fc = nn.Sequential( nn.Linear(num_features, 256), nn.ReLU(), nn.Dropout(0.3), nn.Linear(256, num_classes) ) model_dlr = model_dlr.to(device) # Groupes de parametres avec lr differents param_groups = [ # Couches basses (conv1, bn1, layer1, layer2) : lr tres bas {'params': list(model_dlr.conv1.parameters()) + list(model_dlr.bn1.parameters()) + list(model_dlr.layer1.parameters()) + list(model_dlr.layer2.parameters()), 'lr': 1e-5}, # Couches hautes (layer3, layer4) : lr modere {'params': list(model_dlr.layer3.parameters()) + list(model_dlr.layer4.parameters()), 'lr': 1e-4}, # Classificateur (fc) : lr eleve {'params': list(model_dlr.fc.parameters()), 'lr': 1e-3} ] optimizer_dlr = optim.Adam(param_groups, weight_decay=1e-4) # Afficher les groupes for i, group in enumerate(optimizer_dlr.param_groups): n_params = sum(p.numel() for p in group['params']) print(f"Groupe {i} : lr={group['lr']:.1e} | {n_params:,} parametres")
Groupe 0 : lr=1.0e-05 | 4,728,832 parametres Groupe 1 : lr=1.0e-04 | 6,447,680 parametres Groupe 2 : lr=1.0e-03 | 133,386 parametres

Analyse des resultats

  • Feature Extraction : le plus rapide mais limite en performance. Bon pour le prototypage rapide.
  • Fine-Tuning progressif : meilleur que FE grace a l'adaptation du backbone. Necessite un lr decroissant par phase.
  • LR Differentiel : plus elegant et performant. Les couches s'adaptent chacune a leur rythme optimal.
  • LR Diff + Augmentation : la meilleure approche. L'augmentation agressive compense le petit dataset.

Partie 9 : Recapitulatif et points clefs

Ce que nous avons appris

  1. Transfer Learning : reutiliser un modele pre-entraine pour une nouvelle tache avec peu de donnees.
  2. Feature Extraction : geler le backbone, entrainer uniquement le classificateur. Rapide mais limite.
  3. Fine-Tuning progressif : degeler les couches graduellement avec des lr decroissants.
  4. LR Differentiel : attribuer des lr differents selon la profondeur des couches.
  5. Data Augmentation : essentielle pour les petits datasets, complementaire au Transfer Learning.

Guide pratique : quelle approche choisir ?

SituationApproche recommandee
Tres peu de donnees (<500)Feature Extraction (backbone gele)
Peu de donnees (500-5000)Fine-tuning des couches hautes + FE
Dataset moyen (5K-50K)Fine-tuning complet avec lr differentiel
Grand dataset (>50K)Fine-tuning complet ou entrainement from scratch
Domaine tres different d'ImageNetFine-tuning agressif, lr plus eleve
Domaine similaire a ImageNetFeature Extraction ou fine-tuning leger
Ne pas oublier d'utiliser les transformations de normalisation d'ImageNet quand on utilise un modele pre-entraine sur ImageNet. Oublier cette etape est une erreur classique qui degrade fortement les performances.
Le Transfer Learning est la technique la plus utilisee en pratique industrielle. La majorite des projets de vision par ordinateur partent d'un modele pre-entraine (ResNet, EfficientNet, ViT) et l'adaptent au cas d'usage specifique. Maitrisez les differentes strategies de degelage et de learning rate pour obtenir les meilleurs resultats.
