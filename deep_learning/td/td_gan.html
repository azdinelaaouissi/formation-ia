TD Pratique - Generation d'Images avec GAN

Implementer un DCGAN pour generer des images avec PyTorch

Partie 1 : Recapitulatif theorique

Un Generative Adversarial Network (GAN) est un framework compose de deux reseaux en competition : un Generateur (G) qui cree de fausses images, et un Discriminateur (D) qui tente de distinguer les vraies images des fausses. Les deux s'entrainent mutuellement dans un jeu minimax.

1.1 Le jeu minimax

Fonction objectif du GAN

min_G max_D V(D, G) = E[log(D(x))] + E[log(1 - D(G(z)))]

  • D(x) : probabilite que x soit une vraie image (le discriminateur veut la maximiser)
  • D(G(z)) : probabilite que l'image generee soit vraie (le discriminateur veut la minimiser, le generateur veut la maximiser)
  • z : vecteur de bruit latent echantillonne aleatoirement

1.2 Intuition

Analogie : le faussaire et l'expert

Imaginez un faussaire (Generateur) qui tente de creer de faux billets de banque, et un expert en authentification (Discriminateur) qui les examine. Au debut, le faussaire est mauvais et l'expert les detecte facilement. Mais a force de competition, le faussaire s'ameliore jusqu'a produire des billets indiscernables des vrais, tandis que l'expert ne peut plus faire la difference (D(x) = 0.5).

1.3 DCGAN (Deep Convolutional GAN)

Le DCGAN utilise des couches convolutives dans le Generateur et le Discriminateur, avec des bonnes pratiques empiriques :

Principes du DCGAN

  • Generateur : utilise ConvTranspose2d (deconvolution) pour upscaler le bruit en image.
  • Discriminateur : utilise Conv2d classiques avec stride pour downscaler l'image en decision.
  • BatchNorm partout sauf dans la derniere couche de G et la premiere couche de D.
  • ReLU dans G (sauf la sortie : Tanh), LeakyReLU dans D.
  • Pas de couches Fully Connected (sauf la derniere de D dans certaines implementations).

1.4 Chronologie des GAN

2014
GAN original (Goodfellow et al.) - Premiere formulation du framework adversarial.
2015
DCGAN (Radford et al.) - Architecture convolutive stable et reproductible.
2017
WGAN (Arjovsky et al.) - Wasserstein distance pour un entrainement plus stable.
2018
StyleGAN (Karras et al.) - Generation de visages photoralistes haute resolution.
2020
StyleGAN2 - Amelioration de la qualite et suppression des artefacts.

ConvTranspose2d (deconvolution)

L'operation inverse de la convolution : elle augmente la resolution spatiale.

  • Taille_sortie = (Taille_entree - 1) * stride - 2 * padding + kernel_size
  • Exemple : entree 7x7, stride 2, padding 1, kernel 4 -> sortie = (7-1)*2 - 2 + 4 = 14x14

5.1 Binary Cross-Entropy Loss

# BCE Loss pour la classification binaire (vrai/faux) criterion = nn.BCELoss() # Labels real_label = 1.0 fake_label = 0.0 # Optimizers avec les hyperparametres DCGAN optimizerD = optim.Adam(netD.parameters(), lr=lr, betas=(beta1, 0.999)) optimizerG = optim.Adam(netG.parameters(), lr=lr, betas=(beta1, 0.999)) print("Loss : BCELoss") print(f"Optimizer D : Adam(lr={lr}, betas=({beta1}, 0.999))") print(f"Optimizer G : Adam(lr={lr}, betas=({beta1}, 0.999))")

5.2 Comprendre les losses du GAN

Entrainement du Discriminateur

  1. Sur les vraies images : loss_real = BCE(D(x), 1) -> D doit predire 1 pour les vraies images.
  2. Sur les fausses images : loss_fake = BCE(D(G(z)), 0) -> D doit predire 0 pour les fausses.
  3. Loss totale D = (loss_real + loss_fake) / 2

Entrainement du Generator

  1. Generer de fausses images : fake = G(z)
  2. Loss G = BCE(D(fake), 1) -> G veut que D predise 1 (que les fausses soient vues comme vraies).
Le beta1=0.5 au lieu de 0.9 (defaut Adam) est une recommandation empirique du papier DCGAN. Un beta1 plus bas stabilise l'entrainement des GAN en reduisant le momentum, ce qui evite des oscillations entre G et D.

Interpreter les metriques

  • D(x) -> 0.5 : le discriminateur ne distingue plus les vraies des fausses images (equilibre Nash).
  • D(G(z)) -> 0.5 : les images generees sont aussi convaincantes que les vraies.
  • Loss_D et Loss_G : devraient osciller autour de valeurs stables, pas converger vers 0.
Si la Loss_D tombe a 0 rapidement, le discriminateur est trop fort et le generateur ne peut plus apprendre. Si la Loss_G tombe a 0, c'est l'inverse. Un bon entrainement montre un equilibre entre les deux.

Partie 8 : Interpolation dans l'espace latent et mode collapse

8.1 Interpolation lineaire

L'interpolation dans l'espace latent consiste a generer des images a partir de points intermediaires entre deux vecteurs latents z1 et z2. Si le generateur a bien appris, les images intermediaires devraient montrer une transition douce entre les deux extremites.
def interpolate_latent(z1, z2, n_steps=10): """Interpolation lineaire entre deux vecteurs latents.""" alphas = torch.linspace(0, 1, n_steps) interpolations = [] for alpha in alphas: z_interp = (1 - alpha) * z1 + alpha * z2 interpolations.append(z_interp) return torch.stack(interpolations) # Choisir deux points de depart z1 = torch.randn(1, latent_dim, 1, 1).to(device) z2 = torch.randn(1, latent_dim, 1, 1).to(device) # Interpoler n_steps = 12 z_interp = interpolate_latent(z1, z2, n_steps) # Generer les images netG.eval() with torch.no_grad(): imgs_interp = netG(z_interp.to(device)).cpu() # Afficher fig, axes = plt.subplots(1, n_steps, figsize=(2 * n_steps, 2)) for i, ax in enumerate(axes): img = imgs_interp[i].squeeze() img = (img + 1) / 2 # [-1,1] -> [0,1] ax.imshow(img, cmap='gray') ax.axis('off') if i == 0: ax.set_title('z1', color='#ff6b35') elif i == n_steps - 1: ax.set_title('z2', color='#569cd6') plt.suptitle('Interpolation lineaire dans l\'espace latent', fontsize=14) plt.tight_layout() plt.show()

8.2 Interpolation spherique (SLERP)

def slerp(z1, z2, alpha): """Interpolation spherique entre deux vecteurs.""" z1_flat = z1.view(-1) z2_flat = z2.view(-1) # Normaliser z1_norm = z1_flat / z1_flat.norm() z2_norm = z2_flat / z2_flat.norm() # Angle entre les deux vecteurs omega = torch.acos(torch.clamp(torch.dot(z1_norm, z2_norm), -1, 1)) if omega.abs() < 1e-6: return (1 - alpha) * z1 + alpha * z2 interp = (torch.sin((1 - alpha) * omega) / torch.sin(omega)) * z1_flat + \ (torch.sin(alpha * omega) / torch.sin(omega)) * z2_flat return interp.view_as(z1) # Interpolation SLERP alphas = torch.linspace(0, 1, 12) z_slerp = torch.stack([slerp(z1, z2, a) for a in alphas]) with torch.no_grad(): imgs_slerp = netG(z_slerp.to(device)).cpu() fig, axes = plt.subplots(1, 12, figsize=(24, 2)) for i, ax in enumerate(axes): img = (imgs_slerp[i].squeeze() + 1) / 2 ax.imshow(img, cmap='gray') ax.axis('off') plt.suptitle('Interpolation SLERP (spherique)', fontsize=14) plt.tight_layout() plt.show()
L'interpolation SLERP est souvent preferee a l'interpolation lineaire car les vecteurs latents suivent une distribution gaussienne, donc les points les plus probables se trouvent sur une hypersphere, pas sur une droite. SLERP respecte cette geometrie.

8.3 Detecter le mode collapse

Mode Collapse

Le mode collapse est un probleme frequent des GAN ou le generateur ne produit qu'un petit nombre de sorties differentes, au lieu de couvrir toute la diversite des donnees reelles. Signes a surveiller :

  • Toutes les images generees se ressemblent visuellement.
  • La variance des pixels generees est tres faible.
  • La correlation entre images aleatoires est tres elevee (>0.8).
  • La Loss_G chute a 0 tandis que la Loss_D monte.

8.4 Solutions contre le mode collapse

TechniqueDescription
Label SmoothingUtiliser 0.9 au lieu de 1.0 pour les vrais labels (deja fait ci-dessus).
Instance NoiseAjouter du bruit aux entrees du discriminateur pour le rendre moins confiant.
Feature MatchingMinimiser la distance entre features intermediaires de D pour les vrais et faux.
Minibatch DiscriminationAjouter une couche dans D qui detecte si toutes les images du batch sont identiques.

Partie 9 : Recapitulatif et points clefs

Ce que nous avons appris

  1. Framework GAN : deux reseaux en competition - Generator (cree) et Discriminator (critique).
  2. DCGAN : ConvTranspose2d pour G, Conv2d pour D, avec BatchNorm et activations specifiques.
  3. Entrainement alterne : entrainer D puis G a chaque iteration, avec label smoothing pour la stabilite.
  4. Hyperparametres clefs : lr=0.0002, beta1=0.5, normalisation [-1,1], Tanh en sortie de G.
  5. Interpolation latente : transition douce entre images grace a SLERP.
  6. Mode collapse : diagnostic par variance et correlation, solutions multiples.
