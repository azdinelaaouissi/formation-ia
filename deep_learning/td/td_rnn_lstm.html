TD Pratique - Prediction de Sequences avec LSTM

Predire des series temporelles et classifier du texte avec des reseaux recurrents

Partie 1 : Recapitulatif theorique

Un Reseau de Neurones Recurrent (RNN) est une architecture concue pour traiter des donnees sequentielles. A chaque pas de temps, il prend en entree un element de la sequence et un etat cache (hidden state) provenant du pas precedent, ce qui lui permet de conserver une "memoire" du contexte passe.

1.1 Le probleme du RNN simple

Le RNN classique souffre de deux problemes majeurs lors de l'entrainement sur de longues sequences :

Vanishing & Exploding Gradients

  • Vanishing gradients : les gradients deviennent exponentiellement petits lors de la retropropagation a travers de nombreux pas de temps. Le reseau "oublie" les informations lointaines.
  • Exploding gradients : les gradients croissent exponentiellement, rendant l'entrainement instable. On peut y remedier avec le gradient clipping.

1.2 Architecture LSTM

Le Long Short-Term Memory (LSTM) resout le probleme du vanishing gradient grace a un mecanisme de portes (gates) qui controlent le flux d'information :

Les 3 portes du LSTM

  • Forget Gate (ft) : decide quelles informations oublier de l'etat cellulaire. ft = sigmoid(Wf * [ht-1, xt] + bf)
  • Input Gate (it) : decide quelles nouvelles informations stocker. it = sigmoid(Wi * [ht-1, xt] + bi)
  • Output Gate (ot) : decide quelles informations extraire de l'etat cellulaire. ot = sigmoid(Wo * [ht-1, xt] + bo)

L'etat cellulaire Ct est la "memoire longue" qui traverse la sequence avec des modifications minimales (additions/multiplications element par element).

1.3 LSTM vs GRU

CaracteristiqueLSTMGRU
Nombre de portes3 (forget, input, output)2 (reset, update)

ParametresPlus (4 * hidden * (input + hidden))Moins (3 * hidden * (input + hidden))

VitessePlus lentPlus rapide

Analogie : le LSTM comme un secretaire

Imaginez un secretaire qui prend des notes au fil d'une reunion (sequence). La forget gate efface les notes devenues obsoletes, l'input gate ajoute de nouvelles informations importantes, et l'output gate choisit quelles notes communiquer au patron quand il pose une question.

3.2 Sliding window pour creer les sequences

La technique de sliding window consiste a decouper la serie temporelle en sous-sequences de longueur fixe. Chaque fenetre de taille L produit une paire (X, y) ou X est la sequence d'entree de longueur L et y est la valeur suivante a predire.
class TimeSeriesDataset(Dataset): def __init__(self, data, seq_length): self.data = torch.FloatTensor(data) self.seq_length = seq_length def __len__(self): return len(self.data) - self.seq_length def __getitem__(self, idx): x = self.data[idx:idx + self.seq_length].unsqueeze(-1) # [seq_len, 1] y = self.data[idx + self.seq_length] # valeur suivante return x, y # Parametres seq_length = 50 # Diviser train / test (80% / 20%) split_idx = int(len(data_norm) * 0.8) train_data = data_norm[:split_idx] test_data = data_norm[split_idx:] # Creer les datasets train_dataset = TimeSeriesDataset(train_data, seq_length) test_dataset = TimeSeriesDataset(test_data, seq_length) train_loader = DataLoader(train_dataset, batch_size=32, shuffle=True) test_loader = DataLoader(test_dataset, batch_size=32, shuffle=False) # Verifier les dimensions x_sample, y_sample = train_dataset[0] print(f"Shape entree (x) : {x_sample.shape}") print(f"Shape sortie (y) : {y_sample.shape}") print(f"Nombre de sequences train : {len(train_dataset)}") print(f"Nombre de sequences test : {len(test_dataset)}")

Partie 4 : Construire le modele LSTM

4.1 Architecture du modele

class LSTMPredictor(nn.Module): def __init__(self, input_size=1, hidden_size=64, num_layers=2, dropout=0.2): super(LSTMPredictor, self).__init__() self.hidden_size = hidden_size self.num_layers = num_layers # Couche LSTM self.lstm = nn.LSTM( input_size=input_size, hidden_size=hidden_size, num_layers=num_layers, batch_first=True, dropout=dropout if num_layers > 1 else 0 ) # Couches de sortie self.fc = nn.Sequential( nn.Linear(hidden_size, 32), nn.ReLU(), nn.Linear(32, 1) ) def forward(self, x): # x shape: [batch, seq_len, input_size] # Initialiser les etats caches h0 = torch.zeros(self.num_layers, x.size(0), self.hidden_size).to(x.device) c0 = torch.zeros(self.num_layers, x.size(0), self.hidden_size).to(x.device) # Forward pass du LSTM lstm_out, (hn, cn) = self.lstm(x, (h0, c0)) # lstm_out : [batch, seq_len, hidden_size] # Prendre la sortie du dernier pas de temps last_output = lstm_out[:, -1, :] # [batch, hidden_size] # Passer dans les couches FC prediction = self.fc(last_output) # [batch, 1] return prediction.squeeze(-1) # Instancier le modele model = LSTMPredictor( input_size=1, hidden_size=64, num_layers=2, dropout=0.2 ).to(device) print(model) total_params = sum(p.numel() for p in model.parameters()) print(f"\nParametres totaux : {total_params:,}")
LSTMPredictor( (lstm): LSTM(1, 64, num_layers=2, batch_first=True, dropout=0.2) (fc): Sequential( (0): Linear(in_features=64, out_features=32, bias=True) (1): ReLU() (2): Linear(in_features=32, out_features=1, bias=True) ) ) Parametres totaux : 36,961

Parametres cles de nn.LSTM

  • input_size : dimension de chaque element de la sequence (1 pour une serie univariee)
  • hidden_size : dimension de l'etat cache (64 ici). Plus grand = plus de capacite mais plus lent.
  • num_layers : nombre de couches LSTM empilees. Plus de couches = plus de profondeur.
  • batch_first=True : format des tenseurs [batch, seq, features] au lieu de [seq, batch, features].
  • dropout : applique entre les couches LSTM (pas apres la derniere).

Partie 5 : Entrainement

5.1 Configuration

criterion = nn.MSELoss() optimizer = optim.Adam(model.parameters(), lr=0.001) scheduler = optim.lr_scheduler.ReduceLROnPlateau(optimizer, mode='min', patience=5, factor=0.5)
Pour la prediction de series temporelles (regression), on utilise la MSELoss (Mean Squared Error) et non la CrossEntropyLoss (classification). Le ReduceLROnPlateau reduit le learning rate quand la loss stagne.

5.2 Boucle d'entrainement

def train_epoch(model, loader, criterion, optimizer, device): model.train() total_loss = 0.0 for x_batch, y_batch in loader: x_batch, y_batch = x_batch.to(device), y_batch.to(device) optimizer.zero_grad() predictions = model(x_batch) loss = criterion(predictions, y_batch) loss.backward() # Gradient clipping pour eviter l'explosion torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0) optimizer.step() total_loss += loss.item() * x_batch.size(0) return total_loss / len(loader.dataset) def evaluate_epoch(model, loader, criterion, device): model.eval() total_loss = 0.0 with torch.no_grad(): for x_batch, y_batch in loader: x_batch, y_batch = x_batch.to(device), y_batch.to(device) predictions = model(x_batch) loss = criterion(predictions, y_batch) total_loss += loss.item() * x_batch.size(0) return total_loss / len(loader.dataset) # Entrainement num_epochs = 50 history = {'train_loss': [], 'val_loss': []} best_val_loss = float('inf') for epoch in range(num_epochs): train_loss = train_epoch(model, train_loader, criterion, optimizer, device) val_loss = evaluate_epoch(model, test_loader, criterion, device) scheduler.step(val_loss) history['train_loss'].append(train_loss) history['val_loss'].append(val_loss) if val_loss < best_val_loss: best_val_loss = val_loss torch.save(model.state_dict(), 'best_lstm.pth') if (epoch + 1) % 10 == 0: lr = optimizer.param_groups[0]['lr'] print(f"Epoch [{epoch+1}/{num_epochs}] Train Loss: {train_loss:.6f} | " f"Val Loss: {val_loss:.6f} | LR: {lr:.6f}") print(f"\nMeilleure val loss : {best_val_loss:.6f}")

6.3 Prediction multi-pas (autoregressive)

def predict_multi_step(model, initial_seq, n_steps, device): """Prediction autoregressive : utiliser les predictions comme entrees.""" model.eval() predictions = [] current_seq = initial_seq.clone() with torch.no_grad(): for _ in range(n_steps): x = current_seq.unsqueeze(0).to(device) # [1, seq_len, 1] pred = model(x).item() predictions.append(pred) # Decaler la fenetre : retirer le premier, ajouter la prediction new_point = torch.tensor([[pred]]) current_seq = torch.cat([current_seq[1:], new_point], dim=0) return np.array(predictions) # Prendre les 50 premiers points du test comme point de depart initial_seq = torch.FloatTensor(test_data[:seq_length]).unsqueeze(-1) n_future = 200 future_preds = predict_multi_step(model, initial_seq, n_future, device) # Denormaliser future_preds_real = future_preds * (data_max - data_min) + data_min test_real = test_data * (data_max - data_min) + data_min plt.figure(figsize=(14, 5)) plt.plot(range(seq_length + n_future), test_real[:seq_length + n_future], label='Reel', color='#569cd6') plt.plot(range(seq_length, seq_length + n_future), future_preds_real, label='Predictions multi-pas', color='#ff6b35', linestyle='--') plt.axvline(x=seq_length, color='gray', linestyle=':', label='Debut des predictions') plt.title('Prediction autoregressive (multi-pas)') plt.legend() plt.grid(True, alpha=0.3) plt.show()
En mode autoregressive, les erreurs s'accumulent a chaque pas. La prediction se degrade rapidement apres 50-100 pas. C'est une limitation fondamentale de cette approche.

7.2 Modele LSTM pour la classification de texte

class TextLSTM(nn.Module): def __init__(self, vocab_size, embed_dim, hidden_size, num_classes): super(TextLSTM, self).__init__() # Couche d'embedding : transforme les indices en vecteurs denses self.embedding = nn.Embedding(vocab_size, embed_dim, padding_idx=0) # LSTM self.lstm = nn.LSTM(embed_dim, hidden_size, batch_first=True, bidirectional=True) # Classificateur (bidirectional = hidden * 2) self.classifier = nn.Sequential( nn.Linear(hidden_size * 2, 32), nn.ReLU(), nn.Dropout(0.3), nn.Linear(32, 1), nn.Sigmoid() ) def forward(self, x): # x : [batch, seq_len] (indices de mots) embedded = self.embedding(x) # [batch, seq_len, embed_dim] lstm_out, (hn, _) = self.lstm(embedded) # Concatener le dernier hidden des deux directions hn = torch.cat((hn[-2], hn[-1]), dim=1) # [batch, hidden*2] output = self.classifier(hn).squeeze(-1) return output text_model = TextLSTM( vocab_size=len(vocab), embed_dim=32, hidden_size=32, num_classes=1 ).to(device) print(text_model)

Partie 8 : Comparer LSTM vs GRU

8.1 Modele GRU

class GRUPredictor(nn.Module): def __init__(self, input_size=1, hidden_size=64, num_layers=2, dropout=0.2): super(GRUPredictor, self).__init__() self.hidden_size = hidden_size self.num_layers = num_layers self.gru = nn.GRU( input_size=input_size, hidden_size=hidden_size, num_layers=num_layers, batch_first=True, dropout=dropout if num_layers > 1 else 0 ) self.fc = nn.Sequential( nn.Linear(hidden_size, 32), nn.ReLU(), nn.Linear(32, 1) ) def forward(self, x): h0 = torch.zeros(self.num_layers, x.size(0), self.hidden_size).to(x.device) gru_out, hn = self.gru(x, h0) last_output = gru_out[:, -1, :] prediction = self.fc(last_output) return prediction.squeeze(-1) gru_model = GRUPredictor(input_size=1, hidden_size=64, num_layers=2).to(device) lstm_params = sum(p.numel() for p in model.parameters()) gru_params = sum(p.numel() for p in gru_model.parameters()) print(f"Parametres LSTM : {lstm_params:,}") print(f"Parametres GRU : {gru_params:,}") print(f"Difference : {lstm_params - gru_params:,} ({(lstm_params-gru_params)/lstm_params*100:.1f}%)")
Parametres LSTM : 36,961 Parametres GRU : 28,129 Difference : 8,832 (23.9%)

Conclusions de la comparaison

  • Le GRU est ~20% plus rapide a entrainer grace a ses 2 portes au lieu de 3.
  • Le LSTM atteint une loss legerement inferieure sur notre serie sinusoidale.
  • En pratique, le GRU est souvent le meilleur compromis pour des sequences de longueur moderee.
  • Pour des sequences tres longues (>500 pas), le LSTM conserve generalement un avantage.

Partie 9 : Recapitulatif et points clefs

Ce que nous avons appris

  1. Donnees sequentielles : sliding window pour creer des paires (sequence, cible) a partir d'une serie temporelle.
  2. LSTM : architecture avec 3 portes et un cell state pour memoriser les dependances a long terme.
  3. Entrainement : MSE loss pour la regression, gradient clipping pour la stabilite.
  4. Prediction autoregressive : utiliser les predictions comme entrees pour predire le futur, avec accumulation d'erreurs.
  5. Classification de texte : embedding + LSTM bidirectionnel pour l'analyse de sentiment.
  6. LSTM vs GRU : le GRU est plus rapide avec des performances comparables.

Les RNN/LSTM sont sequentiels par nature et ne peuvent pas etre facilement parallelises. Pour les taches de NLP modernes, les Transformers (avec attention) les ont largement remplaces.
Maitrisez bien les LSTM avant de passer aux Transformers. Les mecanismes de gating et de memoire que vous avez etudies ici sont les fondations conceptuelles de l'attention et des architectures modernes.
