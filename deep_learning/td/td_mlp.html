TD Pratique - Classification avec MLP (PyTorch)

Construire, entrainer et evaluer un Multi-Layer Perceptron de A a Z

Partie 1 : Recapitulatif theorique

Un Multi-Layer Perceptron (MLP) est un reseau de neurones entierement connecte compose d'au moins trois couches : une couche d'entree, une ou plusieurs couches cachees et une couche de sortie. Chaque neurone d'une couche est connecte a tous les neurones de la couche suivante.

1.1 Architecture d'un MLP

Le MLP est le modele fondamental du Deep Learning. Il realise une suite de transformations lineaires suivies de fonctions d'activation non lineaires :

Transformation d'une couche

Sortie = activation(W * x + b)

  • W : matrice de poids (parametres apprenables)
  • x : vecteur d'entree
  • b : vecteur de biais
  • activation : fonction non lineaire (ReLU, Sigmoid, Tanh...)

1.2 Fonctions d'activation

FonctionFormuleUsage typique
ReLUmax(0, x)Couches cachees (standard)
Sigmoid1 / (1 + exp(-x))Classification binaire (sortie)
Tanh(exp(x) - exp(-x)) / (exp(x) + exp(-x))Couches cachees (donnees centrees)
Softmaxexp(xi) / sum(exp(xj))Classification multi-classes (sortie)

1.3 Backpropagation et descente de gradient

L'apprentissage du MLP se fait en deux phases :

  1. Forward pass : les donnees traversent le reseau couche par couche pour produire une prediction.
  2. Backward pass : le gradient de la loss est calcule par rapport a chaque poids grace a la regle de la chaine, puis les poids sont mis a jour.
La backpropagation est un algorithme d'application de la regle de derivation en chaine. PyTorch la gere automatiquement avec autograd. Vous n'aurez jamais a calculer les gradients a la main !

1.4 Fonction de perte (Loss)

Pour la classification multi-classes, on utilise la Cross-Entropy Loss :

CrossEntropyLoss = -sum(yi * log(pi))

Ou yi est le label reel (one-hot) et pi la probabilite predite pour la classe i. En PyTorch, nn.CrossEntropyLoss combine LogSoftmax et NLLLoss en une seule operation numeriquement stable.

Partie 4 : Construire le modele MLP

4.1 Definition de l'architecture

En PyTorch, on definit un modele en heritant de nn.Module. On declare les couches dans __init__ et on definit le passage des donnees dans forward.
class MLP(nn.Module): def __init__(self, input_size, hidden1, hidden2, num_classes, dropout_rate=0.2): super(MLP, self).__init__() # Premiere couche cachee self.fc1 = nn.Linear(input_size, hidden1) self.bn1 = nn.BatchNorm1d(hidden1) self.relu1 = nn.ReLU() self.drop1 = nn.Dropout(dropout_rate) # Deuxieme couche cachee self.fc2 = nn.Linear(hidden1, hidden2) self.bn2 = nn.BatchNorm1d(hidden2) self.relu2 = nn.ReLU() self.drop2 = nn.Dropout(dropout_rate) # Couche de sortie self.fc3 = nn.Linear(hidden2, num_classes) def forward(self, x): # Aplatir l'image 28x28 en vecteur de 784 x = x.view(x.size(0), -1) # Couche 1 x = self.fc1(x) x = self.bn1(x) x = self.relu1(x) x = self.drop1(x) # Couche 2 x = self.fc2(x) x = self.bn2(x) x = self.relu2(x) x = self.drop2(x) # Sortie (pas de softmax, integre dans CrossEntropyLoss) x = self.fc3(x) return x # Instancier le modele model = MLP( input_size=784, # 28 * 28 pixels hidden1=512, hidden2=256, num_classes=10, dropout_rate=0.2 ).to(device) print(model)
MLP( (fc1): Linear(in_features=784, out_features=512, bias=True) (bn1): BatchNorm1d(512) (relu1): ReLU() (drop1): Dropout(p=0.2) (fc2): Linear(in_features=512, out_features=256, bias=True) (bn2): BatchNorm1d(256) (relu2): ReLU() (drop2): Dropout(p=0.2) (fc3): Linear(in_features=256, out_features=10, bias=True) )

Un MLP avec des images necessite d'aplatir chaque image en vecteur. Cela perd l'information spatiale et cree beaucoup de parametres. C'est pourquoi les CNN sont preferes pour les images.

4.3 Comprendre le Dropout

Dropout comme regularisation

Le Dropout desactive aleatoirement une fraction des neurones pendant l'entrainement. Cela force le reseau a ne pas dependre d'un seul chemin et reduit le sur-apprentissage.

  • Entrainement : neurones desactives aleatoirement avec probabilite p
  • Evaluation : tous les neurones actifs, sorties mises a l'echelle automatiquement

Optimizer : Adam ( Parameter Group 0 amsgrad: False betas: (0.9, 0.999) eps: 1e-08 lr: 0.001 weight_decay: 1e-05 )

Parametres d'Adam

  • lr (learning rate) : pas d'apprentissage. Typiquement 0.001 pour Adam.
  • betas (0.9, 0.999) : coefficients pour les moyennes mobiles du gradient et de son carre.
  • weight_decay : regularisation L2 sur les poids. Empeche les poids de devenir trop grands.
  • eps : terme de stabilite numerique pour eviter la division par zero.

Partie 6 : Boucle d'entrainement

6.1 Fonctions d'entrainement et de validation

def train_epoch(model, loader, criterion, optimizer, device): model.train() running_loss = 0.0 correct = 0 total = 0 for images, labels in loader: images, labels = images.to(device), labels.to(device) # Forward pass outputs = model(images) loss = criterion(outputs, labels) # Backward pass et mise a jour optimizer.zero_grad() loss.backward() optimizer.step() # Statistiques running_loss += loss.item() * images.size(0) _, predicted = torch.max(outputs, 1) total += labels.size(0) correct += (predicted == labels).sum().item() epoch_loss = running_loss / total epoch_acc = correct / total return epoch_loss, epoch_acc def evaluate(model, loader, criterion, device): model.eval() running_loss = 0.0 correct = 0 total = 0 with torch.no_grad(): for images, labels in loader: images, labels = images.to(device), labels.to(device) outputs = model(images) loss = criterion(outputs, labels) running_loss += loss.item() * images.size(0) _, predicted = torch.max(outputs, 1) total += labels.size(0) correct += (predicted == labels).sum().item() epoch_loss = running_loss / total epoch_acc = correct / total return epoch_loss, epoch_acc
N'oubliez jamais model.train() avant l'entrainement et model.eval() avant l'evaluation. Le Dropout et le BatchNorm se comportent differemment entre les deux modes.

8.3 Comparer les fonctions d'activation

class MLPCustomActivation(nn.Module): def __init__(self, activation_fn): super().__init__() self.net = nn.Sequential( nn.Linear(784, 512), activation_fn(), nn.Linear(512, 256), activation_fn(), nn.Linear(256, 10) ) def forward(self, x): return self.net(x.view(x.size(0), -1)) activations = { 'ReLU': nn.ReLU, 'Tanh': nn.Tanh, 'LeakyReLU': nn.LeakyReLU, 'ELU': nn.ELU, 'GELU': nn.GELU, } for name, act_fn in activations.items(): model_act = MLPCustomActivation(act_fn).to(device) opt_act = optim.Adam(model_act.parameters(), lr=0.001) for _ in range(5): train_epoch(model_act, train_loader, criterion, opt_act, device) _, val_acc = evaluate(model_act, test_loader, criterion, device) print(f"Activation {name:12s} -> Val Acc: {val_acc:.4f}")

Resultats de l'experimentation

  • Architecture : un reseau plus grand n'est pas toujours meilleur. Le gain marginal diminue rapidement.
  • Learning rate : 0.001 est le point ideal pour Adam sur ce probleme. Trop eleve = instabilite, trop bas = convergence lente.
  • Activations : ReLU reste le choix standard, mais GELU montre un leger avantage.

Partie 9 : Recapitulatif et points clefs

Ce que nous avons appris

  1. Architecture MLP : enchainement de couches Linear + activation + regularisation (Dropout, BatchNorm).
  2. Pipeline PyTorch : Dataset -> DataLoader -> Model -> Loss -> Optimizer -> Training Loop.
  3. Bonnes pratiques : normaliser les donnees, utiliser model.train()/model.eval(), torch.no_grad() pour l'evaluation.
  4. Evaluation : aller au-dela de l'accuracy globale avec confusion matrix, precision/recall par classe, analyse des erreurs.
  5. Experimentation : tester differentes architectures, learning rates et activations pour trouver la meilleure configuration.
