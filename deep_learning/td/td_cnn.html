TD Pratique - Classification d'Images avec CNN

Construire un CNN pour classifier CIFAR-10 avec PyTorch

Partie 1 : Recapitulatif theorique

Un Reseau de Neurones Convolutif (CNN) est une architecture specialisee pour le traitement d'images. Contrairement au MLP qui aplatit l'image, le CNN exploite la structure spatiale grace a des filtres de convolution qui detectent des motifs locaux (bords, textures, formes).

1.1 L'operation de convolution

La convolution applique un filtre (ou noyau) sur l'image d'entree. Le filtre glisse sur l'image et calcule a chaque position un produit scalaire entre le filtre et la region locale de l'image.

Parametres d'une couche de convolution

  • in_channels : nombre de canaux d'entree (3 pour RGB, 1 pour grayscale)
  • out_channels : nombre de filtres (= nombre de feature maps en sortie)
  • kernel_size : taille du filtre (ex: 3x3, 5x5)
  • stride : pas de deplacement du filtre (defaut: 1)
  • padding : ajout de zeros autour de l'image pour controler la taille de sortie

1.2 Formule de la taille de sortie

Taille_sortie = (Taille_entree - kernel_size + 2 * padding) / stride + 1

Exemple : entree 32x32, kernel 3x3, padding 1, stride 1 -> sortie = (32 - 3 + 2) / 1 + 1 = 32x32

1.3 Pooling

Le pooling reduit la dimension spatiale des feature maps tout en conservant les informations les plus importantes :

TypeOperationEffet
Max PoolingPrend le maximum de chaque regionConserve les activations fortes
Average PoolingPrend la moyenne de chaque regionLisse les activations
Global Average PoolingMoyenne sur toute la feature mapReduit a 1x1 par canal

1.4 Architecture typique d'un CNN

Etape 1 : Blocs convolutifs (Conv2d + BatchNorm + ReLU + MaxPool) - extraction de features
Etape 2 : Aplatissement (Flatten) - passage de 3D a 1D
Etape 3 : Couches fully connected (Linear + ReLU + Dropout) - classification

Analogie : le CNN comme un systeme visuel

Les premieres couches detectent des motifs simples (bords, couleurs), les couches intermediaires combinent ces motifs en textures et formes, et les couches profondes reconnaissent des objets complets. C'est exactement comme le systeme visuel humain !

Partie 4 : Construire le CNN

4.1 Architecture du modele

class CNN(nn.Module): def __init__(self, num_classes=10): super(CNN, self).__init__() # Bloc convolutif 1 : 3 -> 32 canaux self.conv_block1 = nn.Sequential( nn.Conv2d(3, 32, kernel_size=3, padding=1), nn.BatchNorm2d(32), nn.ReLU(inplace=True), nn.Conv2d(32, 32, kernel_size=3, padding=1), nn.BatchNorm2d(32), nn.ReLU(inplace=True), nn.MaxPool2d(kernel_size=2, stride=2), nn.Dropout2d(0.25) ) # Bloc convolutif 2 : 32 -> 64 canaux self.conv_block2 = nn.Sequential( nn.Conv2d(32, 64, kernel_size=3, padding=1), nn.BatchNorm2d(64), nn.ReLU(inplace=True), nn.Conv2d(64, 64, kernel_size=3, padding=1), nn.BatchNorm2d(64), nn.ReLU(inplace=True), nn.MaxPool2d(kernel_size=2, stride=2), nn.Dropout2d(0.25) ) # Bloc convolutif 3 : 64 -> 128 canaux self.conv_block3 = nn.Sequential( nn.Conv2d(64, 128, kernel_size=3, padding=1), nn.BatchNorm2d(128), nn.ReLU(inplace=True), nn.Conv2d(128, 128, kernel_size=3, padding=1), nn.BatchNorm2d(128), nn.ReLU(inplace=True), nn.MaxPool2d(kernel_size=2, stride=2), nn.Dropout2d(0.25) ) # Classificateur (couches fully connected) self.classifier = nn.Sequential( nn.Flatten(), nn.Linear(128 * 4 * 4, 512), nn.BatchNorm1d(512), nn.ReLU(inplace=True), nn.Dropout(0.5), nn.Linear(512, num_classes) ) def forward(self, x): x = self.conv_block1(x) # 32x32 -> 16x16 x = self.conv_block2(x) # 16x16 -> 8x8 x = self.conv_block3(x) # 8x8 -> 4x4 x = self.classifier(x) return x model = CNN(num_classes=10).to(device) print(model)

Partie 5 : Entrainement avec scheduler

5.1 Definir loss, optimizer et scheduler

criterion = nn.CrossEntropyLoss() optimizer = optim.Adam(model.parameters(), lr=0.001, weight_decay=1e-4) # Scheduler : CosineAnnealingLR reduit progressivement le lr scheduler = optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=50) print(f"Learning rate initial : {optimizer.param_groups[0]['lr']}")
Learning rate initial : 0.001

CosineAnnealingLR

Ce scheduler fait varier le learning rate selon une courbe cosinus. Il commence haut, diminue progressivement jusqu'a eta_min, puis peut remonter si on continue l'entrainement.

  • T_max : nombre d'epochs pour un demi-cycle du cosinus
  • eta_min : learning rate minimal (defaut: 0)

5.2 Fonctions d'entrainement et d'evaluation

def train_one_epoch(model, loader, criterion, optimizer, device): model.train() running_loss = 0.0 correct = 0 total = 0 for batch_idx, (images, labels) in enumerate(loader): images, labels = images.to(device), labels.to(device) optimizer.zero_grad() outputs = model(images) loss = criterion(outputs, labels) loss.backward() optimizer.step() running_loss += loss.item() * images.size(0) _, predicted = torch.max(outputs, 1) total += labels.size(0) correct += (predicted == labels).sum().item() return running_loss / total, correct / total def evaluate_model(model, loader, criterion, device): model.eval() running_loss = 0.0 correct = 0 total = 0 with torch.no_grad(): for images, labels in loader: images, labels = images.to(device), labels.to(device) outputs = model(images) loss = criterion(outputs, labels) running_loss += loss.item() * images.size(0) _, predicted = torch.max(outputs, 1) total += labels.size(0) correct += (predicted == labels).sum().item() return running_loss / total, correct / total

5.3 Boucle d'entrainement complete

num_epochs = 30 best_val_acc = 0.0 history = {'train_loss': [], 'train_acc': [], 'val_loss': [], 'val_acc': [], 'lr': []} for epoch in range(num_epochs): train_loss, train_acc = train_one_epoch(model, train_loader, criterion, optimizer, device) val_loss, val_acc = evaluate_model(model, test_loader, criterion, device) current_lr = optimizer.param_groups[0]['lr'] # Mise a jour du scheduler scheduler.step() # Sauvegarder le meilleur modele if val_acc > best_val_acc: best_val_acc = val_acc torch.save(model.state_dict(), 'best_cnn_cifar10.pth') history['train_loss'].append(train_loss) history['train_acc'].append(train_acc) history['val_loss'].append(val_loss) history['val_acc'].append(val_acc) history['lr'].append(current_lr) if (epoch + 1) % 5 == 0: print(f"Epoch [{epoch+1}/{num_epochs}] LR: {current_lr:.6f} | " f"Train: {train_acc:.4f} | Val: {val_acc:.4f}") print(f"\nMeilleure accuracy de validation : {best_val_acc:.4f}")

Pourquoi le CNN est superieur au MLP pour les images ?

  1. Partage de parametres : un filtre 3x3 s'applique partout dans l'image. Le MLP a un poids different pour chaque pixel.
  2. Invariance a la translation : un chat detecte en haut a gauche sera aussi detecte en bas a droite.
  3. Hierarchie de features : les couches successives detectent des motifs de plus en plus complexes.
  4. Moins de parametres : malgre une bien meilleure performance, le CNN utilise 3x moins de parametres.

Partie 9 : Recapitulatif et points clefs

Ce que nous avons appris

  1. Architecture CNN : blocs Conv2d + BatchNorm + ReLU + MaxPool empiles, suivis de couches FC.
  2. Data augmentation : flips, crops, rotations augmentent la robustesse et reduisent le sur-apprentissage.
  3. Scheduler : CosineAnnealingLR permet une reduction douce du learning rate pour un fine-tuning en fin d'entrainement.
  4. Feature maps : les couches profondes capturent des features de plus en plus abstraites.
  5. CNN >> MLP : pour les images, le CNN est nettement superieur grace a l'exploitation de la structure spatiale.
