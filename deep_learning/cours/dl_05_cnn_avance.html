1. Au-dela de la classification

Dans le cours precedent, nous avons construit des CNN capables de classer une image entiere dans une categorie unique. Mais la vision par ordinateur ne se limite pas a dire "c'est un chat" ou "c'est un chien". En realite, les applications industrielles et scientifiques exigent des taches bien plus riches et complexes.

La vision par ordinateur (Computer Vision) regroupe l'ensemble des techniques permettant a une machine d'extraire, analyser et comprendre des informations a partir d'images ou de videos numeriques.

1.1 Les grandes taches en vision par ordinateur

Voici les quatre taches fondamentales que nous allons explorer dans ce cours :

  • Classification : attribuer une etiquette unique a une image entiere (resolu au cours 4)
  • Detection d'objets : localiser et identifier chaque objet dans l'image avec une bounding box
  • Segmentation semantique : classifier chaque pixel de l'image en une categorie
  • Segmentation d'instance : distinguer chaque instance individuelle d'objet au niveau pixel
Chacune de ces taches necessite une architecture specifique. On ne peut pas simplement prendre un ResNet et lui demander de detecter des objets. Il faut adapter la sortie du reseau a la nature de la tache.

1.2 Un probleme commun : le manque de donnees

Entrainer un CNN performant depuis zero (from scratch) exige generalement des millions d'images etiquetees. Or, dans la plupart des cas pratiques, on dispose de quelques centaines ou milliers d'echantillons. C'est la que le transfer learning devient indispensable.

2. Transfer Learning

Le transfer learning (apprentissage par transfert) consiste a reutiliser un modele pre-entraine sur un large jeu de donnees (ex : ImageNet, 1.2 million d'images, 1000 classes) comme point de depart pour resoudre une tache differente mais connexe.

2.1 Pourquoi ca marche ?

Les premieres couches d'un CNN apprennent des caracteristiques generiques et universelles : contours, textures, formes geometriques simples. Ces features sont utiles pour n'importe quelle tache de vision. Les couches profondes, elles, capturent des patterns de plus en plus specifiques au dataset d'entrainement.

Analogie : apprendre une langue

Imaginez que vous parlez couramment le francais et que vous voulez apprendre l'espagnol. Vous ne repartez pas de zero : votre connaissance de la grammaire, de l'alphabet latin, et des concepts linguistiques generaux se transfere. Vous n'avez qu'a apprendre les specificites de l'espagnol. Le transfer learning fonctionne exactement de la meme maniere pour les reseaux de neurones.

2.2 Hierarchie des features dans un CNN

2.3 Deux strategies principales

Feature Extraction

On gele toutes les couches convolutives et on ne remplace que le classifieur final.

Ideal quand on a peu de donnees et que la tache est proche d'ImageNet.

Fine-Tuning

On degele une partie des couches convolutives et on re-entraine tout avec un petit learning rate.

Ideal quand on a plus de donnees ou que la tache est eloignee d'ImageNet.

Lors du fine-tuning, utilisez toujours un learning rate tres petit (ex : 1e-5) pour les couches pre-entrainees. Un learning rate trop grand detruirait les poids appris sur ImageNet, ce qu'on appelle le catastrophic forgetting.

# ============================================ # Fine-Tuning : degeler les dernieres couches # ============================================ model = models.resnet50(weights=models.ResNet50_Weights.IMAGENET1K_V2) # Geler tout d'abord for param in model.parameters(): param.requires_grad = False # Degeler layer4 (les derniers blocs convolutifs) for param in model.layer4.parameters(): param.requires_grad = True # Remplacer le classifieur model.fc = nn.Sequential( nn.Linear(model.fc.in_features, 256), nn.ReLU(), nn.Linear(256, num_classes) ) # Learning rates differentiels optimizer = optim.Adam([ {'params': model.layer4.parameters(), 'lr': 1e-5}, # Tres petit LR {'params': model.fc.parameters(), 'lr': 1e-3}, # LR normal ]) print("Fine-tuning configure avec learning rates differentiels")
Le fine-tuning avec des learning rates differentiels est la meilleure pratique. Les couches pre-entrainees recoivent un LR 100x plus petit pour preserver les features apprises, tandis que le nouveau classifieur apprend rapidement.

4.2 Techniques avancees : CutOut et MixUp

CutOut / Random Erasing

CutOut masque aleatoirement une region rectangulaire de l'image, forcant le modele a ne pas se reposer sur une seule partie de l'objet. En PyTorch, transforms.RandomErasing implemente cette technique.

# ============================================ # Implementation de MixUp # ============================================ def mixup_data(x, y, alpha=0.2): """Applique MixUp : melange deux images et leurs labels.""" if alpha > 0: lam = torch.distributions.Beta(alpha, alpha).sample() else: lam = 1.0 batch_size = x.size(0) index = torch.randperm(batch_size).to(x.device) mixed_x = lam * x + (1 - lam) * x[index] y_a, y_b = y, y[index] return mixed_x, y_a, y_b, lam def mixup_criterion(criterion, pred, y_a, y_b, lam): """Calcul de la loss avec MixUp.""" return lam * criterion(pred, y_a) + (1 - lam) * criterion(pred, y_b) # Utilisation dans la boucle d'entrainement for images, labels in train_loader: images, labels = images.to(device), labels.to(device) mixed_images, y_a, y_b, lam = mixup_data(images, labels) outputs = model(mixed_images) loss = mixup_criterion(criterion, outputs, y_a, y_b, lam) loss.backward() optimizer.step()
MixUp est particulierement utile quand vous avez peu de donnees. Il regularise le modele en creant des echantillons "intermediaires" entre les classes, ce qui lisse les frontieres de decision et reduit l'overfitting.

5. Detection d'objets - Introduction

La detection d'objets (object detection) consiste a localiser et classifier chaque objet present dans une image. Pour chaque objet, le modele predit une bounding box (x, y, largeur, hauteur) et une classe.

5.1 Bounding Boxes et format

Une bounding box est generalement representee par quatre coordonnees. Deux formats principaux existent :

Format (x1, y1, x2, y2)

Coin superieur gauche + Coin inferieur droit

Utilise par Faster R-CNN, torchvision

Format (cx, cy, w, h)

Centre + Largeur + Hauteur

Utilise par YOLO, SSD

5.2 Intersection over Union (IoU)

L'IoU (Intersection over Union) mesure le chevauchement entre deux bounding boxes. C'est la metrique standard pour evaluer la qualite d'une detection : IoU = Aire(Intersection) / Aire(Union).
def compute_iou(box1, box2): """ Calcule l'IoU entre deux bounding boxes. Format : (x1, y1, x2, y2) """ # Coordonnees de l'intersection x1 = max(box1[0], box2[0]) y1 = max(box1[1], box2[1]) x2 = min(box1[2], box2[2]) y2 = min(box1[3], box2[3]) # Aire de l'intersection inter_area = max(0, x2 - x1) * max(0, y2 - y1) # Aires des deux boxes area1 = (box1[2] - box1[0]) * (box1[3] - box1[1]) area2 = (box2[2] - box2[0]) * (box2[3] - box2[1]) # IoU union_area = area1 + area2 - inter_area iou = inter_area / union_area if union_area > 0 else 0 return iou # Exemple pred_box = (50, 50, 200, 200) true_box = (60, 40, 210, 190) print(f"IoU = {compute_iou(pred_box, true_box):.4f}")
IoU = 0.7045

5.3 Non-Maximum Suppression (NMS)

Un detecteur produit souvent plusieurs bounding boxes pour le meme objet. Le NMS supprime les detections redondantes en ne gardant que celle avec le score de confiance le plus eleve.

Etape 1 : Trier toutes les detections par score de confiance decroissant.
Etape 2 : Prendre la detection avec le score le plus eleve.
Etape 3 : Calculer l'IoU avec toutes les detections restantes.
Etape 4 : Supprimer celles dont l'IoU depasse un seuil (ex: 0.5).
Etape 5 : Repeter jusqu'a ce qu'il ne reste plus de detections.
from torchvision.ops import nms # Exemple de NMS avec PyTorch boxes = torch.tensor([ [100, 100, 210, 210], [105, 98, 215, 208], [300, 150, 400, 300], [298, 155, 405, 295], ], dtype=torch.float32) scores = torch.tensor([0.95, 0.90, 0.88, 0.75]) keep = nms(boxes, scores, iou_threshold=0.5) print(f"Indices conserves : {keep}") print(f"Boxes finales :\n{boxes[keep]}")
Indices conserves : tensor([0, 2]) Boxes finales : tensor([[100., 100., 210., 210.], [300., 150., 400., 300.]])

Two-stage vs One-stage

Les methodes two-stage (R-CNN, Faster R-CNN) proposent d'abord des regions d'interet puis les classifient. Elles sont precises mais lentes. Les methodes one-stage (YOLO, SSD) predisent directement les bounding boxes et les classes en une seule passe du reseau. Elles sont rapides mais historiquement moins precises (ecart qui s'est reduit avec YOLOv3+).

7. YOLO en detail

YOLO (You Only Look Once) reformule la detection d'objets comme un probleme de regression unique. L'image est divisee en une grille S x S. Chaque cellule de la grille predit B bounding boxes, des scores de confiance et des probabilites de classe, le tout en une seule passe du reseau.

7.2 Anchor Boxes

A partir de YOLOv2, le concept d'anchor boxes (ou prior boxes) est introduit. Plutot que de predire des dimensions absolues, le reseau predit des offsets par rapport a des boxes predefinies de tailles et proportions variees. Cela facilite grandement la convergence.

Comment determiner les anchors ?

On utilise l'algorithme K-Means sur les dimensions des bounding boxes du dataset d'entrainement. Pour YOLO, on utilise generalement K=5 ou K=9 anchors. La metrique de distance utilisee est d(box, centroid) = 1 - IoU(box, centroid) plutot que la distance euclidienne, car cette derniere favorise les grandes boxes.

7.3 La loss function de YOLO

La loss de YOLO combine trois composantes :

Loss = Loss_coord + Loss_conf + Loss_class

  • Loss_coord : erreur de localisation (MSE sur x, y, sqrt(w), sqrt(h))
  • Loss_conf : erreur de confiance (BCE, avec poids different pour obj/no-obj)
  • Loss_class : erreur de classification (Cross-Entropy par cellule responsable)
La racine carree sur w et h dans la loss de localisation est une astuce importante de YOLOv1. Elle permet de penaliser davantage les erreurs sur les petits objets : une erreur de 10 pixels sur un objet de 30px est bien plus grave qu'une erreur de 10 pixels sur un objet de 300px.

8. Segmentation semantique

La segmentation semantique consiste a attribuer une classe a chaque pixel de l'image. Contrairement a la detection, il n'y a pas de bounding box : on obtient un masque dense ou chaque pixel est colore selon sa classe.

8.1 Fully Convolutional Networks (FCN)

L'idee cle des FCN (Long et al., 2015) est de remplacer les couches fully-connected d'un CNN de classification par des couches convolutives 1x1. Cela permet de produire une carte de segmentation de taille spatiale quelconque, pas seulement un vecteur de classe.

Probleme de la resolution

Les CNN classiques reduisent progressivement la resolution spatiale (via pooling et strides). Pour la segmentation, il faut retrouver la resolution originale. Deux approches :

  • Upsampling / Deconvolution : on augmente progressivement la taille des feature maps
  • Skip connections : on combine les features haute resolution des couches superficielles avec les features semantiques des couches profondes

8.2 Architecture U-Net

U-Net (Ronneberger et al., 2015) est l'architecture de segmentation la plus influente. Sa forme en "U" est reconnaissable : un encodeur contractant suivi d'un decodeur expansif, relies par des skip connections symetriques.

8.3 U-Net en PyTorch

import torch import torch.nn as nn class DoubleConv(nn.Module): """Bloc de base : Conv -> BN -> ReLU -> Conv -> BN -> ReLU""" def __init__(self, in_ch, out_ch): super().__init__() self.conv = nn.Sequential( nn.Conv2d(in_ch, out_ch, 3, padding=1), nn.BatchNorm2d(out_ch), nn.ReLU(inplace=True), nn.Conv2d(out_ch, out_ch, 3, padding=1), nn.BatchNorm2d(out_ch), nn.ReLU(inplace=True), ) def forward(self, x): return self.conv(x) class UNet(nn.Module): def __init__(self, in_channels=3, num_classes=21): super().__init__() # Encodeur self.enc1 = DoubleConv(in_channels, 64) self.enc2 = DoubleConv(64, 128) self.enc3 = DoubleConv(128, 256) self.enc4 = DoubleConv(256, 512) self.pool = nn.MaxPool2d(2) # Bottleneck self.bottleneck = DoubleConv(512, 1024) # Decodeur self.up4 = nn.ConvTranspose2d(1024, 512, 2, stride=2) self.dec4 = DoubleConv(1024, 512) # 512+512 par skip self.up3 = nn.ConvTranspose2d(512, 256, 2, stride=2) self.dec3 = DoubleConv(512, 256) self.up2 = nn.ConvTranspose2d(256, 128, 2, stride=2) self.dec2 = DoubleConv(256, 128) self.up1 = nn.ConvTranspose2d(128, 64, 2, stride=2) self.dec1 = DoubleConv(128, 64) # Classifieur pixel-wise self.final = nn.Conv2d(64, num_classes, 1) def forward(self, x): # Encodeur avec sauvegarde pour skip connections e1 = self.enc1(x) e2 = self.enc2(self.pool(e1)) e3 = self.enc3(self.pool(e2)) e4 = self.enc4(self.pool(e3)) # Bottleneck b = self.bottleneck(self.pool(e4)) # Decodeur avec skip connections d4 = self.dec4(torch.cat([self.up4(b), e4], dim=1)) d3 = self.dec3(torch.cat([self.up3(d4), e3], dim=1)) d2 = self.dec2(torch.cat([self.up2(d3), e2], dim=1)) d1 = self.dec1(torch.cat([self.up1(d2), e1], dim=1)) return self.final(d1) # Test model = UNet(in_channels=3, num_classes=21) x = torch.randn(1, 3, 256, 256) output = model(x) print(f"Entree : {x.shape}") print(f"Sortie : {output.shape}")
Entree : torch.Size([1, 3, 256, 256]) Sortie : torch.Size([1, 21, 256, 256])
La sortie a la meme resolution spatiale que l'entree (256x256) mais avec 21 canaux, un par classe. Pour obtenir la segmentation finale, on applique argmax sur la dimension des canaux : chaque pixel recoit la classe avec le score le plus eleve.

8.4 Loss pour la segmentation

# Cross-Entropy pixel-wise (la plus courante) criterion = nn.CrossEntropyLoss() # fonctionne directement sur les logits # Dice Loss (tres utilisee en segmentation medicale) class DiceLoss(nn.Module): def __init__(self, smooth=1.0): super().__init__() self.smooth = smooth def forward(self, pred, target): pred = torch.softmax(pred, dim=1) # One-hot encode target target_oh = torch.nn.functional.one_hot( target, num_classes=pred.size(1) ).permute(0, 3, 1, 2).float() intersection = (pred * target_oh).sum(dim=(2, 3)) union = pred.sum(dim=(2, 3)) + target_oh.sum(dim=(2, 3)) dice = (2 * intersection + self.smooth) / (union + self.smooth) return 1 - dice.mean()

9. Segmentation d'instance et panoptique

La segmentation d'instance combine la detection d'objets et la segmentation semantique : elle identifie chaque objet individuellement avec un masque pixel-precis. Chaque instance de "personne" ou de "voiture" recoit son propre masque distinct.

9.1 Mask R-CNN

Mask R-CNN (He et al., 2017) etend Faster R-CNN en ajoutant une branche de segmentation parallele a la branche de classification et de regression des bounding boxes. Pour chaque region d'interet (RoI), le reseau predit simultanement :

  1. La classe de l'objet
  2. La bounding box raffinee
  3. Un masque binaire de segmentation (28x28 pixels par RoI)

RoI Align vs RoI Pooling

Une innovation cle de Mask R-CNN est le RoI Align, qui remplace le RoI Pooling de Faster R-CNN. Le RoI Pooling effectue des arrondis lors de la quantification des coordonnees, ce qui introduit un desalignement spatial prejudiciable pour la segmentation pixel-precis. RoI Align utilise l'interpolation bilineaire pour echantillonner les features a des positions exactes, evitant toute perte de precision.

9.2 Segmentation panoptique

La segmentation panoptique (Kirillov et al., 2019) unifie la segmentation semantique et la segmentation d'instance en un seul framework :

Stuff (decor)

Regions amorphes : ciel, route, herbe

Segmentation semantique classique

Things (objets)

Objets comptables : voiture, personne

Segmentation d'instance

La segmentation panoptique est aujourd'hui le standard dans les applications de conduite autonome. Chaque pixel de l'image recoit a la fois une classe semantique et, s'il appartient a un objet "countable", un identifiant d'instance unique.

10.2 Guide pratique : quelle strategie choisir ?

Decision flowchart

  1. Peu de donnees (< 1000 images) : Transfer learning avec feature extraction. Ne touchez pas aux couches convolutives.
  2. Donnees moderees (1000 - 10 000) : Fine-tuning des dernieres couches + data augmentation agressive.
  3. Beaucoup de donnees (> 50 000) : Entrainement from scratch possible, ou fine-tuning complet d'un modele pre-entraine.
  4. Domaine tres different d'ImageNet (imagerie medicale, satellite) : Fine-tuning profond necessaire, eventuellement from scratch.

10.3 Erreurs frequentes a eviter

1. Oublier de normaliser les images avec les memes statistiques qu'ImageNet (mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225]) lors du transfer learning.
2. Utiliser le meme learning rate pour les couches pre-entrainees et les nouvelles couches. Les couches gelees ont besoin d'un LR 10-100x plus petit.
3. Appliquer la data augmentation sur le set de validation. Les transformations aleatoires ne doivent etre appliquees que pendant l'entrainement.
4. Ne pas appeler model.eval() et torch.no_grad() pendant l'inference. Cela affecte le comportement du BatchNorm et du Dropout, et gaspille de la memoire.

10.4 Perspectives et tendances actuelles

Le domaine evolue rapidement. Voici les tendances majeures :

  • Vision Transformers (ViT) : les Transformers remplacent progressivement les CNN pour la classification, et s'etendent a la detection (DETR) et la segmentation (SegFormer).
  • Foundation Models : des modeles pre-entraines massifs (SAM, DINOv2) permettent la segmentation zero-shot sur n'importe quelle image.
  • Modeles multimodaux : CLIP et ses descendants associent vision et langage, ouvrant la voie a la detection guidee par le texte.
  • Efficience : distillation, quantification et NAS produisent des modeles deployables sur mobile et embarque.
Dans le prochain cours, nous quitterons la vision par ordinateur pour explorer les reseaux recurrents (RNN, LSTM, GRU), essentiels pour le traitement des donnees sequentielles comme le texte et les series temporelles. Ces architectures posent les bases pour comprendre les Transformers, que nous etudierons au cours 7.
