1. L'Idee de l'Autoencoder

1.1 Compression et reconstruction

Un autoencoder est un reseau de neurones entraine a reproduire son entree en sortie. Cela semble trivial, mais l'astuce est de forcer les donnees a passer par un goulot d'etranglement (bottleneck) de dimension bien inferieure a l'entree. Le reseau doit alors apprendre une representation comprimee mais suffisamment informative pour reconstruire l'original.

Un autoencoder est un reseau de neurones compose de deux parties : un encodeur f qui comprime l'entree x en une representation latente z = f(x) de dimension reduite, et un decodeur g qui reconstruit l'entree a partir de z : x_hat = g(z) = g(f(x)). L'objectif est de minimiser l'erreur de reconstruction ||x - x_hat||^2.

Analogie : Le Telegraphe

Imaginez que vous devez envoyer une photographie detaillee par telegraphe (qui ne transmet que du texte court). Vous devez :

  • Encoder : Decrire la photographie en quelques mots cles essentiels ("femme, robe rouge, jardin, soleil couchant, sourire")
  • Decoder : Le destinataire reconstitue l'image mentale a partir de ces mots cles

L'autoencoder fait exactement cela : il apprend quels "mots cles" (features latentes) sont les plus importants pour decrire et reconstruire les donnees.

1.2 Architecture de base

1.3 A quoi sert un autoencoder ?

Applications principales

  • Reduction de dimensionnalite : Alternative non lineaire a PCA - la representation latente z capture les features les plus importantes
  • Debruitage : Entrainer l'AE a reconstruire des donnees propres a partir de donnees bruitees
  • Detection d'anomalies : Les donnees anormales sont mal reconstruites (haute erreur de reconstruction)
  • Generation de donnees : Avec les VAE, on peut echantillonner dans l'espace latent pour generer de nouvelles donnees
  • Pre-entrainement : L'encodeur apprend des representations utiles transferables a d'autres taches
  • Compression : Stockage efficace de donnees en haute dimension

2. Autoencoder Simple (Undercomplete)

2.1 Autoencoder undercomplete

Un autoencoder undercomplete est un autoencoder dont la dimension de l'espace latent est strictement inferieure a celle de l'entree : dim(z) < dim(x). C'est la forme la plus simple et la plus courante. La contrainte de dimension force le reseau a apprendre une compression informative des donnees.
Si l'espace latent a la meme dimension que l'entree (ou plus grande), l'autoencoder peut simplement apprendre la fonction identite sans compression. Il faut toujours choisir une dimension latente significativement plus petite que l'entree pour forcer l'apprentissage de representations utiles.

2.2 Fonction de perte (Loss)

La loss la plus courante pour les autoencoders est l'erreur quadratique moyenne (MSE) :

L_reconstruction = (1/N) * sum_i ||x_i - g(f(x_i))||^2

Pour les donnees binaires (images en noir et blanc), on utilise souvent la Binary Cross-Entropy (BCE) :

L_BCE = -sum [x * log(x_hat) + (1-x) * log(1-x_hat)]

2.3 Implementation PyTorch d'un autoencoder simple

import torch import torch.nn as nn class SimpleAutoencoder(nn.Module): """Autoencoder simple fully-connected pour MNIST""" def __init__(self, input_dim=784, latent_dim=32): super().__init__() # Encodeur : 784 -> 512 -> 256 -> 32 self.encoder = nn.Sequential( nn.Linear(input_dim, 512), nn.ReLU(), nn.Linear(512, 256), nn.ReLU(), nn.Linear(256, latent_dim), nn.ReLU() ) # Decodeur : 32 -> 256 -> 512 -> 784 self.decoder = nn.Sequential( nn.Linear(latent_dim, 256), nn.ReLU(), nn.Linear(256, 512), nn.ReLU(), nn.Linear(512, input_dim), nn.Sigmoid() # Sortie entre [0, 1] pour images normalisees ) def encode(self, x): return self.encoder(x) def decode(self, z): return self.decoder(z) def forward(self, x): z = self.encode(x) x_hat = self.decode(z) return x_hat, z # Test ae = SimpleAutoencoder(input_dim=784, latent_dim=32) x = torch.randn(16, 784) x_hat, z = ae(x) print(f"Entree: {x.shape}") print(f"Espace latent: {z.shape}") print(f"Reconstruction: {x_hat.shape}") print(f"Compression: {784/32:.1f}x")

3. L'Espace Latent

3.1 Representation apprise

L'espace latent est la representation comprimee apprise par l'encodeur. Chaque point dans cet espace correspond a une "description" compacte d'un exemple d'entree. Les dimensions de l'espace latent capturent les facteurs de variation les plus importants dans les donnees.

Proprietes d'un bon espace latent

  • Compacite : Les donnees sont representees avec un nombre minimal de dimensions
  • Regularite : Les points proches dans l'espace latent correspondent a des donnees similaires
  • Separabilite : Les differentes classes sont bien separees dans l'espace latent
  • Continuite : Idealement, interpoler entre deux points produit des donnees coherentes

3.4 Comparaison avec PCA

PCA (lineaire)

Projection lineaire sur les composantes principales

Capture uniquement les correlations lineaires

Rapide, deterministe, solution analytique

Autoencoder (non lineaire)

Transformation non lineaire via reseau de neurones

Capture les varietes non lineaires

Plus lent, necessite un entrainement, plus expressif

Un autoencoder avec des activations lineaires et une loss MSE apprend essentiellement la meme chose que PCA. C'est la non-linearite (ReLU, etc.) qui donne aux autoencoders leur pouvoir expressif superieur. En pratique, pour les donnees simples et de faible dimension, PCA est souvent suffisant. Les autoencoders brillent sur les donnees complexes et haute dimension (images, sequences).

4.2 Autoencoder Sparse

Un autoencoder sparse ajoute une penalite de parcimonie sur les activations latentes. L'objectif est que seul un petit nombre de neurones latents soient actifs pour chaque entree, encourageant le reseau a apprendre des features specialisees et interpretables.
# Loss avec regularisation L1 pour la sparsity def sparse_ae_loss(x, x_hat, z, lambda_sparse=1e-3): # Loss de reconstruction recon_loss = nn.MSELoss()(x_hat, x) # Penalite de parcimonie L1 sur les activations latentes sparsity_loss = torch.mean(torch.abs(z)) return recon_loss + lambda_sparse * sparsity_loss

4.3 Autoencoder Contractive

L'autoencoder contractive penalise la norme de Frobenius du Jacobien de l'encodeur par rapport a l'entree. Cela force les representations a etre insensibles aux petites perturbations de l'entree :

L_contractive = L_reconstruction + lambda * ||J_f(x)||_F^2
ou J_f(x) est la matrice jacobienne de l'encodeur.

5. Denoising Autoencoder

5.1 Principe du debruitage

Analogie : Le restaurateur d'art

Imaginez un restaurateur d'art qui apprend a retrouver l'oeuvre originale a partir de versions endommagees (taches, craquelures, decolorations). A force de voir de nombreuses paires "oeuvre endommagee / oeuvre originale", il developpe une expertise pour reconstituer les details manquants ou corrompus.

Le denoising autoencoder fonctionne de la meme maniere : il apprend a reconstruire les donnees propres a partir de versions corrompues.

5.2 Types de corruption

Methodes de corruption de l'entree

  • Bruit gaussien : Ajouter x_noisy = x + epsilon, avec epsilon ~ N(0, sigma^2). Le plus courant.
  • Masquage aleatoire : Mettre aleatoirement certains pixels a 0 (dropout de l'entree).
  • Salt and pepper : Remplacer aleatoirement des pixels par 0 ou 1.
  • Corruption structuree : Masquer des regions entieres (patches) de l'image.

5.3 Implementation PyTorch

class DenoisingAutoencoder(nn.Module): """Denoising Autoencoder convolutionnel""" def __init__(self): super().__init__() # Encodeur convolutionnel self.encoder = nn.Sequential( nn.Conv2d(1, 32, 3, stride=2, padding=1), # (32, 14, 14) nn.ReLU(), nn.Conv2d(32, 64, 3, stride=2, padding=1), # (64, 7, 7) nn.ReLU(), nn.Conv2d(64, 128, 7), # (128, 1, 1) nn.ReLU() ) # Decodeur convolutionnel self.decoder = nn.Sequential( nn.ConvTranspose2d(128, 64, 7), # (64, 7, 7) nn.ReLU(), nn.ConvTranspose2d(64, 32, 3, stride=2, padding=1, output_padding=1), nn.ReLU(), # (32, 14, 14) nn.ConvTranspose2d(32, 1, 3, stride=2, padding=1, output_padding=1), nn.Sigmoid() # (1, 28, 28) ) def forward(self, x): z = self.encoder(x) x_hat = self.decoder(z) return x_hat def add_noise(images, noise_factor=0.3): """Ajouter du bruit gaussien aux images""" noisy = images + noise_factor * torch.randn_like(images) return torch.clamp(noisy, 0.0, 1.0) # Entrainement du denoising autoencoder dae = DenoisingAutoencoder().to(device) optimizer = torch.optim.Adam(dae.parameters(), lr=1e-3) criterion = nn.MSELoss() for epoch in range(20): total_loss = 0 for imgs, _ in train_loader: imgs = imgs.to(device) # Corrompre les images noisy_imgs = add_noise(imgs, noise_factor=0.3) # Reconstruire les images propres a partir des corrompues reconstructed = dae(noisy_imgs) loss = criterion(reconstructed, imgs) # Loss vs images PROPRES optimizer.zero_grad() loss.backward() optimizer.step() total_loss += loss.item() print(f"Epoch {epoch+1}, Loss: {total_loss/len(train_loader):.6f}")

Le denoising autoencoder apprend des representations plus robustes qu'un autoencoder standard, car il doit "comprendre" la structure sous-jacente des donnees pour distinguer le signal du bruit. En pratique, les representations apprises par les denoising autoencoders sont souvent meilleures pour les taches en aval (classification, clustering).

6. Variational Autoencoder (VAE) en Detail

6.1 Le probleme de l'autoencoder classique pour la generation

Un autoencoder classique apprend un espace latent non structure. Les points de l'espace latent n'ont pas de signification probabiliste : si on echantillonne un point aleatoire z dans cet espace, rien ne garantit que le decodeur produira une image coherente. L'espace latent peut contenir de vastes "zones mortes" ou le decodeur n'a jamais ete entraine.

Avec un autoencoder classique, l'interpolation et la generation sont aleatoires : le decodeur peut produire des artefacts ou des images absurdes pour des points z qui ne correspondent a aucune donnee d'entrainement.

6.2 L'idee du VAE

Le Variational Autoencoder (Kingma & Welling, 2013) resout ce probleme en forcant l'espace latent a suivre une distribution gaussienne. Au lieu d'encoder une entree vers un point fixe z, le VAE l'encode vers une distribution q(z|x) = N(mu, sigma^2), definie par une moyenne mu et une variance sigma^2.

Differences cles : AE classique vs VAE

  • Autoencoder : Encode x -> z (un point fixe dans l'espace latent)
  • VAE : Encode x -> (mu, log_sigma^2), puis z est echantillonne depuis N(mu, sigma^2)

La regularisation KL force les distributions q(z|x) a rester proches de N(0, I), ce qui structure l'espace latent : chaque region a une signification, l'interpolation est fluide, et on peut generer en echantillonnant z ~ N(0, I).

7. La Loss du VAE

7.1 ELBO - Evidence Lower BOund

La loss du VAE est derivee de la maximisation de l'ELBO (Evidence Lower Bound), qui est une borne inferieure de la log-vraisemblance des donnees :

Derivation de la loss VAE

On veut maximiser log p(x), la log-vraisemblance des donnees. En utilisant l'inference variationnelle :

log p(x) >= E_q[log p(x|z)] - KL(q(z|x) || p(z)) = ELBO

La loss a minimiser est donc :

L_VAE = -E_q[log p(x|z)] + KL(q(z|x) || p(z))

7.2 Les deux termes de la loss

Terme 1 - Loss de reconstruction : -E_q[log p(x|z)]
C'est l'erreur de reconstruction entre l'entree et la sortie. Pour des images, on utilise typiquement MSE ou BCE. Ce terme pousse le VAE a bien reconstruire les donnees.
Terme 2 - Divergence KL : KL(q(z|x) || p(z))
Mesure la "distance" entre la distribution apprise q(z|x) = N(mu, sigma^2) et le prior p(z) = N(0, I). Ce terme regularise l'espace latent pour le garder structure et continu.

7.3 Formule fermee de la KL divergence

Quand q(z|x) est gaussienne et le prior p(z) = N(0, I), la KL divergence a une formule fermee elegante :

KL = -0.5 * sum(1 + log(sigma^2) - mu^2 - sigma^2)

En utilisant log_var = log(sigma^2) :

KL = -0.5 * sum(1 + log_var - mu^2 - exp(log_var))

7.4 Le Reparameterization Trick

Le reparameterization trick est une astuce mathematique qui rend l'echantillonnage stochastique differentiable. Au lieu d'echantillonner directement z ~ N(mu, sigma^2), on ecrit :
z = mu + sigma * epsilon, ou epsilon ~ N(0, I).
L'aleatoire est "externalise" dans epsilon, et les operations sur mu et sigma restent differentiables, permettant la retropropagation.
Sans le reparameterization trick, on ne pourrait pas retropropager le gradient a travers l'operation d'echantillonnage. C'est l'innovation technique cle qui rend le VAE entrainable par descente de gradient standard. L'astuce consiste a deplacer la source d'aleatoire hors du chemin de calcul.

7.5 Le compromis reconstruction vs regularisation

Balance entre les deux termes

  • Si la reconstruction domine : Le VAE produit des reconstructions nettes mais l'espace latent est desorganise (comme un AE classique). La generation est mauvaise.
  • Si la KL domine : L'espace latent est parfaitement gaussien mais les reconstructions sont floues. Le VAE "oublie" les details.
  • beta-VAE : Introduit un facteur beta devant la KL : L = Recon + beta * KL. beta > 1 encourage un espace latent plus disentangled (chaque dimension controle un facteur de variation independant).
Les images generees par les VAE sont souvent plus floues que celles des GANs. Cela est du a la nature de la loss de reconstruction (MSE tend a moyenner les possibilites) et a la regularisation KL. Des techniques comme les VAE-GAN combinent les deux approches pour obtenir des images nettes avec un espace latent structure.

8. Implementation VAE Complete en PyTorch

8.1 Architecture du VAE

import torch import torch.nn as nn import torch.nn.functional as F class VAE(nn.Module): """Variational Autoencoder pour MNIST""" def __init__(self, input_dim=784, hidden_dim=400, latent_dim=20): super().__init__() # ---- Encodeur ---- self.fc1 = nn.Linear(input_dim, hidden_dim) self.fc2 = nn.Linear(hidden_dim, hidden_dim) # Deux sorties : mu et log_var self.fc_mu = nn.Linear(hidden_dim, latent_dim) self.fc_logvar = nn.Linear(hidden_dim, latent_dim) # ---- Decodeur ---- self.fc3 = nn.Linear(latent_dim, hidden_dim) self.fc4 = nn.Linear(hidden_dim, hidden_dim) self.fc5 = nn.Linear(hidden_dim, input_dim) def encode(self, x): """Encoder x vers (mu, log_var)""" h = F.relu(self.fc1(x)) h = F.relu(self.fc2(h)) mu = self.fc_mu(h) log_var = self.fc_logvar(h) return mu, log_var def reparameterize(self, mu, log_var): """Reparameterization trick : z = mu + sigma * epsilon""" std = torch.exp(0.5 * log_var) # sigma = exp(0.5 * log(sigma^2)) eps = torch.randn_like(std) # epsilon ~ N(0, I) z = mu + std * eps return z def decode(self, z): """Decoder z vers x_hat""" h = F.relu(self.fc3(z)) h = F.relu(self.fc4(h)) x_hat = torch.sigmoid(self.fc5(h)) return x_hat def forward(self, x): mu, log_var = self.encode(x) z = self.reparameterize(mu, log_var) x_hat = self.decode(z) return x_hat, mu, log_var # Test de l'architecture vae = VAE(input_dim=784, hidden_dim=400, latent_dim=20) x = torch.randn(32, 784) x_hat, mu, log_var = vae(x) print(f"Entree: {x.shape}") print(f"mu: {mu.shape}, log_var: {log_var.shape}") print(f"Reconstruction: {x_hat.shape}") print(f"Parametres: {sum(p.numel() for p in vae.parameters()):,}")

8.2 Fonction de loss du VAE

def vae_loss(x, x_hat, mu, log_var, beta=1.0): """ Loss du VAE = Reconstruction + beta * KL Divergence Args: x: images originales (batch, 784) x_hat: images reconstruites (batch, 784) mu: moyenne de q(z|x) (batch, latent_dim) log_var: log-variance de q(z|x) (batch, latent_dim) beta: coefficient pour la KL (1.0 = VAE standard, >1 = beta-VAE) """ # Loss de reconstruction (BCE pour images binaires) recon_loss = F.binary_cross_entropy(x_hat, x, reduction='sum') # KL Divergence : KL(N(mu, sigma^2) || N(0, I)) # = -0.5 * sum(1 + log(sigma^2) - mu^2 - sigma^2) kl_loss = -0.5 * torch.sum(1 + log_var - mu.pow(2) - log_var.exp()) # Loss totale total_loss = recon_loss + beta * kl_loss return total_loss, recon_loss, kl_loss

9. Applications des Autoencoders

9.1 Generation d'images

Avec les VAE, on peut generer de nouvelles images en echantillonnant dans l'espace latent. Les images generees sont moins nettes que celles des GANs mais beneficient d'un espace latent structure et interpretable.

VAE pour la generation

Avantage : Espace latent continu et interpole bien

Inconvenient : Images legrement floues

Mesure : ELBO (borne sur la vraisemblance)

GAN pour la generation

Avantage : Images tres nettes et realistes

Inconvenient : Mode collapse, espace latent non structure

Mesure : FID, IS (pas de vraisemblance)

9.2 Interpolation dans l'espace latent

L'une des applications les plus fascinantes des VAE est l'interpolation fluide entre des images. Grace a la regularisation KL, le deplacement continu dans l'espace latent produit des transitions visuellement coherentes :

Types d'interpolation

  • Interpolation lineaire : z_interp = (1-t) * z1 + t * z2 pour t dans [0, 1]. Simple mais peut traverser des zones de basse probabilite.
  • Interpolation spherique (slerp) : Interpole le long du grand cercle reliant z1 et z2 sur la sphere. Meilleur pour les distributions gaussiennes car reste dans les zones de haute probabilite.

9.3 Compression de donnees

Un autoencoder peut servir de compresseur lossy : l'encodeur comprime les donnees en un vecteur latent compact, et le decodeur les reconstruit. Le ratio de compression est dim(x) / dim(z).

Exemples de compression

  • Image MNIST 28x28 (784 pixels) -> z de 32 dimensions = compression 24.5x
  • Image 128x128x3 (49152 pixels) -> z de 256 dimensions = compression 192x
  • Spectre audio (16000 points) -> z de 64 dimensions = compression 250x

9.4 Pre-entrainement non supervise

L'encodeur d'un autoencoder pre-entraine peut servir d'extracteur de features pour une tache supervisee en aval. Cette approche est particulierement utile quand :

  • On dispose de beaucoup de donnees non etiquetees mais peu de donnees etiquetees
  • On veut initialiser un reseau avec des representations significatives au lieu de poids aleatoires
  • On cherche des representations generiques transferables entre taches

10. Detection d'Anomalies avec Autoencoders

10.1 Le principe

La detection d'anomalies par autoencoder repose sur une idee simple : un autoencoder entraine sur des donnees normales reconstruira bien les donnees normales mais mal les donnees anormales (jamais vues pendant l'entrainement). L'erreur de reconstruction sert donc de score d'anomalie.

Analogie : Le copiste

Imaginez un copiste qui a passe sa vie a reproduire des paysages. Si on lui montre un nouveau paysage, il le reproduira fidelement car il a appris les "regles" des paysages. Mais si on lui montre un portrait, sa reproduction sera mauvaise car il n'a jamais appris a copier des visages. L'ecart entre l'original et la copie revele l'anomalie.

10.2 Pipeline de detection

Etape 1 - Entrainement : Entrainer l'autoencoder uniquement sur des donnees normales.
Etape 2 - Inference : Pour chaque nouvelle donnee, calculer l'erreur de reconstruction : score = ||x - AE(x)||^2.
Etape 3 - Seuil : Definir un seuil tau (typiquement le percentile 95 ou 99 des erreurs sur les donnees d'entrainement).
Etape 4 - Classification : Si score > tau, la donnee est classee comme anomalie.

10.3 Implementation PyTorch complete

import torch import torch.nn as nn import numpy as np class AnomalyDetectorAE(nn.Module): """Autoencoder pour la detection d'anomalies""" def __init__(self, input_dim, latent_dim=16): super().__init__() self.encoder = nn.Sequential( nn.Linear(input_dim, 128), nn.ReLU(), nn.Linear(128, 64), nn.ReLU(), nn.Linear(64, latent_dim), nn.ReLU() ) self.decoder = nn.Sequential( nn.Linear(latent_dim, 64), nn.ReLU(), nn.Linear(64, 128), nn.ReLU(), nn.Linear(128, input_dim), nn.Sigmoid() ) def forward(self, x): z = self.encoder(x) x_hat = self.decoder(z) return x_hat class AnomalyDetector: """Systeme complet de detection d'anomalies""" def __init__(self, input_dim, latent_dim=16, threshold_percentile=95): self.model = AnomalyDetectorAE(input_dim, latent_dim) self.threshold = None self.threshold_percentile = threshold_percentile self.device = torch.device('cuda' if torch.cuda.is_available() else 'cpu') self.model.to(self.device) def train(self, normal_data, epochs=50, lr=1e-3): """Entrainer sur les donnees normales uniquement""" self.model.train() optimizer = torch.optim.Adam(self.model.parameters(), lr=lr) criterion = nn.MSELoss(reduction='none') for epoch in range(epochs): x = normal_data.to(self.device) x_hat = self.model(x) loss = criterion(x_hat, x).mean() optimizer.zero_grad() loss.backward() optimizer.step() if (epoch + 1) % 10 == 0: print(f"Epoch {epoch+1}/{epochs}, Loss: {loss.item():.6f}") # Calculer le seuil sur les donnees d'entrainement self._compute_threshold(normal_data) def _compute_threshold(self, normal_data): """Calculer le seuil d'anomalie""" self.model.eval() with torch.no_grad(): x = normal_data.to(self.device) x_hat = self.model(x) # Erreur par echantillon errors = ((x - x_hat) ** 2).mean(dim=1).cpu().numpy() self.threshold = np.percentile(errors, self.threshold_percentile) print(f"Seuil d'anomalie (percentile {self.threshold_percentile}): {self.threshold:.6f}") def predict(self, data): """Detecter les anomalies. Retourne (is_anomaly, scores)""" self.model.eval() with torch.no_grad(): x = data.to(self.device) x_hat = self.model(x) scores = ((x - x_hat) ** 2).mean(dim=1).cpu().numpy() is_anomaly = scores > self.threshold return is_anomaly, scores # Exemple d'utilisation # Donnees normales (chiffre 0 dans MNIST) from torchvision import datasets, transforms train_data = datasets.MNIST('./data', train=True, transform=transforms.ToTensor(), download=True) # Extraire uniquement les 0 (donnees "normales") normal_indices = (train_data.targets == 0).nonzero().squeeze() normal_imgs = train_data.data[normal_indices].float() / 255.0 normal_imgs = normal_imgs.view(-1, 784) # Entrainer le detecteur detector = AnomalyDetector(input_dim=784, latent_dim=16, threshold_percentile=95) detector.train(normal_imgs, epochs=50) # Tester sur des donnees mixtes (0 = normal, autres = anomalies) test_data = datasets.MNIST('./data', train=False, transform=transforms.ToTensor()) test_imgs = test_data.data.float().view(-1, 784) / 255.0 test_labels = test_data.targets is_anomaly, scores = detector.predict(test_imgs[:100]) for i in range(10): status = "ANOMALIE" if is_anomaly[i] else "Normal" print(f"Image {i} (chiffre={test_labels[i]}): score={scores[i]:.4f} -> {status}")

10.4 Cas d'usage industriels

Applications de la detection d'anomalies par autoencoder

  • Controle qualite : Detecter les defauts de fabrication sur les lignes de production (pieces defectueuses, rayures, deformations)
  • Cybersecurite : Identifier les comportements reseau anormaux (intrusions, attaques DDoS, exfiltration de donnees)
  • Sante : Detecter les anomalies dans les images medicales (tumeurs, lesions, arythmies cardiaques sur ECG)
  • Finance : Detection de fraude sur les transactions bancaires
  • IoT / Maintenance predictive : Detecter les pannes imminentes a partir des capteurs de machines industrielles
L'avantage majeur de la detection d'anomalies par autoencoder est qu'elle ne necessite que des donnees normales pour l'entrainement. C'est un apprentissage semi-supervise qui est tres pratique en industrie, ou les anomalies sont rares et couteuses a obtenir. Cependant, le choix du seuil et de la dimension latente reste critique et necessite un travail de validation soigneux.

Resume : Points cles des Autoencoders et VAE

  • Les autoencoders apprennent des representations comprimees en reconstruisant leur entree a travers un goulot d'etranglement
  • L'espace latent capture les facteurs de variation les plus importants des donnees
  • Les denoising autoencoders apprennent des representations plus robustes grace a la corruption de l'entree
  • Les VAE structurent l'espace latent avec une regularisation gaussienne, permettant la generation
  • La loss du VAE combine reconstruction + KL divergence (ELBO)
  • Le reparameterization trick rend l'echantillonnage differentiable
  • La detection d'anomalies par autoencoder exploite l'erreur de reconstruction comme score d'anomalie
  • Les autoencoders sont utilises en compression, generation, debruitage, pre-entrainement et detection d'anomalies
